# LES-291: Tailwind config fontSize hardcodes px instead of referencing CSS tokens

## Symptom
- `tailwind.config.js` defines fontSize with raw pixel values like `'48px'`, `'32px'`
- Changing the fluid scale in `_tokens.css` doesn't affect Tailwind-generated classes
- Two sources of truth for typography sizes (CSS vars vs Tailwind config)

## Root Cause
When extending Tailwind's fontSize, developers copy pixel values from the design spec instead of referencing the CSS custom properties. This creates a disconnect between the fluid `clamp()` scale in `_tokens.css` and the static values in Tailwind config.

## Wrong Pattern
```javascript
// tailwind.config.js — SAI
fontSize: {
  'display': ['48px', { lineHeight: '1.1', fontWeight: '800' }],
  'h1':      ['32px', { lineHeight: '1.2', fontWeight: '700' }],
  'h2':      ['24px', { lineHeight: '1.3', fontWeight: '700' }],
  'body-md': ['16px', { lineHeight: '1.6', fontWeight: '400' }],
}
```

## Correct Pattern
```javascript
// tailwind.config.js — ĐÚNG: reference CSS vars
fontSize: {
  'display': ['var(--text-4xl)', { lineHeight: '1.1', fontWeight: '800' }],
  'h1':      ['var(--text-3xl)', { lineHeight: '1.2', fontWeight: '700' }],
  'h2':      ['var(--text-2xl)', { lineHeight: '1.3', fontWeight: '700' }],
  'body-md': ['var(--text-base)', { lineHeight: '1.6', fontWeight: '400' }],
}
```

## Fix Checklist
1. Open `tailwind.config.js` → find `fontSize` extend section
2. Replace each hardcoded px value with corresponding `var(--text-*)` token
3. Mapping: 48px→var(--text-4xl), 32px→var(--text-3xl), 24px→var(--text-2xl), 20px→var(--text-xl), 18px→var(--text-lg), 16px→var(--text-base/--text-md), 14px→var(--text-sm), 12px→var(--text-xs)
4. Rebuild Tailwind CSS to verify no visual regression
