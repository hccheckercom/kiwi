## Bad
```ts
export const useCart = create<CartStore>()(
  persist(
    (set, get) => ({
      items: [],
      isOpen: false,  // UI state persisted → hydration mismatch
      addItem: (item) => set(...),
    }),
    { name: 'cart' }
    // No partialize → everything serialized including functions (silently fails)
  )
)
```

## Good (pattern from lib/stores/ind-cart.ts)
```ts
export const useIndCart = create<CartStore>()(
  persist(
    (set, get) => ({
      items: [],
      isOpen: false,
      addItem: (item, qty = 1) => { ... },
      clearCart: () => set({ items: [] }),
    }),
    {
      name: 'ind-cart',
      partialize: (state) => ({ items: state.items }),
      // Only persist data — not UI state (isOpen) or functions
    }
  )
)
```

## Why
Real pattern from ind-cart.ts. Without partialize: (1) UI state like isOpen persists across page loads → hydration mismatch in SSR, (2) functions get serialized as undefined → silent corruption, (3) localStorage grows with every new field added to store. Only persist data that must survive page reload.
