## Auth patterns in webstore-vn

| Pattern | Use when | How it works |
|---------|----------|--------------|
| `withPlatformKey(handler)` | Platform API (v1/ecommerce/*) | Reads X-Platform-Key header, validates, injects store |
| `requireAdmin(req)` | Admin-only routes | Throws redirect if not admin — MUST re-throw in catch |
| `requireStoreOwner(req)` | Store owner routes | Checks user owns the store |
| `supabase.auth.getUser()` | User-specific routes | Returns user or null — manual check needed |
| `withWzseoAuth(handler)` | WzSEO API routes | API key auth for SEO service |

## Bad
```ts
// Platform API using requireAdmin — wrong! Platform stores don't have admin users
export async function GET(req: Request) {
  await requireAdmin(req) // Platform API key holders are NOT admin users
  // ...
}
```

## Good
```ts
// Platform API — use withPlatformKey
export const GET = withPlatformKey(async (req, store) => {
  // store is validated, scoped to this tenant
})
```

## Why
Real bug: commit 7116896. Using requireAdmin in platform API routes causes 500 because it throws redirect (not JSON error). Platform routes need withPlatformKey which returns JSON errors. Each auth pattern has different error behavior.
