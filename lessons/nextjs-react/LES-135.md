## Bad
```ts
// Looks fine locally but breaks CI when Supabase types regenerate
const slug = template.slug as string  // slug is actually string | null after schema change
const url = `/template/${slug}`       // CI: Type 'string | null' not assignable to 'string'
```

## Good
```ts
const slug = template.slug
if (!slug) return notFound()
const url = `/template/${slug}`  // TypeScript knows slug is string here
```

## Pattern: CI-blocking type errors in webstore-vn
```ts
// Common offenders:
// 1. Supabase select() return type changes after migration
// 2. PaginationMeta field name mismatch (total vs totalCount)
// 3. Template slug nullable after schema update

// Fix: narrow with runtime check, not cast
```

## Why
Real bug: commit ae2b9f6. `as` casts suppress TypeScript errors locally but CI with fresh types fails. In webstore-vn, Supabase types are auto-generated — schema changes make casts silently wrong. Always narrow with runtime checks instead of casting.
