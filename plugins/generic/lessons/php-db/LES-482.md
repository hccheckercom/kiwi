## Bad
```php
$data = json_decode( wp_remote_retrieve_body( $response ), true );
if ( ! is_array( $data ) ) {
    return null;
}
```

## Good
```php
$body = wp_remote_retrieve_body( $response );
$data = json_decode( $body, true );
if ( json_last_error() !== JSON_ERROR_NONE ) {
    error_log( 'JSON decode failed: ' . json_last_error_msg() );
    return null;
}
if ( ! is_array( $data ) ) {
    return null;
}
```

## Why
json_decode() returns null on invalid JSON. Without json_last_error() check, you cannot distinguish between valid null and decode failure. Always check json_last_error() and log errors for debugging.

## Grep
`json_decode\([^)]+\)(?!.*json_last_error)` — in `**/*.php`
