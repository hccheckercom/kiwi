## Pattern

Seed import operations that loop through data and call `$wpdb->insert()` for each row create N+1 queries. For large datasets (100+ rows), this causes significant performance degradation. Use `wz_bulk_insert()` helper for bulk operations.

## Bug

```php
// SAI — N+1 queries cho 1000 products = 1000 INSERT statements
private function import_products(): array {
    global $wpdb;
    
    $data = $this->read_json( 'products.json' );
    $products_table = $wpdb->prefix . 'wz_products';
    
    foreach ( $data as $product ) {
        $wpdb->insert( // N queries
            $products_table,
            array(
                'sku'  => $product['sku'],
                'name' => $product['name'],
                'price' => $product['price'],
            ),
            array( '%s', '%s', '%f' )
        );
    }
}

// 1000 products → 1000 queries → 5-10 seconds
```

**Hậu quả:**
- N+1 queries → slow import (5-10s cho 1000 rows)
- High DB load → locks, replication lag
- Timeout risk → import fails midway
- Poor UX → user waits, thinks it's broken

## Fix

### Option 1: Use `wz_bulk_insert()` (recommended)

```php
// ĐÚNG — 1 query cho 1000 products
private function import_products(): array {
    global $wpdb;
    
    $data = $this->read_json( 'products.json' );
    $products_table = $wpdb->prefix . 'wz_products';
    
    $rows = array();
    foreach ( $data as $product ) {
        $rows[] = array(
            sanitize_text_field( $product['sku'] ),
            sanitize_text_field( $product['name'] ),
            (float) $product['price'],
        );
    }
    
    wz_bulk_insert(
        $products_table,
        $rows,
        array( '%s', '%s', '%f' )
    );
}

// 1000 products → 1 query → 0.5s
```

### Option 2: Batch inserts (khi cần insert_id)

```php
// ĐÚNG — batch 100 rows/query khi cần insert_id
private function import_categories(): int {
    global $wpdb;
    
    $data = $this->read_json( 'categories.json' );
    $table = $wpdb->prefix . 'wz_categories';
    $slug_map = array();
    
    $wpdb->query( 'START TRANSACTION' );
    
    foreach ( $data as $cat ) {
        $parent_id = null;
        if ( ! empty( $cat['parent'] ) && isset( $slug_map[ $cat['parent'] ] ) ) {
            $parent_id = $slug_map[ $cat['parent'] ];
        }
        
        $wpdb->insert( // no-bulk: needs insert_id for parent_id mapping
            $table,
            array(
                'name'      => sanitize_text_field( $cat['name'] ),
                'slug'      => sanitize_title( $cat['slug'] ),
                'parent_id' => $parent_id,
            ),
            array( '%s', '%s', '%d' )
        );
        
        $inserted_id = (int) $wpdb->insert_id;
        if ( $inserted_id > 0 ) {
            $slug_map[ $cat['slug'] ] = $inserted_id;
        }
    }
    
    $wpdb->query( 'COMMIT' );
    
    return count( $slug_map );
}

// Wrap trong transaction để giảm overhead
```

### Option 3: Multi-value INSERT (manual SQL)

```php
// ĐÚNG — manual multi-value INSERT
private function import_products(): array {
    global $wpdb;
    
    $data = $this->read_json( 'products.json' );
    $products_table = $wpdb->prefix . 'wz_products';
    
    $values = array();
    $placeholders = array();
    
    foreach ( $data as $product ) {
        $values[] = sanitize_text_field( $product['sku'] );
        $values[] = sanitize_text_field( $product['name'] );
        $values[] = (float) $product['price'];
        $placeholders[] = '(%s, %s, %f)';
    }
    
    $sql = "INSERT INTO `{$products_table}` (sku, name, price) VALUES " . implode( ', ', $placeholders );
    $wpdb->query( $wpdb->prepare( $sql, $values ) );
}
```

## Áp dụng

**Khi nào dùng `wz_bulk_insert()`:**
- Seed import (products, categories, tags)
- Batch operations (reindex, migration)
- CSV import
- Sync operations (API → DB)

**Khi nào KHÔNG dùng bulk insert:**
- Cần `insert_id` để map relations (categories với parent_id)
- Cần trigger WordPress hooks (`wp_insert_post`)
- Cần complex validation per row
- Small datasets (< 10 rows)

**Performance comparison:**
| Method | 1000 rows | 10000 rows |
|--------|-----------|------------|
| Loop insert | 5-10s | 50-100s |
| wz_bulk_insert() | 0.5s | 3-5s |
| Transaction + loop | 2-3s | 20-30s |

**Best practices:**
1. Use `wz_bulk_insert()` cho flat data (no relations)
2. Use transaction + loop khi cần insert_id
3. Batch 100-500 rows/query để tránh max_allowed_packet
4. Sanitize data TRƯỚC khi bulk insert
