## Bad
```php
class IconsAdminPage {
    private function enqueue_admin_assets(): void {
        wp_enqueue_style(
            'wezone-icons-admin',
            WEZONE_ICONS_URL . 'assets/css/admin/icons-admin.css'
        );
    }
}
```

## Good
```php
class IconsAdminPage {
    private function enqueue_admin_assets(): void {
        if ( ! defined( 'WEZONE_ICONS_URL' ) ) {
            return;
        }
        wp_enqueue_style(
            'wezone-icons-admin',
            WEZONE_ICONS_URL . 'assets/css/admin/icons-admin.css'
        );
    }
}
```

## Why
Plugin constants are defined in entry point (wezone-icons.php). If class is loaded before plugin init (e.g., via Composer autoload from another plugin), constants are undefined â†’ fatal error. Always guard with defined() check in classes outside entry point.

`(?<!defined\s*\(\s*['\"])WEZONE_ICONS_(?:URL|DIR|VERSION)(?!\s*\))` — in `src/**/*.php`
