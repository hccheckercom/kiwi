## Pattern

Reorder operations that loop through an array of IDs and execute individual `$wpdb->update()` for each item create N database round-trips. This is O(N) queries where a single CASE WHEN query would be O(1).

## Bug

```php
// SAI — N queries for N items
public static function reorder( array $ids ): bool {
    global $wpdb;
    $table = self::table();

    foreach ( $ids as $order => $id ) {
        $wpdb->update( $table, array( 'sort_order' => (int) $order ), array( 'id' => absint( $id ) ) );
    }

    return true;
}
// 100 items = 100 UPDATE queries
```

## Fix

```php
// ĐÚNG — single query with CASE WHEN
public static function reorder( array $ids ): bool {
    global $wpdb;
    $table = self::table();

    if ( empty( $ids ) ) {
        return false;
    }

    $cases       = '';
    $id_list     = array();
    $placeholders = array();

    foreach ( $ids as $order => $id ) {
        $safe_id      = absint( $id );
        $safe_order   = (int) $order;
        $cases       .= $wpdb->prepare( ' WHEN id = %d THEN %d', $safe_id, $safe_order );
        $id_list[]    = $safe_id;
        $placeholders[] = '%d';
    }

    $in_clause = implode( ',', $placeholders );

    $wpdb->query(
        $wpdb->prepare(
            "UPDATE {$table} SET sort_order = CASE{$cases} END WHERE id IN ({$in_clause})",
            ...$id_list
        )
    );

    return true;
}
// 100 items = 1 query
```

## Áp dụng

- Reorder endpoints with > 10 items → use CASE WHEN
- Alternative: build VALUES list and use INSERT ... ON DUPLICATE KEY UPDATE
- Always wrap batch updates in transaction if data integrity matters
