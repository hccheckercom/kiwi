# Pattern: Inline Styles in Admin Dashboard Widget

## Problem

Dashboard widgets with extensive inline `style=` attributes:

1. **CSP violation** — sites with `Content-Security-Policy: style-src 'self'` will block inline styles
2. **Unmaintainable** — style changes require editing PHP template, not CSS
3. **No dark mode** — hardcoded colors (`#666`, `#eee`, `#fff8e1`) break WordPress admin dark mode schemes

**Example (BAD):**
```php
public function render_dashboard_widget(): void {
    ?>
    <div style="text-align:center;padding:10px;">
        <p style="margin:10px 0 0;font-size:14px;color:#666;">/100 điểm</p>
    </div>
    <table style="width:100%;border-collapse:collapse;margin-top:15px;">
        <tr style="border-bottom:1px solid #eee;">
    <?php
}
```

## Fix

Move all styles to the enqueued CSS file and use classes:

```php
public function render_dashboard_widget(): void {
    ?>
    <div class="wezone-score-summary">
        <p class="wezone-score-subtitle">/100 điểm</p>
    </div>
    <table class="wezone-score-table">
        <tr class="wezone-score-row">
    <?php
}
```

```css
/* score.css */
.wezone-score-summary { text-align: center; padding: 10px; }
.wezone-score-subtitle { margin: 10px 0 0; font-size: 14px; color: var(--wz-score-text-muted); }
.wezone-score-table { width: 100%; border-collapse: collapse; margin-top: 15px; }
.wezone-score-row { border-bottom: 1px solid var(--wz-score-border); }
```

## Impact

- CSP compliant
- Dark mode compatible via CSS custom properties
- Single source of truth for visual styles
