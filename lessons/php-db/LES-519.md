## Bad
```php
$wpdb->query( "TRUNCATE TABLE {$table}" );
foreach ( $products as $product ) {
    $wpdb->insert( $table, $product );
}
```

## Good
```php
$tmp = $table . '_tmp';
$wpdb->query( "CREATE TABLE {$tmp} LIKE {$table}" );
wz_bulk_insert( $tmp, $rows, $format );
$wpdb->query( "RENAME TABLE {$table} TO {$table}_old, {$tmp} TO {$table}" );
$wpdb->query( "DROP TABLE {$table}_old" );
```

## Why
TRUNCATE + insert leaves table empty if insert fails. Use temp table swap: CREATE tmp â†’ INSERT â†’ RENAME â†’ DROP old. Zero downtime, rollback-safe.
