## Bad
```php
// WZ_Coupon has no interface — callers can't type-hint or mock
class WZ_Coupon {
    public function create( string $code, array $data = array() ): int { ... }
}
```

## Good
```php
// Define contract in Contracts/
interface CouponInterface {
    public function create( string $code, array $data = array() ): int;
}

class WZ_Coupon implements CouponInterface { ... }
```

## Why
WZ_Product, WZ_Order, WZ_Cart, WZ_Customer all implement interfaces. WZ_Coupon, WZ_ProductRelations, WZ_ProductWriter do not. This breaks consistency and prevents dependency injection / testing.
