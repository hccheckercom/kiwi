## Bad
```php
useEffect(() => {
  loadGraph()
}, [])

// WebSocket reconnects â†’ component re-renders â†’ useEffect runs again â†’ loadGraph() called â†’ infinite loading
```

## Good
```php
const hasLoadedRef = useRef(false)

useEffect(() => {
  if (!hasLoadedRef.current) {
    hasLoadedRef.current = true
    loadGraph()
  }
}, [])

// Or: add loading state to deps
useEffect(() => {
  if (!loading && !graphData) {
    loadGraph()
  }
}, [loading, graphData])
```

## Why
When WebSocket reconnects after tab visibility change, it triggers component re-render. If useEffect has empty deps [] and calls async data loading, the component enters infinite loading state because the effect doesn't track whether data is already loading or loaded.
