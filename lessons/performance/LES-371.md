# Bug: Transient Purge Gây Full Table Scan

## Vấn đề

`DELETE FROM wp_options WHERE option_name LIKE '_transient_%'` → full table scan → slow trên DB lớn.

**Ví dụ lỗi:**
```php
// ❌ SAI — full table scan
private function purge_transients(): int {
    global $wpdb;
    
    $count = (int) $wpdb->query(
        $wpdb->prepare(
            'DELETE FROM %i WHERE option_name LIKE %s OR option_name LIKE %s',
            $wpdb->options,
            $wpdb->esc_like( '_transient_wz' ) . '%',
            $wpdb->esc_like( '_site_transient_wz' ) . '%'
        )
    );
    
    return $count;
}
```

**EXPLAIN output:**
```sql
EXPLAIN DELETE FROM wp_options WHERE option_name LIKE '_transient_wz%';
+----+-------------+------------+------+---------------+------+---------+------+--------+-------------+
| id | select_type | table      | type | possible_keys | key  | key_len | ref  | rows   | Extra       |
+----+-------------+------------+------+---------------+------+---------+------+--------+-------------+
|  1 | SIMPLE      | wp_options | ALL  | NULL          | NULL | NULL    | NULL | 150000 | Using where |
+----+-------------+------------+------+---------------+------+---------+------+--------+-------------+
```

**Hậu quả:**
- 150k rows table → 2-5s query time
- Locks table → blocks other queries
- High CPU usage → affects all requests

## Root Cause

1. **`option_name` không có index** → MySQL phải scan toàn bộ table
2. **LIKE với wildcard prefix** → không thể dùng index (ngay cả khi có)
3. **Transients lẫn với options** → không thể optimize riêng

## Fix

### 1. Add index on option_name (quick win)

```sql
-- ✅ ĐÚNG — add index (helps with exact match, not LIKE with %)
ALTER TABLE wp_options ADD INDEX idx_option_name (option_name(191));
```

**Lưu ý:** Index này chỉ giúp với exact match hoặc prefix match (`LIKE 'abc%'`), KHÔNG giúp với `LIKE '%abc%'` hoặc `LIKE '_transient_%'` (vì `_` là wildcard).

### 2. Use separate transient table (recommended)

```php
// ✅ ĐÚNG — separate table cho transients
function wz_create_transient_table(): void {
    global $wpdb;
    
    $table_name = $wpdb->prefix . 'wz_transients';
    $charset    = $wpdb->get_charset_collate();
    
    $sql = "CREATE TABLE IF NOT EXISTS {$table_name} (
        transient_key varchar(191) NOT NULL,
        transient_value longtext NOT NULL,
        transient_expiration bigint(20) unsigned NOT NULL,
        PRIMARY KEY (transient_key),
        KEY expiration (transient_expiration)
    ) {$charset};";
    
    require_once ABSPATH . 'wp-admin/includes/upgrade.php';
    dbDelta( $sql );
}

// Custom transient functions
function wz_set_transient( string $key, $value, int $expiration = 0 ): bool {
    global $wpdb;
    
    $table = $wpdb->prefix . 'wz_transients';
    $exp   = $expiration ? time() + $expiration : 0;
    
    return (bool) $wpdb->replace(
        $table,
        array(
            'transient_key'        => $key,
            'transient_value'      => maybe_serialize( $value ),
            'transient_expiration' => $exp,
        ),
        array( '%s', '%s', '%d' )
    );
}

function wz_get_transient( string $key ) {
    global $wpdb;
    
    $table = $wpdb->prefix . 'wz_transients';
    $row   = $wpdb->get_row(
        $wpdb->prepare(
            "SELECT transient_value, transient_expiration FROM {$table} WHERE transient_key = %s",
            $key
        )
    );
    
    if ( ! $row ) {
        return false;
    }
    
    if ( $row->transient_expiration && $row->transient_expiration < time() ) {
        wz_delete_transient( $key );
        return false;
    }
    
    return maybe_unserialize( $row->transient_value );
}

function wz_delete_transient( string $key ): bool {
    global $wpdb;
    
    $table = $wpdb->prefix . 'wz_transients';
    return (bool) $wpdb->delete( $table, array( 'transient_key' => $key ), array( '%s' ) );
}

// Purge all — FAST với separate table
function wz_purge_all_transients(): int {
    global $wpdb;
    
    $table = $wpdb->prefix . 'wz_transients';
    return (int) $wpdb->query( "TRUNCATE TABLE {$table}" );
}

// Purge expired — có index trên expiration
function wz_purge_expired_transients(): int {
    global $wpdb;
    
    $table = $wpdb->prefix . 'wz_transients';
    return (int) $wpdb->query(
        $wpdb->prepare(
            "DELETE FROM {$table} WHERE transient_expiration > 0 AND transient_expiration < %d",
            time()
        )
    );
}
```

### 3. Batch delete với LIMIT (fallback nếu không thể tạo table mới)

```php
// ✅ ĐÚNG — batch delete để tránh long lock
private function purge_transients(): int {
    global $wpdb;
    
    $total   = 0;
    $batch   = 1000;
    $timeout = time() + 5; // max 5s
    
    while ( time() < $timeout ) {
        $deleted = (int) $wpdb->query(
            $wpdb->prepare(
                'DELETE FROM %i WHERE option_name LIKE %s LIMIT %d',
                $wpdb->options,
                $wpdb->esc_like( '_transient_wz' ) . '%',
                $batch
            )
        );
        
        $total += $deleted;
        
        if ( $deleted < $batch ) {
            break; // No more rows
        }
    }
    
    return $total;
}
```
