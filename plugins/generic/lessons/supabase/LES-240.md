## Pattern sai

```typescript
await supabase.from('orders').insert({
  template_id: templateId, customer_id: user?.id ?? null,
  // ...
})
```

## Phải làm

```typescript
await supabase.from('orders').insert({
  template_id: templateId, user_id: user?.id ?? null,
  // ...
})
```

## Lý do

Supabase PostgREST silently rejects inserts when a non-existent column is passed — returns error object but no throw. The column in `orders` table is `user_id`, not `customer_id`. This causes `create_failed` redirect without any visible server error.

## Grep check

```regex
\.(insert|update|upsert)\(\{[^}]*customer_id
```

Match = likely wrong column name. Verify against DB types in `types/database.types.ts`.
