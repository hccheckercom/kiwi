# Bug: SHOW TABLES In Hot-Path / Loop

## Problem

Running `SHOW TABLES LIKE %s` inside `format_item()` or any method called per-item in a list is an N+1 schema query. If listing 20 products, that's 20 redundant `SHOW TABLES` queries — the table either exists or it doesn't for the entire request.

**Example (BAD):**
```php
private function format_item( $post ): array {
    $downloads_table = $GLOBALS['wpdb']->prefix . 'wz_theme_downloads';
    // This runs for EVERY item in the list!
    if ( $GLOBALS['wpdb']->get_var( $GLOBALS['wpdb']->prepare( 'SHOW TABLES LIKE %s', $downloads_table ) ) === $downloads_table ) {
        $downloads = (int) $GLOBALS['wpdb']->get_var( ... );
    }
}
```

## Fix

Cache the result in a static variable — table existence doesn't change mid-request:

```php
private static array $table_exists_cache = array();

private function table_exists( string $table ): bool {
    if ( ! isset( self::$table_exists_cache[ $table ] ) ) {
        global $wpdb;
        self::$table_exists_cache[ $table ] = $wpdb->get_var(
            $wpdb->prepare( 'SHOW TABLES LIKE %s', $table )
        ) === $table;
    }
    return self::$table_exists_cache[ $table ];
}
```

## Impact

- 20 products × 1 SHOW TABLES = **20 queries → 1 query**
- Reduces DB round-trips on every list_items() call
