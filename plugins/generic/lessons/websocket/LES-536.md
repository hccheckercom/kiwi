## Bad

```typescript
useEffect(() => {
  const ws = new WebSocket(url)
  wsRef.current = ws

  ws.onopen = () => setConnected(true)
  ws.onclose = () => setConnected(false)

  return () => {
    ws.close()
  }
}, [url])
```

## Good

```typescript
useEffect(() => {
  let ws: WebSocket | null = null
  let reconnectTimer: ReturnType<typeof setTimeout> | null = null
  let isVisible = true

  const connect = () => {
    if (ws && ws.readyState === WebSocket.OPEN) return

    ws = new WebSocket(url)
    wsRef.current = ws

    ws.onopen = () => {
      setConnected(true)
      console.log('WebSocket connected')
    }

    ws.onclose = () => {
      setConnected(false)
      console.log('WebSocket disconnected')

      if (isVisible && !reconnectTimer) {
        reconnectTimer = setTimeout(() => {
          reconnectTimer = null
          connect()
        }, 2000)
      }
    }
  }

  const handleVisibilityChange = () => {
    isVisible = !document.hidden

    if (isVisible) {
      if (!ws || ws.readyState !== WebSocket.OPEN) {
        console.log('Tab visible, reconnecting WebSocket...')
        connect()
      }
    } else {
      if (ws && ws.readyState === WebSocket.OPEN) {
        console.log('Tab hidden, closing WebSocket')
        ws.close()
      }
      if (reconnectTimer) {
        clearTimeout(reconnectTimer)
        reconnectTimer = null
      }
    }
  }

  document.addEventListener('visibilitychange', handleVisibilityChange)
  connect()

  return () => {
    isVisible = false
    document.removeEventListener('visibilitychange', handleVisibilityChange)
    if (reconnectTimer) clearTimeout(reconnectTimer)
    if (ws) ws.close()
  }
}, [url])
```

## Why

When user switches browser tabs, WebSocket connection can become stale:
- Server may close idle connections
- Network state may change
- Connection appears open but is actually broken

Without tab visibility handling:
- Stale connections remain open when tab is hidden
- No reconnection when tab becomes visible again
- Messages fail silently with "Missing nonce verification" or similar errors
- UI shows "Connected" but connection is broken

**Impact:** Users see connection status as "Connected" but real-time updates fail. Switching tabs breaks WebSocket permanently until page reload.

## How to Fix

1. **Track tab visibility** with `document.hidden` and `visibilitychange` event
2. **Close connection when tab hidden** to avoid stale state
3. **Reconnect when tab visible** if connection is closed
4. **Auto-reconnect on close** with exponential backoff (2s delay)
5. **Clean up listeners** in useEffect cleanup

**Key points:**
- Use `isVisible` flag to prevent reconnection when tab is hidden
- Clear reconnect timer when tab becomes hidden
- Check `ws.readyState` before reconnecting to avoid duplicate connections
- Use `ReturnType<typeof setTimeout>` for TypeScript timer type (not `NodeJS.Timeout`)
