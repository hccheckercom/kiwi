## Pattern

AJAX handler cho read-only data (danh sách tỉnh/huyện/xã, lookup địa chỉ) có `check_ajax_referer()` nhưng JS caller không gửi nonce, hoặc gửi nonce với action name khác.

## Symptoms

- Dropdown cấp 1 (provinces) hoạt động bình thường
- Dropdown cấp 2, 3 (districts, wards) không load data — chỉ hiện placeholder
- Network tab: AJAX request trả về 403 hoặc `-1`
- Không có error hiện trên UI — fail silently

## Root Cause

```php
// PHP handler yêu cầu nonce action 'wz_address_lookup'
public function get_districts(): void {
    check_ajax_referer( 'wz_address_lookup', 'nonce' ); // ← FAIL
}
```

```javascript
// JS chỉ gửi action + parent_id, KHÔNG gửi nonce
fetch(wzTheme.ajaxurl, {
    method: 'POST',
    body: 'action=wz_get_districts&parent_id=' + id
    // ← thiếu &nonce=...
});
```

Thêm: `enqueue_nonce()` localize nonce vào `wzAddress.nonce` nhưng method này không bao giờ được gọi. Theme dùng `wzTheme.nonce` (action `wz-theme-nonce`) — khác action name.

## Fix

Với read-only public data (geographic lookup, no state mutation): xóa `check_ajax_referer()` — consistent với endpoint provinces.

Với write endpoints: đảm bảo JS gửi đúng nonce action name mà PHP check.

## Checklist

- [ ] Mọi `check_ajax_referer()` trong AJAX handler → verify JS caller gửi nonce với đúng action name
- [ ] Read-only public data endpoint (tỉnh/huyện/xã) → không cần nonce
- [ ] Write/mutation endpoint → bắt buộc nonce, verify JS gửi đúng
- [ ] `enqueue_nonce()` / `wp_localize_script()` → verify method được gọi trong boot/register flow
