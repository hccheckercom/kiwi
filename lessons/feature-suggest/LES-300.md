## Bad
```php
// Theme has no exit-intent mechanism at all
// User moves mouse to close tab → nothing happens
// Cart items silently abandoned with zero recovery attempt
```
No exit-intent popup means every abandoning visitor leaves without any intervention — no email capture, no discount offer, no last-chance CTA.

## Good
```javascript
// assets/js/exit-intent.js
(function() {
    let shown = false;
    const startTime = Date.now();

    function shouldShow() {
        if (shown) return false;
        if (Date.now() - startTime < 30000) return false;
        if (document.cookie.includes('wz_exit_dismissed')) return false;
        if (!window.wzCart || window.wzCart.count === 0) return false;
        return true;
    }

    document.addEventListener('mouseleave', function(e) {
        if (e.clientY < 0 && shouldShow()) {
            shown = true;
            document.getElementById('wz-exit-popup').classList.add('is-visible');
            dataLayer.push({ event: 'exit_intent_shown', cart_value: window.wzCart.total });
        }
    });

    // Mobile: back button
    window.addEventListener('popstate', function() {
        if (shouldShow()) {
            history.pushState(null, '', location.href);
            document.getElementById('wz-exit-popup').classList.add('is-visible');
        }
    });
})();
```

## Why
Blueprint `conversion-recovery.md` Section 2 requires exit-intent popup with: (1) desktop mouseleave trigger, (2) mobile popstate/scroll trigger, (3) escalating offers based on cart value, (4) 7-day dismiss cookie, (5) dataLayer tracking. Without this, the theme has zero chance of recovering the 70% of carts that are abandoned — the single highest-ROI conversion recovery mechanism.
