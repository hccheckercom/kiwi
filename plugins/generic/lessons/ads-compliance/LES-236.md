## Bad
```tsx
// Popup appears immediately on page load
useEffect(() => {
  setTimeout(() => setShowPopup(true), 1000) // 1 second delay
}, [])

// Exit-intent popup blocks navigation
window.addEventListener('beforeunload', (e) => {
  showExitPopup()
  e.preventDefault()
})

// Full-screen interstitial before content
<div className="fixed inset-0 z-50 bg-black/80">
  <div>ĐĂNG KÝ NGAY để nhận ưu đãi!</div>
</div>
```
Google penalizes "intrusive interstitials" that block content access. This reduces Quality Score and can trigger landing page rejection.

## Good
```tsx
// Non-intrusive banner at bottom (doesn't block content)
<div className="fixed bottom-0 left-0 right-0 bg-sky-500 p-3 text-center">
  <p>Nhận tư vấn miễn phí <Link href="/contact">Liên hệ →</Link></p>
</div>

// Popup only after meaningful engagement (scroll 50%+)
useEffect(() => {
  const handleScroll = () => {
    if (window.scrollY > document.body.scrollHeight * 0.5) {
      setShowCTA(true)
    }
  }
  window.addEventListener('scroll', handleScroll)
  return () => window.removeEventListener('scroll', handleScroll)
}, [])

// Cookie consent and age verification are exempt
<CookieConsent /> // OK — legally required
```

## Why
Google's "Intrusive Interstitial" penalty (since 2017) affects both organic rankings and ad Quality Score. Prohibited: (1) popups that cover main content immediately on load, (2) standalone interstitials before content access, (3) above-fold layouts that look like interstitials. Exempt: legally required notices (cookie consent, age verification), login dialogs for gated content, small banners that don't block content. Best practice: delay popups until user engagement (scroll, time on page > 30s), use non-blocking formats (bottom bar, slide-in), and always provide easy dismiss.
