## Bad
```php
$consent_cookie = $_COOKIE['wezone_cookie_consent'] ?? '';
if ( $consent_cookie === 'all' || $consent_cookie === 'marketing' ) {
    return true;
}
// â†’ Attacker sets cookie to: all<script>alert(1)</script>
```

## Good
```php
$consent_cookie = isset( $_COOKIE['wezone_cookie_consent'] ) 
    ? sanitize_text_field( wp_unslash( $_COOKIE['wezone_cookie_consent'] ) )
    : '';
if ( $consent_cookie === 'all' || $consent_cookie === 'marketing' ) {
    return true;
}
```

## Why
$_COOKIE values can be manipulated by user. Direct comparison without sanitization can lead to XSS if cookie value is later echoed, or logic bypass if attacker sets unexpected values.

## Grep
`\$_COOKIE\[[^\]]+\]\s*\?\?[^;]+;(?!.*sanitize)` — in `**/*.php`
