## Bad

```python
with open(filepath) as f:
    content = f.read()

with open(output, "w") as f:
    f.write(data)
```

## Good

```python
with open(filepath, encoding="utf-8") as f:
    content = f.read()

with open(output, "w", encoding="utf-8") as f:
    f.write(data)
```

## Why

On Windows, `open()` defaults to the system locale encoding (CP1252/ANSI), not UTF-8. Vietnamese text, emoji, or any non-ASCII content will cause `UnicodeDecodeError` or produce mojibake. Always specify `encoding="utf-8"` explicitly. Related: LES-612 (subprocess encoding).
