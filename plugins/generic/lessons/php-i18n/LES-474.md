## Bad
```php
wp_localize_script( 'my-script', 'myData', array(
    'i18n' => array(
        'loading' => __( 'Loading...', 'my-plugin' ), // Hardcoded key
        'error' => __( 'Error', 'my-plugin' ),
    ),
) );
```

## Good
```php
$i18n = apply_filters( 'my_plugin_js_i18n', array(
    'loading' => __( 'Loading...', 'my-plugin' ),
    'error' => __( 'Error', 'my-plugin' ),
) );
wp_localize_script( 'my-script', 'myData', array( 'i18n' => $i18n ) );
```

## Why
Hardcoded i18n keys in wp_localize_script cannot be rebranded without editing PHP. Use filter hooks or dynamic keys to allow JS text customization.

`wp_localize_script\([\s\S]{0,300}['"]i18n['"][\s\S]{0,300}__\(` — in `**/*.php`
