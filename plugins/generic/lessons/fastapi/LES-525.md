# Bad Code

```python
@app.post("/users")
async def create_user(user: UserCreate):
    db_user = User(**user.dict())
    db.add(db_user)
    db.commit()
    
    # BAD: Blocks response until email sent
    await send_welcome_email(db_user.email)
    await send_slack_notification(f"New user: {db_user.email}")
    
    return db_user  # User waits 2-3 seconds for emails

@app.post("/orders")
async def create_order(order: OrderCreate):
    db_order = Order(**order.dict())
    db.add(db_order)
    
    # BAD: Blocks response
    await update_analytics(db_order)
    await send_confirmation_email(db_order)
    
    return db_order
```

# Good Code

```python
from fastapi import BackgroundTasks

@app.post("/users")
async def create_user(
    user: UserCreate,
    background_tasks: BackgroundTasks
):
    db_user = User(**user.dict())
    db.add(db_user)
    db.commit()
    
    # GOOD: Runs after response sent
    background_tasks.add_task(send_welcome_email, db_user.email)
    background_tasks.add_task(send_slack_notification, f"New user: {db_user.email}")
    
    return db_user  # Instant response

@app.post("/orders")
async def create_order(
    order: OrderCreate,
    background_tasks: BackgroundTasks
):
    db_order = Order(**order.dict())
    db.add(db_order)
    
    background_tasks.add_task(update_analytics, db_order)
    background_tasks.add_task(send_confirmation_email, db_order)
    
    return db_order

# For long-running tasks, use Celery/RQ
@app.post("/reports")
async def generate_report(
    report: ReportCreate,
    background_tasks: BackgroundTasks
):
    # Quick tasks: BackgroundTasks
    background_tasks.add_task(log_report_request, report.id)
    
    # Long tasks: Queue
    celery_app.send_task("generate_report", args=[report.id])
    
    return {"status": "processing", "report_id": report.id}
```

# Why

**Without BackgroundTasks:**
- Slow response times (user waits for emails, analytics)
- Timeout errors (if task takes >30s)
- Poor UX (loading spinners)
- Wasted resources (connection held open)

**Performance impact:**
```python
# Without BackgroundTasks: 2500ms response
@app.post("/users")
async def create_user(user: UserCreate):
    db.add(user)  # 50ms
    await send_email(user)  # 2000ms
    await log_analytics(user)  # 500ms
    return user

# With BackgroundTasks: 50ms response
@app.post("/users")
async def create_user(user: UserCreate, bg: BackgroundTasks):
    db.add(user)  # 50ms
    bg.add_task(send_email, user)  # Runs after response
    bg.add_task(log_analytics, user)
    return user
```

**When to use BackgroundTasks:**
- Sending emails/notifications
- Logging/analytics
- Cache warming
- Cleanup tasks
- Non-critical operations

**When NOT to use BackgroundTasks:**
- Critical operations (payment processing)
- Long-running tasks (>30s) → use Celery/RQ
- Tasks that need retry logic → use queue
- Tasks that need monitoring → use queue
