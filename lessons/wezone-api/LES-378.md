# Bug: recalculate() Dùng Stale Price — Giá Cũ Sau Khi Admin Sửa

## Vấn đề

`recalculate()` tính subtotal dùng `$item['price']` đã lưu lúc add. Nếu admin thay đổi giá sản phẩm sau khi user thêm vào cart → user vẫn mua giá cũ.

**Hậu quả:**
- Flash sale hết hạn nhưng cart vẫn giữ giá sale
- Admin tăng giá nhưng orders vẫn ở giá cũ
- Admin giảm giá nhưng user không được hưởng

**Ví dụ lỗi:**
```php
// ❌ SAI — dùng stored price
private function recalculate( array &$cart ): void {
    foreach ( $cart['items'] as &$item ) {
        $item['subtotal'] = $item['price'] * $item['qty'];  // ← stale price!
        $subtotal += $item['subtotal'];
    }
}
```

## Fix

```php
// ✅ ĐÚNG — refresh price từ DB
private function recalculate( array &$cart ): void {
    $subtotal = 0.0;
    foreach ( $cart['items'] as &$item ) {
        $fresh_price = $this->resolvePrice(
            (int) $item['product_id'],
            (int) ( $item['variation_id'] ?? 0 )
        );
        $item['price']    = $fresh_price;
        $item['subtotal'] = round( $fresh_price * (int) $item['qty'], 4 );
        $subtotal         += $item['subtotal'];
    }
    unset( $item );
    // ... rest of recalculate
}
```

**Trade-off:** Mỗi recalculate thêm N queries (1/item). Mitigation: batch query tất cả product_ids một lần.
