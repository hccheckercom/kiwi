## Bad
```php
// header.php — no Facebook Pixel
<!DOCTYPE html>
<html>
<head>
  <?php wp_head(); ?>
  <!-- No fbq init -->
</head>
```
Meta Ads campaigns run blind — no conversion data, no remarketing audiences, no optimization signal.

## Good
```php
// inc/facebook-pixel.php
function wz_facebook_pixel_init() {
    $config = wz_get_store_config();
    $pixel_id = $config['fb_pixel_id'] ?? '';
    if ( empty( $pixel_id ) ) return;
    ?>
    <script>
    !function(f,b,e,v,n,t,s){if(f.fbq)return;n=f.fbq=function(){n.callMethod?
    n.callMethod.apply(n,arguments):n.queue.push(arguments)};if(!f._fbq)f._fbq=n;
    n.push=n;n.loaded=!0;n.version='2.0';n.queue=[];t=b.createElement(e);t.async=!0;
    t.src=v;s=b.getElementsByTagName(e)[0];s.parentNode.insertBefore(t,s)}(window,
    document,'script','https://connect.facebook.net/en_US/fbevents.js');
    fbq('init', '<?php echo esc_js( $pixel_id ); ?>');
    fbq('track', 'PageView');
    </script>
    <?php
}
add_action( 'wp_head', 'wz_facebook_pixel_init', 2 );
```

## Why
Facebook Pixel is required for: (1) Meta Ads conversion tracking, (2) Custom Audiences for remarketing, (3) Lookalike Audiences, (4) conversion optimization (CPA/ROAS bidding). Without fbq: ad spend has no feedback loop — Meta cannot optimize delivery. Blueprint requires both client-side pixel AND server-side CAPI for ad-blocker resilience.
