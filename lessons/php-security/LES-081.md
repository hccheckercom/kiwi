## Bad
```php
function handle_payment_ipn() {
    $data = json_decode(file_get_contents('php://input'), true);
    update_order_status($data['order_id'], 'paid');
}
```

## Good
```php
function handle_payment_ipn() {
    $data = json_decode(file_get_contents('php://input'), true);
    $signature = $_SERVER['HTTP_X_SIGNATURE'] ?? '';
    if ( ! verify_hmac($data, $signature, $secret_key) ) {
        wp_send_json_error('Invalid signature', 403);
    }
    update_order_status($data['order_id'], 'paid');
}
```

## Why
IPN/webhook handler without signature verification — attacker can forge payment confirmations.
