# LES-112: Dashboard Client-Side Aggregation Anti-Pattern

## Problem

Dashboard page fetch toàn bộ raw data (100 orders + 100 products + 100 customers) rồi tính stats bằng JavaScript:
- 3 API calls nặng, mỗi call trả về full object (billing, shipping, items, images...)
- Client phải filter, reduce, group — chậm trên mobile/low-end devices
- Bandwidth lãng phí: trả 300KB JSON chỉ để hiển thị 4 con số

## Fix

Tạo dedicated dashboard endpoints với SQL aggregation:

```php
// /dashboard/stats — 4 simple COUNT/SUM queries
$revenue = $wpdb->get_var("SELECT COALESCE(SUM(total), 0) FROM wz_orders
    WHERE DATE(created_at) = CURDATE() AND status IN ('completed','processing')");

// /dashboard/revenue-chart — 1 GROUP BY query
$rows = $wpdb->get_results("SELECT DATE(created_at) as date, SUM(total) as revenue
    FROM wz_orders WHERE DATE(created_at) >= DATE_SUB(CURDATE(), INTERVAL 30 DAY)
    AND status IN ('completed','processing') GROUP BY DATE(created_at)");

// /dashboard/recent-orders — SELECT chỉ cần fields hiển thị, LIMIT 5
// /dashboard/low-stock — SELECT chỉ products có stock < 5, LIMIT 5
```

Frontend chỉ cần 5 lightweight API calls thay vì 3 heavy calls.

## Impact

- Response size: ~300KB → ~5KB (giảm 98%)
- Queries: ~500 (N+1 trong format) → ~6 (aggregation)
- Time: 2-5s → <200ms

## Detection

Tìm pattern: `per_page: '100'` hoặc `per_page: 100` trong Dashboard/Stats components.
