# Bug: wz_apply_coupon() TOCTOU — getByCode() Có Thể Return Null

## Vấn đề

`wz_apply_coupon()` gọi `validate()` → rồi `getByCode()` → rồi `calculateDiscount()`. Giữa validate và getByCode, coupon có thể bị xóa bởi admin hoặc expired job → `getByCode()` return null → `calculateDiscount(null)` gây TypeError/fatal.

**Impact:** 500 error tại checkout → user mất giỏ hàng.

**Note:** Scanner sẽ detect dòng `$coupon_row = $engine->getByCode()` nhưng nếu có null-check trong 3 dòng tiếp theo (`if ($coupon_row === null)`) thì đây là false positive — code đã được fix đúng.

## Bad

```php
// ❌ SAI — không check null sau getByCode()
$error = $engine->validate( $code, $subtotal );
if ( $error !== '' ) {
    return -1.0;
}
$coupon_row = $engine->getByCode( $code );
return $engine->calculateDiscount( $code, $subtotal, $coupon_row ); // ← $coupon_row có thể null!
```

## Good

```php
// ✅ ĐÚNG — null-check sau getByCode()
$error = $engine->validate( $code, $subtotal, $uid );
if ( $error !== '' ) {
    return -1.0;
}
$coupon_row = $engine->getByCode( $code );
if ( $coupon_row === null ) {
    return -1.0;
}
return $engine->calculateDiscount( $code, $subtotal, $coupon_row );
```
