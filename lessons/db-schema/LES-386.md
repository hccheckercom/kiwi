## Bad
```php
'order_number' => 'varchar(50) NOT NULL',
// ... indexes:
// no UNIQUE on order_number
```

## Good
```php
'order_number' => 'varchar(50) NOT NULL',
// ... indexes:
'UNIQUE idx_order_number' => 'order_number',
```

## Why
order_number is a business identifier shown to customers. Without UNIQUE constraint, two orders can share the same number — causing wrong order lookup, payment mismatch, and customer confusion.
