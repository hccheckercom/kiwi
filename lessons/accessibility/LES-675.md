## Bad
```tsx
<Image src={product.image} width={300} height={300} />
```
Missing `alt` — screen readers skip or announce filename.

## Good
```tsx
<Image src={product.image} width={300} height={300} alt={product.name} />
```
Descriptive alt text for content images.

```tsx
<Image src="/decorative-bg.png" width={1200} height={400} alt="" />
```
Empty alt for decorative images — screen readers skip them.

## Why
WCAG 1.1.1 — all images must have alt text. Content images need descriptive text; decorative images need `alt=""` (not missing alt). Next.js Image component requires alt prop but doesn't enforce meaningful content.
