## Bad
```php
// single.php — product page loads with no tracking
<?php get_header(); ?>
<div class="product-detail">
  <?php // product content ?>
</div>
<?php get_footer(); ?>
```
Product page views are invisible to GA4 — cannot measure product interest or build remarketing audiences.

## Good
```php
// inc/tracking.php
function wz_tracking_product_view() {
    if ( ! wz_is_single_product() ) return;
    $product = wz_get_current_product();
    if ( ! $product ) return;
    ?>
    <script>
    dataLayer.push({ ecommerce: null });
    dataLayer.push({
        event: 'view_item',
        ecommerce: {
            currency: 'VND',
            value: <?php echo (int) ($product['sale_price'] ?: $product['price']); ?>,
            items: [<?php echo wp_json_encode( wz_tracking_item_data( $product ) ); ?>]
        }
    });
    </script>
    <?php
}
add_action( 'wp_footer', 'wz_tracking_product_view' );
```

## Why
`view_item` is the foundation of GA4 ecommerce funnel. Without it: (1) no product-level engagement data, (2) Google Ads dynamic remarketing cannot build product audiences, (3) funnel analysis shows 0% at browse stage. Blueprint requires all 17 GA4 events — view_item is #2 after view_item_list.
