# Bug: Thiếu Cache Stampede Protection

## Vấn đề

Cache miss → nhiều requests cùng rebuild cache → thundering herd → DB overload.

**Ví dụ lỗi:**
```php
// ❌ SAI — no stampede protection
public function get_products( array $args ): array {
    $cache_key = 'wz_products_' . md5( serialize( $args ) );
    $cached    = wp_cache_get( $cache_key, 'wezone' );
    
    if ( false === $cached ) {
        // ← 1000 requests cùng vào đây khi cache miss!
        global $wpdb;
        $cached = $wpdb->get_results( "SELECT * FROM wz_products WHERE ..." );
        wp_cache_set( $cache_key, $cached, 'wezone', 3600 );
    }
    
    return $cached;
}
```

**Timeline khi cache expire:**
```
t=0: Cache expires
t=1: Request A → cache miss → query DB (takes 500ms)
t=2: Request B → cache miss → query DB (takes 500ms)
t=3: Request C → cache miss → query DB (takes 500ms)
...
t=100: Request Z → cache miss → query DB (takes 500ms)

Result: 100 identical DB queries trong 500ms!
```

**Hậu quả:**
- DB connection pool exhausted → 503 errors
- Query queue buildup → latency spike
- CPU/memory spike → cascade failure

## Root Cause

1. **Không có lock khi rebuild** → N requests cùng query DB
2. **Không có stale-while-revalidate** → phải chờ rebuild xong
3. **Không có probabilistic early expiration** → tất cả expire cùng lúc

## Fix

### 1. Lock-based stampede protection (recommended)

```php
// ✅ ĐÚNG — lock với wp_cache_add()
public function get_products( array $args ): array {
    $cache_key = 'wz_products_' . md5( serialize( $args ) );
    $lock_key  = $cache_key . '_lock';
    
    $cached = wp_cache_get( $cache_key, 'wezone' );
    
    if ( false === $cached ) {
        // Try to acquire rebuild lock (atomic)
        if ( wp_cache_add( $lock_key, 1, 'wezone', 10 ) ) {
            // We got the lock → rebuild cache
            global $wpdb;
            $cached = $wpdb->get_results( "SELECT * FROM wz_products WHERE ..." );
            wp_cache_set( $cache_key, $cached, 'wezone', 3600 );
            wp_cache_delete( $lock_key, 'wezone' );
        } else {
            // Another request is rebuilding → wait and retry
            usleep( 50000 ); // 50ms
            $cached = wp_cache_get( $cache_key, 'wezone' );
            
            if ( false === $cached ) {
                // Still not ready → fallback to DB (better than blocking)
                global $wpdb;
                $cached = $wpdb->get_results( "SELECT * FROM wz_products WHERE ..." );
                // Don't cache (let the lock holder do it)
            }
        }
    }
    
    return $cached;
}
```

### 2. Stale-while-revalidate pattern (best UX)

```php
// ✅ ĐÚNG — serve stale data while rebuilding
public function get_products( array $args ): array {
    $cache_key       = 'wz_products_' . md5( serialize( $args ) );
    $lock_key        = $cache_key . '_lock';
    $stale_threshold = 3600; // 1 hour fresh
    $stale_ttl       = 7200; // 2 hours stale
    
    $cached = wp_cache_get( $cache_key, 'wezone' );
    
    if ( false === $cached ) {
        // Hard miss → rebuild with lock
        if ( wp_cache_add( $lock_key, 1, 'wezone', 10 ) ) {
            global $wpdb;
            $data = $wpdb->get_results( "SELECT * FROM wz_products WHERE ..." );
            $cached = array(
                'data'      => $data,
                'timestamp' => time(),
            );
            wp_cache_set( $cache_key, $cached, 'wezone', $stale_ttl );
            wp_cache_delete( $lock_key, 'wezone' );
        } else {
            usleep( 50000 );
            $cached = wp_cache_get( $cache_key, 'wezone' );
        }
    }
    
    // Check if stale
    $age = time() - ( $cached['timestamp'] ?? 0 );
    if ( $age > $stale_threshold ) {
        // Stale → trigger async rebuild (don't block)
        if ( wp_cache_add( $lock_key, 1, 'wezone', 10 ) ) {
            // Spawn background rebuild
            wp_schedule_single_event( time(), 'wz_rebuild_cache', array( $cache_key, $args ) );
        }
        // Return stale data immediately (no blocking!)
    }
    
    return $cached['data'] ?? array();
}
```

### 3. Probabilistic early expiration (prevent simultaneous expiry)

```php
// ✅ ĐÚNG — xprob early expiration
public function get_products( array $args ): array {
    $cache_key = 'wz_products_' . md5( serialize( $args ) );
    $ttl       = 3600;
    $beta      = 1.0; // tuning parameter
    
    $cached = wp_cache_get( $cache_key, 'wezone' );
    
    if ( false !== $cached ) {
        $age = time() - ( $cached['timestamp'] ?? 0 );
        
        // Probabilistic early expiration (XFetch algorithm)
        $xfetch = $age - $ttl + $beta * log( mt_rand() / mt_getrandmax() );
        
        if ( $xfetch >= 0 ) {
            // Trigger early rebuild (probabilistic)
            $cached = false;
        }
    }
    
    if ( false === $cached ) {
        $lock_key = $cache_key . '_lock';
        if ( wp_cache_add( $lock_key, 1, 'wezone', 10 ) ) {
            global $wpdb;
            $data = $wpdb->get_results( "SELECT * FROM wz_products WHERE ..." );
            $cached = array(
                'data'      => $data,
                'timestamp' => time(),
            );
            wp_cache_set( $cache_key, $cached, 'wezone', $ttl );
            wp_cache_delete( $lock_key, 'wezone' );
        }
    }
    
    return $cached['data'] ?? array();
}
```
