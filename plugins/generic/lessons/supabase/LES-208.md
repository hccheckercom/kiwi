## Bad
```ts
const { data } = await supabase.from('orders').select('*')
// Returns ALL orders — 100K+ rows in production → OOM crash
```

## Good
```ts
const { data, count } = await supabase
  .from('orders')
  .select('*', { count: 'exact' })
  .range(page * pageSize, (page + 1) * pageSize - 1)
  .order('created_at', { ascending: false })
```

## Why
Supabase returns all matching rows by default. Without `.range()` or `.limit()`, a query on a large table loads everything into memory — crashing the serverless function. Always paginate list queries. `.single()` and `.eq('id', ...)` are safe because they return 1 row.
