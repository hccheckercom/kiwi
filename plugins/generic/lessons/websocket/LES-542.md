## Bad
```php
async def broadcast(self, message: dict):
    for connection in self.active_connections:
        try:
            await connection.send_json(message)
        except Exception as e:
            import sys
            print(f"WARNING: WebSocket send failed: {e}", file=sys.stderr)
            # Connection still in active_connections!
```

## Good
```php
async def broadcast(self, message: dict):
    dead_connections = []
    
    for connection in self.active_connections:
        try:
            await connection.send_json(message)
        except Exception as e:
            print(f"WARNING: WebSocket send failed: {e}", file=sys.stderr)
            dead_connections.append(connection)
    
    # Clean up dead connections
    for conn in dead_connections:
        self.disconnect(conn)
```

## Why
When send_json fails (client disconnected, network error), connection is dead but remains in active_connections list. Future broadcasts will keep trying to send to dead connection, wasting resources and filling logs. Collect failed connections and call disconnect() after broadcast loop.
