## Bad
```php
// AnalyticsCollector.php:69-75
if ( function_exists( 'wz_bulk_insert' ) && count( $rows ) > 1 ) {
    wz_bulk_insert( $table_events, $rows, $format );
} else {
    foreach ( $rows as $row ) {
        $wpdb->insert( $table_events, $row, $format );
    }
}
```

## Good
```php
// Always use wz_bulk_insert â€” it's a core Wezone function
if ( count( $rows ) > 1 ) {
    wz_bulk_insert( $table_events, $rows, $format );
} else {
    $wpdb->insert( $table_events, $rows[0], $format );
}

// Or fail fast if missing
if ( ! function_exists( 'wz_bulk_insert' ) ) {
    return new \WP_Error( 'missing_dependency', 'wz_bulk_insert() not available' );
}
wz_bulk_insert( $table_events, $rows, $format );
```

## Why
`wz_bulk_insert()` is a core Wezone function defined in `shared/src/Helpers.php`. Checking `function_exists()` and falling back to N+1 `$wpdb->insert()` loop suggests:
1. Missing dependency (should fail fast)
2. Defensive programming that masks real issues

N+1 insert loops cause severe performance degradation (50+ rows = 50 queries vs 1). If `wz_bulk_insert()` is missing, the plugin should error early, not silently degrade.
