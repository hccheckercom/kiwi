## Bad
```php
// header.php — no dataLayer initialization
<!DOCTYPE html>
<html>
<head>
  <?php wp_head(); ?>
</head>
```
Theme loads without dataLayer — all GA4 ecommerce events silently fail.

## Good
```php
// inc/tracking.php or header.php
<script>
window.dataLayer = window.dataLayer || [];
window.dataLayer.push({
  'pageType': '<?php echo wz_get_page_type(); ?>',
  'currencyCode': 'VND'
});
</script>
```

## Why
GA4 Enhanced Ecommerce requires `window.dataLayer` to exist before any `dataLayer.push()` call. Without initialization, all tracking events (view_item, add_to_cart, purchase) are lost silently — no JS error, just zero data in GA4 reports. GTM container also depends on dataLayer being initialized before its snippet loads.
