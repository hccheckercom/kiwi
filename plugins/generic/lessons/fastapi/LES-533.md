## Bad
```php
const manager = ConnectionManager()

@app.websocket("/ws")
async def websocket_endpoint(websocket: WebSocket):
    await manager.connect(websocket)
    try:
        while True:
            data = await websocket.receive_text()
            # Process data...
    except WebSocketDisconnect:
        manager.disconnect(websocket)
```

## Good
```php
const manager = ConnectionManager()

@app.websocket("/ws")
async def websocket_endpoint(websocket: WebSocket):
    try:
        await manager.connect(websocket)
        # Send initial connection success message
        await websocket.send_json({"type": "connected", "status": "ok"})
        
        while True:
            data = await websocket.receive_text()
            # Process data...
    except WebSocketDisconnect:
        manager.disconnect(websocket)
    except Exception as e:
        logger.error(f"WebSocket error: {e}")
        try:
            await websocket.close(code=1011, reason=str(e))
        except:
            pass
        manager.disconnect(websocket)
```

## Why
When WebSocket connection is accepted but no initial message is sent, client may see "WebSocket is closed before the connection is established" error. This happens because client doesn't know if connection succeeded. Solution: send confirmation message immediately after accept(), and add proper error handling for connection failures.
