# LES-242: PowerShell Set-Content UTF-8 BOM breaks PHP strict_types

## Symptom
- WordPress shows "There has been a critical error on this website."
- debug.log: `PHP Fatal error: strict_types declaration must be the very first statement in the script in functions.php on line 8`
- File looks correct in editor but PHP parser rejects it

## Root Cause
`Set-Content $file $content -Encoding UTF8` in **Windows PowerShell 5.1** writes a **UTF-8 BOM** (bytes `EF BB BF`) at the start of the file.

PHP requires `declare(strict_types=1)` to be the absolute first statement — even a BOM before `<?php` violates this rule, causing a Fatal Error.

## Wrong Pattern
```powershell
# SAI — thêm BOM vào đầu file
$content = Get-Content $file -Raw -Encoding UTF8
$content = $content -replace "old", "new"
Set-Content $file $content -Encoding UTF8   # ← BOM được thêm vào
```

## Correct Pattern
```powershell
# ĐÚNG — dùng [System.IO.File] để tránh BOM
$content = [System.IO.File]::ReadAllText($file, [System.Text.Encoding]::UTF8)
$content = $content -replace "old", "new"
$utf8NoBom = New-Object System.Text.UTF8Encoding $false
[System.IO.File]::WriteAllText($file, $content, $utf8NoBom)
```

## Fix khi đã bị BOM
```powershell
$file = "path\to\file.php"
$bytes = [System.IO.File]::ReadAllBytes($file)
if ($bytes[0] -eq 0xEF -and $bytes[1] -eq 0xBB -and $bytes[2] -eq 0xBF) {
    $bytes = $bytes[3..($bytes.Length-1)]
    [System.IO.File]::WriteAllBytes($file, $bytes)
    Write-Host "BOM removed"
}
```

## Scope
Ảnh hưởng tất cả PHP files có `declare(strict_types=1)`:
- `functions.php` của mọi theme
- Plugin entry point files
- Bất kỳ file PHP nào dùng strict types

## Scanner Check
```
Set-Content.*-Encoding UTF8
```
Grep trong PowerShell scripts — cảnh báo khi thấy pattern này trong scripts đang edit PHP files.
