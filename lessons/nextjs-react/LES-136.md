## Bad (Tailwind v3 pattern — broken in v4)
```css
@layer base {
  :root {
    --wz-primary: oklch(0.6 0.2 250);
    --color-accent: oklch(0.7 0.15 30);
    --font-sans: 'Inter', sans-serif;
  }
}
/* Tailwind v4 ignores @layer base for theme tokens — utilities won't resolve */
```

## Good (Tailwind v4 @theme directive)
```css
@import "tailwindcss";

@theme {
  --color-primary: oklch(0.6 0.2 250);
  --color-accent: oklch(0.7 0.15 30);
  --font-sans: 'Inter', sans-serif;
}
/* Now bg-primary, text-accent, font-sans all work as utilities */
```

## Key differences in Tailwind v4
| Feature | v3 | v4 |
|---------|----|----|
| Config | tailwind.config.js | CSS @theme block |
| Custom colors | theme.extend.colors | @theme { --color-* } |
| Fonts | theme.fontFamily | @theme { --font-* } |
| Import | @tailwind base/components/utilities | @import "tailwindcss" |
| Preflight | Separate config | Included by default |

## Why
webstore-vn uses Tailwind CSS v4. The old v3 pattern of defining tokens in @layer base still works for raw CSS variables, but Tailwind v4 utilities (bg-primary, text-accent) only resolve tokens defined in @theme blocks. This causes "class not found" in production builds while appearing to work in dev (where JIT is more permissive).
