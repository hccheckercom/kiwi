# Bug: Cart save() Dùng REPLACE — Race Condition

## Vấn đề

`$wpdb->replace()` = DELETE cũ + INSERT mới. Khi 2 request đồng thời (ví dụ: user click "Add to Cart" nhanh 2 lần):

1. Request A: DELETE old cart
2. Request B: DELETE old cart (nothing to delete)
3. Request A: INSERT cart with item 1
4. Request B: INSERT cart with item 2 → **overwrites** item 1

**Hậu quả:** Item bị mất ngẫu nhiên khi user thao tác nhanh.

## Fix

```php
// ✅ ĐÚNG — INSERT ON DUPLICATE KEY UPDATE
private function save( array $cart ): bool {
    global $wpdb;

    $key     = $this->session->getKey();
    $user_id = $this->session->getUserId();
    $table   = $wpdb->prefix . 'wz_carts';

    $result = $wpdb->query(
        $wpdb->prepare(
            "INSERT INTO {$table} (session_key, user_id, cart_data, expires_at, updated_at)
             VALUES (%s, %d, %s, %s, %s)
             ON DUPLICATE KEY UPDATE
               user_id = VALUES(user_id),
               cart_data = VALUES(cart_data),
               expires_at = VALUES(expires_at),
               updated_at = VALUES(updated_at)",
            $key,
            $user_id > 0 ? $user_id : 0,
            wp_json_encode( $cart ),
            gmdate( 'Y-m-d H:i:s', time() + 48 * HOUR_IN_SECONDS ),
            gmdate( 'Y-m-d H:i:s' )
        )
    );

    wp_cache_delete( 'wz_cart_' . $key, 'wezone_carts' );
    return $result !== false;
}
```
