## Bad

```python
result = ""
for item in items:
    result += f"{item.name}: {item.value}\n"
```

## Good

```python
result = "\n".join(f"{item.name}: {item.value}" for item in items)
```

## Why

String concatenation in a loop creates O(n²) intermediate strings because Python strings are immutable — each `+=` copies the entire string so far. `str.join()` pre-allocates the final buffer and is O(n). For small loops the difference is negligible, but it becomes significant at scale.
