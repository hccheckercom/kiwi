**Pattern sai:**
```php
// inc/account.php — KHÔNG có handler nào cho wz_change_password
// JS gọi action=wz_change_password nhưng PHP không register
// → fetch trả về 0 (WP default khi action không tồn tại)
```
**Phải làm:**
```php
add_action( 'wp_ajax_wz_change_password', 'wezone_funilux_ajax_change_password' );

function wezone_funilux_ajax_change_password(): void {
    check_ajax_referer( 'wz-change-password', 'wz_changepw_nonce' );

    if ( ! is_user_logged_in() ) {
        wp_send_json_error( [ 'message' => 'Vui lòng đăng nhập' ] );
    }

    $current = wp_unslash( $_POST['current_password'] ?? '' );
    $new     = wp_unslash( $_POST['new_password'] ?? '' );
    $confirm = wp_unslash( $_POST['confirm_password'] ?? '' );

    if ( ! $current || ! $new || ! $confirm ) {
        wp_send_json_error( [ 'message' => 'Vui lòng điền đầy đủ thông tin' ] );
    }
    if ( mb_strlen( $new ) < 8 ) {
        wp_send_json_error( [ 'message' => 'Mật khẩu mới phải có ít nhất 8 ký tự' ] );
    }
    if ( $new !== $confirm ) {
        wp_send_json_error( [ 'message' => 'Mật khẩu xác nhận không khớp' ] );
    }

    $user = wp_get_current_user();
    if ( ! wp_check_password( $current, $user->user_pass, $user->ID ) ) {
        wp_send_json_error( [ 'message' => 'Mật khẩu hiện tại không đúng' ] );
    }

    wp_set_password( $new, $user->ID );
    wp_send_json_success( [ 'message' => 'Đổi mật khẩu thành công' ] );
}
```
**Lý do:** WP trả về `0` khi `action` không được register qua `wp_ajax_*`. Form submit không báo lỗi rõ ràng nên khó debug. Mọi form dùng AJAX custom phải có handler tương ứng trong PHP — nonce, auth check, và logic xử lý.
**Grep check:** Tìm `action=wz_` trong JS files, đối chiếu với `add_action.*wp_ajax_` trong PHP — nếu thiếu cặp → CRITICAL
