**Triệu chứng:**
- Bộ lọc sidebar hiển thị "Chất liệu: Da thật, Da công nghiệp, Vải lưới, Nhựa, Gỗ, Kim loại" — không liên quan đến sản phẩm thật
- Bộ lọc không nhất quán với data admin — sản phẩm "Béc tưới" không có thuộc tính chất liệu nào trong số đó
- Filter checkbox click không có tác dụng vì data không match product attributes trong DB
**Root cause:**
- `sidebar-filters.php:67` và `mobile-filter-drawer.php:86` hardcode mảng PHP: `$materials = ['Da thật', 'Da công nghiệp', ...]`
- Không query `wz_product_meta` table để lấy thuộc tính thực tế của sản phẩm
**Pattern sai:**
```php
$materials = [ 'Da thật', 'Da công nghiệp', 'Vải lưới', 'Nhựa', 'Gỗ', 'Kim loại' ];
foreach ( $materials as $mat ) :
```
**Phải làm:**
```php
$cat_id_for_filter = isset( $category ) && ! empty( $category['id'] ) ? (int) $category['id'] : 0;
$filter_attrs = function_exists( 'wz_get_filter_attributes' ) ? wz_get_filter_attributes( $cat_id_for_filter ) : [];
foreach ( $filter_attrs as $attr_name => $attr_values ) :
```
**Lý do:** Bộ lọc phải phản ánh đúng thuộc tính sản phẩm từ database. Hardcode data tạo UX sai lệch — user click filter nhưng không có kết quả vì không product nào match.
**Grep check:** `\$materials\s*=\s*\[` hoặc `Da thật|Da công nghiệp` trong `template-parts/archive/` → nếu có → HIGH
**Ảnh hưởng:** Mọi theme clone từ funilux sẽ kế thừa mockup data sai — cần dùng `wz_get_filter_attributes()` helper
