## Bad
```
app/admin/
├── layout.tsx      (no error.tsx)
├── products/
│   └── page.tsx    (if this throws, entire admin crashes)
└── orders/
    └── page.tsx
```

## Good
```
app/admin/
├── layout.tsx
├── error.tsx       (catches errors in admin section)
├── products/
│   ├── page.tsx
│   └── error.tsx   (granular error boundary for products)
└── orders/
    └── page.tsx
```

## Why
Next.js App Router uses `error.tsx` as error boundaries. Without them, an unhandled error in any component bubbles up to the nearest boundary — potentially the root layout, crashing the entire app. Each major section needs its own error.tsx.
