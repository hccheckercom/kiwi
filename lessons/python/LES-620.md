## Bad

```python
try:
    result = process(data)
except Exception:
    pass
```

## Good

```python
try:
    result = process(data)
except (ValueError, KeyError) as e:
    logger.warning("Processing failed: %s", e)
    result = default_value
```

## Why

`except Exception:` catches `ValueError`, `TypeError`, `AttributeError`, `FileNotFoundError`, and dozens more. Without logging or re-raising, bugs hide silently. Catch the specific exceptions you expect, log unexpected ones, and let truly unexpected errors propagate.

## Grep

```bash
rg 'except\s+Exception\s*:' --type py -g '!test_*.py'
```
