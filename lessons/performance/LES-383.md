# Bug: wp_cache_flush() Xóa Toàn Bộ Redis Bao Gồm Sites Khác

## Vấn đề

`wp_cache_flush()` trên Redis → `FLUSHDB` → xóa **toàn bộ keys** trong Redis database, bao gồm:
- Cache của site khác (multisite)
- Session data
- Custom application cache
- Mọi thứ trong cùng Redis DB

**Ví dụ lỗi:**
```php
// ❌ SAI — xóa toàn bộ Redis
wp_cache_flush();

// ❌ SAI — trong purge_all()
public function purge_all(): array {
    wp_cache_flush(); // ← nuclear option!
}
```

**Hậu quả:**
- Multisite: site A purge → site B mất cache → tất cả sites slow
- Session data mất → users bị logout
- Cache stampede: 100% cache miss → DB overload

## Root Cause

1. `wp_cache_flush()` maps to Redis `FLUSHDB` — deletes ALL keys
2. WordPress không có concept "flush only my keys"
3. Redis Object Cache plugin dùng key prefix nhưng flush vẫn xóa hết

## Fix

### 1. Dùng wp_cache_flush_group() (WP 6.1+)

```php
// ✅ ĐÚNG — chỉ xóa group cụ thể
wp_cache_flush_group( 'wezone' );

// ✅ ĐÚNG — xóa nhiều groups
$groups = array( 'wezone', 'wezone-products', 'wezone-orders' );
foreach ( $groups as $group ) {
    wp_cache_flush_group( $group );
}
```

### 2. Dùng wp_cache_flush_runtime() (WP 6.0+)

```php
// ✅ ĐÚNG — chỉ xóa runtime cache (PHP memory), KHÔNG xóa Redis
wp_cache_flush_runtime();
```

### 3. Dùng selective delete thay flush

```php
// ✅ ĐÚNG — xóa theo pattern
private function purge_object_cache(): string {
    // Xóa specific keys thay vì flush all
    $keys = array(
        'wz_products_list',
        'wz_categories',
        'wz_settings',
    );
    
    foreach ( $keys as $key ) {
        wp_cache_delete( $key, 'wezone' );
    }
    
    return 'selective_cleared';
}
```

### 4. Nếu BẮT BUỘC phải flush — dùng Redis SELECT DB riêng

```php
// wp-config.php — tách WordPress cache vào DB riêng
define( 'WP_REDIS_DATABASE', 1 ); // DB 1 cho WordPress, DB 0 cho app khác
```

## Verification

```bash
# Grep wp_cache_flush calls
rg 'wp_cache_flush\s*\(' --type php

# Check nếu có group-based flush
rg 'wp_cache_flush_group' --type php
```
