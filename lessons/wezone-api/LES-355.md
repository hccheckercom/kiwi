## Bad
```php
$rows = $wpdb->get_results( $wpdb->prepare(
    "SELECT ri.id, ri.product_id, p.name AS product_name, p.image AS product_image
     FROM {$table} ri LEFT JOIN {$prods} p ON ri.product_id = p.id
     WHERE ri.request_id = %d",
    $request_id
) );
// p.image does not exist in wz_products table → always NULL
```

## Good
```php
$rows = $wpdb->get_results( $wpdb->prepare(
    "SELECT ri.id, ri.product_id, p.name AS product_name, p.image_id AS product_image_id
     FROM {$table} ri LEFT JOIN {$prods} p ON ri.product_id = p.id
     WHERE ri.request_id = %d",
    $request_id
) );
// Then convert image_id to URL: wp_get_attachment_url( $r['product_image_id'] )
```

## Why
The `wz_products` table has `image_id` column (INT, foreign key to wp_posts attachments), not `image` (string URL). The wrong column name returns NULL silently, making return request items show no product images.
