## Pattern

React SPA admin load danh sách entity (categories, attributes, tags...) một lần khi component mount. Nếu user cần tạo entity mới mà chưa có, phải mở tab khác → tạo → quay lại → F5 để reload.

## Bug

```tsx
// SAI — chỉ load 1 lần, không có cách tạo mới inline
useEffect(() => {
  wzApi.get('products/categories', { per_page: '100' }).then(res => {
    setAllCategories(buildCategoryTree(res));
  });
}, []);

// Không có nút "Tạo mới" → user phải F5 sau khi tạo ở tab khác
<MultiSelect options={allCategories} ... />
```

## Fix

Thêm nút "+ Tạo mới" inline, sau khi POST thành công gọi lại GET để reload state:

```tsx
const handleCreateCategory = async () => {
  await wzApi.post('products/categories', { name: newCatName.trim() });
  // Reload realtime — không cần F5
  const res = await wzApi.get<Category[]>('products/categories', { per_page: '100' });
  setAllCategories(buildCategoryTree(Array.isArray(res) ? res : (res as any).data ?? []));
  setShowNewCatModal(false);
  toast('success', `Đã tạo danh mục "${newCatName.trim()}"`);
};

// UI: nút inline bên cạnh label
<div className="flex items-center justify-between mb-1">
  <label>Danh mục</label>
  <button onClick={() => setShowNewCatModal(true)} className="text-xs text-wz-primary hover:underline">
    + Tạo mới
  </button>
</div>
```

## Áp dụng

Bất kỳ form nào trong admin SPA có dropdown/select chọn entity (categories, attributes, tags, labels...) mà entity đó có thể cần tạo mới trong luồng làm việc → phải có inline create + realtime reload.
