## Bad
```tsx
'use client'
import { createServerClient } from '@/lib/supabase/server'

export function UserProfile() {
  // This will fail at build time or leak service_role key to client bundle
}
```

## Good
```tsx
'use client'
import { createBrowserClient } from '@/lib/supabase/client'

export function UserProfile() {
  const supabase = createBrowserClient()
  // Uses anon key, safe for client
}
```

## Why
Server-only modules (using service_role keys, accessing env vars) must never be imported in 'use client' components. Next.js tree-shakes server code, but incorrect imports can leak secrets into the client bundle or cause build errors.
