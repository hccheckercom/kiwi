# Bug: wz_format_dimensions() Loose Coercion — String "0" Treated As Empty

## Vấn đề

`wz_format_dimensions()` dùng `! $l` để check empty. Nếu dimension là string "0" (từ DB hoặc API), `! "0"` = true → dimension bị skip dù là giá trị hợp lệ.

**Impact:** Edge case — sản phẩm có dimension = 0 (flat item) hiển thị sai.

## Bad

```php
// ❌ SAI — loose coercion
function wz_format_dimensions( array $dims ): string {
    $l = $dims['length'] ?? 0;
    $w = $dims['width'] ?? 0;
    $h = $dims['height'] ?? 0;

    if ( ! $l && ! $w && ! $h ) { // ← "0" string = truthy in ! check
        return '';
    }

    return "{$l} × {$w} × {$h} cm";
}
```

## Good

```php
// ✅ ĐÚNG — strict comparison
function wz_format_dimensions( array $dims ): string {
    $l = (float) ( $dims['length'] ?? 0 );
    $w = (float) ( $dims['width'] ?? 0 );
    $h = (float) ( $dims['height'] ?? 0 );

    if ( $l === 0.0 && $w === 0.0 && $h === 0.0 ) {
        return '';
    }

    return "{$l} × {$w} × {$h} cm";
}
```
