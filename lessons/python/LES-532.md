# Bad Code

```python
def get_user(email):
    # CRITICAL: SQL injection vulnerability
    query = f"SELECT * FROM users WHERE email = '{email}'"
    cursor.execute(query)
    return cursor.fetchone()

def search_products(keyword):
    # CRITICAL: SQL injection
    query = "SELECT * FROM products WHERE name LIKE '%" + keyword + "%'"
    cursor.execute(query)
    return cursor.fetchall()

def delete_user(user_id):
    # CRITICAL: SQL injection
    cursor.execute(f"DELETE FROM users WHERE id = {user_id}")
```

# Good Code

```python
def get_user(email):
    # GOOD: Parameterized query
    query = "SELECT * FROM users WHERE email = %s"
    cursor.execute(query, (email,))
    return cursor.fetchone()

def search_products(keyword):
    # GOOD: Parameterized query with LIKE
    query = "SELECT * FROM products WHERE name LIKE %s"
    cursor.execute(query, (f"%{keyword}%",))
    return cursor.fetchall()

def delete_user(user_id):
    # GOOD: Parameterized query
    cursor.execute("DELETE FROM users WHERE id = %s", (user_id,))

# Using ORM (best)
def get_user_orm(email):
    return session.query(User).filter(User.email == email).first()
```

# Why

**SQL injection attack:**
```python
# Attacker input
email = "admin@example.com' OR '1'='1"

# Bad code produces
query = "SELECT * FROM users WHERE email = 'admin@example.com' OR '1'='1'"
# Returns ALL users!

# Worse attack
email = "'; DROP TABLE users; --"
query = "SELECT * FROM users WHERE email = ''; DROP TABLE users; --'"
# Deletes entire table!
```

**Real-world impact:**
- Data breach (steal all data)
- Data loss (DROP TABLE)
- Privilege escalation (OR 1=1)
- Authentication bypass
- Remote code execution (xp_cmdshell)

**Always use parameterized queries:**
```python
# PostgreSQL (psycopg2)
cursor.execute("SELECT * FROM users WHERE id = %s", (user_id,))

# MySQL (mysql-connector)
cursor.execute("SELECT * FROM users WHERE id = %s", (user_id,))

# SQLite
cursor.execute("SELECT * FROM users WHERE id = ?", (user_id,))

# SQLAlchemy ORM
session.query(User).filter(User.id == user_id).first()
```

**Never concatenate user input:**
- No f-strings with SQL
- No % formatting with SQL
- No + concatenation with SQL
- No .format() with SQL
