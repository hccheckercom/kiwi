## Bad
```tsx
<span className="text-red-500">{order.status}</span>
<span className="text-green-500">{order.status}</span>
```
Status differentiated only by color — invisible to colorblind users.

## Good
```tsx
<span className="text-red-500 flex items-center gap-1">
  <XCircleIcon className="w-4 h-4" aria-hidden="true" />
  {order.status}
</span>
<span className="text-green-500 flex items-center gap-1">
  <CheckCircleIcon className="w-4 h-4" aria-hidden="true" />
  {order.status}
</span>
```
Icon + color provides redundant visual cue.

## Why
WCAG 1.4.1 — color must not be the only visual means of conveying information. ~8% of males are colorblind. Status indicators, error states, and success messages need icon or text differentiation beyond color.
