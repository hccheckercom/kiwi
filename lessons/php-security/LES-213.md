## Bad
```php
register_rest_route('wezone/v1', '/orders', [
    'methods'  => 'POST',
    'callback' => [ $this, 'create_order' ],
    'permission_callback' => '__return_true', // Anyone can create orders!
]);
```

## Good
```php
register_rest_route('wezone/v1', '/orders', [
    'methods'  => 'POST',
    'callback' => [ $this, 'create_order' ],
    'permission_callback' => function() {
        return current_user_can( 'manage_wezone_orders' );
    },
]);
```

## Why
`__return_true` as permission_callback means ANY visitor (including bots) can access the endpoint. For write operations (POST/PUT/DELETE), always check `current_user_can()`. Only use `__return_true` for genuinely public read-only endpoints (e.g., product catalog).
