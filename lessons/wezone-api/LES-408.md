# Repository sanitization pattern

## Problem
Repository create/update methods must sanitize all input data before DB operations. Missing sanitization allows malicious input to corrupt data or cause security issues.

## Pattern
1. Create private `sanitizeData()` method that handles all fields
2. Use appropriate sanitization for each field type:
   - Text fields → `sanitize_text_field()`
   - Textarea → `sanitize_textarea_field()`
   - HTML → `wp_kses_post()`
   - Slugs → `sanitize_title()`
   - Integers → `(int)` or `absint()`
   - Floats → `(float)`
   - Enums → `in_array()` whitelist check
3. Only allow known fields (whitelist approach)
4. Call `sanitizeData()` in both `create()` and `update()`

## Bad Examples

```php
// BAD: No sanitization
public function create( array $data ): int {
    global $wpdb;
    $wpdb->insert( $wpdb->prefix . 'wz_products', $data );
    return (int) $wpdb->insert_id;
}
```

**Issue**: Raw user input goes directly to database.

```php
// BAD: Partial sanitization
public function update( int $id, array $data ): bool {
    global $wpdb;
    if ( isset( $data['name'] ) ) {
        $data['name'] = sanitize_text_field( $data['name'] );
    }
    // Other fields not sanitized
    return $wpdb->update( $wpdb->prefix . 'wz_products', $data, array( 'id' => $id ) ) !== false;
}
```

**Issue**: Only sanitizes `name`, other fields like `description`, `price` are not sanitized.

## Good Examples

```php
// GOOD: Comprehensive sanitization method
private function sanitizeData( array $data ): array {
    $string_fields = array( 'sku', 'name', 'slug', 'type', 'status', 'stock_status' );
    $text_fields   = array( 'short_desc' );
    $html_fields   = array( 'description' );
    $int_fields    = array( 'stock_qty', 'image_id', 'manage_stock' );
    $float_fields  = array( 'price', 'regular_price', 'sale_price', 'weight' );
    
    $all_allowed = array_merge( $string_fields, $text_fields, $html_fields, $int_fields, $float_fields );
    
    $clean = array();
    foreach ( $all_allowed as $field ) {
        if ( ! array_key_exists( $field, $data ) ) {
            continue;
        }
        if ( in_array( $field, $string_fields, true ) ) {
            $clean[ $field ] = sanitize_text_field( (string) $data[ $field ] );
        } elseif ( in_array( $field, $text_fields, true ) ) {
            $clean[ $field ] = sanitize_textarea_field( (string) $data[ $field ] );
        } elseif ( in_array( $field, $html_fields, true ) ) {
            $clean[ $field ] = wp_kses_post( (string) $data[ $field ] );
        } elseif ( in_array( $field, $int_fields, true ) ) {
            $clean[ $field ] = $data[ $field ] !== null ? (int) $data[ $field ] : null;
        } elseif ( in_array( $field, $float_fields, true ) ) {
            $clean[ $field ] = (float) $data[ $field ];
        }
    }
    
    return $clean;
}

public function create( array $data ): int {
    global $wpdb;
    $insert = $this->sanitizeData( $data );
    $insert['created_at'] = gmdate( 'Y-m-d H:i:s' );
    $wpdb->insert( $wpdb->prefix . 'wz_products', $insert );
    return (int) $wpdb->insert_id;
}
```

```php
// GOOD: Enum validation with whitelist
private function sanitizeData( array $data ): array {
    $clean = array();
    
    if ( array_key_exists( 'stock_status', $data ) ) {
        $clean['stock_status'] = in_array( 
            $data['stock_status'], 
            array( 'instock', 'outofstock', 'onbackorder' ), 
            true 
        ) ? $data['stock_status'] : 'outofstock';
    }
    
    if ( array_key_exists( 'status', $data ) ) {
        $clean['status'] = in_array( 
            $data['status'], 
            array( 'publish', 'draft', 'private' ), 
            true 
        ) ? $data['status'] : 'draft';
    }
    
    return $clean;
}
```

## Detection Rules
- Repository `create()` or `update()` without calling `sanitizeData()`
- Direct `$wpdb->insert()` or `$wpdb->update()` with unsanitized `$data`
- Missing `sanitize_*()` calls for text/HTML fields
- Missing type casting for numeric fields
- Missing whitelist validation for enum fields

## Why This Matters
- **Security**: Prevents XSS, SQL injection, data corruption
- **Data integrity**: Ensures consistent data types in database
- **Validation**: Catches invalid enum values before DB errors
