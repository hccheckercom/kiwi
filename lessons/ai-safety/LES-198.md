## Bad
```ts
const response = await anthropic.messages.create({ ... })
const data = JSON.parse(response.content[0].text)
await supabase.from('products').update({ seo: data.meta }).eq('id', productId)
```
AI sometimes returns markdown-wrapped JSON, extra text, or malformed output → JSON.parse throws → unhandled error.

## Good
```ts
const response = await anthropic.messages.create({ ... })
const raw = response.content[0].text

// Extract JSON from potential markdown wrapping
const jsonMatch = raw.match(/```json?\s*([\s\S]*?)```/) || [null, raw]
const parsed = seoSchema.safeParse(JSON.parse(jsonMatch[1].trim()))

if (!parsed.success) {
  console.error('AI returned invalid structure', parsed.error)
  return null // Don't save malformed data
}
await supabase.from('products').update({ seo: parsed.data.meta }).eq('id', productId)
```

## Why
AI outputs are non-deterministic — even with JSON mode, responses can include markdown fences, extra whitespace, or missing fields. Always: (1) handle JSON extraction, (2) validate with a schema (Zod), (3) handle parse failures gracefully. Never trust AI output shape.
