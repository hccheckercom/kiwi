# Bug: AJAX Handler Without Nonce Verification

## Problem

Functions that handle AJAX requests (call `wp_send_json*` / `wp_die`) without `check_ajax_referer()` or `wp_verify_nonce()` are vulnerable to CSRF.

## AST Detection

Tree-sitter finds function/method bodies containing AJAX indicators (`wp_send_json`, `wp_die`) but missing nonce verification calls.

## Bad

```php
function handle_add_to_cart() {
    $product_id = absint( $_POST['product_id'] );
    wz_add_to_cart( $product_id );
    wp_send_json_success();
}
```

## Good

```php
function handle_add_to_cart() {
    check_ajax_referer( 'wz_cart', 'nonce' );
    $product_id = absint( $_POST['product_id'] );
    wz_add_to_cart( $product_id );
    wp_send_json_success();
}
```
