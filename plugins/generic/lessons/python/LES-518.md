# Bad Code

```python
class UserService:
    @staticmethod
    def validate_email(email):
        return '@' in email and '.' in email
    
    @staticmethod
    def hash_password(password):
        return hashlib.sha256(password.encode()).hexdigest()
    
    def create_user(self, email, password):
        if not self.validate_email(email):
            raise ValueError("Invalid email")
        hashed = self.hash_password(password)
        return User(email=email, password=hashed)
```

# Good Code

```python
# Option 1: Module-level functions (best for utilities)
def validate_email(email):
    return '@' in email and '.' in email

def hash_password(password):
    return hashlib.sha256(password.encode()).hexdigest()

class UserService:
    def create_user(self, email, password):
        if not validate_email(email):
            raise ValueError("Invalid email")
        hashed = hash_password(password)
        return User(email=email, password=hashed)

# Option 2: Keep as methods if they access class state
class UserService:
    def __init__(self, password_hasher):
        self.hasher = password_hasher
    
    def validate_email(self, email):
        # Could use self.email_validator in future
        return '@' in email and '.' in email
    
    def create_user(self, email, password):
        if not self.validate_email(email):
            raise ValueError("Invalid email")
        hashed = self.hasher.hash(password)
        return User(email=email, password=hashed)
```

# Why

**`@staticmethod` is rarely needed:**
- Doesn't access `self` or `cls`
- Can't be overridden meaningfully
- Adds namespace pollution
- Makes testing harder (can't mock easily)

**When to use `@staticmethod`:**
- Factory methods that don't need class state
- Grouping related utilities under a class namespace
- Framework requirements (Django, FastAPI)

**Better alternatives:**
- Module-level functions for pure utilities
- Regular methods if might need `self` later
- `@classmethod` for alternative constructors

**Code smell indicators:**
- All methods are `@staticmethod`
- Class has no `__init__`
- Methods don't share state
- → Should be module-level functions
