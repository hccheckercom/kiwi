## Problem

Deploy theme mới chỉ upload files lên VPS nhưng quên 1 trong 4 layers → theme "không thấy" hoặc "lỗi":

1. **VPS files** — theme code + symlink
2. **WP Multisite blog** — `wp site create` + `wp theme activate`
3. **Static HTML** — crawl homepage → `/opt/wezone-static/{slug}/index.html`
4. **Next.js listing** — thêm theme vào `THEMES[]` array trong `WordPressThemesPage.tsx`

Thiếu layer 1-2: 404 khi truy cập demo
Thiếu layer 3: mỗi request hit PHP (chậm, không scale)
Thiếu layer 4: theme không hiển thị trên trang `wezone.vn/wordpress`

## Correct Deploy Flow

```bash
# 1. Upload theme to VPS
rsync -avz ./themes/{category}/{slug}/ root@VPS:/opt/wezone-plugins/themes/{slug}/

# 2. Create symlink + register blog + activate theme
ssh root@VPS "ln -sfn /opt/wezone-plugins/themes/{slug} /var/www/wp.wezone.vn/wp-content/themes/{slug}"
ssh root@VPS "bash /opt/wezone-plugins/scripts/register-theme-demo.sh {slug}"

# 3. Crawl static HTML
ssh root@VPS "bash /opt/wezone-plugins/scripts/crawl-theme-static.sh {slug} {blog-slug}"

# 4. Update Next.js listing (WordPressThemesPage.tsx THEMES array)
# Then build + deploy Next.js
```

## Automated Script

`themes/scripts/deploy-theme.ps1 -Theme wezone-xxx` handles layers 1-3 automatically.
Layer 4 (Next.js listing) requires manual edit + Next.js redeploy.

## Scale Note

Static rendering (layer 3) enables 3000+ themes on single VPS:
- 3000 × 100KB = 300MB static files
- Nginx serve static = 10,000+ req/s
- No PHP/MySQL per request
- WP only needed for admin/editing
