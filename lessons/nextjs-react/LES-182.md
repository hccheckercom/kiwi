## Pattern sai

Using double-quoted here-strings `@"..."@` to embed bash scripts in PowerShell 5.1 deploy scripts:

```powershell
$remoteScript = @"
cp $DEPLOY_DIR/.env.production $ENV_SAFE/main.env 2>/dev/null || true
pm2 delete wezone 2>/dev/null || true
Write-Host "[ERROR] Failed" -ForegroundColor Red
"@
```

PowerShell 5.1 parses `$DEPLOY_DIR` as PS variable, `||` as invalid operator, and `[ERROR]` as array index → parse errors.

## Phải làm

Use single-quoted here-strings `@'...'@` — content is treated as literal text, no interpolation:

```powershell
$remoteScript = @'
cp $DEPLOY_DIR/.env.production $ENV_SAFE/main.env 2>/dev/null || true
pm2 delete wezone 2>/dev/null || true
'@

# For PS strings with brackets, escape or avoid:
Write-Host "Remote command failed (exit $exitCode)" -ForegroundColor Red
```

## Lý do

PowerShell 5.1 (Windows default) does NOT support `||` pipeline chain operator (added in PS7+). Double-quoted here-strings expand `$var` and parse `[text]` as array indexer. Any deploy script embedding bash via SSH must use `@'...'@` to prevent PS from parsing bash syntax.

## Grep check

```
@"[\s\S]*?\|\|[\s\S]*?"@
```

Matches double-quoted here-strings containing `||` — indicates bash embedded in PS double-quoted context.
