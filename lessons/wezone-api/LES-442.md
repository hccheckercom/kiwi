## Bad
```php
$wpdb->get_results( $wpdb->prepare(
    "SELECT q.*, p.post_title AS product_name
     FROM {$table} q LEFT JOIN {$wpdb->posts} p ON q.product_id = p.ID
     WHERE q.status = 'pending'",
) );
// wp_posts does NOT contain Wezone Commer products → product_name is always NULL
```

## Good
```php
$rows = $wpdb->get_results( $wpdb->prepare(
    "SELECT q.*, p.name AS product_name
     FROM {$table} q LEFT JOIN {$wpdb->prefix}wz_products p ON q.product_id = p.id
     WHERE q.status = 'pending'",
) );
// OR: fetch product name separately via wz_get_product()
```

## Why
Wezone Commer stores products in `wz_products` table (custom), NOT `wp_posts`. Joining on `wp_posts` returns NULL for product_name because product CPT no longer exists in WP posts table. This affects admin display — QA pending list shows blank product names.
