## Problem

WeZone Router (`Router.php`) load template qua `TemplateLoader` — nó set data qua `set_query_var('wz_tpl_{key}', $value)` nhưng **không set global `$post`**. Nếu template gọi `get_the_ID()`, hàm này trả về 0 vì không có WP post context.

```php
// SAI — get_the_ID() trả 0, wz_get_product(0) trả null
$product_id = get_the_ID();
$wz_product = wz_get_product( $product_id ); // null → "Sản phẩm không tồn tại"
```

## Correct Pattern

Đọc data từ query var mà Router đã set:

```php
// ĐÚNG — Router đã fetch product, chỉ cần đọc query var
$wz_product = get_query_var( 'wz_tpl_product', null );
$product_id = is_array( $wz_product ) ? (int) ( $wz_product['id'] ?? 0 ) : 0;
```

## Key Mapping

| `$loader->set(key, value)` | Query var |
|---|---|
| `set('product', $product)` | `wz_tpl_product` |
| `set('category', $category)` | `wz_tpl_category` |
| `set('section', $section)` | `wz_tpl_section` |
| `set('order_id', $id)` | `wz_tpl_order_id` |

## Why

WeZone dùng custom table `wp_wz_products` — không có WP post tương ứng. Router fetch product bằng `wz_get_product_by_slug()` rồi pass qua query var. Template không cần fetch lại.

## Grep Check

`get_the_ID` trong `wezone-templates/` → kiểm tra xem có dùng để fetch wz_product không → HIGH
