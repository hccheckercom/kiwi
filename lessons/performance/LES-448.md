# Bug: Transient Without Invalidation Hook

## Problem

Caching results with `set_transient()` but not registering hooks to invalidate when underlying data changes. Users see stale scores/data until the transient expires naturally.

**Example (BAD):**
```php
// Cache score for 10 minutes
set_transient('wz_score_result', $result, 10 * MINUTE_IN_SECONDS);
// But NO hook registered for:
// - Product added/deleted → SEO/image score changes
// - Plugin activated/deactivated → backup score changes
// - SSL status change → SSL score changes
// → User adds products, score stays at 0/20 for 10 minutes
```

## Fix

Register invalidation hooks for all data sources that affect the cached value:

```php
// In Plugin::boot()
add_action('save_post_wz_product', [$this, 'invalidate_score']);
add_action('delete_post',          [$this, 'invalidate_score']);
add_action('activated_plugin',     [$this, 'invalidate_score']);
add_action('deactivated_plugin',   [$this, 'invalidate_score']);
add_action('update_option_wezone-analytics_enabled', [$this, 'invalidate_score']);

public function invalidate_score(): void {
    delete_transient('wz_score_result');
}
```

## Detection

Look for `set_transient()` calls and verify that corresponding `delete_transient()` is called from relevant WordPress hooks.

## Impact

- User experience: changes reflected immediately instead of delayed 10 minutes
- Trust: admin dashboard shows accurate store health at all times
