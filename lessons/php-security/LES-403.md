# Bug: wz_add_notice() Không Whitelist $type — Stored XSS Potential

## Vấn đề

`wz_add_notice()` chấp nhận bất kỳ string nào cho `$type` parameter. Nếu rendering code dùng `$type` trong class name hoặc attribute mà không escape → stored XSS.

**Impact:** XSS nếu rendering code không escape `$type`.

## Bad

```php
// ❌ SAI — không validate $type
function wz_add_notice( string $message, string $type = 'info' ): void {
    $key       = wz_notices_key();
    $notices   = get_transient( $key ) ?: array();
    $notices[] = array(
        'message' => $message,
        'type'    => $type, // ← attacker truyền 'info" onclick="alert(1)'
    );
    set_transient( $key, $notices, 60 );
}
```

## Good

```php
// ✅ ĐÚNG — whitelist $type
function wz_add_notice( string $message, string $type = 'info' ): void {
    $allowed_types = array( 'info', 'success', 'warning', 'error' );
    if ( ! in_array( $type, $allowed_types, true ) ) {
        $type = 'info';
    }
    
    $key       = wz_notices_key();
    $notices   = get_transient( $key ) ?: array();
    $notices[] = array(
        'message' => $message,
        'type'    => $type,
    );
    set_transient( $key, $notices, 60 );
}
```
