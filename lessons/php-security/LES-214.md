## Bad
```php
$results = $wpdb->get_results(
    "SELECT * FROM {$wpdb->prefix}wz_orders WHERE customer_id = {$customer_id}"
);
```
`$customer_id` injected directly into SQL — SQL injection if user-controlled.

## Good
```php
$results = $wpdb->get_results(
    $wpdb->prepare(
        "SELECT * FROM {$wpdb->prefix}wz_orders WHERE customer_id = %d",
        $customer_id
    )
);
```

## Why
Direct variable interpolation in SQL queries is the #1 WordPress security vulnerability. Always use `$wpdb->prepare()` with type placeholders (%d, %s, %f). DDL statements (CREATE TABLE, ALTER TABLE) are exceptions — they don't take user input.
