**Pattern sai:**
```php
// Database.php — create_table_raw()
foreach ( $indexes as $idx_name => $idx_columns ) {
    $sql .= ",\nUNIQUE KEY {$idx_name} ({$idx_columns})";
}
// dbDelta() cố ADD KEY đã tồn tại → "Duplicate key name" error x100 mỗi WP-CLI command
```
**Phải làm:**
```php
$existing_indexes = array();
if ( $wpdb->get_var( $wpdb->prepare( 'SHOW TABLES LIKE %s', $full_table ) ) === $full_table ) {
    $rows = $wpdb->get_results( "SHOW INDEX FROM `{$full_table}`", ARRAY_A );
    if ( is_array( $rows ) ) {
        foreach ( $rows as $row ) {
            $existing_indexes[ $row['Key_name'] ] = true;
        }
    }
}
foreach ( $indexes as $idx_name => $idx_columns ) {
    if ( ! isset( $existing_indexes[ $real_idx_name ] ) ) {
        $sql .= ",\nUNIQUE KEY {$real_idx_name} ({$idx_columns})";
    }
}
```
**Lý do:** WordPress `dbDelta()` không handle UNIQUE KEY idempotently — nếu index đã tồn tại, nó vẫn cố ALTER TABLE ADD → MySQL error "Duplicate key name". Khi nhiều plugins gọi `force_migrate()` (analytics, loyalty, migrator), mỗi lần WP-CLI chạy sẽ spam hàng trăm dòng error, che mất lỗi thật trong deploy log. Fix: check `SHOW INDEX` trước khi include index trong CREATE TABLE SQL.
