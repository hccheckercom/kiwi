## Bad

```python
def scan_file(path):
    print(f"Scanning {path}...")
    if errors:
        print(f"Found {len(errors)} errors")
    print("Done!")
```

## Good

```python
import logging
logger = logging.getLogger(__name__)

def scan_file(path):
    logger.info("Scanning %s", path)
    if errors:
        logger.warning("Found %d errors", len(errors))
    logger.debug("Scan complete")
```

## Why

`print()` goes to stdout unconditionally — no log levels, no filtering, no structured output. In libraries and modules (not CLI entry points), use `logging` so consumers can control verbosity. CLI tools and scripts are exempt from this rule.
