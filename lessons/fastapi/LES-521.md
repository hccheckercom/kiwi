# Bad Code

```python
from fastapi import FastAPI
from database import get_db_connection

app = FastAPI()

@app.get("/users/{user_id}")
async def get_user(user_id: int):
    # BAD: Direct database access, hard to test
    db = get_db_connection()
    user = db.query(User).filter(User.id == user_id).first()
    return user

@app.post("/users")
async def create_user(name: str, email: str):
    # BAD: Direct service instantiation
    service = UserService()
    return service.create_user(name, email)
```

# Good Code

```python
from fastapi import FastAPI, Depends
from typing import Annotated
from database import get_db
from services import UserService

app = FastAPI()

# Dependency functions
def get_user_service(db: Annotated[Session, Depends(get_db)]) -> UserService:
    return UserService(db)

@app.get("/users/{user_id}")
async def get_user(
    user_id: int,
    db: Annotated[Session, Depends(get_db)]
):
    user = db.query(User).filter(User.id == user_id).first()
    if not user:
        raise HTTPException(status_code=404, detail="User not found")
    return user

@app.post("/users")
async def create_user(
    name: str,
    email: str,
    service: Annotated[UserService, Depends(get_user_service)]
):
    return service.create_user(name, email)
```

# Why

**Dependency Injection benefits:**
- **Testability:** Mock dependencies in tests
- **Flexibility:** Swap implementations (dev/prod databases)
- **Reusability:** Share dependencies across endpoints
- **Lifecycle management:** FastAPI handles cleanup automatically

**Without DI:**
- Hard to test (can't mock database)
- Tight coupling to implementations
- Resource leaks (connections not closed)
- Duplicate setup code

**FastAPI DI features:**
```python
# Dependency with cleanup
async def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

# Nested dependencies
def get_current_user(token: str = Depends(oauth2_scheme)):
    return decode_token(token)

def get_admin_user(user: User = Depends(get_current_user)):
    if not user.is_admin:
        raise HTTPException(403)
    return user

# Class-based dependencies
class Pagination:
    def __init__(self, skip: int = 0, limit: int = 100):
        self.skip = skip
        self.limit = limit
```

**Testing with DI:**
```python
def test_get_user():
    def mock_db():
        return MockSession()
    
    app.dependency_overrides[get_db] = mock_db
    response = client.get("/users/1")
    assert response.status_code == 200
```
