## Bad
```php
// 'product' is NOT a valid twitter:card value
<meta name="twitter:card" content="<?php echo $is_product ? 'product' : 'summary_large_image'; ?>">
```

## Good
```php
// Valid values: summary, summary_large_image, app, player
// For e-commerce, always use summary_large_image for maximum image visibility
<meta name="twitter:card" content="summary_large_image">
```

## Why
Twitter/X only supports 4 card types: `summary`, `summary_large_image`, `app`, `player`. The value `product` is invalid and will cause Twitter to fall back to `summary` (small image). Blueprint `seo.md` specifies `summary_large_image` for all pages. Using the correct value ensures product images display at full width when shared on Twitter/X.
