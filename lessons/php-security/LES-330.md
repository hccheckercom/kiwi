**Pattern sai:**
```php
// change_password() — không check empty trước wp_check_password()
$current_password = $request->get_param('current_password');
if ( ! wp_check_password( $current_password, $user->user_pass, $user->ID ) ) {
    return new WP_REST_Response( array( 'error' => 'Wrong password' ), 400 );
}
```
**Phải làm:**
```php
$current_password = trim( (string) $request->get_param('current_password') );
if ( '' === $current_password ) {
    return new WP_REST_Response( array( 'error' => 'Current password required' ), 400 );
}
if ( ! wp_check_password( $current_password, $user->user_pass, $user->ID ) ) {
    return new WP_REST_Response( array( 'error' => 'Wrong password' ), 400 );
}
```
**Lý do:** Empty/null password có thể bypass hash check trong một số edge cases — phải reject trước khi gọi wp_check_password()
**Grep check:** `if ( '' === $current_password )` hoặc `if ( empty($current_password) )` — thiếu trong REST API password endpoint → CRITICAL
