## Bad
```ts
export async function POST(req: Request) {
  const response = await anthropic.messages.create({
    model: 'claude-sonnet-4-6',
    max_tokens: 2048,
    messages: [{ role: 'user', content: prompt }]
  })
  return Response.json({ text: response.content[0].text })
}
```
If AI provider is slow or down, request hangs until Vercel/server timeout (60s+). User sees spinner forever.

## Good
```ts
export async function POST(req: Request) {
  const controller = new AbortController()
  const timeoutId = setTimeout(() => controller.abort(), 30000) // 30s max

  try {
    const response = await anthropic.messages.create({
      model: 'claude-sonnet-4-6',
      max_tokens: 2048,
      messages: [{ role: 'user', content: prompt }]
    }, { signal: controller.signal })

    return Response.json({ text: response.content[0].text })
  } catch (err) {
    if (err.name === 'AbortError') {
      return Response.json({ error: 'AI request timed out, please try again' }, { status: 504 })
    }
    throw err
  } finally {
    clearTimeout(timeoutId)
  }
}
```

## Why
AI providers have variable latency (2-60s). Without explicit timeouts, slow responses block the event loop, consume server resources, and leave users waiting. Set timeouts shorter than your platform's request timeout to return a useful error message.
