## Bad
```ts
const search = searchParams.get('search')
// User sends search="%%" → matches ALL rows (wildcard)
// User sends search="___" → matches any 3-char value
query = query.or(`customer_name.ilike.%${search}%`)
```

## Good (sanitize pattern from orders.handlers.ts)
```ts
const rawSearch = searchParams.get('search')
// Truncate + strip SQL wildcard chars
const search = rawSearch ? rawSearch.slice(0, 100).replace(/[%_\\]/g, '') : null

if (search) {
  query = query.or(`order_code.ilike.%${search}%,customer_name.ilike.%${search}%`)
}
```

## Why
Real pattern from orders.handlers.ts. Supabase .ilike() passes the pattern to PostgreSQL LIKE. Characters % and _ are wildcards — unsanitized user input can match all rows (data leak) or cause slow full-table scans (DoS). Always strip wildcards and truncate length.
