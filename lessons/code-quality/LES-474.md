## Bad

```php
// Fatal error: PHP constant names cannot contain hyphens
define( 'THEME-NAME_VERSION', '1.0.0' );
define( 'MY-PLUGIN_DIR', __DIR__ );

// Also fails in usage
require_once THEME-NAME_DIR . '/inc/helpers.php';
```

## Good

```php
// Use underscores for PHP constant names
define( 'THEME_NAME_VERSION', '1.0.0' );
define( 'MY_PLUGIN_DIR', __DIR__ );

// Valid usage
require_once THEME_NAME_DIR . '/inc/helpers.php';
```

## Why

PHP constant names must follow identifier rules: letters, numbers, and underscores only. Hyphens are treated as minus operators, causing parse errors.

This commonly occurs when converting slugs (which use hyphens) to constant names without replacing hyphens with underscores.

**Impact:** Fatal error on theme/plugin activation, site crashes.

## Grep

```bash
# Find constant definitions with hyphens
grep -rn "define\s*(\s*['\"][A-Z0-9_]*-[A-Z0-9_-]*['\"]" --include="*.php"

# Find constant usage with hyphens (less reliable due to false positives)
grep -rn "[A-Z][A-Z0-9_]*-[A-Z0-9_-]*_[A-Z]" --include="*.php"
```

## Pattern

```regex
define\s*\(\s*['"]([A-Z0-9_]*-[A-Z0-9_-]*)['"]
```

## Auto-fix

Replace hyphens with underscores in constant names:

```php
// Before
define( 'KIWI-GENERATOR_VERSION', '1.0.0' );

// After
define( 'KIWI_GENERATOR_VERSION', '1.0.0' );
```
