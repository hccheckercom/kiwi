## Bad
```ts
// Step 1: Create order
const { data: order } = await supabase.from('orders').insert({ customer_id, total }).select().single()
// Step 2: Create items
await supabase.from('order_items').insert(items.map(i => ({ order_id: order.id, ...i })))
// Step 3: Decrement stock
for (const item of items) {
  await supabase.from('products').update({ stock: item.stock - item.qty }).eq('id', item.id)
}
```
If step 2 or 3 fails, order exists without items or with wrong stock. No rollback.

## Good
```ts
const { data, error } = await supabase.rpc('create_order', {
  p_customer_id: customerId,
  p_items: JSON.stringify(items),
  p_total: total
})
// RPC wraps all 3 operations in a single transaction — all succeed or all fail
```

## Why
Checkout is a multi-table operation: orders + order_items + inventory + coupons. Without a database transaction, partial failures leave data inconsistent — orders without items, stock decremented for failed orders, coupons used but order not created.
