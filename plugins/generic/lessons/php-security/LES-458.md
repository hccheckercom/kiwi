# Bug: Unescaped Variable in echo/print — XSS

## Problem

Echoing a PHP variable without escaping allows XSS injection.

## AST Detection

Tree-sitter parses `echo_statement` nodes, checks if they contain `$variable` references without wrapping in escape functions (`esc_html`, `esc_attr`, `esc_url`, `wp_kses_post`, etc.).

## Bad

```php
echo $user_name;
echo '<a href="' . $url . '">';
```

## Good

```php
echo esc_html( $user_name );
echo '<a href="' . esc_url( $url ) . '">';
```
