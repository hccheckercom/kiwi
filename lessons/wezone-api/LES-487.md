## Bad
```php
function wezoneAcceptCookies(level) {
    document.cookie = 'wezone_cookie_consent=' + level + ';expires=...';
    // No nonce check â†’ CSRF: <img src="javascript:wezoneAcceptCookies('marketing')">
}
```

## Good
```php
// Option 1: AJAX with nonce
function wezoneAcceptCookies(level) {
    fetch('/wp-json/wezone/v1/consent', {
        method: 'POST',
        headers: { 'X-WP-Nonce': wezone_ajax.nonce },
        body: JSON.stringify({ level: level })
    });
}

// Option 2: Form POST
<form method="POST">
    <?php wp_nonce_field( 'wezone_consent' ); ?>
    <button name="consent" value="marketing">Accept</button>
</form>
```

## Why
Inline JS setting consent cookie without nonce validation allows CSRF â€” attacker can trick user into accepting tracking via malicious link. Use AJAX with nonce or form POST with wp_nonce_field().

## Grep
`document\.cookie\s*=.*consent.*(?!nonce)` — in `**/*.php`
