## Bad
```ts
export async function GET() {
  const { data } = await supabase.from('categories').select('*')
  return Response.json({ data })
}
// Every request queries the database — categories rarely change
```

## Good
```ts
import { CACHE_LONG } from '@/lib/api-cache'

export async function GET() {
  const { data } = await supabase.from('categories').select('*')
  return Response.json({ data }, {
    headers: { 'Cache-Control': CACHE_LONG } // s-maxage=3600
  })
}
```

## Why
Public GET endpoints serving static-ish data (categories, settings, navigation) should be cached. Without Cache-Control, every page load triggers a database query. Use CACHE_LONG for rarely changing data, CACHE_SHORT for frequently updated data.
