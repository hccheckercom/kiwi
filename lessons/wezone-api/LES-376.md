# Bug: resolvePrice() Trả 0.0 Khi Product Không Tồn Tại — Free Item

## Vấn đề

`resolvePrice()` query DB cho product price. Nếu product_id không tồn tại hoặc bị xóa → `$wpdb->get_var()` trả `null` → cast `(float)` = `0.0` → item thêm vào cart với giá 0.

**Ví dụ lỗi:**
```php
// ❌ SAI — không validate price > 0
private function resolvePrice( int $product_id, int $variation_id ): float {
    // ...
    $price = (float) $wpdb->get_var( ... ); // null → 0.0

    if ( $price === 0.0 ) {
        // Fallback to product price — cũng có thể = 0.0!
        $price = (float) $wpdb->get_var( ... );
    }

    return (float) apply_filters( Hooks::PRODUCT_PRICE, $price, $product_id );
    // → 0.0 if product doesn't exist
}
```

## Fix

```php
// ✅ ĐÚNG — validate price > 0, throw nếu product không tồn tại
private function resolvePrice( int $product_id, int $variation_id ): float {
    global $wpdb;
    $price = 0.0;

    if ( $variation_id > 0 ) {
        $price = (float) $wpdb->get_var( ... );
    }

    if ( $price <= 0.0 ) {
        $result = $wpdb->get_var( ... );
        if ( $result === null ) {
            throw new \RuntimeException( "Product {$product_id} not found" );
        }
        $price = (float) $result;
    }

    if ( $price <= 0.0 ) {
        throw new \RuntimeException( "Product {$product_id} has invalid price" );
    }

    return (float) apply_filters( Hooks::PRODUCT_PRICE, $price, $product_id );
}
```

Và trong `add()`:
```php
try {
    $price = $this->resolvePrice( $product_id, $variation_id );
} catch ( \RuntimeException $e ) {
    wz_log( 'CartEngine::add() — ' . $e->getMessage(), 'error' );
    return '';
}
```
