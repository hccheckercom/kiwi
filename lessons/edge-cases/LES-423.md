# Bug Pattern

Using `current_time('mysql')` for time-sensitive features (flash sales, countdowns) without considering timezone mismatches between server and client.

**Why:** Server time may differ from user's local time, causing sales to appear active/ended at wrong times in client-side countdowns.

## Detection

```regex
current_time\(\s*['"]mysql['"][\s\S]{0,500}(?:flash[_-]?sale|countdown|timer|schedule|start[_-]?time|end[_-]?time)
```

Match `current_time('mysql')` usage in time-sensitive features without timezone conversion.

## Examples

### ❌ BAD: Server time without timezone context

```php
public function get_active_sales(): array {
    global $wpdb;
    $table = $wpdb->prefix . 'wezone_flash_sales';
    $now   = current_time( 'mysql' ); // Server timezone, not user timezone!
    
    $sales = $wpdb->get_results(
        $wpdb->prepare(
            "SELECT * FROM {$table} WHERE status = 'active' AND start_time <= %s AND end_time >= %s",
            $now,
            $now
        ),
        ARRAY_A
    );
    
    return $sales;
}

// Frontend JS countdown
wp_localize_script( 'flash-sale', 'flashSaleData', array(
    'endTime' => $sale['end_time'], // MySQL datetime string, no timezone info!
) );
```

**Problem:**
- Server in UTC, user in GMT+7 → countdown shows wrong time
- User sees "Sale ends in 7 hours" but sale already ended on server

### ✅ GOOD: Use Unix timestamps for client-side

```php
public function get_active_sales(): array {
    global $wpdb;
    $table = $wpdb->prefix . 'wezone_flash_sales';
    $now   = current_time( 'mysql' );
    
    $sales = $wpdb->get_results(
        $wpdb->prepare(
            "SELECT * FROM {$table} WHERE status = 'active' AND start_time <= %s AND end_time >= %s",
            $now,
            $now
        ),
        ARRAY_A
    );
    
    // Convert MySQL datetime to Unix timestamp for client-side
    foreach ( $sales as &$sale ) {
        $sale['start_timestamp'] = strtotime( $sale['start_time'] );
        $sale['end_timestamp']   = strtotime( $sale['end_time'] );
        $sale['server_time']     = time(); // Current Unix timestamp
    }
    unset( $sale );
    
    return $sales;
}

// Frontend JS countdown
wp_localize_script( 'flash-sale', 'flashSaleData', array(
    'endTimestamp' => $sale['end_timestamp'], // Unix timestamp, timezone-agnostic
    'serverTime'   => time(), // For client-server time sync
) );
```

**Frontend JS:**
```javascript
// Calculate time difference using Unix timestamps
const serverTime = flashSaleData.serverTime * 1000; // Convert to milliseconds
const clientTime = Date.now();
const timeDiff = clientTime - serverTime; // Client-server time offset

const endTime = flashSaleData.endTimestamp * 1000;
const timeLeft = endTime - Date.now() + timeDiff; // Adjust for server-client offset

if (timeLeft > 0) {
    // Show countdown
} else {
    // Sale ended
}
```

### ✅ BETTER: Provide timezone info + server time API

```php
// API endpoint for server time sync
public function get_server_time( \WP_REST_Request $request ): \WP_REST_Response {
    return new \WP_REST_Response(
        array(
            'timestamp' => time(),
            'datetime'  => current_time( 'mysql' ),
            'timezone'  => wp_timezone_string(), // e.g., 'Asia/Ho_Chi_Minh'
            'offset'    => get_option( 'gmt_offset' ) * 3600, // Seconds from UTC
        )
    );
}

// Frontend: Sync with server time every 5 minutes
async function syncServerTime() {
    const response = await fetch('/wp-json/wezone/v1/flash-sale/server-time');
    const data = await response.json();
    
    // Calculate client-server time difference
    const serverTime = data.timestamp * 1000;
    const clientTime = Date.now();
    window.timeOffset = clientTime - serverTime;
    
    console.log(`Time offset: ${window.timeOffset}ms`);
}

// Sync on page load and every 5 minutes
syncServerTime();
setInterval(syncServerTime, 5 * 60 * 1000);

// Use offset in countdown
function updateCountdown() {
    const now = Date.now() - (window.timeOffset || 0); // Adjust for server time
    const endTime = flashSaleData.endTimestamp * 1000;
    const timeLeft = endTime - now;
    
    if (timeLeft > 0) {
        // Show countdown
    } else {
        // Sale ended, refresh data
        location.reload();
    }
}
```

## Fix Strategy

1. **Store times as Unix timestamps** in API responses
2. **Provide server time API** for client-server sync
3. **Calculate time offset** between client and server
4. **Use offset in countdowns** to show accurate time remaining
5. **Refresh data** when countdown reaches zero (don't trust client time)
6. **Test across timezones** using VPN or browser timezone override
