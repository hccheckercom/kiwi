## Pattern sai

```css
/* Position variants applied globally — breaks mobile */
.haven-hero__visual--products-right .haven-hero__products { left: auto; right: 16px; max-width: 50%; }
.haven-hero__visual--products-left .haven-hero__products { right: auto; left: 16px; max-width: 50%; }
.haven-hero__visual--products-bottom .haven-hero__products { left: 16px; right: 16px; max-width: 100%; }
```

## Pattern đúng

```css
/* Position variants — desktop only (mobile always full-width bottom strip) */
@media (min-width: 768px) {
    .haven-hero__visual--products-right .haven-hero__products { left: auto; right: 16px; max-width: 50%; }
    .haven-hero__visual--products-left .haven-hero__products { right: auto; left: 16px; max-width: 50%; }
    .haven-hero__visual--products-bottom .haven-hero__products { left: 16px; right: 16px; max-width: 100%; }
}
```

## Lý do

Position variant modifiers (right/left/bottom/overlay) are desktop layout choices. On mobile, elements like product strips must always be full-width (`left: 0; right: 0`). Applying `max-width: 50%` or `left: auto` globally overrides the mobile-first base styles, causing:
1. Products strip not fullwidth on mobile
2. Body background visible as gap (horizontal overflow illusion)
3. Content clipped or misaligned

## Grep check

Detect variant modifier selectors outside `@media` blocks. Manual review needed — check that all `--products-{position}` selectors are inside `@media (min-width: 768px)`.
