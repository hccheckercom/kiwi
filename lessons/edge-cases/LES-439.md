## Problem

Plugin constants like `WEZONE_POPUP_URL` are defined in the entry point file (e.g., `wezone-popup.php`). If the class is loaded without the entry point (autoloader, unit tests, or require from another plugin), the undefined constant triggers a PHP warning and `constant()` returns the constant name as string — producing broken URLs and version strings.

## Bad

```php
wp_enqueue_style(
    'wezone-popup',
    WEZONE_POPUP_URL . 'assets/css/popup.css', // PHP warning if undefined
    array(),
    WEZONE_POPUP_VERSION // Returns "WEZONE_POPUP_VERSION" string
);
```

## Good

```php
wp_enqueue_style(
    'wezone-popup',
    ( defined( 'WEZONE_POPUP_URL' ) ? WEZONE_POPUP_URL : '' ) . 'assets/css/popup.css',
    array(),
    defined( 'WEZONE_POPUP_VERSION' ) ? WEZONE_POPUP_VERSION : '1.0.0'
);
```

## Apply

- Any file using `WEZONE_*_URL` or `WEZONE_*_VERSION` outside the defining file MUST guard with `defined()`
- Alternatively, provide a safe fallback method in the Plugin class
