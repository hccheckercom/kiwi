## Pattern

Two admins open same product/order simultaneously. Both edit, both save. Last write wins silently — first admin's changes lost without warning. No version check or updated_at comparison.

## Fix

```php
public function update_product( WP_REST_Request $request ): WP_REST_Response {
    global $wpdb;
    $id         = (int) $request->get_param( 'id' );
    $updated_at = sanitize_text_field( $request->get_param( 'updated_at' ) );

    // Optimistic lock: chỉ update nếu updated_at khớp
    $affected = $wpdb->query( $wpdb->prepare(
        "UPDATE {$wpdb->prefix}wz_products
         SET name = %s, updated_at = NOW()
         WHERE id = %d AND updated_at = %s",
        sanitize_text_field( $request->get_param( 'name' ) ),
        $id,
        $updated_at
    ) );

    if ( 0 === $affected ) {
        return new WP_REST_Response(
            array( 'error' => 'conflict', 'message' => 'Dữ liệu đã bị thay đổi bởi người khác. Vui lòng tải lại.' ),
            409
        );
    }

    return new WP_REST_Response( array( 'success' => true ) );
}
```

## Áp dụng

Mọi admin update endpoint cho shared resources (products, orders, coupons, settings) PHẢI có optimistic locking via `updated_at` comparison. Frontend gửi `updated_at` kèm payload, backend reject nếu mismatch.
