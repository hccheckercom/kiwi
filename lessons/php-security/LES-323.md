## Pattern

All admin REST endpoints use `current_user_can('manage_options')` — the highest WordPress capability (only administrators). When shop needs staff roles (warehouse manager views orders, content editor manages products), they can't access anything. No granular capabilities = all-or-nothing access.

## Scan

```yaml
pattern: "permission_callback.*manage_options"
scope: "**/AdminUI/**/*.php|**/src/**/*Controller*.php"
exclude_line: "manage_wezone_|wz_cap_|custom_cap"
message: "Admin endpoint uses manage_options — no granular capability for staff roles"
```

## Bug

```php
// SAI — warehouse staff cần xem orders nhưng không cần manage_options
register_rest_route( 'wezone/v1', '/orders', array(
    'methods'             => 'GET',
    'callback'            => array( $ctrl, 'get_orders' ),
    'permission_callback' => fn() => current_user_can( 'manage_options' ), // Admin only!
) );

// Cùng capability cho view orders VÀ delete orders — không phân biệt
register_rest_route( 'wezone/v1', '/orders/(?P<id>\d+)', array(
    'methods'             => 'DELETE',
    'callback'            => array( $ctrl, 'delete_order' ),
    'permission_callback' => fn() => current_user_can( 'manage_options' ),
) );
```

## Fix

```php
// Đăng ký custom capabilities khi plugin activate
function wz_register_capabilities(): void {
    $admin = get_role( 'administrator' );
    $caps  = array(
        'wz_view_orders', 'wz_edit_orders', 'wz_delete_orders',
        'wz_view_products', 'wz_edit_products', 'wz_delete_products',
        'wz_view_customers', 'wz_manage_settings', 'wz_view_analytics',
        'wz_manage_coupons', 'wz_manage_flash_sales',
    );
    foreach ( $caps as $cap ) {
        $admin->add_cap( $cap );
    }
}
register_activation_hook( __FILE__, 'wz_register_capabilities' );

// Dùng granular capabilities
register_rest_route( 'wezone/v1', '/orders', array(
    'methods'             => 'GET',
    'callback'            => array( $ctrl, 'get_orders' ),
    'permission_callback' => fn() => current_user_can( 'wz_view_orders' ),
) );

register_rest_route( 'wezone/v1', '/orders/(?P<id>\d+)', array(
    'methods'             => 'DELETE',
    'callback'            => array( $ctrl, 'delete_order' ),
    'permission_callback' => fn() => current_user_can( 'wz_delete_orders' ),
) );
```

## Áp dụng

Phase 1 (hiện tại): `manage_options` chấp nhận được khi chỉ có 1 admin. Phase 2 (khi cần staff): migrate sang custom capabilities `wz_*`. Mỗi resource cần ít nhất: view, edit, delete. Settings/payment luôn giữ `manage_options`.
