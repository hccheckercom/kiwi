# Bad Code

```python
@app.post("/users")
async def create_user(name: str, email: str, age: int):
    # BAD: No validation, accepts any input
    # age=-1, email="not-an-email" both accepted
    user = User(name=name, email=email, age=age)
    db.add(user)
    return user

@app.post("/orders")
async def create_order(data: dict):
    # BAD: Accepts any dict structure
    # Missing fields, wrong types all accepted
    order = Order(**data)
    return order
```

# Good Code

```python
from pydantic import BaseModel, EmailStr, Field, validator

class UserCreate(BaseModel):
    name: str = Field(..., min_length=1, max_length=100)
    email: EmailStr
    age: int = Field(..., ge=0, le=150)
    
    @validator('name')
    def name_must_not_be_empty(cls, v):
        if not v.strip():
            raise ValueError('Name cannot be empty')
        return v.strip()

class OrderCreate(BaseModel):
    user_id: int = Field(..., gt=0)
    items: list[OrderItem] = Field(..., min_items=1)
    total: float = Field(..., gt=0)
    
    @validator('total')
    def validate_total(cls, v, values):
        if 'items' in values:
            calculated = sum(item.price * item.quantity for item in values['items'])
            if abs(v - calculated) > 0.01:
                raise ValueError('Total does not match items')
        return v

@app.post("/users")
async def create_user(user: UserCreate):
    # Pydantic validates automatically
    # Invalid data returns 422 with detailed errors
    db_user = User(**user.dict())
    db.add(db_user)
    return db_user

@app.post("/orders")
async def create_order(order: OrderCreate):
    db_order = Order(**order.dict())
    return db_order
```

# Why

**Without Pydantic validation:**
- **Security:** SQL injection, XSS, command injection
- **Data corruption:** Invalid data in database
- **Runtime errors:** Type errors, missing fields
- **Poor UX:** Generic 500 errors instead of validation feedback

**Real-world attacks:**
```python
# Without validation
@app.post("/users")
async def create_user(email: str, role: str):
    # Attacker sends: role="admin"
    user = User(email=email, role=role)  # Privilege escalation!

# With validation
class UserCreate(BaseModel):
    email: EmailStr
    # role field not exposed - set internally
```

**Pydantic features:**
```python
# Field constraints
age: int = Field(..., ge=18, le=100)
email: EmailStr  # Built-in email validation
url: HttpUrl  # Built-in URL validation

# Custom validators
@validator('password')
def password_strength(cls, v):
    if len(v) < 8:
        raise ValueError('Password too short')
    return v

# Nested models
class Address(BaseModel):
    street: str
    city: str

class User(BaseModel):
    name: str
    address: Address  # Validates nested structure
```

**Error responses:**
```json
{
  "detail": [
    {
      "loc": ["body", "age"],
      "msg": "ensure this value is greater than or equal to 0",
      "type": "value_error.number.not_ge"
    }
  ]
}
```
