# LES-111: N+1 Query Trong REST List Endpoints

## Problem

Khi trả về danh sách (products, orders, customers), mỗi record được format bằng individual queries:
- `forProduct($id)` — 1 query/product cho categories
- `get_post_meta($id, ...)` — 3-5 queries/product cho meta
- `wp_get_attachment_url($id)` — 1 query/product cho image
- `getItems($order_id)` — 1 query/order cho line items
- `get_userdata($id)` — 1 query/customer

Với 20 products/page → 100+ queries. Với 100 records → 500+ queries.

## Fix

Batch load trước khi format:

```php
// ĐÚNG — 1 query cho tất cả categories
$product_ids = array_map(fn($r) => (int) $r['id'], $rows);
$placeholders = implode(',', array_fill(0, count($product_ids), '%d'));
$sql = "SELECT pc.product_id, c.id, c.name
        FROM {$wpdb->prefix}wz_product_categories pc
        JOIN {$wpdb->prefix}wz_categories c ON c.id = pc.category_id
        WHERE pc.product_id IN ({$placeholders})";
$cats_map = []; // group by product_id

// ĐÚNG — 1 query cho tất cả meta
$sql = "SELECT post_id, meta_key, meta_value FROM {$wpdb->postmeta}
        WHERE post_id IN ({$placeholders}) AND meta_key IN (...)";

// ĐÚNG — 1 query cho tất cả order items
$sql = "SELECT * FROM {$wpdb->prefix}wz_order_items
        WHERE order_id IN ({$placeholders})";

// SAI — N queries
foreach ($rows as $row) {
    $cats = $this->categories->forProduct($row['id']); // 1 query
    $meta = get_post_meta($row['id'], '_wz_gallery', true); // 1 query
}
```

## Detection

Tìm pattern: `array_map([$this, 'format_...'], $rows)` trong REST controller mà format method chứa individual DB queries.
