## Vấn đề

`sanitize_textarea_field()` được thiết kế cho text thuần, **không phải JSON**. Nó:
- Strip HTML tags
- Encode `<`, `>`, `&`, `"` thành HTML entities
- Remove extra whitespace

Khi áp dụng lên JSON payload từ `$_POST`, các ký tự hợp lệ trong JSON bị biến đổi → `json_decode()` trả về `null` hoặc dữ liệu sai.

## Sai

```php
$raw = sanitize_textarea_field( wp_unslash( $_POST['tokens'] ) );
$tokens = json_decode( $raw, true );
```

## Đúng

```php
// wp_unslash() → json_decode() → validate structure
$raw = wp_unslash( $_POST['tokens'] );
if ( ! is_string( $raw ) ) {
    wp_send_json_error( array( 'message' => 'Invalid payload' ) );
}
$tokens = json_decode( $raw, true );
if ( ! is_array( $tokens ) ) {
    wp_send_json_error( array( 'message' => 'Invalid JSON' ) );
}
// Sanitize từng value SAU khi decode
foreach ( $tokens as $key => $val ) {
    $tokens[ $key ] = sanitize_text_field( (string) $val );
}
```

## Tại sao quan trọng

- JSON chứa `"` (double quotes) bị encode thành `&quot;` → parse fail
- CSS value như `rgba(0,0,0,0.5)` bị strip dấu ngoặc
- Bug chỉ xảy ra khi user save token có ký tự đặc biệt → khó reproduce
