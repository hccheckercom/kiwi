## Bad
```php
$key = get_option('vnpay_secret_key');
$hash = hash_hmac('sha256', $data, $key);
```

## Good
```php
$key = get_option('vnpay_secret_key');
if ( empty($key) ) {
    wz_log('Payment key not configured', 'error');
    return new WP_Error('config', 'Payment not configured');
}
$hash = hash_hmac('sha256', $data, $key);
```

## Why
HMAC with empty key — hash_hmac('sha256', data, '') produces a valid hash that attacker can reproduce.
