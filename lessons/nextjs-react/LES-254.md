## Pattern sai

Hai `THEMES` array riêng biệt: một trong listing component, một trong routing page — maintained độc lập, dễ lệch nhau.

```tsx
// components/wordpress/WordPressThemesPage.tsx — 9 themes (listing UI)
const THEMES: WPTheme[] = [
  { id: 'haven', slug: 'haven', ... },   // ← có ở đây
  { id: 'funilux', slug: 'funilux', ... },
  // ...9 themes
]

// app/wordpress/[slug]/page.tsx — chỉ 7 themes (routing + generateStaticParams)
export const THEMES: ThemeData[] = [
  { slug: 'funilux', ... },
  // ...7 themes — THIẾU haven, noir-beauty
]

export function generateStaticParams() {
  return THEMES.map((t) => ({ slug: t.slug })) // haven không được generate → 404
}
```

## Phải làm

Single source of truth — một array duy nhất, import vào cả hai nơi:

```tsx
// lib/data/wordpress-themes.ts — nguồn duy nhất
export const WORDPRESS_THEMES: ThemeData[] = [
  { slug: 'haven', ... },
  { slug: 'funilux', ... },
  // tất cả themes
]

// app/wordpress/[slug]/page.tsx
import { WORDPRESS_THEMES } from '@/lib/data/wordpress-themes'
export function generateStaticParams() {
  return WORDPRESS_THEMES.map((t) => ({ slug: t.slug }))
}

// components/wordpress/WordPressThemesPage.tsx
import { WORDPRESS_THEMES } from '@/lib/data/wordpress-themes'
```

## Lý do

`generateStaticParams()` dùng array riêng → slug mới thêm vào listing nhưng quên thêm vào routing array → Next.js không pre-generate page → 404 khi user click "Xem chi tiết".

## Grep check

`const THEMES` — tìm nhiều hơn 1 khai báo `THEMES` array trong cùng project → HIGH
