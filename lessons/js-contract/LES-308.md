## Bad
```javascript
const API_BASE = '/wp-json/wezone/v1/flash-sale';

function loadSales() {
    $.get(API_BASE + '/sales', function(response) {
        renderSalesList(response.items || []);
    });
}
```

## Good
```javascript
const API_BASE = '/wp-json/wezone/v1/flash-sale';

// Setup AJAX with nonce
$.ajaxSetup({
    beforeSend: function(xhr) {
        if (typeof wezoneFlashSaleAdmin !== 'undefined' && wezoneFlashSaleAdmin.nonce) {
            xhr.setRequestHeader('X-WP-Nonce', wezoneFlashSaleAdmin.nonce);
        }
    }
});

function loadSales() {
    $.get(API_BASE + '/sales', function(response) {
        renderSalesList(response.items || []);
    });
}
```

## Why
WordPress REST API requires nonce authentication for protected endpoints. Without `X-WP-Nonce` header, requests fail with 404 "No route was found" even though routes are registered. Admin UI becomes completely non-functional.

**Root cause:** Routes exist but permission callback (`manage_options`) rejects unauthenticated requests silently.

**Fix pattern:**
1. Backend: `wp_localize_script()` must pass nonce via `wp_create_nonce('wp_rest')`
2. Frontend: Set `X-WP-Nonce` header in `$.ajaxSetup()` or per-request `beforeSend`

**Discovered in:** wezone-core Flash Sale admin (2026-05-22)
