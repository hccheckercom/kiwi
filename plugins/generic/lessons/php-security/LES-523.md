## Bad
```php
$response = wp_remote_post( $webhook_url, [
    'body' => json_encode( $data ),
    'headers' => [ 'Content-Type' => 'application/json' ],
] );
```

## Good
```php
$response = wp_remote_post( $webhook_url, [
    'body' => json_encode( $data ),
    'headers' => [ 'Content-Type' => 'application/json' ],
    'timeout' => 5,
    'sslverify' => true,
] );

if ( is_wp_error( $response ) ) {
    error_log( 'Webhook failed: ' . $response->get_error_message() );
    return false;
}
```

## Why
wp_remote_* without timeout can hang indefinitely if remote server is slow/down. Always set timeout (5-10s for webhooks, 30s for APIs) and handle WP_Error.

## Grep
`wp_remote_(get|post)\([^)]+\)(?![^;]*timeout)` — in `**/*.php`
