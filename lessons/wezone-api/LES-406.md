# Bug: Helper Singleton Functions Không Reset Được — Breaks Unit Testing

## Vấn đề

`wz_product_repo()`, `wz_cart()`, `wz_order_engine()` dùng `static $repo = null` pattern. Trong unit tests, static variable persist giữa các test cases → test thứ 2 nhận state từ test thứ 1 → flaky tests.

**Impact:** Testability — không thể isolate unit tests.

## Bad

```php
// ❌ SAI — không có cách reset
function wz_product_repo(): ProductRepository {
    static $repo = null;
    if ( $repo === null ) {
        $repo = new ProductRepository();
    }
    return $repo;
}
```

## Good

```php
// ✅ ĐÚNG — thêm reset function cho testing
function wz_product_repo( bool $reset = false ): ProductRepository {
    static $repo = null;
    if ( $reset ) {
        $repo = null;
    }
    if ( $repo === null ) {
        $repo = new ProductRepository();
    }
    return $repo;
}

// Trong test:
public function tearDown(): void {
    wz_product_repo( true ); // reset singleton
    parent::tearDown();
}
```
