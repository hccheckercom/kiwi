## Bad
```ts
export async function POST(req: Request) {
  const { url } = await req.json()
  // User sends url = "http://169.254.169.254/latest/meta-data/" → AWS credentials leaked
  // User sends url = "http://127.0.0.1:5432" → internal DB port scanned
  const res = await fetch(url)
  return Response.json({ html: await res.text() })
}
```

## Good (pattern from lib/googleads/checker/url-validator.ts)
```ts
const PRIVATE_IP_PATTERNS = [
  /^127\./,                    // localhost
  /^10\./,                     // Class A private
  /^172\.(1[6-9]|2\d|3[01])\./, // Class B private
  /^192\.168\./,               // Class C private
  /^169\.254\./,               // Link-local (AWS metadata!)
  /^100\.(6[4-9]|[7-9]\d|1[01]\d|12[0-7])\./, // CGNAT
]

const BLOCKED_HOSTS = [
  'localhost', 'metadata.google.internal', '169.254.169.254', 'instance-data',
]

export async function validateUrl(rawUrl: string): Promise<ServiceResult<string>> {
  const parsed = new URL(rawUrl)
  if (!['http:', 'https:'].includes(parsed.protocol)) return error('Only HTTP/HTTPS')
  if (BLOCKED_HOSTS.includes(parsed.hostname)) return error('Blocked')
  if (PRIVATE_IP_PATTERNS.some(p => p.test(parsed.hostname))) return error('Private IP')
  return { data: rawUrl, error: null }
}
```

## Checklist for any "fetch user URL" feature
```
✓ Protocol whitelist: only http: and https:
✓ Block private IP ranges (127.x, 10.x, 172.16-31.x, 192.168.x)
✓ Block cloud metadata endpoints (169.254.169.254)
✓ Block localhost aliases
✓ Timeout: AbortSignal.timeout(10_000)
✓ Don't follow redirects to private IPs (redirect: 'manual' + re-validate)
```

## Why
Real pattern from Google Ads checker (url-validator.ts). Any feature that fetches a user-supplied URL is an SSRF vector. Attacker can probe internal network, read cloud metadata (AWS/GCP credentials), or access internal services. The URL validator must run BEFORE fetch, not after.
