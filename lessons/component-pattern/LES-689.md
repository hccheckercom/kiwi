## Bad
```tsx
<div className="grid grid-cols-3 gap-4">
  {products.map(p => <ProductCard key={p.id} {...p} />)}
</div>
```
When products is empty, renders an empty grid — user sees blank space with no guidance.

## Good
```tsx
{products.length === 0 ? (
  <EmptyState
    icon={<PackageIcon />}
    title="Chưa có sản phẩm"
    description="Thêm sản phẩm đầu tiên để bắt đầu bán hàng"
    action={<Button href="/admin/products/new">Thêm sản phẩm</Button>}
  />
) : (
  <div className="grid grid-cols-3 gap-4">
    {products.map(p => <ProductCard key={p.id} {...p} />)}
  </div>
)}
```

## Why
Empty states are a UX opportunity, not a bug to ignore. They guide users toward the next action (add first item, adjust filters, etc.) instead of leaving them confused by blank space.
