## Bad
```php
public function expire_old_points(): void {
    $expired = $this->repo->get_expired_rows( gmdate('Y-m-d H:i:s') );
    foreach ( $expired as $row ) {
        $balance = $this->repo->get_balance( $row->user_id ) - $row->points;
        $this->repo->log_transaction( $row->user_id, 'expire', -$row->points, $balance );
    }
    // No clear_expired_flags() - points expire again next run
}
```

## Good
```php
public function expire_old_points(): void {
    $now = gmdate('Y-m-d H:i:s');
    $expired = $this->repo->get_expired_rows( $now );
    
    foreach ( $expired as $row ) {
        $user_id = (int) $row->user_id;
        $points  = (int) $row->total_points;
        
        $balance = max( 0, $this->repo->get_balance( $user_id ) - $points );
        $this->repo->log_transaction( $user_id, 'expire', -$points, $balance );
        $this->repo->clear_expired_flags( $user_id, $now ); // Mark as processed
        $this->repo->update_balance( $user_id, $this->tiers );
    }
}
```

## Why
Cron job queries WHERE expires_at <= NOW(), deducts points, but does not clear expires_at flag. Next run finds same rows again, deducts again. Must UPDATE expires_at = NULL after processing.

## Grep
`function\s+expire.*?points.*?\{(?!.*clear_expired|mark_processed|UPDATE.*expires_at.*NULL)` — in `**/*.php`
