## In-process vs DB-backed rate limiting

| Approach | Use when | Limitation |
|----------|----------|-----------|
| In-process Map (lib/rate-limit.ts) | AI routes, single instance | Resets on restart, per-instance only |
| DB-backed (gads_rate_limits table) | Public tools, multi-instance | Slower (DB query per check), but persistent |

## DB-backed pattern (from lib/googleads/services/scan.cache.ts)
```ts
const RATE_LIMITS = { anonymous: 3, user: 10 } // per day

export async function checkRateLimit(userId: string | null, ip: string): Promise<{ allowed: boolean; remaining: number }> {
  const identifier = userId ? `user:${userId}` : `ip:${ip}`
  const limit = userId ? RATE_LIMITS.user : RATE_LIMITS.anonymous
  const today = new Date().toISOString().split('T')[0]

  const { data } = await supabase.from('gads_rate_limits')
    .select('scan_count')
    .eq('identifier', identifier)
    .eq('scan_date', today)
    .single()

  const currentCount = data?.scan_count ?? 0
  return currentCount >= limit
    ? { allowed: false, remaining: 0 }
    : { allowed: true, remaining: limit - currentCount }
}

export async function incrementRateLimit(userId: string | null, ip: string): Promise<void> {
  const identifier = userId ? `user:${userId}` : `ip:${ip}`
  const today = new Date().toISOString().split('T')[0]

  const { data: existing } = await supabase.from('gads_rate_limits')
    .select('id, scan_count')
    .eq('identifier', identifier)
    .eq('scan_date', today)
    .maybeSingle()

  if (existing) {
    await supabase.from('gads_rate_limits').update({ scan_count: existing.scan_count + 1 }).eq('id', existing.id)
  } else {
    await supabase.from('gads_rate_limits').insert({ identifier, scan_date: today, scan_count: 1 })
  }
}
```

## Usage in route handler
```ts
const { allowed, remaining } = await checkRateLimit(userId, ip)
if (!allowed) {
  return Response.json({ error: 'Rate limit exceeded', remaining: 0 }, {
    status: 429,
    headers: { 'Retry-After': '86400', 'X-RateLimit-Remaining': '0' }
  })
}
// ... do work ...
await incrementRateLimit(userId, ip)
```

## Why
Real pattern from Google Ads checker. Public tools (no auth required) need rate limiting to prevent abuse. In-process Map resets on deploy and doesn't work across multiple instances. DB-backed rate limits persist across restarts and are shared across all server instances. Daily window (scan_date) auto-resets without cleanup jobs.
