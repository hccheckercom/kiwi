# Bug Pattern

Deleting existing rows then bulk inserting new rows without a transaction can leave tables empty if the insert fails.

**Why:** If `DELETE` succeeds but `INSERT` fails (constraint violation, disk full, etc.), the table is left in an inconsistent state with data loss.

## Detection

```regex
wpdb->delete\([^;]+;[\s\S]{0,200}(?:wz_bulk_insert|wpdb->insert)\(
```

Match `$wpdb->delete()` followed by insert operations without transaction wrapper.

## Examples

### ❌ BAD: Delete then insert without transaction

```php
public static function sync_products( int $sale_id, array $products ): void {
    global $wpdb;
    $table = $wpdb->prefix . 'wezone_flash_sale_products';
    
    // CRITICAL: Delete all existing products for this sale
    $wpdb->delete( $table, array( 'sale_id' => $sale_id ) );
    
    // If this fails (e.g., constraint violation), sale has ZERO products!
    $insert_rows = array();
    foreach ( $products as $item ) {
        $insert_rows[] = array(
            'sale_id'    => $sale_id,
            'product_id' => (int) $item['product_id'],
            'sale_price' => (float) $item['sale_price'],
        );
    }
    
    wz_bulk_insert( $table, $insert_rows, array( '%d', '%d', '%f' ) );
}
```

**Failure scenario:**
1. `DELETE` removes 10 products from sale
2. `wz_bulk_insert()` fails (e.g., invalid product_id foreign key)
3. Sale now has 0 products → broken state

### ✅ GOOD: Wrap in transaction

```php
public static function sync_products( int $sale_id, array $products ): void {
    global $wpdb;
    $table = $wpdb->prefix . 'wezone_flash_sale_products';
    
    // Preserve existing sold_count before delete
    $existing_sold = array();
    $rows = $wpdb->get_results(
        $wpdb->prepare( "SELECT product_id, sold_count FROM {$table} WHERE sale_id = %d", $sale_id ),
        ARRAY_A
    );
    foreach ( $rows as $row ) {
        $existing_sold[ (int) $row['product_id'] ] = (int) $row['sold_count'];
    }
    
    // Start transaction
    $wpdb->query( 'START TRANSACTION' );
    
    // Delete existing products
    $wpdb->delete( $table, array( 'sale_id' => $sale_id ) );
    
    // Build insert rows
    $insert_rows = array();
    foreach ( $products as $item ) {
        $product_id = (int) ( $item['product_id'] ?? 0 );
        if ( $product_id <= 0 ) {
            continue;
        }
        
        $product = wz_get_product( $product_id );
        if ( ! $product ) {
            continue; // Skip invalid products
        }
        
        $insert_rows[] = array(
            'sale_id'        => $sale_id,
            'product_id'     => $product_id,
            'sale_price'     => (float) ( $item['sale_price'] ?? 0 ),
            'original_price' => (float) $product['regular_price'],
            'sold_count'     => $existing_sold[ $product_id ] ?? 0,
        );
    }
    
    // Insert new products
    if ( ! empty( $insert_rows ) ) {
        $result = wz_bulk_insert(
            $table,
            $insert_rows,
            array( '%d', '%d', '%f', '%f', '%d' )
        );
        
        if ( $result === false ) {
            $wpdb->query( 'ROLLBACK' );
            wz_log( "FlashSale: sync_products failed for sale_id={$sale_id}", 'error' );
            return;
        }
    }
    
    // Commit transaction
    $wpdb->query( 'COMMIT' );
}
```

### ✅ BETTER: Use temp table swap for zero-downtime updates

```php
public static function sync_products( int $sale_id, array $products ): void {
    global $wpdb;
    $table      = $wpdb->prefix . 'wezone_flash_sale_products';
    $temp_table = $table . '_tmp_' . $sale_id;
    
    // Create temp table with same schema
    $wpdb->query( "CREATE TEMPORARY TABLE {$temp_table} LIKE {$table}" );
    
    // Insert into temp table (no data loss if this fails)
    $insert_rows = array();
    foreach ( $products as $item ) {
        $product_id = (int) ( $item['product_id'] ?? 0 );
        if ( $product_id <= 0 ) continue;
        
        $product = wz_get_product( $product_id );
        if ( ! $product ) continue;
        
        $insert_rows[] = array(
            'sale_id'        => $sale_id,
            'product_id'     => $product_id,
            'sale_price'     => (float) ( $item['sale_price'] ?? 0 ),
            'original_price' => (float) $product['regular_price'],
        );
    }
    
    if ( empty( $insert_rows ) ) {
        $wpdb->query( "DROP TEMPORARY TABLE {$temp_table}" );
        return;
    }
    
    $result = wz_bulk_insert( $temp_table, $insert_rows, array( '%d', '%d', '%f', '%f' ) );
    
    if ( $result === false ) {
        $wpdb->query( "DROP TEMPORARY TABLE {$temp_table}" );
        wz_log( "FlashSale: sync_products insert failed for sale_id={$sale_id}", 'error' );
        return;
    }
    
    // Atomic swap: delete old + insert new in single transaction
    $wpdb->query( 'START TRANSACTION' );
    $wpdb->delete( $table, array( 'sale_id' => $sale_id ) );
    $wpdb->query( "INSERT INTO {$table} SELECT * FROM {$temp_table}" );
    $wpdb->query( 'COMMIT' );
    
    $wpdb->query( "DROP TEMPORARY TABLE {$temp_table}" );
}
```

## Fix Strategy

1. **Always use transactions** for delete + insert patterns
2. **Preserve critical data** (e.g., sold_count) before delete
3. **Validate inserts** before committing transaction
4. **Use temp tables** for large updates to avoid locking main table
5. **Log failures** and rollback on error
