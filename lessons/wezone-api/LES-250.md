## Problem

In WordPress Multisite, `define('WP_HOME')` and `define('WP_SITEURL')` in wp-config.php only override the **main site** (blog_id=1). Subsites get their URLs from `wp_X_options` table (`home` and `siteurl` option_name).

Setting these constants when trying to serve subsites under a different domain/path causes:
- Main site works, subsites still redirect to original domain
- Or worse: subsites load but with broken internal routing (404 on all pages)

## Wrong Approach

```php
// wp-config.php — DOES NOT WORK for subsites
if ( $_SERVER['HTTP_X_FORWARDED_HOST'] === 'wezone.vn' ) {
    define( 'WP_HOME', 'https://wezone.vn/wordpress/' . $slug . '/demo' );
    define( 'WP_SITEURL', 'https://wezone.vn/wordpress/' . $slug . '/demo' );
}
```

## Correct Approaches

### Option A: mu-plugin with option filters (if WordPress must be aware)

```php
// wp-content/mu-plugins/url-override.php
add_filter( 'option_home', fn() => $new_url );
add_filter( 'option_siteurl', fn() => $new_url );
```

### Option B: Proxy-level URL rewrite (recommended)

Let WordPress run on its original domain. Use nginx `sub_filter` + `proxy_redirect` to rewrite URLs in the response. WordPress never knows about the new URL structure.

## Why Option B is Better

- No PHP code changes needed
- No risk of breaking multisite blog detection
- Works for all subsites without per-site configuration
- WordPress internal routing (canonical redirects, permalink matching) stays intact
