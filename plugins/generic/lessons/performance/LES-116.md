# LES-116: Unnecessary Module Boot on REST API Requests

## Problem

`plugins_loaded` hook boot toàn bộ hệ thống cho MỌI request type:
- ThemeEngine (Router, TokenLoader, DesignSystem, TemplateRegistry, ComponentCSS, PresetManager)
- Login, Account, Email, PageBuilder, Tracking, FlashSale, Banner plugins
- I18n, License checker, Updater

Trên REST API request (`/wp-json/`), chỉ cần:
- Commerce core (products, orders, customers)
- AdminUI routes
- Security + Cache
- API layer

Các module frontend-only tốn 50-200ms boot time mỗi request.

## Fix

Detect REST request sớm, skip modules không cần:

```php
// Detect REST early — trước khi boot modules
$is_rest = ( defined( 'REST_REQUEST' ) && REST_REQUEST )
    || ( isset( $_SERVER['REQUEST_URI'] ) && str_contains( $_SERVER['REQUEST_URI'], '/wp-json/' ) );

// Always boot — needed for REST
( new AdminUI\Plugin() )->boot();
( new Cache\Plugin() )->boot();
( new Security\Plugin() )->boot();

// Skip on REST — frontend-only
if ( ! $is_rest ) {
    ( new Account\Plugin() )->boot();
    ( new Login\Plugin() )->boot();
    ( new Email\Plugin() )->boot();
    ( new FlashSale\Plugin() )->boot();
    ( new Banner\Plugin() )->boot();
    ( new PageBuilder\Plugin() )->boot();
    ( new Tracking\Plugin() )->boot();
    // ThemeEngine modules...
    $i18n->init();
    $license->init();
    $updater->init();
}
```

## Impact

- REST response time: giảm 50-200ms per request
- Memory: giảm ~5MB per REST request
- Đặc biệt quan trọng khi admin SPA gọi 5+ API calls song song

## Detection

Tìm `plugins_loaded` callback mà boot tất cả modules không có conditional check cho REST_REQUEST.
