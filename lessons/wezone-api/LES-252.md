# LES-252 — Missing `single-wz_product.php` Template

## Bug Pattern

WeZone Router handles `/product/{slug}` via rewrite rules and loads `wezone-templates/product/single.php`. But WordPress also generates `?p=ID` links (admin "View" button, REST API `link` field, `get_permalink()` before flush_rewrite_rules). These use WordPress template hierarchy which looks for `single-wz_product.php` → `single.php` → `index.php`.

If `single-wz_product.php` doesn't exist and `single.php` is a blog post template, the product shows 404 or renders as a blog post.

## Root Cause

- CPT `wz_product` registered by wezone-core plugin
- Router intercepts `/product/{slug}` but NOT `?p=ID`
- WordPress template hierarchy for CPT: `single-{post_type}.php` → `single.php` → `index.php`
- Theme only has `wezone-templates/product/single.php` (Router path) but no WP hierarchy file

## Fix

Create `single-wz_product.php` in theme root that includes the Router template:

```php
<?php
// single-wz_product.php — WP template hierarchy fallback for wz_product CPT
include get_theme_file_path( 'wezone-templates/product/single.php' );
```
