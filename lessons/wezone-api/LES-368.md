# Bug: Banner get_by_position thiếu cache → query DB mỗi pageview

## Mô tả
`BannerRepository::get_by_position()` được gọi mỗi pageview (home hero, mini banners) nhưng không có cache → query DB mỗi lần. Cần cache với TTL 5 phút.

## Tại sao nguy hiểm
- **Performance**: 1000 pageviews/phút = 1000 queries
- **Database load**: Banner query có JOIN với date comparison
- **Scalability**: Không scale khi traffic tăng

## Code sai (BannerRepository.php:202-224)
```php
public static function get_by_position( string $position ): array {
    global $wpdb;
    $table = self::table();
    $now   = current_time( 'mysql' );

    $results = $wpdb->get_results(
        $wpdb->prepare(
            "SELECT * FROM {$table}
            WHERE position = %s
            AND status = 'active'
            AND (start_date IS NULL OR start_date <= %s)
            AND (end_date IS NULL OR end_date >= %s)
            ORDER BY sort_order ASC",
            $position,
            $now,
            $now
        ),
        ARRAY_A
    );

    return $results ?: array();
}
```

**Vấn đề**: Không có cache → query DB mỗi pageview.

## Code đúng
```php
public static function get_by_position( string $position ): array {
    // Try cache first
    $cache_key = 'wz_banners_' . $position;
    $cached = wp_cache_get( $cache_key, 'wezone_banners' );
    
    if ( false !== $cached ) {
        return $cached;
    }

    global $wpdb;
    $table = self::table();
    $now   = current_time( 'mysql' );

    $results = $wpdb->get_results(
        $wpdb->prepare(
            "SELECT * FROM {$table}
            WHERE position = %s
            AND status = 'active'
            AND (start_date IS NULL OR start_date <= %s)
            AND (end_date IS NULL OR end_date >= %s)
            ORDER BY sort_order ASC",
            $position,
            $now,
            $now
        ),
        ARRAY_A
    );

    $results = $results ?: array();
    
    // Cache for 5 minutes
    wp_cache_set( $cache_key, $results, 'wezone_banners', 300 );

    return $results;
}
```

## Invalidate cache khi update
```php
// In BannerRepository::create(), update(), delete(), reorder()
private static function clear_cache(): void {
    $positions = array( 'home_hero', 'home_mini_1', 'home_mini_2' );
    foreach ( $positions as $position ) {
        wp_cache_delete( 'wz_banners_' . $position, 'wezone_banners' );
    }
}

public static function create( array $data ): int {
    // ... existing code
    $id = (int) $wpdb->insert_id;
    self::clear_cache();
    return $id;
}
```

## Cách fix
1. Thêm `wp_cache_get()` đầu function
2. Cache với group `wezone_banners` và TTL 300s
3. Invalidate cache trong create/update/delete/reorder
4. Dùng `wp_cache_set()` thay `set_transient()` (faster)

## Áp dụng cho
- Mọi query được gọi nhiều lần per pageview
- Đặc biệt: categories, settings, config
- Không cache: user-specific data, cart, session
