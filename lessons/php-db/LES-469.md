## Bad
```php
// Delete files first
foreach ( $files as $file ) {
    wp_delete_file( $file );
}
// Then delete DB record â€” if this fails, files already gone
$wpdb->delete( $table, array( 'id' => $id ) );
```

## Good
```php
// Delete DB record first
$deleted = $wpdb->delete( $table, array( 'id' => $id ) );
if ( ! $deleted ) {
    return false; // DB delete failed, files still intact
}
// Then delete files
foreach ( $files as $file ) {
    wp_delete_file( $file );
}
```

## Why
If files are deleted first and DB delete fails, files are lost but DB record remains (orphaned record). Reverse order or transaction prevents data loss.

## Grep
`wp_delete_file\(|unlink\([\s\S]{0,200}\$wpdb->delete\(` — in `**/*.php`
