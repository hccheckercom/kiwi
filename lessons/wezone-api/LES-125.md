# LES-125: Hardcoded Banner Content in Theme

## Problem

Theme có banner hero với nội dung (title, subtitle, badge, CTA, trust indicators) được hardcode trực tiếp trong template PHP. Khi cần thay đổi nội dung banner, phải sửa code và deploy lại.

## Root Cause

Developer code banner trực tiếp trong `front-page.php` hoặc hero template thay vì sử dụng hệ thống admin banner (`wp_wz_banners` table + REST API).

## Detection

```php
// SAI — hardcoded trong template
<h1 class="...">Ghế Văn Phòng Đẳng Cấp Giám Đốc</h1>
<p class="...">Thiết kế công thái học đỉnh cao...</p>
```

## Fix

```php
// ĐÚNG — load từ admin banner system
$hero_banners = function_exists( 'wz_get_banners' ) ? wz_get_banners( 'home_hero' ) : [];
$hero = ! empty( $hero_banners ) ? $hero_banners[0] : null;

if ( $hero && 'gradient' === $hero['type'] ) :
    $meta = $hero['meta'] ?? [];
    // render dynamic content from $hero['title'], $hero['subtitle'], etc.
endif;
```

## Key Points

- Admin banner system hỗ trợ 4 types: `image`, `product`, `category`, `gradient`
- Type `gradient` dùng cho banner text-only với gradient background (kiểu funilux)
- Luôn có fallback content nếu chưa có banner nào active trong admin
- Dùng `wz_get_banners('home_hero')` để query active banners theo position
- Meta field (JSON) chứa dữ liệu mở rộng: gradient_to, cta_text_2, trust_items
