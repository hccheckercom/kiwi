## Bad
```php
// File starts with EF BB BF bytes (invisible BOM)
<?php declare(strict_types=1);
```

## Good
```php
// File starts clean — no BOM
<?php declare(strict_types=1);
```

## Why
UTF-8 BOM before <?php causes output before headers. With strict_types, PHP throws fatal error.
