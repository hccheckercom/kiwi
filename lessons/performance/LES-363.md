## Problem

`DELETE FROM table WHERE date < cutoff` without `LIMIT` acquires a row-level lock on all matching rows. On tables with millions of historical rows, this blocks all writes for the duration of the DELETE — causing timeouts on frontend requests.

## Bad

```php
$wpdb->query( $wpdb->prepare( "DELETE FROM {$daily_table} WHERE date < %s", $cutoff ) );
```

## Good

```php
do {
    $deleted = $wpdb->query(
        $wpdb->prepare( "DELETE FROM {$daily_table} WHERE date < %s LIMIT 1000", $cutoff )
    );
} while ( $deleted > 0 );
```

## Why

Batch deletion releases the lock between batches, allowing concurrent writes to proceed. This is especially critical for cron jobs that run alongside live traffic.
