# Bad Code

```python
def validate_status(status):
    if status is "active":  # BUG: Compares identity, not value
        return True
    return False

def check_count(count):
    if count is 0:  # BUG: May fail for some 0 values
        return "empty"
    return "has items"

def is_enabled(flag):
    if flag is True:  # Works but wrong pattern
        return "enabled"
    return "disabled"
```

# Good Code

```python
def validate_status(status):
    if status == "active":  # Correct: Value comparison
        return True
    return False

def check_count(count):
    if count == 0:  # Correct: Value comparison
        return "empty"
    return "has items"

def is_enabled(flag):
    if flag:  # Best: Truthy check
        return "enabled"
    return "disabled"

# `is` is ONLY for None checks
def get_value(data):
    if data is None:  # Correct use of `is`
        return default_value
    return data
```

# Why

**`is` checks identity (same object), not equality (same value):**
- `is` compares memory addresses with `id()`
- `==` compares values with `__eq__()`
- Only use `is` for singleton checks: `None`, `True`, `False`

**Why it breaks:**
```python
# Small integers are cached (-5 to 256)
a = 100
b = 100
a is b  # True (same cached object)

# Large integers are not cached
a = 1000
b = 1000
a is b  # False (different objects)

# Strings are sometimes interned
a = "hello"
b = "hello"
a is b  # True (interned)

a = "hello world"
b = "hello world"
a is b  # False (not interned)
```

**Real-world bugs:**
```python
# Production bug: Worked in dev, failed in prod
def process_order(status):
    if status is "pending":  # Worked with small test data
        send_notification()
    # Failed in prod with different string objects!

# Subtle bug with booleans
def check_flag(value):
    if value is True:  # Fails for truthy values like 1, "yes"
        return "enabled"
    # Expected: check_flag(1) → "enabled"
    # Actual: check_flag(1) → None
```

**Correct patterns:**
```python
# None check (ONLY valid use of `is`)
if value is None:
    ...

# Value comparison
if status == "active":
    ...

# Truthy check (best for booleans)
if flag:
    ...

# Falsy check
if not flag:
    ...
```

**Linters catch this:**
- `pylint`: `singleton-comparison`
- `flake8`: `E711`, `E712`
- `ruff`: `SIM101`
