## Triệu chứng

Deploy lên VPS thành công (smoke test HTTP 200), nhưng trang hiển thị thiếu section (voucher, banner, v.v.). Trông giống code cũ nhưng thực ra code mới đã deploy — chỉ thiếu DB table/data nên template `return` sớm.

## Root Cause

Template part query DB trực tiếp (`$wpdb->get_results(...)`) rồi check `if (empty($results)) { return; }`. Khi bảng chưa tồn tại hoặc chưa có data, query trả về empty → template không render → **silent failure**.

Multisite WordPress dùng prefix `wp_{blog_id}_` (VD: `wp_14_wz_coupons` cho blog_id=14). Nếu tạo bảng sai prefix (VD: `wp_2_` thay vì `wp_14_`) → cũng silent failure.

## Bad

```php
// template-parts/voucher/section.php
$vouchers = $wpdb->get_results("SELECT ... FROM {$wpdb->prefix}wz_coupons WHERE ...");
if ( empty( $vouchers ) ) {
    return; // Silent — không log, không hiển thị gì
}
```

## Good

```php
// Option 1: Log khi table missing (dev mode)
$table = $wpdb->prefix . 'wz_coupons';
$exists = $wpdb->get_var( $wpdb->prepare( "SHOW TABLES LIKE %s", $table ) );
if ( ! $exists ) {
    if ( function_exists( 'wz_log' ) ) {
        wz_log( "Table {$table} not found — voucher section skipped", 'warning', 'template' );
    }
    return;
}

// Option 2: Plugin activation hook tạo table (đúng cách)
register_activation_hook( __FILE__, function() {
    global $wpdb;
    $table = $wpdb->prefix . 'wz_coupons';
    $wpdb->query( "CREATE TABLE IF NOT EXISTS {$table} (...)" );
});
```

## Checklist deploy mới

1. **Trước deploy**: kiểm tra template parts nào query custom tables
2. **Sau deploy**: verify tables tồn tại trên VPS với đúng prefix (`wp_{blog_id}_`)
3. **Multisite**: dùng `$wpdb->prefix` (đã include blog_id), KHÔNG hardcode `wp_2_`
4. **Seed data**: nếu template cần data để render (coupons, banners) → seed sau khi tạo table

## Grep check

Template parts có `$wpdb->get_results` + `if (empty(...)) { return; }` mà KHÔNG có table existence check hoặc fallback UI → CRITICAL (silent deploy failure)
