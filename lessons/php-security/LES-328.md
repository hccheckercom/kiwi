## Pattern

Dashboard API returns all business metrics (revenue, customer count, order details) to any authenticated admin user. When shop adds staff roles (warehouse, content editor), they see sensitive financial data they shouldn't access. Single endpoint returns everything — no granular widget-level permissions.

## Bug

```php
// SAI — mọi admin user đều thấy revenue, customer data
register_rest_route( 'wezone/v1', '/dashboard/stats', array(
    'methods'             => 'GET',
    'callback'            => array( $ctrl, 'get_stats' ),
    'permission_callback' => fn() => current_user_can( 'manage_options' ),
    // Khi thêm staff roles → họ cũng thấy hết
) );

// get_stats() trả về MỌI THỨ trong 1 response
public function get_stats(): WP_REST_Response {
    return new WP_REST_Response( array(
        'revenue'        => $this->get_revenue(),      // Sensitive!
        'pending_orders' => $this->get_pending(),
        'new_customers'  => $this->get_new_customers(), // PII count
        'low_stock'      => $this->get_low_stock(),
    ) );
}
```

## Fix

```php
public function get_stats( WP_REST_Request $request ): WP_REST_Response {
    $user = wp_get_current_user();
    $data = array();

    // Revenue — chỉ owner/admin
    if ( current_user_can( 'wz_view_revenue' ) ) {
        $data['revenue'] = $this->get_revenue();
    }

    // Orders — warehouse + admin
    if ( current_user_can( 'wz_view_orders' ) ) {
        $data['pending_orders'] = $this->get_pending();
    }

    // Customers — admin only
    if ( current_user_can( 'wz_view_customers' ) ) {
        $data['new_customers'] = $this->get_new_customers();
    }

    // Stock — warehouse + admin
    if ( current_user_can( 'wz_view_products' ) ) {
        $data['low_stock'] = $this->get_low_stock();
    }

    return new WP_REST_Response( $data );
}
```

## Áp dụng

Dashboard stats PHẢI filter theo user capabilities: (1) revenue/financial → owner only, (2) orders → order managers, (3) products/stock → warehouse, (4) customers → CRM role. API trả về chỉ data user được phép xem — frontend render conditional.
