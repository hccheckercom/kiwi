## Bad
```php
// Canonical always uses get_permalink() or current URL — page 2 canonicals to itself
if ( apply_filters( 'wezone_seo_show_canonical', true ) ) :
    <link rel="canonical" href="<?php echo esc_url( $url ); ?>">
endif;
// No pagination handling — /danh-muc/ao/?page=2 canonicals to /danh-muc/ao/?page=2
```

## Good
```php
// Canonical for paginated category/archive pages must point to page 1
function wezone_theme_canonical_url(): string {
    if ( is_singular() ) {
        return get_permalink();
    }

    if ( is_front_page() ) {
        return home_url( '/' );
    }

    // Archive/category: always canonical to page 1 (no ?page= or /page/N/)
    if ( is_archive() || is_tax() || is_category() ) {
        $term = get_queried_object();
        if ( $term && ! is_wp_error( $term ) ) {
            return get_term_link( $term );
        }
        return get_post_type_archive_link( get_post_type() );
    }

    return home_url( $_SERVER['REQUEST_URI'] );
}
```

## Why
Blueprint `seo.md` Canonical URL Rules: "Category page 2+ → `/danh-muc/{slug}/` (page 1, NOT self)". When page 2+ canonicals to itself, Google treats each page as a separate URL competing for the same keyword — diluting ranking signals. Consolidating to page 1 tells Google all pagination pages support the main category URL.
