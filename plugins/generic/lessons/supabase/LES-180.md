## Bad
```ts
export const handlePostProducts = withPlatformKey(async (req, store) => {
  const body = await req.json()
  // No validation — attacker sends { price: "abc", name: "" }
  const { data, error } = await supabase.from('platform_products').insert(body)
  return apiSuccess(data)
})
```

## Good (validation pattern from platform handlers)
```ts
export const handlePostProducts = withPlatformKey(async (req, store) => {
  let body: CreateProductInput
  try { body = await req.json() } catch {
    return apiError('INVALID_JSON', 'Request body must be valid JSON')
  }

  // 1. Required field validation
  if (!body.name?.trim()) return apiError('VALIDATION_ERROR', 'Product name is required', 400, 'name')
  if ((body.price ?? -1) < 0) return apiError('VALIDATION_ERROR', 'Invalid price', 400, 'price')

  // 2. Always inject store_id — never trust client
  const { data, error } = await supabase.from('platform_products').insert({
    store_id: store.id,  // ← ALWAYS from auth, never from body
    name: body.name.trim(),
    price: Math.round(body.price),  // Coerce to integer (VND)
    slug: body.slug ?? slugify(body.name),
    // ... other fields with defaults
  }).select().single()

  if (error) return apiError('DB_ERROR', error.message, 500)

  // 3. Return minimal response — don't leak internal fields
  return Response.json(
    { success: true, data: { id: data.id, slug: data.slug } },
    { status: 201, headers: corsHeaders(req) }
  )
})
```

## Checklist for platform write handlers
```
✓ Parse body with try/catch (invalid JSON → 400)
✓ Validate required fields (empty string → 400 with field name)
✓ Inject store_id from auth (never from request body)
✓ Coerce types (Math.round for VND, .trim() for strings)
✓ Return minimal response (id + slug, not full DB row)
✓ Include corsHeaders(req) in response
```

## Why
Real pattern from orders.handlers.ts, flash-sales.handlers.ts. Platform API is public (authenticated by API key, not session). Without validation: (1) empty names in DB, (2) NaN prices, (3) cross-tenant data injection via store_id in body. The store_id must ALWAYS come from the authenticated store object, never from the request.
