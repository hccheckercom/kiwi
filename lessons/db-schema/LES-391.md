## Bad
```php
// wz_notifications — individual indexes
'idx_user'    => 'user_id',
'idx_is_read' => 'is_read',
'idx_created' => 'created_at',
```

## Good
```php
// Composite index covers: WHERE user_id=? AND is_read=0 ORDER BY created_at DESC
'idx_user_unread' => 'user_id, is_read, created_at',
'idx_type'        => 'type',
```

## Why
The most common notification query is "unread notifications for user X, newest first". Three individual indexes force MySQL to pick only one; a composite (user_id, is_read, created_at) covers the full WHERE + ORDER BY in a single index scan.
