# Bug: Shipping Rate Không Validate — Negative Rate Bypass

## Vấn đề

REST API `set-shipping` endpoint nhận `rate` param trực tiếp cast `(float)` mà không validate giá trị >= 0. Attacker gửi `rate: -50000` → shipping_total âm → total giảm dưới giá thật.

**Ví dụ lỗi:**
```php
// ❌ SAI — không validate rate
public function set_shipping( WP_REST_Request $request ): WP_REST_Response {
    $method = (string) $request->get_param( 'method' );
    $rate   = (float) $request->get_param( 'rate' );   // ← -99999 OK!
    $label  = (string) $request->get_param( 'label' );

    wz_cart()->setShipping( $method, $rate, $label );
    // ...
}
```

`recalculate()` có `max(0, subtotal + shipping_total - discount)` nên total không âm, nhưng shipping_total âm vẫn stored và hiển thị sai.

## Fix

```php
// ✅ ĐÚNG — validate rate >= 0
public function set_shipping( WP_REST_Request $request ): WP_REST_Response {
    $method = (string) $request->get_param( 'method' );
    $rate   = max( 0.0, (float) $request->get_param( 'rate' ) );
    $label  = (string) $request->get_param( 'label' );

    wz_cart()->setShipping( $method, $rate, $label );
    // ...
}
```

Hoặc validate trong `CartEngine::setShipping()`:
```php
public function setShipping( string $method, float $rate, string $label ): void {
    $rate = abs( $rate ); // hoặc max( 0.0, $rate )
    // ...
}
```
