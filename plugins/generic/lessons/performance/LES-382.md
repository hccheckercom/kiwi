# Bug: Redis Không Giới Hạn Memory → OOM Kill

## Vấn đề

Redis mặc định `maxmemory 0` + `noeviction` → dùng hết RAM server → Linux OOM killer giết process → toàn bộ web down.

**Config nguy hiểm (default):**
```
maxmemory 0              # ← không giới hạn!
maxmemory-policy noeviction  # ← khi đầy → trả lỗi, không xóa gì
```

**Timeline sự cố:**
```
t=0:   Redis 1MB, mọi thứ OK
t=30d: Redis 500MB, vẫn OK
t=60d: Redis 1.5GB, bắt đầu swap
t=61d: RAM hết → OOM killer → redis-server killed → wp_cache_get() trả false
       → mọi request query DB → DB overload → 503
```

## Root Cause

1. WordPress Object Cache plugin KHÔNG set maxmemory
2. Redis default không giới hạn memory
3. `noeviction` policy → khi đầy trả error thay vì evict old keys
4. Không có TTL mặc định cho cached items

## Fix

### 1. Set maxmemory + eviction policy (BẮT BUỘC)

```bash
# Trên VPS
redis-cli CONFIG SET maxmemory 256mb
redis-cli CONFIG SET maxmemory-policy allkeys-lru
redis-cli CONFIG REWRITE

# Verify
redis-cli CONFIG GET maxmemory
redis-cli CONFIG GET maxmemory-policy
```

**Chọn policy:**
| Policy | Khi nào dùng |
|--------|-------------|
| `allkeys-lru` | **Recommended** — evict key ít dùng nhất |
| `volatile-lru` | Chỉ evict keys có TTL |
| `allkeys-lfu` | Evict key ít dùng nhất (frequency-based) |
| `noeviction` | **KHÔNG dùng cho production** |

### 2. Set default TTL trong wp-config.php

```php
// wp-config.php
define( 'WP_REDIS_MAXTTL', 86400 ); // 24 hours max TTL
```

### 3. Monitor memory usage

```bash
# Cron job: alert khi Redis dùng > 80% maxmemory
redis-cli INFO memory | grep used_memory_human
```

## Sizing Guide

| VPS RAM | Recommended maxmemory |
|---------|----------------------|
| 1 GB    | 128mb               |
| 2 GB    | 256mb               |
| 4 GB    | 512mb               |
| 8 GB    | 1gb                 |

Rule: Redis nên dùng **10-25% tổng RAM** — phần còn lại cho PHP-FPM, MySQL, OS.

## Verification

```bash
# Check config
redis-cli CONFIG GET maxmemory
redis-cli CONFIG GET maxmemory-policy

# Check current usage
redis-cli INFO memory | grep used_memory_human

# Check eviction stats
redis-cli INFO stats | grep evicted_keys
```
