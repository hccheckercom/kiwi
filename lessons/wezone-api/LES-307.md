## Bug

`wz_icon()` signature is `wz_icon(string $name, array $options = [])`.

Calling it with a CSS class string as 2nd argument causes PHP TypeError fatal:

```php
// WRONG — fatal error
wz_icon('shopping_cart', 'text-primary text-2xl');

// CORRECT
wz_icon('shopping_cart', ['size' => 'lg', 'color' => 'primary', 'class' => 'text-2xl']);
```

## Impact

- Page shows "critical error" — entire site broken
- Error only appears at runtime (no static analysis catches it)
- Common when copying icon calls from Tailwind-style components

## Fix

Always pass an array with named keys: `size`, `color`, `class`, `filled`, `label`, `echo`.
