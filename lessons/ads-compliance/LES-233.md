## Bad
```tsx
<Link href="">Xem chi tiết</Link>
<a href="#TODO">Liên hệ ngay</a>
<a href="javascript:void(0)">Mua ngay</a>

// Or: page returns 404/500 for valid product URLs
if (!product) return notFound()  // Without proper fallback
```
Broken links on landing pages cause immediate ad rejection. Google AdsBot crawls all links on the landing page — any 404/500 = "Non-functional destination" violation.

## Good
```tsx
// All links point to real, working pages
<Link href="/baogia">Xem chi tiết</Link>
<Link href="/contact">Liên hệ ngay</Link>

// Proper fallback for dynamic pages
if (!product) {
  redirect('/shop')  // Redirect to valid page instead of 404
}
```

## Why
Google Ads "Destination Requirements" policy states: "All URLs must be functional and lead to a working page." AdsBot-Google crawls landing pages AND all links on them. If any link returns 4xx/5xx, the ad is rejected with "Non-functional destination" error. This includes: (1) href="" or href="#" links, (2) links to pages that 404, (3) links to pages behind auth without proper handling, (4) javascript:void links. Fix: audit all links on pages used as ad destinations, ensure every href resolves to a 200 response.
