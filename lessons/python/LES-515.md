# Bad Code

```python
def build_html(items):
    html = ""
    for item in items:
        html += f"<li>{item}</li>"  # O(n²) - creates new string each iteration
    return f"<ul>{html}</ul>"

def format_log(entries):
    log = ""
    for entry in entries:
        log += f"{entry['timestamp']}: {entry['message']}\n"
    return log

# With 10,000 items: ~5 seconds
# With 100,000 items: ~8 minutes (!)
```

# Good Code

```python
def build_html(items):
    parts = []
    for item in items:
        parts.append(f"<li>{item}</li>")
    return f"<ul>{''.join(parts)}</ul>"

# Better: List comprehension
def build_html_v2(items):
    html = ''.join(f"<li>{item}</li>" for item in items)
    return f"<ul>{html}</ul>"

def format_log(entries):
    parts = [f"{e['timestamp']}: {e['message']}" for e in entries]
    return '\n'.join(parts)

# With 10,000 items: ~0.01 seconds
# With 100,000 items: ~0.1 seconds
```

# Why

**String concatenation with `+=` is O(n²):**
- Strings are immutable in Python
- Each `+=` creates a new string, copies all previous content
- With n iterations: 1 + 2 + 3 + ... + n = n²/2 operations

**Performance impact:**
```python
# Benchmark with 10,000 items
import timeit

# Bad: String concatenation
def bad():
    s = ""
    for i in range(10000):
        s += str(i)
    return s

# Good: List + join
def good():
    parts = []
    for i in range(10000):
        parts.append(str(i))
    return ''.join(parts)

print(timeit.timeit(bad, number=100))   # ~5.2 seconds
print(timeit.timeit(good, number=100))  # ~0.08 seconds
# 65x faster!
```

**When it matters:**
- Loops with >100 iterations
- Building HTML, JSON, CSV, logs
- Processing large files
- API responses with many items

**When it doesn't matter:**
- <10 iterations
- One-time scripts
- Known small datasets

**Alternatives:**
```python
# List comprehension + join (best)
result = ''.join(f"<li>{item}</li>" for item in items)

# io.StringIO (for complex building)
from io import StringIO
buf = StringIO()
for item in items:
    buf.write(f"<li>{item}</li>")
result = buf.getvalue()

# f-string with unpacking (small lists)
result = f"<ul>{''.join(items)}</ul>"
```
