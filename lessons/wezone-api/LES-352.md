## Bad
```php
$data[] = array(
    // ...
    'orders_count' => 0,        // Always zero — useless
    'total_spent'  => '0',      // Always zero — useless
);
```

## Good
```php
// Batch compute orders_count and total_spent from wz_orders
$stats_sql = "SELECT customer_id, COUNT(*) as cnt, COALESCE(SUM(total),0) as spent
              FROM {$table} WHERE customer_id IN ({$placeholders})
              AND status IN ('completed','processing')
              GROUP BY customer_id";
$stats = $wpdb->get_results( $wpdb->prepare( $stats_sql, $int_ids ), ARRAY_A );
$stats_map = [];
foreach ( $stats as $s ) {
    $stats_map[ (int) $s['customer_id'] ] = $s;
}

// Then in the loop:
'orders_count' => (int) ( $stats_map[ $uid ]['cnt'] ?? 0 ),
'total_spent'  => (string) ( $stats_map[ $uid ]['spent'] ?? '0' ),
```

## Why
Admin dashboard displays customer list with 0 orders and 0 spent — misleading for store operators. The data exists in `wz_orders` table; compute it with a single batch query.
