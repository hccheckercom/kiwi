## Triệu chứng

Chạy `deploy.ps1` → git push OK, VPS build OK, health check 200, nhưng giao diện trên production vẫn là bản cũ. Purge Cloudflare cache + xóa ISR cache + restart PM2 đều không fix.

## Root Cause

Deploy script flow: `git push` → VPS `git pull` → `npm run build`. Nếu file mới chưa được `git add` + `git commit` trước khi chạy deploy → git push chỉ push commit cũ → VPS pull không có file mới → build dùng code cũ → **deploy "thành công" nhưng UI không đổi**.

Đặc biệt nguy hiểm khi:
- Build local pass (vì file tồn tại trên local)
- Deploy script không check uncommitted/untracked files
- Health check pass (app chạy bình thường, chỉ thiếu UI mới)

## Bad

```powershell
# Tạo 20+ file mới → build local OK → deploy ngay
npm run build  # ✓ pass (file có trên local)
.\scripts\deploy.ps1  # ✓ "DEPLOY OK" nhưng VPS không có file mới
```

## Good

```powershell
# LUÔN commit trước khi deploy
git status  # Kiểm tra untracked/modified files
git add <files>
git commit -m "feat: ..."
.\scripts\deploy.ps1  # Giờ VPS mới có code mới
```

## Checklist trước deploy

1. `git status` — không còn file untracked/modified liên quan feature
2. `git log --oneline -1` — commit cuối chứa code cần deploy
3. Mới chạy `deploy.ps1`

## Prevention

Deploy script nên thêm guard:
```powershell
# Đầu deploy.ps1 — warn nếu có uncommitted changes
$dirty = git status --porcelain
if ($dirty) {
    Write-Warning "⚠️ Uncommitted changes detected! Deploy sẽ KHÔNG bao gồm:"
    $dirty | ForEach-Object { Write-Warning "  $_" }
    $confirm = Read-Host "Tiếp tục deploy? (y/N)"
    if ($confirm -ne 'y') { exit 1 }
}
```
