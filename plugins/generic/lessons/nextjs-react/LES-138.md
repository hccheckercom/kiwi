## Bad
```tsx
function MyComponent() {
  const supabase = createClient() // New instance every render!
  useEffect(() => {
    supabase.from('items').select('*').then(...)
  }, [supabase]) // supabase changes every render → infinite loop
}
```

## Good (pattern from hooks/useAuth.ts)
```tsx
function useAuth() {
  const supabase = useMemo(() => createClient(), [])
  // Single instance for component lifetime

  useEffect(() => {
    const { data: { subscription } } = supabase.auth.onAuthStateChange(...)
    return () => { subscription.unsubscribe() }
  }, [supabase]) // supabase is stable — effect runs once
}
```

## Why
Real pattern from useAuth.ts. createClient() allocates GoTrue client + cookie handler. In component body without useMemo, it creates a new instance every render. If used as useEffect dependency, it triggers infinite re-renders. useMemo ensures single instance.
