## Bad
```php
wp_localize_script( 'wezone-loyalty', 'wzLoyalty', [
    'i18n' => [
        'loading' => 'Loading...', // Hardcode English
        'noData'  => 'No data',
    ],
] );
```

## Good
```php
wp_localize_script( 'wezone-loyalty', 'wzLoyalty', [
    'i18n' => [
        'loading' => __( 'Loading...', 'wezone-loyalty' ),
        'noData'  => __( 'No data', 'wezone-loyalty' ),
    ],
] );
```

## Why
wp_localize_script() passes strings to JS. If hardcode 'Loading...' instead of __('Loading...', 'domain') then JS cannot translate when user changes language.

## Grep
`wp_localize_script\s*\([^)]+array\s*\([^)]*['"](loading|error|success|confirm)['"]\s*=>\s*['"][^'"]+['"](?!\s*__\()` — in `**/*.php`
