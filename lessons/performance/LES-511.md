## Good
```php
public function export_products( int $limit = 100, int $offset = 0 ): array {
    global $wpdb;
    $table = $wpdb->prefix . 'wz_products';
    
    // Validate limit to prevent memory exhaustion
    $limit = min( max( 1, $limit ), 500 );
    
    $rows = $wpdb->get_results(
        $wpdb->prepare(
            "SELECT * FROM {$table} WHERE status = 'publish' LIMIT %d OFFSET %d",
            $limit,
            $offset
        ),
        ARRAY_A
    );
    
    $data = array();
    foreach ( $rows as $row ) {
        $data[] = $this->build_product_export( $row );
    }
    
    return array(
        'data'   => $data,
        'limit'  => $limit,
        'offset' => $offset,
        'total'  => (int) $wpdb->get_var( $wpdb->prepare( "SELECT COUNT(*) FROM {$table} WHERE status = 'publish'" ) ),
    );
}
```

## Why
Large dataset (10k+ products) â†’ export loads all into memory â†’ 256MB limit exceeded â†’ fatal error. Must support pagination with configurable limit.
