## Bad
```ts
// Same page submitted multiple times with different URL formats
await pingIndexNow(['/blog/my-post'])
await pingIndexNow(['https://wezone.vn/blog/my-post'])
await pingIndexNow(['https://wezone.vn/blog/my-post/'])
// 3 submissions for the same page — wastes daily quota
```

## Good (pattern from lib/indexnow.ts)
```ts
const INDEXNOW_KEY = process.env.INDEXNOW_KEY ?? ''
const SITE_URL = process.env.NEXT_PUBLIC_APP_URL ?? 'https://wezone.vn'

export async function pingIndexNow(urls: string[]): Promise<boolean> {
  if (!INDEXNOW_KEY || urls.length === 0) return false

  try {
    const res = await fetch('https://api.indexnow.org/IndexNow', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json; charset=utf-8' },
      body: JSON.stringify({
        host: new URL(SITE_URL).hostname,
        key: INDEXNOW_KEY,
        keyLocation: `${SITE_URL}/${INDEXNOW_KEY}.txt`,
        // Normalize: relative paths → absolute URLs
        urlList: urls.map(u => u.startsWith('http') ? u : `${SITE_URL}${u}`),
      }),
    })
    return res.ok || res.status === 202
  } catch {
    return false
  }
}
```

## When to ping IndexNow
| Event | Ping? | URLs |
|-------|-------|------|
| Blog post published | Yes | `/blog/{slug}`, `/blog` (list page) |
| Product updated | Yes | `/san-pham/{slug}` |
| Page deleted | Yes (tells engines to de-index) | The deleted URL |
| Draft saved | No | — |
| Admin settings changed | No | — |

## Why
Real pattern from lib/indexnow.ts. IndexNow has daily submission limits. Without normalization, the same page gets submitted multiple times with different URL formats (relative vs absolute, trailing slash vs not). The function normalizes relative paths to absolute URLs. Also: guard against empty INDEXNOW_KEY (dev/CI environments).
