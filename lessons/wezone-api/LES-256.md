### Pattern sai

Admin SPA Tailwind config dùng hardcoded hex colors:

```js
// tailwind.config.js
colors: {
  wz: {
    primary: '#6366f1',
    sidebar: '#0F172A',
  }
}
```

Kết quả: mọi theme đều có admin panel màu indigo — không nhận diện được theme nào đang quản lý.

### Phải làm

Dùng CSS custom properties với fallback:

```js
// tailwind.config.js
colors: {
  wz: {
    primary: 'var(--wz-admin-primary, #6366f1)',
    sidebar: 'var(--wz-admin-sidebar, #0F172A)',
  }
}
```

PHP inject CSS variables từ `store-config.php`:

```php
// AdminPage.php — render_spa()
$theme_colors = $this->get_theme_colors();
echo '<style>:root{';
foreach ($theme_colors as $key => $value) {
    echo '--wz-admin-' . esc_attr($key) . ':' . esc_attr($value) . ';';
}
echo '}</style>';
```

### Lý do

- Mỗi theme có bảng màu riêng trong `store-config.php` (primary, accent, header_bg)
- Admin panel phải phản ánh brand identity của theme
- Khi bán theme cho khách hàng, admin tự động đồng bộ — zero config
- Bidirectional: Design System > Colors save → cập nhật cả admin lẫn theme

### Grep check

```
# Kiểm tra tailwind.config.js có dùng CSS variables chưa
grep "var(--wz-admin" wezone-plugins/packages/wezone-core/tailwind.config.js
# Kiểm tra AdminPage.php có inject CSS vars chưa
grep "wz-admin-primary" wezone-plugins/packages/wezone-core/src/AdminUI/AdminPage.php
```
