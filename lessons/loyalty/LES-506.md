## Bad
```php
public function earn_points( int $order_id ): void {
    $order = wz_get_order( $order_id );
    $points = floor( $order['total'] / 1000 );
    $this->repo->log_transaction( $user_id, 'earn', $points, $balance );
    // No duplicate check - concurrent requests earn 2x points
}
```

## Good
```php
public function earn_points( int $order_id ): void {
    global $wpdb;
    $wpdb->query( 'START TRANSACTION' );
    
    if ( $this->repo->has_earned_for_order( $order_id ) ) {
        $wpdb->query( 'ROLLBACK' );
        return;
    }
    
    $order = wz_get_order( $order_id );
    $points = floor( $order['total'] / 1000 );
    $ok = $this->repo->log_transaction( $user_id, 'earn', $points, $balance, $order_id );
    
    if ( ! $ok ) {
        $wpdb->query( 'ROLLBACK' );
        return;
    }
    
    $wpdb->query( 'COMMIT' );
}
```

## Why
Order completion webhook can fire twice (payment gateway retry, network timeout). Without has_earned_for_order() check in transaction, user earns 2x points. Must check + log in atomic transaction.

## Grep
`function\s+(?:earn|award|grant)_points.*?\{(?!.*has_earned_for_order|has_points_for|already_earned)` — in `**/*.php`
