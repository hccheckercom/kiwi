## Bad
```javascript
// tracking.js — has purchase/add_to_cart but no recovery events
dataLayer.push({ event: 'purchase', value: total });
dataLayer.push({ event: 'add_to_cart', ecommerce: { items } });
// No exit_intent_shown, no cart_recovered, no recovery funnel tracking
```
Recovery mechanisms exist but their effectiveness is invisible — cannot measure popup conversion rate, email recovery rate, or overall recovery revenue.

## Good
```javascript
// tracking.js — full recovery event coverage
// Exit-intent popup shown
dataLayer.push({ event: 'exit_intent_shown', cart_value: window.wzCart.total });

// Exit-intent converted (user clicked CTA)
dataLayer.push({ event: 'exit_intent_converted', cart_value: window.wzCart.total });

// Exit-intent dismissed
dataLayer.push({ event: 'exit_intent_dismissed' });

// Cart recovered (from email link or popup)
dataLayer.push({ event: 'cart_recovered', source: 'email', value: total });
dataLayer.push({ event: 'cart_recovered', source: 'exit_intent', value: total });

// Recovery email events (server-side push)
dataLayer.push({ event: 'recovery_email_opened', email_number: 1 });
dataLayer.push({ event: 'recovery_link_clicked', email_number: 2 });
```

## Why
Blueprint `conversion-recovery.md` Section 9.2 requires these dataLayer events for measuring recovery KPIs: exit-intent conversion rate (target 3-5%), email recovery rate (target 5-10%), overall recovery rate (target 10-15%). Without these events, GA4 cannot report on recovery funnel performance, Google Ads cannot optimize for recovered conversions, and the team cannot iterate on offer tiers.
