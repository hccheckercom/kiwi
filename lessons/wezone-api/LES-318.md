## Pattern

Admin actions (change order status, edit product price, delete customer, modify settings) execute without logging who did what. When issues arise (wrong price published, order cancelled unexpectedly), no way to trace which admin made the change or when.

## Bug

```php
// SAI — ai đổi status? khi nào? không ai biết
public function update_order_status( WP_REST_Request $request ): WP_REST_Response {
    $wpdb->update(
        $wpdb->prefix . 'wz_orders',
        array( 'status' => $new_status ),
        array( 'id' => $order_id )
    );
    return new WP_REST_Response( array( 'success' => true ) );
}
```

## Fix

```php
public function update_order_status( WP_REST_Request $request ): WP_REST_Response {
    $order_id   = (int) $request->get_param( 'id' );
    $new_status = sanitize_text_field( $request->get_param( 'status' ) );
    $old_status = $wpdb->get_var( $wpdb->prepare(
        "SELECT status FROM {$wpdb->prefix}wz_orders WHERE id = %d", $order_id
    ) );

    $wpdb->update(
        $wpdb->prefix . 'wz_orders',
        array( 'status' => $new_status ),
        array( 'id' => $order_id )
    );

    // Audit trail
    wz_log( 'order.status_changed', array(
        'order_id'   => $order_id,
        'old_status' => $old_status,
        'new_status' => $new_status,
        'admin_id'   => get_current_user_id(),
        'admin_name' => wp_get_current_user()->display_name,
        'ip'         => $_SERVER['REMOTE_ADDR'] ?? '',
    ) );

    return new WP_REST_Response( array( 'success' => true ) );
}
```

## Áp dụng

Mọi admin write operation (create/update/delete) trên business entities PHẢI gọi `wz_log()` với: entity type, entity id, action, old value, new value, admin_id. Đặc biệt quan trọng cho: order status, product price, coupon limits, payment settings.
