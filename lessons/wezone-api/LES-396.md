## Pattern

`list()` supports filtering by `customer_id`, `status`, and `search`, but `count()` only supports `status` and `search`. When paginating a customer's orders, the total count is wrong — showing total orders across ALL customers instead of just theirs.

## Wrong

```php
public function count(array $args = []): int {
    // Only handles 'status' and 'search' — missing 'customer_id'
    if (!empty($args['status'])) { /* ... */ }
    if (!empty($args['search'])) { /* ... */ }
    // ← customer_id filter missing
}
```

## Correct

```php
public function count(array $args = []): int {
    // ... status filter ...
    if (!empty($args['customer_id'])) {
        $where[]  = 'customer_id = %d';
        $params[] = (int) $args['customer_id'];
    }
    // ... search filter ...
}
```

## Why

Admin pagination shows "Page 1 of 50" for a customer who has 3 orders. Frontend "My Orders" page breaks with phantom pages. Any filter in `list()` must also exist in `count()`.
