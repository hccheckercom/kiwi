## Bad
```php
public function updatePrice( int $id, float $price ): bool {
    return $this->repo->update(
        $id,
        array(
            'regular_price' => $price,
            'price'         => $price,
            // sale_price still set from previous sale!
        )
    );
}
```

## Good
```php
public function updatePrice( int $id, float $price ): bool {
    return $this->repo->update(
        $id,
        array(
            'regular_price' => $price,
            'sale_price'    => null,
            'price'         => $price,
        )
    );
}
```

## Why
If a product had `sale_price=80000` and admin updates regular price to 100000 via `updatePrice()`, the old sale_price remains. Theme may still show "Was X, Now 80000d" even though the sale ended. Always clear sale_price when setting the new base price.
