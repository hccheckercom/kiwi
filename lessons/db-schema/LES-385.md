## Bad
```php
Database::register_table(
    'wezone-core',
    'wz_product_categories',
    array(
        'product_id'  => 'bigint(20) unsigned NOT NULL',
        'category_id' => 'bigint(20) unsigned NOT NULL',
    ),
    array(
        'idx_product'  => 'product_id',
        'idx_category' => 'category_id',
    )
);
```

## Good
```php
Database::register_table(
    'wezone-core',
    'wz_product_categories',
    array(
        'product_id'  => 'bigint(20) unsigned NOT NULL',
        'category_id' => 'bigint(20) unsigned NOT NULL',
    ),
    array(
        'UNIQUE idx_product_category' => 'product_id, category_id',
        'idx_category'                => 'category_id',
    )
);
```

## Why
Junction tables without `id AUTO_INCREMENT` have no PRIMARY KEY (dbDelta infers PK from AUTO_INCREMENT). Without UNIQUE composite index, duplicate rows are allowed. InnoDB creates a hidden clustered index (GEN_CLUST_INDEX) — poor performance. Fix: add `UNIQUE` on the FK pair.
