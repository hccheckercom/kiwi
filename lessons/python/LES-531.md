# Bad Code

```python
def process_order(order_id):
    print(f"Processing order {order_id}")
    order = get_order(order_id)
    print(f"Order status: {order.status}")
    
    if order.status == "pending":
        print("Charging payment")
        charge_payment(order)
        print("Payment charged successfully")
    
    print(f"Order {order_id} completed")
    return order
```

# Good Code

```python
import logging

logger = logging.getLogger(__name__)

def process_order(order_id):
    logger.info("Processing order %s", order_id)
    order = get_order(order_id)
    logger.debug("Order status: %s", order.status)
    
    if order.status == "pending":
        logger.info("Charging payment for order %s", order_id)
        try:
            charge_payment(order)
            logger.info("Payment charged successfully for order %s", order_id)
        except PaymentError as e:
            logger.error("Payment failed for order %s: %s", order_id, e)
            raise
    
    logger.info("Order %s completed", order_id)
    return order
```

# Why

**Logging advantages over print:**
- **Levels:** DEBUG, INFO, WARNING, ERROR, CRITICAL
- **Filtering:** Control verbosity per module
- **Formatting:** Timestamps, module names, line numbers
- **Destinations:** Files, syslog, cloud services
- **Performance:** Can be disabled in production
- **Context:** Structured logging with extra fields

**Production issues with print:**
- No timestamps or context
- Can't filter by severity
- Goes to stdout (mixed with app output)
- Can't route to log aggregation
- No log rotation
- Performance overhead (always executes)

**Logging best practices:**
```python
# Use lazy formatting (% not f-strings)
logger.info("User %s logged in", user_id)  # Good
logger.info(f"User {user_id} logged in")   # Bad (always formats)

# Use appropriate levels
logger.debug("Detailed state: %s", state)  # Development
logger.info("User action: %s", action)     # Normal flow
logger.warning("Deprecated API used")      # Warnings
logger.error("Failed to save: %s", err)    # Errors
logger.critical("Database down!")          # Critical

# Add context with extra
logger.info("Order processed", extra={
    "order_id": order_id,
    "user_id": user_id,
    "amount": total
})
```
