## Pattern

Constructors that accept path-related parameters (`$theme_slug`, `$dir`, `$folder`) without sanitization are vulnerable to path traversal attacks. If the parameter is used to construct file paths, an attacker can inject `../` sequences to access files outside the intended directory.

## Bug

```php
// SAI — $theme_slug không sanitize
class SeedImporter {
    private string $seed_dir;

    public function __construct( string $theme_slug ) {
        $this->seed_dir = WP_CONTENT_DIR . '/themes/' . $theme_slug . '/demo-seed';
    }

    public function import(): array {
        $manifest = $this->read_json( 'seed.json' );
        // ...
    }
}

// Attacker exploit
$importer = new SeedImporter( '../../uploads/malicious' );
// $seed_dir = '/var/www/wp-content/themes/../../uploads/malicious/demo-seed'
// = '/var/www/wp-content/uploads/malicious/demo-seed'
// → có thể đọc/import file từ uploads/ thay vì themes/
```

**Hậu quả:**
- Path traversal → đọc file ngoài intended directory
- Arbitrary file read → leak sensitive data (wp-config.php, .env)
- Malicious import → inject data từ attacker-controlled files
- Privilege escalation → bypass theme directory restriction

## Fix

### Option 1: Sanitize với `sanitize_title()` (recommended cho theme slugs)

```php
// ĐÚNG — sanitize_title() loại bỏ ../
class SeedImporter {
    private string $seed_dir;

    public function __construct( string $theme_slug ) {
        $theme_slug = sanitize_title( $theme_slug );
        $this->seed_dir = WP_CONTENT_DIR . '/themes/' . $theme_slug . '/demo-seed';
    }
}

// sanitize_title('../../malicious') → 'malicious' (loại bỏ ../)
```

### Option 2: Whitelist validation

```php
// ĐÚNG — validate against installed themes
class SeedImporter {
    private string $seed_dir;

    public function __construct( string $theme_slug ) {
        $themes = wp_get_themes();
        if ( ! isset( $themes[ $theme_slug ] ) ) {
            throw new \InvalidArgumentException( 'Invalid theme slug' );
        }
        
        $this->seed_dir = WP_CONTENT_DIR . '/themes/' . $theme_slug . '/demo-seed';
    }
}
```

### Option 3: Use `basename()` để strip path components

```php
// ĐÚNG — basename() loại bỏ directory separators
class SeedImporter {
    private string $seed_dir;

    public function __construct( string $theme_slug ) {
        $theme_slug = basename( $theme_slug );
        $this->seed_dir = WP_CONTENT_DIR . '/themes/' . $theme_slug . '/demo-seed';
    }
}

// basename('../../malicious') → 'malicious'
```

### Option 4: `realpath()` validation (defense-in-depth)

```php
// ĐÚNG — realpath() resolve .. và validate path
class SeedImporter {
    private string $seed_dir;

    public function __construct( string $theme_slug ) {
        $theme_slug = sanitize_title( $theme_slug );
        $seed_dir = WP_CONTENT_DIR . '/themes/' . $theme_slug . '/demo-seed';
        
        // Validate resolved path starts with themes/
        $real_path = realpath( $seed_dir );
        $themes_dir = realpath( WP_CONTENT_DIR . '/themes' );
        
        if ( ! $real_path || strpos( $real_path, $themes_dir ) !== 0 ) {
            throw new \InvalidArgumentException( 'Invalid theme directory' );
        }
        
        $this->seed_dir = $real_path;
    }
}
```

## Áp dụng

**Mọi constructor parameter dùng để construct file paths PHẢI sanitize:**
- Theme slugs → `sanitize_title()`
- Directory names → `sanitize_key()` hoặc `basename()`
- File paths → `realpath()` validation
- User-provided paths → whitelist validation

**Defense-in-depth:**
1. Sanitize input (first line of defense)
2. Validate resolved path (second line)
3. Check file exists before use (third line)
4. Restrict file operations to allowed directories

**Exceptions (không cần sanitize):**
- Hardcoded paths (không từ user input)
- Paths từ trusted sources (DB với proper validation)
- Paths đã sanitize ở caller (nhưng nên sanitize lại để defense-in-depth)
