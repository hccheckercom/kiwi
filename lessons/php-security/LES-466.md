# Bad Code

```php
// CheckoutController.php
public function process( WP_REST_Request $request ): WP_REST_Response {
    // ✅ Logged-in users: nonce verified
    if ( is_user_logged_in() ) {
        $nonce = $request->get_header( 'X-WP-Nonce' ) ?? '';
        if ( ! wp_verify_nonce( $nonce, 'wp_rest' ) ) {
            return new WP_REST_Response( array( 'success' => false, 'errors' => array( 'Invalid nonce.' ) ), 403 );
        }
    }
    
    // ❌ HIGH: Guest users — no session validation
    // → Attacker có thể guess session ID, hijack cart của guest khác
    $cart_items = $this->cart->getItems(); // Reads from $_COOKIE['wezone_session']
    
    // ... process checkout
}
```

**Attack scenario:**
1. Victim (guest) adds iPhone 15 Pro to cart → session ID = `wz_sess_abc123` (predictable)
2. Attacker brute-force session IDs: `wz_sess_abc120`, `wz_sess_abc121`, `wz_sess_abc122`, `wz_sess_abc123` ✓
3. Attacker sends checkout request with victim's session cookie
4. Order created with attacker's billing address → attacker nhận iPhone của victim

# Good Code

## Option 1: Session Regeneration (Recommended)

```php
// CheckoutController.php
public function process( WP_REST_Request $request ): WP_REST_Response {
    if ( is_user_logged_in() ) {
        $nonce = $request->get_header( 'X-WP-Nonce' ) ?? '';
        if ( ! wp_verify_nonce( $nonce, 'wp_rest' ) ) {
            return new WP_REST_Response( array( 'success' => false, 'errors' => array( 'Invalid nonce.' ) ), 403 );
        }
    } else {
        // ✅ Guest checkout: Regenerate session ID before processing
        // Prevents session fixation — attacker's pre-set session ID becomes invalid
        if ( function_exists( 'wz_regenerate_session' ) ) {
            wz_regenerate_session();
        }
        
        // ✅ Rate limiting at nginx level (not in PHP — too late)
        // nginx: limit_req_zone $binary_remote_addr zone=checkout:10m rate=5r/m;
    }
    
    $cart_items = $this->cart->getItems();
    
    if ( empty( $cart_items ) ) {
        return new WP_REST_Response( array( 'success' => false, 'errors' => array( 'Cart empty.' ) ), 422 );
    }
    
    // ... process checkout
}
```

## Option 2: CSRF Token for Guest (Alternative)

```php
// CheckoutController.php
public function process( WP_REST_Request $request ): WP_REST_Response {
    if ( is_user_logged_in() ) {
        $nonce = $request->get_header( 'X-WP-Nonce' ) ?? '';
        if ( ! wp_verify_nonce( $nonce, 'wp_rest' ) ) {
            return new WP_REST_Response( array( 'success' => false, 'errors' => array( 'Invalid nonce.' ) ), 403 );
        }
    } else {
        // ✅ Guest: Verify CSRF token (generated when cart created)
        $csrf_token = $request->get_header( 'X-CSRF-Token' ) ?? '';
        $stored     = wz_session_get( 'checkout_csrf_token' );
        
        if ( ! $csrf_token || ! hash_equals( $stored, $csrf_token ) ) {
            return new WP_REST_Response( array( 'success' => false, 'errors' => array( 'Invalid CSRF token.' ) ), 403 );
        }
    }
    
    // ... process checkout
}
```

# Why

**Security impact:**
- **Cart hijacking:** Attacker steals guest's cart, completes checkout with own address
- **Payment fraud:** Victim pays, attacker receives goods
- **PII leak:** Attacker sees victim's cart contents (privacy violation)

**Why session fixation is possible:**
1. **Predictable session IDs:** `wz_sess_{timestamp}_{counter}` → brute-forceable
2. **No session regeneration:** Session ID không đổi từ lúc add-to-cart → checkout
3. **No CSRF protection for guests:** Logged-in users có nonce, guests không có gì

**Why rate limiting alone is NOT enough:**
- Rate limiting chỉ slow down attacker, không prevent session fixation
- Attacker có thể dùng distributed IPs (botnet) để bypass rate limit
- Proper fix: Session regeneration + CSRF token

**Real-world incidents:**
- Magento 2018: Guest checkout session fixation → $200k fraud
- WooCommerce 2020: Predictable session IDs → cart hijacking campaign
