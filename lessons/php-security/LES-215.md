## Bad
```php
add_action('wp_ajax_wz_delete_order', [$this, 'handle_delete_order']);

public function handle_delete_order() {
    $order_id = intval($_POST['order_id']);
    $this->delete_order($order_id);
    wp_send_json_success();
}
```
No nonce check — attacker can craft a page that auto-submits this request while admin is logged in.

## Good
```php
public function handle_delete_order() {
    check_ajax_referer('wz_delete_order_nonce', 'security');
    $order_id = intval($_POST['order_id']);
    $this->delete_order($order_id);
    wp_send_json_success();
}
```

## Why
CSRF attacks trick authenticated users into executing unwanted actions. WordPress nonces expire after 24h and are tied to the user session. Every AJAX handler that modifies data must verify a nonce. Read-only handlers can skip this in specific cases.
