**Required fields cho OrderEngine::create():**
```
billing.first_name (string, non-empty)
billing.email (string, valid email)
billing.phone (string, non-empty)
items (array, non-empty — mỗi item cần: product_id, name, qty, price, subtotal)
payment_method (string: 'cod', 'bank_transfer', 'vnpay', etc.)
```
**Pattern sai:**
```php
// Checkout handler gửi thiếu field
$order_id = $engine->create([
    'billing' => [ 'email' => $email ],  // thiếu first_name, phone
    'items'   => $cart_items,
    'payment_method' => 'cod',
]);
// → $order_id = 0, đơn hàng KHÔNG được tạo, user thấy lỗi generic
```
**Phải làm:**
```php
$order_id = $engine->create([
    'billing' => [
        'first_name' => sanitize_text_field( $_POST['first_name'] ?? '' ),
        'email'      => sanitize_email( $_POST['email'] ?? '' ),
        'phone'      => sanitize_text_field( $_POST['phone'] ?? '' ),
    ],
    'shipping'       => $shipping_data,
    'items'          => $cart_items,
    'payment_method' => sanitize_text_field( $_POST['payment_method'] ?? 'cod' ),
    'currency'       => 'VND',
]);
```
**Lý do:** OrderEngine validate required fields trước khi INSERT. Thiếu bất kỳ field nào → return 0 ngay, không có error message cụ thể. Theme checkout form phải collect đủ fields VÀ validate client-side trước khi submit.
**Grep check:** `wz_shim_rest_checkout\|OrderEngine.*create` — verify payload có đủ billing.first_name + billing.email + billing.phone + items + payment_method → HIGH
