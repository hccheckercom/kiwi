## Bad
```php
public function restore_database( string $file ): void {
    $handle = fopen( $file, 'r' );
    while ( ( $line = fgets( $handle ) ) !== false ) {
        $wpdb->query( $line ); // No validation â€” corrupt backup = corrupt DB
    }
}
```

## Good
```php
public function restore_database( string $file, bool $dry_run = false ): void {
    $handle = fopen( $file, 'r' );
    $statements = array();
    while ( ( $line = fgets( $handle ) ) !== false ) {
        if ( $this->is_valid_sql( $line ) ) {
            $statements[] = $line;
        } else {
            throw new \Exception( 'Invalid SQL in backup' );
        }
    }
    if ( ! $dry_run ) {
        foreach ( $statements as $stmt ) {
            $wpdb->query( $stmt );
        }
    }
}
```

## Why
Executing SQL from backup files without validation can corrupt DB if backup is malformed or tampered. Add dry-run validation pass before applying.
