## Bad
```php
// AdminPage.php — registers submenu
add_submenu_page( 'wezone-admin', 'Mẫu thẻ giảm giá', ..., 'wezone-voucher-template', ... );

// VoucherTemplateAdminPage.php — registers SAME submenu again
add_submenu_page( 'wezone-admin', 'Mẫu thẻ giảm giá', ..., 'wezone-voucher-template', ... );

// Plugin.php boot() calls both → duplicate menu entry in WP admin
```

## Good
```php
// Only register submenu in ONE place — either AdminPage or VoucherTemplateAdminPage, not both
// Remove from AdminPage.php, keep in VoucherTemplateAdminPage.php since that's the dedicated class
```

## Why
WordPress silently accepts duplicate `add_submenu_page` with same slug. The second registration overwrites the first callback. While not a crash, it causes confusion and the duplicate appears in the menu.
