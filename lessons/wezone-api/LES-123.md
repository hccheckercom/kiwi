## Pattern

Template gọi `get_template_part('template-parts/product/card')` mà **không truyền `$args`**. Card component đọc `$args['product_id']` — khi thiếu args, `$product_id = 0` → card return early → trang trắng hoặc không hiển thị sản phẩm.

## Bad

```php
<?php foreach ( $wishlist as $product_id ) :
    $product = wz_get_product( (int) $product_id );
    if ( ! $product ) { continue; }
    set_query_var( 'wz_card_product', $product );
    get_template_part( 'template-parts/product/card' );
endforeach; ?>
```

## Good

```php
<?php foreach ( $wishlist as $product_id ) :
    get_template_part( 'template-parts/product/card', null, [
        'product_id' => (int) $product_id,
    ] );
endforeach; ?>
```

## Checklist

- [ ] `get_template_part` call includes 3rd argument `$args` with required keys
- [ ] Card component documents required `$args` keys in file header
- [ ] No reliance on `set_query_var` for passing data to template parts (WP 5.5+ supports `$args`)

## Root cause

Coder dùng pattern cũ (`set_query_var` + `get_template_part` 2 args) nhưng card component đã chuyển sang đọc `$args` (WP 5.5+ third parameter). Kết quả: data không đến được card → render trống.

## Scan rule

