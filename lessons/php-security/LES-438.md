## Problem

Logging raw email addresses violates PII handling principles. Log files are often stored unencrypted, shared with third-party log aggregators, and accessible by multiple team members. A data breach of log storage exposes all customer emails.

## Bad

```php
wz_log( "SubscriberManager::subscribe() DB insert failed for email={$email}", 'error' );
```

## Good

```php
$masked = substr_replace( $email, '***', 2, strpos( $email, '@' ) - 2 );
wz_log( "SubscriberManager::subscribe() DB insert failed for email={$masked}", 'error' );
```

## Apply

- Every `wz_log()` call that includes email, phone, or name MUST mask the value
- Use `substr_replace()` or a shared `wz_mask_email()` helper
- Pattern: keep first 2 chars + `***` + `@domain.com`
