## Problem

REST API returns raw customer email addresses. Even with `manage_options` permission, REST responses can be cached by CDN, logged by WAF, or intercepted — exposing PII unnecessarily.

## Bad

```php
$customers[] = array(
    'email' => $row->email ?? '',
    'name'  => $row->first_name . ' ' . $row->last_name,
);
```

## Good

```php
$customers[] = array(
    'email' => wz_mask_email( $row->email ?? '' ),
    'name'  => $row->first_name . ' ' . $row->last_name,
);

// In shared/src/Helpers.php:
function wz_mask_email( string $email ): string {
    $parts = explode( '@', $email );
    if ( count( $parts ) !== 2 ) return '***';
    $local = $parts[0];
    return substr( $local, 0, 2 ) . str_repeat( '*', max( 1, strlen( $local ) - 2 ) ) . '@' . $parts[1];
}
```

## Why

GDPR/data protection: PII should be minimized in API responses. Admin can see full emails on the customer detail page, not in analytics summary lists.
