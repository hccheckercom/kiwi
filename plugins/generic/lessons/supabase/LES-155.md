## Bad
```ts
await supabase.from('ai_providers').insert({
  name: 'Claude',
  api_key: 'sk-ant-api03-xxxxx', // Plaintext in DB — anyone with DB access sees it
})
```

## Good (pattern from lib/ai/provider-helpers.ts)
```ts
import { encryptApiKey, decryptApiKey, maskApiKey } from '@/lib/ai/provider-helpers'

// Store encrypted
await supabase.from('ai_providers').insert({
  name: 'Claude',
  api_key: encryptApiKey('sk-ant-api03-xxxxx'),
  // Stored as: "base64iv:base64tag:base64ciphertext"
})

// Read and decrypt when needed
const provider = await getProvider('claude')
const realKey = decryptApiKey(provider.api_key)

// Show to admin UI — masked
const display = maskApiKey(realKey) // "sk-a***xxxx"
```

## Encryption details (AES-256-GCM)
```ts
const ALGO = 'aes-256-gcm'
// Key derived from: AI_ENCRYPTION_KEY env || SUPABASE_SERVICE_ROLE_KEY || fallback
// Format: "iv:authTag:ciphertext" (all base64)
// Decrypt gracefully handles unencrypted legacy values (no ':' separator)
```

## Why
Real pattern from provider-helpers.ts. AI API keys (Claude, Gemini) cost $$ per call. If DB is compromised (SQL injection, backup leak, Supabase dashboard access), plaintext keys = unlimited spend. AES-256-GCM with auth tag prevents both reading and tampering. Mask for admin UI display.
