## Problem

Plugin `boot()` calls `wz_config()` but doesn't check `wezone_is_active()`. When admin disables the module, all hooks/cron/tracker are still registered — wasting resources and potentially causing errors.

## Bad

```php
public function boot(): void {
    if ( function_exists( 'wz_config' ) ) {
        wz_config( 'wezone-analytics' );
    }
    $this->register_settings();  // Runs even if disabled!
    $this->register_tracker();   // Registers hooks even if disabled!
}
```

## Good

```php
public function boot(): void {
    if ( function_exists( 'wz_config' ) ) {
        wz_config( 'wezone-analytics' );
    }
    if ( ! wezone_is_active( 'analytics' ) ) {
        return;
    }
    $this->register_settings();
    $this->register_tracker();
}
```

## Why

Feature-disabled modules should not register hooks, cron jobs, or database operations. This follows the Wezone plugin architecture pattern where `wezone_is_active()` is the master switch.
