# Bad Code

```python
import os

def load_config():
    config_dir = os.path.dirname(__file__)
    config_path = os.path.join(config_dir, 'config', 'settings.json')
    
    if os.path.exists(config_path) and os.path.isfile(config_path):
        with open(config_path) as f:
            return json.load(f)
    return {}

def get_data_files(base_dir):
    files = []
    for name in os.listdir(base_dir):
        path = os.path.join(base_dir, name)
        if os.path.isfile(path) and name.endswith('.json'):
            files.append(path)
    return files
```

# Good Code

```python
from pathlib import Path

def load_config():
    config_path = Path(__file__).parent / 'config' / 'settings.json'
    
    if config_path.is_file():
        return json.loads(config_path.read_text())
    return {}

def get_data_files(base_dir):
    base = Path(base_dir)
    return [f for f in base.iterdir() if f.suffix == '.json']

# Or using glob
def get_data_files_v2(base_dir):
    return list(Path(base_dir).glob('*.json'))
```

# Why

**`pathlib` advantages:**
- Object-oriented, chainable API
- Cross-platform (handles `/` vs `\\` automatically)
- More readable with `/` operator
- Built-in methods: `.read_text()`, `.write_text()`, `.glob()`
- Type-safe (Path objects vs strings)

**Common operations comparison:**
```python
# os.path (old way)
path = os.path.join(os.path.dirname(__file__), 'data', 'file.txt')
if os.path.exists(path):
    with open(path) as f:
        content = f.read()

# pathlib (modern way)
path = Path(__file__).parent / 'data' / 'file.txt'
if path.exists():
    content = path.read_text()
```

**When to still use `os.path`:**
- Legacy code compatibility
- Performance-critical code (pathlib has slight overhead)
- Working with string paths from external APIs

**Migration tips:**
```python
# Convert string to Path
path = Path('/some/path')

# Convert Path to string
str(path)

# Works with open()
with open(path) as f:  # Path objects work here
    ...
```
