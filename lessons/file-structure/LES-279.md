## Bad
```
main.css imports _dark-mode.css, _animations.css, _conversion.css
main.bundle.css does NOT contain those styles
→ functions.php prefers bundle → features invisible in browser
```

Theme adds new CSS partial and imports it in `main.css`, but forgets to rebuild `main.bundle.css`. Since `functions.php` prefers the bundle file when it exists, the new styles never reach the browser.

## Good
After adding/modifying any CSS partial, always rebuild the bundle:
```powershell
# PowerShell — concatenate all partials in order
$cssDir = "themes/{name}/assets/css"
$partials = Get-Content "$cssDir/main.css" | Where-Object { $_ -match "@import" } |
  ForEach-Object { ($_ -replace "^@import\s*['""](.+?)['""];.*", '$1') }
$output = "/* Bundled CSS (auto-generated) */`n"
foreach ($f in $partials) {
  $path = Join-Path $cssDir $f
  if (Test-Path $path) { $output += "`n/* === $f === */`n" + (Get-Content $path -Raw) + "`n" }
}
[IO.File]::WriteAllText("$cssDir/main.bundle.css", $output, [Text.UTF8Encoding]::new($false))
```

## Why
`functions.php` uses this pattern:
```php
$bundle_css = THEME_DIR . '/assets/css/main.bundle.css';
$main_css   = file_exists( $bundle_css ) ? $bundle_css : THEME_DIR . '/assets/css/main.css';
```
If `main.bundle.css` exists but is stale, the browser gets old CSS. Dark mode, animations, or any new feature becomes completely invisible despite the source files being correct.

## Detection
Compare `@import` list in `main.css` against actual content in `main.bundle.css`. If any imported partial's key selectors are missing from the bundle, it's stale.

## Real incident
- **Theme:** wezone-trunganh-v2 (2026-05-22)
- **Symptom:** Dark mode toggle button visible, JS works (data-theme changes), but zero visual change
- **Root cause:** `_dark-mode.css` added to `main.css` imports but `main.bundle.css` never rebuilt
- **Fix:** Rebuild bundle by concatenating all 46 partials in correct order
