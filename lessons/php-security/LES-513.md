## Good
```php
public function import_products( array $items ): array {
    $success = 0;
    $failed  = 0;
    $errors  = array();
    
    // Validate batch size from settings
    $max_batch = (int) ( \WeZone\Core\Settings::get( 'wezone-migrator', 'import_batch_size', '50' ) );
    $max_batch = min( max( 1, $max_batch ), 200 ); // Cap at 200
    
    if ( count( $items ) > $max_batch ) {
        return array(
            'total'   => 0,
            'success' => 0,
            'failed'  => 0,
            'errors'  => array( sprintf( 'Batch size %d exceeds limit %d', count( $items ), $max_batch ) ),
        );
    }
    
    foreach ( $items as $item ) {
        try {
            $product_id = wz_create_product( $item );
            ++$success;
        } catch ( \Exception $e ) {
            ++$failed;
            $errors[] = $e->getMessage();
        }
    }
    
    return array(
        'total'   => count( $items ),
        'success' => $success,
        'failed'  => $failed,
        'errors'  => $errors,
    );
}
```

## Why
Attacker sends 10k products in single request â†’ memory exhausted â†’ fatal error. Must validate batch size against configurable limit.
