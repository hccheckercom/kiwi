## Good
```php
public function sideload( string $url, int $post_id ): ?int {
    require_once ABSPATH . 'wp-admin/includes/media.php';
    require_once ABSPATH . 'wp-admin/includes/file.php';
    require_once ABSPATH . 'wp-admin/includes/image.php';
    
    // Validate URL scheme
    if ( ! filter_var( $url, FILTER_VALIDATE_URL ) || ! in_array( wp_parse_url( $url, PHP_URL_SCHEME ), array( 'http', 'https' ), true ) ) {
        return null;
    }
    
    $tmp = download_url( $url );
    if ( is_wp_error( $tmp ) ) {
        return null;
    }
    
    // Validate MIME type before sideload
    $filetype = wp_check_filetype( basename( $url ), array(
        'jpg|jpeg|jpe' => 'image/jpeg',
        'png'          => 'image/png',
        'gif'          => 'image/gif',
        'webp'         => 'image/webp',
    ) );
    
    if ( ! $filetype['type'] ) {
        @unlink( $tmp );
        return null;
    }
    
    $file = array(
        'name'     => basename( $url ),
        'type'     => $filetype['type'],
        'tmp_name' => $tmp,
    );
    
    $id = media_handle_sideload( $file, $post_id );
    @unlink( $tmp );
    
    return is_wp_error( $id ) ? null : $id;
}
```

## Why
Attacker uploads malicious file (PHP shell, SVG with XSS) disguised as image â†’ RCE or XSS. Must validate MIME type + URL scheme before sideload.
