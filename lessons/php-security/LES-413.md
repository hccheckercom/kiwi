## Pattern

Public methods that modify WordPress state (options, database, files) without nonce verification are vulnerable to CSRF attacks. If these methods are callable from AJAX handlers, form submissions, or URL parameters, an attacker can craft malicious requests that execute with the victim's credentials.

## Bug

```php
// SAI — public method without nonce check
class Commerce {
    public static function disable( string $module ): void {
        if ( ! current_user_can( 'manage_options' ) ) {
            return;
        }
        
        if ( in_array( $module, self::MODULES, true ) ) {
            update_option( "wezone_commerce_{$module}_enabled", false, false );
        }
    }
}

// AJAX handler gọi method này
add_action( 'wp_ajax_disable_module', function() {
    $module = $_POST['module'] ?? '';
    Commerce::disable( $module ); // CSRF vulnerability!
    wp_send_json_success();
});

// Attacker crafts malicious request
// <img src="https://victim-site.com/wp-admin/admin-ajax.php?action=disable_module&module=products">
// Khi admin load trang có img này → products module bị tắt
```

**Hậu quả:**
- CSRF attack → attacker có thể trigger state changes với victim's credentials
- No user intent verification → actions execute automatically
- Persistent damage → state changes survive page reload
- Privilege escalation → low-privilege user tricks admin into executing action

## Fix

### Option 1: Add nonce check in AJAX handler (recommended for AJAX)

```php
// ĐÚNG — nonce check trong AJAX handler
add_action( 'wp_ajax_disable_module', function() {
    // Verify nonce
    check_ajax_referer( 'wezone_commerce_settings', 'nonce' );
    
    $module = sanitize_key( $_POST['module'] ?? '' );
    Commerce::disable( $module );
    wp_send_json_success();
});

// Frontend JS phải gửi nonce
fetch( ajaxurl, {
    method: 'POST',
    body: new URLSearchParams({
        action: 'disable_module',
        module: 'products',
        nonce: wzAdmin.nonce, // từ wp_localize_script
    }),
});
```

### Option 2: Add nonce check inside method (defense-in-depth)

```php
// ĐÚNG — nonce check trong method
class Commerce {
    public static function disable( string $module ): void {
        // Capability check
        if ( ! current_user_can( 'manage_options' ) ) {
            wp_die( 'Unauthorized', 403 );
        }
        
        // Nonce check (defense-in-depth)
        if ( ! check_admin_referer( 'wezone_commerce_settings' ) ) {
            wp_die( 'Invalid nonce', 403 );
        }
        
        $module = sanitize_key( $module );
        if ( in_array( $module, self::MODULES, true ) ) {
            update_option( "wezone_commerce_{$module}_enabled", false, false );
        }
    }
}
```

### Option 3: Make method private, expose via nonce-protected API

```php
// ĐÚNG — private method, public API có nonce check
class Commerce {
    private static function disable( string $module ): void {
        $module = sanitize_key( $module );
        if ( in_array( $module, self::MODULES, true ) ) {
            update_option( "wezone_commerce_{$module}_enabled", false, false );
        }
    }
    
    // Public API với nonce + capability check
    public static function admin_disable_module( string $module, string $nonce ): bool {
        if ( ! current_user_can( 'manage_options' ) ) {
            return false;
        }
        
        if ( ! wp_verify_nonce( $nonce, 'wezone_commerce_settings' ) ) {
            return false;
        }
        
        self::disable( $module );
        return true;
    }
}
```

### Option 4: REST API with built-in nonce verification

```php
// ĐÚNG — REST API tự động verify nonce qua X-WP-Nonce header
register_rest_route( 'wezone/v1', '/commerce/(?P<module>[a-z-]+)/disable', array(
    'methods'             => 'POST',
    'callback'            => function( $request ) {
        $module = $request['module'];
        Commerce::disable( $module );
        return new WP_REST_Response( array( 'success' => true ), 200 );
    },
    'permission_callback' => fn() => current_user_can( 'manage_options' ),
));

// Frontend JS — WordPress REST API tự động verify nonce
fetch( wpApiSettings.root + 'wezone/v1/commerce/products/disable', {
    method: 'POST',
    headers: {
        'X-WP-Nonce': wpApiSettings.nonce,
    },
});
```

## Áp dụng

**Mọi public method modify state PHẢI có nonce protection nếu:**
1. Callable từ AJAX handlers (, )
2. Callable từ form submissions (admin pages, frontend forms)
3. Callable từ URL parameters (, )

**Nonce verification methods:**
-  — cho admin form submissions
-  — cho AJAX requests
-  — manual verification
- REST API — automatic via  header (no manual check needed)

**Exceptions (không cần nonce):**
- Private methods (không callable từ frontend)
- Methods chỉ gọi từ cron/CLI (không có user session)
- Methods marked `@internal` với clear documentation
- Read-only methods (không modify state)
- **WordPress filter callbacks** (e.g., `wp_mail_from`, `wp_mail_from_name`) — chỉ được gọi bởi WordPress core trong internal flow, không callable từ AJAX/frontend

**Defense-in-depth strategy:**
1. Nonce check trong AJAX handler (first line of defense)
2. Capability check trong method (authorization)
3. Input sanitization (prevent injection)
4. Audit logging (detect abuse)
