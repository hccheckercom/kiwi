## Pattern

Table schema có columns thường xuyên xuất hiện trong WHERE/ORDER BY/GROUP BY nhưng không có index → full table scan.

## Bug Example

```php
// ❌ BAD — không có indexes
return array(
    'wezone_popup_stats' => array(
        'id'          => 'bigint(20) unsigned NOT NULL AUTO_INCREMENT PRIMARY KEY',
        'popup_type'  => 'varchar(50) NOT NULL',      // query by type
        'variant'     => 'varchar(10) NOT NULL',      // query by variant
        'views'       => 'int unsigned NOT NULL DEFAULT 0',
        'conversions' => 'int unsigned NOT NULL DEFAULT 0',
        'date'        => 'date NOT NULL',             // query by date range
    ),
);

// Query sẽ chậm — full table scan
SELECT SUM(conversions) FROM wezone_popup_stats
WHERE popup_type = 'email_collect' AND date >= '2026-05-01';
```

## Why It's Wrong

1. **Performance degradation** — O(n) scan thay vì O(log n) index lookup
2. **Scalability issues** — chậm dần khi data tăng
3. **High CPU usage** — MySQL phải scan toàn bộ table
4. **Slow admin dashboard** — analytics queries timeout

## Correct Approach

```php
// ✅ GOOD — indexes cho query patterns
return array(
    'wezone_popup_stats' => array(
        'id'          => 'bigint(20) unsigned NOT NULL AUTO_INCREMENT PRIMARY KEY',
        'popup_type'  => 'varchar(50) NOT NULL',
        'variant'     => 'varchar(10) NOT NULL',
        'views'       => 'int unsigned NOT NULL DEFAULT 0',
        'conversions' => 'int unsigned NOT NULL DEFAULT 0',
        'date'        => 'date NOT NULL',
        // Single-column indexes
        'KEY popup_type' => '(popup_type)',
        'KEY date' => '(date)',
        // Composite index cho common query pattern
        'KEY type_date' => '(popup_type, date)',
        'KEY type_variant_date' => '(popup_type, variant, date)',
    ),
);
```

**Index strategy:**
1. **Single-column indexes** — cho simple WHERE clauses
2. **Composite indexes** — cho multi-column queries (leftmost prefix rule)
3. **Covering indexes** — include SELECT columns để avoid table lookup

**Query optimization:**
```php
// Query sẽ dùng index type_date
SELECT SUM(conversions) FROM wezone_popup_stats
WHERE popup_type = 'email_collect' AND date >= '2026-05-01';

// Query sẽ dùng index type_variant_date (covering index)
SELECT popup_type, variant, date, SUM(conversions)
FROM wezone_popup_stats
WHERE popup_type = 'email_collect' AND variant = 'A'
GROUP BY date;
```

## Detection (Manual Check)

**Vì sao manual?** Scanner regex không thể detect "column có trong schema nhưng không có KEY entry tương ứng" — cần semantic analysis của toàn bộ array structure.

**Cách check thủ công:**

1. Đọc file `tables.php` hoặc schema definition
2. List tất cả columns trong mỗi table
3. Với mỗi column, check xem có xuất hiện trong WHERE/ORDER BY/GROUP BY queries không
4. Nếu có → check xem có `'KEY column_name' => '(column_name)'` entry hay không
5. Nếu không có KEY/UNIQUE → violation

**Columns thường cần index:**
- `email` (nếu không UNIQUE thì cần KEY)
- `date`, `created_at` (range queries)
- `status`, `type`, `variant` (categorical filters)
- Foreign keys (JOIN performance)
- Columns trong composite queries (cần composite index)

**Ví dụ check:**
```php
// Table có 5 columns
'wezone_popup_stats' => array(
    'id'          => '...',
    'popup_type'  => 'varchar(50) NOT NULL',  // ← check: có query by popup_type?
    'variant'     => 'varchar(10) NOT NULL',  // ← check: có query by variant?
    'date'        => 'date NOT NULL',         // ← check: có query by date?
    // ... có KEY entries cho 3 columns trên không?
),
```

## Impact

- **Performance**: Full table scan → 100-1000x slower khi data lớn
- **Scalability**: Query time tăng tuyến tính với data size
- **User experience**: Admin dashboard timeout, slow reports
