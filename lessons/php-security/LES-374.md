# Bug: Session destroy() Không Clear cached_key — Stale Session Reuse

## Vấn đề

`SessionHandler::destroy()` xóa cookie nhưng **không** clear `$this->cached_key`. Nếu code gọi `destroy()` rồi `getKey()` trong cùng request → trả về key cũ (đã bị xóa cookie) → thao tác tiếp trên session "đã destroy".

**Ví dụ lỗi:**
```php
// ❌ SAI — cached_key vẫn giữ giá trị cũ
public function destroy(): void {
    if ( ! headers_sent() ) {
        setcookie( self::COOKIE_NAME, '', array(
            'expires' => time() - 3600,
            // ...
        ));
    }
    // $this->cached_key vẫn = key cũ!
}
```

## Fix

```php
// ✅ ĐÚNG — clear cached_key
public function destroy(): void {
    $this->cached_key = null;

    if ( ! headers_sent() ) {
        setcookie( self::COOKIE_NAME, '', array(
            'expires' => time() - 3600,
            'path'    => COOKIEPATH,
            'domain'  => COOKIE_DOMAIN,
            'secure'  => is_ssl(),
            'httponly' => true,
            'samesite' => 'Lax',
        ));
    }
}
```
