## Bad
```tsx
// GoogleAnalytics.tsx and MetaPixel.tsx are installed
// BUT no CookieConsent component exists
// Tracking fires immediately without user consent
```
Both Google and Meta require consent mechanisms for tracking cookies. GDPR (EU users) requires explicit opt-in. Google Ads can reject landing pages that fire tracking without consent.

## Good
```tsx
// CookieConsent component with:
// 1. Clear explanation of what cookies are used
// 2. "Accept" and "Decline" buttons
// 3. Link to privacy policy
// 4. Tracking only fires AFTER consent
// 5. Consent stored in localStorage

<CookieConsent
  onAccept={() => { initGA4(); initMetaPixel(); }}
  onDecline={() => { /* no tracking */ }}
  privacyLink="/privacy"
/>
```

## Why
Google's "Consent Mode" (required since March 2024) requires that tracking respects user consent. Meta's "Limited Data Use" flag requires consent before pixel fires. Without consent mechanism: (1) Google Ads may reject landing page, (2) Meta may restrict ad account, (3) GDPR fines up to 4% of revenue for EU-targeting ads. The consent banner must offer genuine choice (not just "OK" button).
