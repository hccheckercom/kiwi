## Bad
```php
$shop_name = wz_config( 'shop_name', 'FUNILUX' );
$phone     = wz_config( 'phone', '1900 xxxx' );
$email     = wz_config( 'email', 'support@funilux.vn' );
```

## Good
```php
$shop_name = wz_config( 'shop_name', get_bloginfo( 'name' ) );
$phone     = wz_config( 'phone', '' );
$email     = wz_config( 'email', '' );
```

## Why
`wz_config()` 2nd parameter là fallback khi config chưa load. Nếu fallback chứa brand text cụ thể (tên shop, SĐT, email riêng), khi clone theme sang shop mới mà store-config.php chưa kịp load → hiển thị brand cũ. 26 files trong funilux đã mắc lỗi này. Fallback đúng: `get_bloginfo('name')` cho tên, `''` cho contact info (template sẽ ẩn nếu rỗng).

## Impact
- Clone theme hiển thị tên/SĐT shop cũ trên 20+ trang
- Google crawl thấy brand không nhất quán → GMC NAP consistency fail (M03-006)
- User thấy thông tin sai → mất trust

## Grep check
`wz_config\(\s*'shop_name',\s*'[A-Z]` hoặc `wz_config\(\s*'phone',\s*'[0-9]` trong *.php (trừ store-config.php) → HIGH
