## Vấn đề

Khi nhiều AJAX action (save, get_history, restore_snapshot) dùng **cùng 1 nonce string**, attacker chỉ cần leak 1 nonce là có thể gọi tất cả actions.

Đặc biệt nguy hiểm khi:
- GET action (read-only) expose nonce qua URL/referrer
- Nonce đó cũng dùng cho write action (save/restore)

## Sai

```php
// Cả 3 action dùng chung 1 nonce
add_action( 'wp_ajax_wz_save_design_tokens', ... );
add_action( 'wp_ajax_wz_get_design_history', ... );
add_action( 'wp_ajax_wz_restore_design_snapshot', ... );

// Trong cả 3 handler:
check_ajax_referer( 'wz_save_design_tokens', 'nonce' );
```

## Đúng

```php
// Mỗi action có nonce riêng
wp_create_nonce( 'wz_save_design_tokens' );
wp_create_nonce( 'wz_get_design_history' );
wp_create_nonce( 'wz_restore_design_snapshot' );

// Handler check đúng nonce
public function ajax_get_history(): void {
    check_ajax_referer( 'wz_get_design_history', 'nonce' );
}

public function ajax_restore_snapshot(): void {
    check_ajax_referer( 'wz_restore_design_snapshot', 'nonce' );
}
```

## Tại sao quan trọng

- CSRF nonce nên isolate theo action — principle of least privilege
- Nếu nonce bị leak từ 1 endpoint, không ảnh hưởng endpoint khác
