# Bad Code

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/api/users")
async def get_users():
    return {"users": []}

# Frontend at http://localhost:3000 gets CORS error
# Browser blocks request from different origin
```

# Good Code

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI()

# Production: Specific origins only
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "https://app.example.com",
        "https://admin.example.com"
    ],
    allow_credentials=True,
    allow_methods=["GET", "POST", "PUT", "DELETE"],
    allow_headers=["*"],
)

# Development: Localhost only
if settings.ENV == "development":
    app.add_middleware(
        CORSMiddleware,
        allow_origins=["http://localhost:3000"],
        allow_credentials=True,
        allow_methods=["*"],
        allow_headers=["*"],
    )

@app.get("/api/users")
async def get_users():
    return {"users": []}
```

# Why

**Without CORS middleware:**
- Browser blocks cross-origin requests
- Frontend can't call API from different domain
- Development blocked (frontend on :3000, API on :8000)

**Security risks of `allow_origins=["*"]`:**
- Any website can call your API
- Credentials exposed to malicious sites
- CSRF attacks possible
- Data leakage

**Correct CORS setup:**
```python
# ✅ GOOD: Specific origins
allow_origins=["https://app.example.com"]

# ❌ BAD: Wildcard in production
allow_origins=["*"]

# ✅ GOOD: Environment-based
origins = ["https://app.example.com"]
if settings.ENV == "dev":
    origins.append("http://localhost:3000")
```

**Credentials handling:**
```python
# If using cookies/auth headers
allow_credentials=True
allow_origins=["https://app.example.com"]  # Must be specific

# Cannot use both
allow_credentials=True
allow_origins=["*"]  # ERROR: Not allowed together
```
