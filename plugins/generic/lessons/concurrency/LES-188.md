## Bad
```tsx
<button type="submit" className="btn-primary">
  Place Order
</button>
```
User double-clicks → two POST requests → two orders created.

## Good
```tsx
const [isPending, startTransition] = useTransition()

<button type="submit" disabled={isPending} className="btn-primary">
  {isPending ? 'Processing...' : 'Place Order'}
</button>
```

## Why
Network latency makes double-submit easy — user clicks, nothing happens for 500ms, clicks again. Without client-side debounce AND server-side idempotency, you get duplicate orders. Client-side is the first line of defense.
