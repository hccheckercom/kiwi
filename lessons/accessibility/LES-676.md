## Bad
```tsx
<input type="email" placeholder="Enter email" className="border p-2 rounded" />
```
Placeholder is not a label — disappears on focus, not announced by all screen readers.

## Good
```tsx
<label htmlFor="email" className="text-sm font-medium">Email</label>
<input id="email" type="email" className="border p-2 rounded" />
```
Or for visually hidden labels:
```tsx
<label htmlFor="search" className="sr-only">Search products</label>
<input id="search" type="search" placeholder="Search..." />
```

## Why
WCAG 1.3.1 + 4.1.2 — form inputs must have programmatically associated labels. Placeholder text is not a substitute: it disappears on input, has poor contrast, and isn't reliably announced.
