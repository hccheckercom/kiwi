# Bad Code

```python
def get_user_name(user_dict):
    try:
        return user_dict['name']
    except KeyError:
        return "Unknown"

def process_config(config):
    try:
        timeout = config['timeout']
    except KeyError:
        timeout = 30
    return timeout
```

# Good Code

```python
def get_user_name(user_dict):
    return user_dict.get('name', 'Unknown')

def process_config(config):
    return config.get('timeout', 30)

# Or use `in` for explicit check
def get_user_name_v2(user_dict):
    if 'name' in user_dict:
        return user_dict['name']
    return 'Unknown'
```

# Why

**EAFP vs LBYL in Python:**
- EAFP (Easier to Ask Forgiveness than Permission): `try/except`
- LBYL (Look Before You Leap): `if key in dict`

**For simple key access, use `.get()`:**
- More readable and concise
- Faster (no exception overhead)
- Pythonic idiom

**When to use `try/except`:**
- Multiple operations that might fail
- Different exceptions need different handling
- Performance-critical code where key usually exists

**Performance comparison:**
```python
# .get() - fastest
value = d.get('key', default)

# in check - fast
if 'key' in d:
    value = d['key']

# try/except - slowest when key missing
try:
    value = d['key']
except KeyError:
    value = default
```
