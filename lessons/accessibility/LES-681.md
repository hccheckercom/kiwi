## Bad
```tsx
<h1>Sản phẩm</h1>
<section>
  <h3>Mô tả</h3>  {/* Skip h2! */}
  <h4>Chi tiết</h4>
</section>
```

## Good
```tsx
<h1>Sản phẩm</h1>
<section>
  <h2>Mô tả</h2>
  <h3>Chi tiết</h3>
</section>
```

## Why
WCAG 1.3.1: heading hierarchy phải sequential (h1→h2→h3). Skip levels khiến screen reader users mất context về document structure. Lưu ý: checker chỉ flag h3+ trong page.tsx — cần manual review heading order.
