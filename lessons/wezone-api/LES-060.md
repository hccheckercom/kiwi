**Triệu chứng:**
- Click "Chi tiết" từ /my-account/ hoặc /my-account/orders/ → URL đúng `/my-account/orders/5/` nhưng bị redirect ngay về `/my-account/orders/`
- Không có error log, không có 404 — silent redirect loop
**Root cause:**
- Router.php add rewrite rules mỗi lần `init` nhưng không gọi `flush_rewrite_rules()` sau khi add
- Nếu rules chưa được flush vào DB (lần đầu activate theme, hoặc sau khi update Router), WordPress không nhận ra rule `^my-account/orders/([0-9]+)/?$`
- URL `/my-account/orders/5/` không match rewrite → `wz_order_id` = 0 → `order-detail.php` redirect về `/my-account/orders/`
- Tracking plugin có flush riêng, nhưng Router chính không có → rules của Router có thể stale
**Pattern sai:**
```php
// Router.php — add rules nhưng không flush
public function add_rewrite_rules(): void {
    add_rewrite_rule( '^my-account/orders/([0-9]+)/?$', '...&wz_order_id=$matches[1]', 'top' );
    // Không có flush → rules chỉ active sau khi admin vào Settings > Permalinks > Save
}
```
**Phải làm:**
```php
public function add_rewrite_rules(): void {
    add_rewrite_rule( '^my-account/orders/([0-9]+)/?$', '...&wz_order_id=$matches[1]', 'top' );
    // One-time flush sau khi add rules
    if ( get_option( 'wz_router_rewrite_flushed' ) !== '1' ) {
        flush_rewrite_rules( false );
        update_option( 'wz_router_rewrite_flushed', '1' );
    }
}
// Và trong theme setup.php — reset flag khi switch theme để force re-flush
add_action( 'after_switch_theme', function() {
    delete_option( 'wz_router_rewrite_flushed' );
    flush_rewrite_rules( false );
} );
```
**Lý do:** Router không flush rewrite rules → WordPress không biết custom routes → URL /my-account/orders/123/ trả 404 hoặc redirect loop vì wz_order_id luôn = 0.
