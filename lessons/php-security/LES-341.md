## Bad
```php
public static function render( string $template, array $data = array() ): string {
    $path = self::getTemplate( $template );
    ob_start();
    extract( $data, EXTR_SKIP ); // if $data has 'path' key, it shadows $path
    include $path;
    return ob_get_clean();
}
```

## Good
```php
public static function render( string $template, array $data = array() ): string {
    $__path = self::getTemplate( $template );
    if ( $__path === '' ) {
        return '';
    }
    ob_start();
    try {
        extract( $data, EXTR_SKIP );
        include $__path;
    } catch ( \Throwable $e ) {
        ob_end_clean();
        return '';
    }
    return ob_get_clean() ?: '';
}
```

## Why
`extract()` with EXTR_SKIP won't overwrite existing vars, but local var names like `$path`, `$template`, `$data` could collide with template data keys. Use double-underscore prefix (`$__path`) for internal vars to avoid shadowing.
