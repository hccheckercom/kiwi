## Bad (denylist — always bypassable)
```ts
function sanitize(html: string) {
  return html
    .replace(/<script>/gi, '')  // Bypassed by: <scr<script>ipt>
    .replace(/onclick/gi, '')   // Bypassed by: ONCLICK, on\x00click
}
```

## Good (allowlist pattern from lib/sanitize.ts)
```ts
const ALLOWED_TAGS = new Set(['p', 'br', 'b', 'i', 'em', 'strong', 'a', 'ul', 'ol', 'li', ...])

export function sanitizeHtml(html: string): string {
  return html
    .replace(/<script\b[^<]*(?:(?!<\/script>)<[^<]*)*<\/script>/gi, '')
    .replace(/<style\b[^<]*(?:(?!<\/style>)<[^<]*)*<\/style>/gi, '')
    .replace(/\s+on\w+\s*=\s*(?:"[^"]*"|'[^']*'|[^\s>]*)/gi, '')
    .replace(/href\s*=\s*(?:"javascript:[^"]*"|'javascript:[^']*')/gi, '')
    .replace(/src\s*=\s*(?:"data:[^"]*"|'data:[^']*')/gi, '')
    .replace(/<iframe\b[^<]*(?:(?!<\/iframe>)<[^<]*)*<\/iframe>/gi, '')
    .replace(/<(?:object|embed|applet)\b[^>]*>/gi, '')
    .replace(/<(?:form|input|button)\b[^>]*>/gi, '')
    .replace(/<svg\b[^<]*(?:(?!<\/svg>)<[^<]*)*<\/svg>/gi, '')
}
```

## When to use which
| Content source | Sanitization |
|---------------|-------------|
| Admin-authored blog HTML | sanitizeHtml() — strip dangerous, keep formatting |
| User comments | Strip ALL HTML — plain text only |
| JSON-LD / analytics scripts | No sanitization needed — hardcoded |
| Color values from postMessage | isValidColorValue() regex — strict format check |

## Why
Real pattern from lib/sanitize.ts. Blog content from admin CMS contains HTML that must be rendered. Denylist approaches (remove `<script>`) are always bypassable. Allowlist approach: remove everything dangerous first (scripts, iframes, event handlers, javascript: URLs, SVG, forms), then render what remains.
