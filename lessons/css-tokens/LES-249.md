## Problem

A hero/banner section uses `width: 100%` expecting fullwidth display, but a parent element (`<main>`, `.wz-container`, or theme engine wrapper) has `max-width`, `padding`, or other constraints that shrink the section to less than viewport width.

Symptoms:
- Banner only covers 60-80% of screen width on mobile
- Body background color visible on sides
- `width: 100%` only fills the parent, not the viewport

## Pattern sai

```css
/* Relies on parent being fullwidth — breaks when parent is constrained */
.haven-hero {
    width: 100%;
    overflow: hidden;
}
```

```php
<!-- Hero inside constrained <main> -->
<main id="main-content">
    <?php get_template_part( 'template-parts/hero/default' ); ?>
</main>
```

## Pattern đúng — Option A: CSS full-bleed breakout

```css
html, body {
    overflow-x: hidden;
}

.haven-hero {
    position: relative;
    width: 100vw;
    margin-left: calc(-50vw + 50%);
    overflow: hidden;
}
```

## Pattern đúng — Option B: Move outside constrained parent (preferred)

```php
<?php get_header(); ?>

<!-- Hero OUTSIDE main — no parent constraints -->
<?php get_template_part( 'template-parts/hero/default' ); ?>

<main id="main-content">
    <?php get_template_part( 'template-parts/home/flash-sale' ); ?>
</main>

<?php get_footer(); ?>
```

## Lý do

- `width: 100%` means 100% of parent, NOT 100% of viewport
- Theme engines, shared design systems, or WordPress wrappers often inject `max-width`, `padding`, or `margin: 0 auto` on parent elements
- The CSS full-bleed pattern (`100vw + margin-left calc`) works but requires `overflow-x: hidden` on html/body to prevent scrollbar-induced offset
- Moving the section outside the constrained parent is the cleanest structural fix

## Checklist

1. Check if parent (`<main>`, wrapper div) has `max-width` or `padding`
2. If yes → either use full-bleed CSS or move section outside parent
3. Always add `overflow-x: hidden` on `html, body` when using `100vw`
4. Test on 375px mobile — body background should NOT be visible beside hero
