## Problem

Using `max(0, min(23, (int) get_option(...)))` to validate cron schedule hour assumes `get_option()` returns numeric value. If option is corrupted (string, array, object), `(int)` cast produces 0, and `strtotime("today 0:00:00")` works but schedules at midnight instead of intended hour. Worse, if option is malicious string like `"5; rm -rf /"`, it passes validation but breaks `strtotime()`.

## Bad

```php
public function init(): void {
    if (!wp_next_scheduled('wezone_analytics_daily_aggregate')) {
        // No type validation — assumes numeric
        $hour = max(0, min(23, (int) get_option('wezone-analytics_aggregate_hour', 3)));
        
        // If option is "abc", $hour becomes 0 → schedules at midnight
        // If option is array, $hour becomes 1 → schedules at 1am
        $timestamp = strtotime("today {$hour}:00:00");
        
        wp_schedule_event($timestamp, 'daily', 'wezone_analytics_daily_aggregate');
    }
}
```

**Risk scenarios:**
- Option corrupted to `"invalid"` → cron runs at midnight instead of 3am
- Option set to `array(5)` → cron runs at 1am (array cast to 1)
- Option set to malicious string → `strtotime()` returns false → cron not scheduled

## Good

```php
public function init(): void {
    if (!wp_next_scheduled('wezone_analytics_daily_aggregate')) {
        $hour_option = get_option('wezone-analytics_aggregate_hour', 3);
        
        // Validate type before casting
        if (!is_numeric($hour_option)) {
            error_log("[Analytics] Invalid aggregate_hour option: " . print_r($hour_option, true));
            $hour_option = 3; // Reset to default
            update_option('wezone-analytics_aggregate_hour', 3);
        }
        
        $hour = max(0, min(23, (int) $hour_option));
        
        $timestamp = strtotime("today {$hour}:00:00");
        
        // Guard against strtotime() failure
        if ($timestamp === false) {
            error_log("[Analytics] strtotime() failed for hour: {$hour}");
            $timestamp = strtotime("today 03:00:00");
        }
        
        if ($timestamp < time()) {
            $timestamp = strtotime("tomorrow {$hour}:00:00");
        }
        
        wp_schedule_event($timestamp, 'daily', 'wezone_analytics_daily_aggregate');
    }
}
```

## Apply

- Any `get_option()` used in `strtotime()` MUST validate with `is_numeric()` first
- Log invalid values and reset to default with `update_option()`
- Guard `strtotime()` result with `=== false` check
- Apply to all cron schedule options (hour, minute, day)
