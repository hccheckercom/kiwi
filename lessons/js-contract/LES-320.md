## Pattern

Admin SPA catches API errors but either: (1) shows raw JSON/status code to admin ("Error: 500"), (2) silently fails (toast never shown), or (3) leaves form in loading state forever. Admin doesn't know what went wrong or how to fix it.

## Bug

```tsx
// SAI — admin thấy "Error: [object Object]" hoặc không thấy gì
const handleSave = async () => {
  try {
    setLoading(true);
    await wzApi.put(`products/${id}`, formData);
    toast('success', 'Đã lưu');
  } catch (err) {
    console.error(err); // Chỉ log console — admin không thấy
    // HOẶC: alert(JSON.stringify(err)) — raw JSON dump
  }
  // BUG: setLoading(false) thiếu trong catch → button disabled mãi mãi
};
```

## Fix

```tsx
const handleSave = async () => {
  try {
    setLoading(true);
    await wzApi.put(`products/${id}`, formData);
    toast('success', 'Đã lưu sản phẩm');
  } catch (err: unknown) {
    const message = err instanceof Error
      ? err.message
      : (err as any)?.data?.message || 'Có lỗi xảy ra, vui lòng thử lại';
    toast('error', message);
  } finally {
    setLoading(false); // LUÔN reset loading state
  }
};

// wzApi wrapper: parse error message từ WP REST response
const wzApi = {
  async put(endpoint: string, data: unknown) {
    const res = await fetch(`${wzAdmin.apiUrl}${endpoint}`, {
      method: 'PUT',
      headers: { 'Content-Type': 'application/json', 'X-WP-Nonce': wzAdmin.nonce },
      body: JSON.stringify(data),
    });
    if (!res.ok) {
      const body = await res.json().catch(() => ({}));
      throw new Error(body.message || body.error || `Lỗi ${res.status}`);
    }
    return res.json();
  }
};
```

## Áp dụng

Mọi admin API call PHẢI: (1) hiển thị error message thân thiện cho admin (không raw JSON), (2) dùng `finally` để reset loading/disabled state, (3) parse `response.data.message` từ WP REST error format. Admin cần biết: lỗi gì, có thể retry không.
