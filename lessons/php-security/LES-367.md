# Bug: Public REST endpoint thiếu rate limiting

## Mô tả
Endpoint `/active` có `permission: 'public'` → không cần auth → có thể bị spam. Cần thêm rate limiting dựa trên IP hoặc user agent.

## Tại sao nguy hiểm
- **DoS attack**: Attacker spam 1000 requests/giây
- **Database overload**: Mỗi request query DB
- **Cache bypass**: Nếu không có cache, mỗi request hit DB

## Code sai (Plugin.php:76-81)
```php
array(
    'route'      => '/active',
    'method'     => 'GET',
    'callback'   => array( $controller, 'get_active_banners' ),
    'permission' => 'public',
),
```

**Vấn đề**: Không có rate limiting → có thể spam.

## Code đúng (thêm rate limit helper)
```php
// In BannerController.php
public function get_active_banners( WP_REST_Request $request ): WP_REST_Response {
    // Rate limit: 60 requests/minute per IP
    $ip = $_SERVER['REMOTE_ADDR'] ?? 'unknown';
    $key = 'wz_banner_rate_' . md5( $ip );
    $attempts = (int) get_transient( $key );

    if ( $attempts >= 60 ) {
        return new WP_REST_Response( 
            array( 'error' => 'Too many requests. Try again later.' ), 
            429 
        );
    }

    set_transient( $key, $attempts + 1, 60 );

    $position = $request->get_param( 'position' );
    if ( ! $position ) {
        return new WP_REST_Response( array( 'error' => 'Position là bắt buộc' ), 400 );
    }

    // ... rest of code
}
```

## Cách fix
1. Thêm rate limit check đầu function
2. Dùng transient với TTL 60s
3. Key = `md5($ip)` để track per-IP
4. Return 429 Too Many Requests nếu vượt limit
5. Limit hợp lý: 60 req/min cho public endpoint

## Áp dụng cho
- Mọi endpoint có `permission: 'public'`
- Đặc biệt: search, filter, autocomplete endpoints
- Login/register endpoints (cần limit thấp hơn: 5 req/min)
