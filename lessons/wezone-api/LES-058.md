**Triệu chứng:**
- Frontend: dropdown Tỉnh/Thành phố hiển thị mã viết tắt (HCM, HCM-4) thay vì tên đầy đủ
- Frontend: dropdown Phường/Xã trống hoặc không load được options sau khi chọn Quận/Huyện
- Admin: cột "Địa chỉ" trong đơn hàng hiển thị ", SNI" / ", LBI" thay vì tên tỉnh/quận đầy đủ
**Root cause:**
- Dữ liệu địa chính (provinces/districts/wards) hardcode trong theme JS hoặc dùng API cũ không cập nhật theo Nghị quyết mới (tách/nhập quận huyện)
- Mỗi theme tự implement riêng → không đồng bộ, không reuse được
**Phải làm:**
```
1. Plugin Vận chuyển (wezone-shipping) phải cung cấp REST API trả danh sách tỉnh/quận/xã:
   GET /wp-json/wezone/v1/shipping/provinces
   GET /wp-json/wezone/v1/shipping/districts?province_id={id}
   GET /wp-json/wezone/v1/shipping/wards?district_id={id}

2. Dữ liệu địa chính lưu tập trung trong plugin (DB table hoặc JSON file), 
   cập nhật theo nguồn chính thức (danhmuchanhchinh.gso.gov.vn)

3. Theme frontend gọi API từ plugin — KHÔNG hardcode danh sách trong JS/PHP
   → Mọi theme đều dùng chung 1 nguồn dữ liệu từ plugin

4. Response format chuẩn:
   { "id": 79, "name": "Thành phố Hồ Chí Minh", "code": "HCM" }
   { "id": 760, "name": "Quận 1", "province_id": 79 }
   { "id": 26734, "name": "Phường Bến Nghé", "district_id": 760 }
```
**Lý do:** Địa chính Việt Nam thay đổi thường xuyên (tách/nhập quận huyện, đổi tên). Nếu mỗi theme tự quản lý data → không đồng bộ, khó cập nhật, user không chọn được địa chỉ → không đặt hàng được. Plugin Vận chuyển là nơi tập trung duy nhất vì mọi theme đều cần shipping + address.
**Grep check:** `provinces\s*=\s*\[` trong theme JS — nếu hardcode array thay vì gọi API plugin → CRITICAL
**Ảnh hưởng:** Mọi theme — checkout, my-account/addresses, admin đơn hàng
