## Bad
```php
public function __construct( string $backup_dir ) {
    $this->backup_dir = $backup_dir; // No validation â€” caller can pass "../../../etc/passwd"
}
```

## Good
```php
public function __construct( string $backup_dir ) {
    $real = realpath( $backup_dir );
    $base = realpath( wp_upload_dir()['basedir'] );
    if ( ! $real || ! $base || ! str_starts_with( $real, $base . DIRECTORY_SEPARATOR ) ) {
        throw new \InvalidArgumentException( 'Invalid backup directory' );
    }
    $this->backup_dir = $real;
}
```

## Why
Constructor path parameters from caller can contain ../ sequences. Validate with realpath() and check prefix before storing to prevent path traversal attacks.
