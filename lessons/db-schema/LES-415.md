## Pattern

Table schema có email column nhưng không có UNIQUE constraint → cho phép duplicate emails.

## Bug Example

```php
// ❌ BAD — email không UNIQUE
return array(
    'wezone_popup_subscribers' => array(
        'id'         => 'bigint(20) unsigned NOT NULL AUTO_INCREMENT PRIMARY KEY',
        'email'      => 'varchar(191) NOT NULL',  // không UNIQUE
        'phone'      => 'varchar(50) DEFAULT NULL',
        'created_at' => 'datetime DEFAULT CURRENT_TIMESTAMP',
    ),
);
```

**Hậu quả:**
- User submit form 2 lần → 2 rows với cùng email
- Gửi email marketing → duplicate sends, tốn cost
- Analytics sai → count subscribers không chính xác

## Why It's Wrong

1. **Data integrity violation** — email nên unique per subscriber
2. **Wasted storage** — duplicate rows
3. **Business logic bugs** — code phải handle duplicates manually
4. **Performance** — queries phải DISTINCT hoặc GROUP BY

## Correct Approach

```php
// ✅ GOOD — email UNIQUE constraint
return array(
    'wezone_popup_subscribers' => array(
        'id'         => 'bigint(20) unsigned NOT NULL AUTO_INCREMENT PRIMARY KEY',
        'email'      => 'varchar(191) NOT NULL UNIQUE',  // UNIQUE constraint
        'phone'      => 'varchar(50) DEFAULT NULL',
        'created_at' => 'datetime DEFAULT CURRENT_TIMESTAMP',
    ),
);

// Hoặc composite UNIQUE nếu cần scope
return array(
    'wezone_popup_subscribers' => array(
        'id'         => 'bigint(20) unsigned NOT NULL AUTO_INCREMENT PRIMARY KEY',
        'email'      => 'varchar(191) NOT NULL',
        'source'     => 'varchar(50) NOT NULL',
        'created_at' => 'datetime DEFAULT CURRENT_TIMESTAMP',
        // UNIQUE per source
        'UNIQUE KEY email_source' => '(email, source)',
    ),
);
```

**Application code phải handle:**
```php
// INSERT với ON DUPLICATE KEY UPDATE
$wpdb->query( $wpdb->prepare(
    "INSERT INTO {$wpdb->prefix}wezone_popup_subscribers (email, source)
     VALUES (%s, %s)
     ON DUPLICATE KEY UPDATE created_at = CURRENT_TIMESTAMP",
    $email, $source
) );
```

## Detection

```regex
'email'\s*=>\s*'varchar\([^)]+\)[^']*'(?!.*UNIQUE)
```

Trong context: table schema array, email column không có UNIQUE keyword

## Impact

- **Data integrity**: Duplicate subscribers → sai analytics, duplicate emails
- **Cost**: Gửi duplicate emails → tốn email service credits
- **Performance**: Queries phải DISTINCT → slower
