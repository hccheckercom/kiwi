# LES-246: Hero Product Cards — CSS/JS Animation & Layout Bugs

## Pattern

When adding admin-configurable product cards to hero banners with dynamic settings (animation, ratio, columns, hover effects), multiple CSS/JS bugs emerge together.

## Bugs (5 co-occurring issues)

### 1. Inline `padding-bottom` overrides CSS class rules
**Wrong:** `<div style="padding-bottom:75%">` — inline specificity kills overlay card style overrides.
**Fix:** Use CSS custom property on container: `--hp-img-ratio: 75%` → CSS: `padding-bottom: var(--hp-img-ratio, 100%)`.

### 2. `animation-name: var(--name)` fails in Safari < 16.4
**Wrong:** `.card.is-visible { animation-name: var(--hp-anim-name); }`
**Fix:** Explicit selectors per animation type:
```css
.card.hp-anim-ready.is-visible[data-anim="fade-up"] { animation-name: hpFadeUp; }
```

### 3. `opacity: 0` on `[data-anim]` causes FOIC without JS
**Wrong:** Cards invisible if JS fails or loads late.
**Fix:** Progressive enhancement — JS adds `.hp-anim-ready` class first, CSS only hides cards with that class.

### 4. Fixed `height` conflicts with dynamic aspect ratio
**Wrong:** `.img-wrap { height: 80px; }` + inline `padding-bottom` = double height.
**Fix:** Use `padding-bottom` technique with `position: relative` + absolute child img. Remove fixed height.

### 5. `white-space: nowrap` conflicts with `-webkit-line-clamp`
**Wrong:** `white-space: nowrap; -webkit-line-clamp: 2;` — nowrap prevents multi-line.
**Fix:** Use `-webkit-box` display + remove `white-space: nowrap`:
```css
display: -webkit-box;
-webkit-line-clamp: var(--hp-line-clamp, 2);
-webkit-box-orient: vertical;
overflow: hidden;
```

## Additional gotchas

- `.haven-hero { overflow: hidden }` clips hover transforms on cards at bottom edge — ensure cards have enough `bottom` offset (≥16px) to absorb translateY(-6px)
- `position: relative` required on card for overlay style info + shine `::after` pseudo-element
- Content `padding-bottom: 180px` wastes space when products are left/right — reduce for non-bottom positions

## Checklist

- [ ] No inline `padding-bottom` on img-wrap — use CSS var
- [ ] No `animation-name: var()` — use explicit selectors
- [ ] Animation hidden state added by JS, not CSS alone
- [ ] No fixed height + padding-bottom conflict
- [ ] Line clamp uses `-webkit-box`, not `white-space: nowrap`
- [ ] Card has `position: relative`
- [ ] Hero overflow doesn't clip hover transforms
