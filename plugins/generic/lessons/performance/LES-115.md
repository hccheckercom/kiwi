# LES-115: O(n²) NOT IN Subquery

## Problem

Query tìm "khách hàng mới hôm nay" dùng pattern:

```sql
SELECT COUNT(DISTINCT customer_id) FROM orders
WHERE customer_id NOT IN (
    SELECT DISTINCT customer_id FROM orders
    WHERE DATE(created_at) < '2026-05-19'
)
AND DATE(created_at) = '2026-05-19'
```

Vấn đề:
- Subquery scan toàn bộ bảng orders để build list "old customers"
- Outer query check mỗi row against toàn bộ list đó
- Complexity: O(n × m) với n = orders today, m = all historical orders
- Với 10K orders → subquery trả về hàng nghìn IDs, mỗi row outer phải check against list

## Fix

Dùng `GROUP BY ... HAVING MIN()` — 1 pass duy nhất:

```sql
-- ĐÚNG — O(n) single pass with GROUP BY
SELECT COUNT(*) FROM (
    SELECT customer_id FROM orders
    WHERE customer_id IS NOT NULL
    GROUP BY customer_id
    HAVING MIN(DATE(created_at)) = '2026-05-19'
) AS new_cust

-- SAI — O(n²) NOT IN subquery
SELECT COUNT(DISTINCT customer_id) FROM orders
WHERE customer_id NOT IN (
    SELECT DISTINCT customer_id FROM orders WHERE DATE(created_at) < '2026-05-19'
)
AND DATE(created_at) = '2026-05-19'
```

## Impact

- Complexity: O(n²) → O(n)
- Time: 2-5s (10K orders) → <100ms
- Scales linearly with table size

## Detection

Tìm `NOT IN (SELECT DISTINCT` trong dashboard/stats queries.
