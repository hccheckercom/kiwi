## Bad
```css
[data-theme="dark"] {
    --color-bg: #0a0a0f;
    --color-surface: #16161f;
    --color-text: #e8e8ed;
}
```
Theme inline styles use `var(--wz-surface)`, `var(--wz-on-surface)` directly — these `--wz-*` tokens are never overridden in dark mode, so UI stays light.

## Good
```css
[data-theme="dark"] {
    /* Override the actual --wz-* tokens used in inline styles */
    --wz-surface: #16161f;
    --wz-surface-container-lowest: #0a0a0f;
    --wz-surface-container-low: #111118;
    --wz-surface-container: #1a1a24;
    --wz-on-surface: #e8e8ed;
    --wz-on-surface-variant: #a0a0b0;
    --wz-outline: #6b6b80;
    --wz-outline-variant: #2a2a3a;
    --wz-background: #0a0a0f;
    --wz-primary: #4d7fff;
    --wz-accent: #fb923c;

    /* ALSO override --color-* aliases for partials that use them */
    --color-bg: #0a0a0f;
    --color-text: #e8e8ed;
    /* ... */
}
```

## Why
TokenLoader injects `--wz-*` tokens on `:root`. Theme inline styles (header, nav, cards) reference these directly. If dark mode only overrides the `--color-*` aliases (defined in `_tokens.css`), the actual rendered elements using `var(--wz-surface)` etc. never change. Both layers must be overridden.

## Detection
Check if `_dark-mode.css` contains `--wz-surface` override inside `[data-theme="dark"]`. If absent, dark mode is broken for all inline-styled elements.
