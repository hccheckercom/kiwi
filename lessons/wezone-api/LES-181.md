## Pattern sai

Plugins that serve virtual files (service workers, manifests, etc.) using exact path comparison:

```php
$path = (string) wp_parse_url( $uri, PHP_URL_PATH );

if ( $path !== '/wezone-push-sw.js' ) {
    return;
}
```

On subpath WP installations (e.g., `wp.wezone.vn/funilux/`), `home_url('/wezone-push-sw.js')` generates `/funilux/wezone-push-sw.js` but the handler only matches `/wezone-push-sw.js` → 404.

## Phải làm

Use `str_ends_with()` to match the filename regardless of path prefix:

```php
$path = (string) wp_parse_url( $uri, PHP_URL_PATH );

if ( ! str_ends_with( $path, '/wezone-push-sw.js' ) ) {
    return;
}
```

## Lý do

WordPress multisite subdirectory or custom permalink structures prepend a subpath to all URLs. Any plugin serving virtual files via `init` hook must handle this. Also requires nginx config to route `.js`/`.json` virtual files to PHP instead of returning 404 from the static file handler.

## Grep check

```
\$path\s*(!==|===)\s*'/
```

Matches any exact path comparison in PHP — review each hit to determine if it should use `str_ends_with()` or `str_starts_with()` instead.
