## Pattern

Methods registered as WordPress hook callbacks (`add_action`, `add_filter`) are called by WordPress core, not directly from frontend/AJAX requests. These methods do NOT need nonce protection because:

1. They're invoked by WordPress core in a trusted context
2. They're not directly callable from user input
3. Adding nonce checks would break the hook system

This lesson serves as a **false positive exclusion** for [[LES-413]] (public state-modifying methods need nonce protection).

## Scan

```yaml
pattern: "add_action\\([^,]+,\\s*array\\(\\s*\\$this,\\s*'([^']+)'\\s*\\)|add_filter\\([^,]+,\\s*array\\(\\s*\\$this,\\s*'([^']+)'\\s*\\)"
scope: "**/src/**/*.php"
exclude_from: ["LES-413"]
message: "Hook callback methods are exempt from nonce checks"
```

## Example

```php
// ĐÚNG — hook callbacks không cần nonce
class DemoUserManager {
    public function init(): void {
        add_filter( 'map_meta_cap', array( $this, 'restrict_demo_caps' ), 10, 4 );
        add_action( 'admin_menu', array( $this, 'remove_restricted_menus' ), 999 );
    }

    // Hook callback — không cần nonce
    public function restrict_demo_caps( array $caps, string $cap, int $user_id, array $args ): array {
        if ( get_current_blog_id() === 1 ) {
            return $caps;
        }

        $restricted = array( 'install_plugins', 'delete_plugins' );
        if ( in_array( $cap, $restricted, true ) ) {
            return array( 'do_not_allow' );
        }

        return $caps;
    }

    // Hook callback — không cần nonce
    public function remove_restricted_menus(): void {
        if ( is_super_admin() || get_current_blog_id() === 1 ) {
            return;
        }

        remove_menu_page( 'plugins.php' );
        remove_menu_page( 'themes.php' );
    }
}
```

## Khi nào KHÔNG cần nonce

**Hook callbacks:**
- `add_action` callbacks
- `add_filter` callbacks
- `register_rest_route` callbacks (REST API has built-in nonce via `X-WP-Nonce`)

**Trusted contexts:**
- Cron jobs (`wp_schedule_event`)
- CLI commands (WP-CLI)
- Internal methods (private/protected)
- Methods marked `@internal`

## Khi nào VẪN CẦN nonce

**AJAX handlers:**
```php
// CẦN nonce — callable từ frontend
add_action( 'wp_ajax_disable_module', function() {
    check_ajax_referer( 'wezone_commerce_settings', 'nonce' );
    Commerce::disable( $_POST['module'] );
    wp_send_json_success();
});
```

**Form submissions:**
```php
// CẦN nonce — callable từ POST request
public function handle_settings_form(): void {
    if ( ! check_admin_referer( 'wezone_settings' ) ) {
        wp_die( 'Invalid nonce', 403 );
    }
    
    update_option( 'wezone_setting', $_POST['value'] );
}
```

**Public API methods:**
```php
// CẦN nonce — callable từ bất kỳ đâu
class Commerce {
    public static function disable( string $module ): void {
        if ( ! current_user_can( 'manage_options' ) ) {
            wp_die( 'Unauthorized', 403 );
        }
        
        if ( ! check_admin_referer( 'wezone_commerce_settings' ) ) {
            wp_die( 'Invalid nonce', 403 );
        }
        
        update_option( "wezone_commerce_{$module}_enabled", false );
    }
}
```

## Scanner implementation

Kiwi scanner should:
1. Detect methods registered via `add_action`/`add_filter`
2. Exclude these methods from LES-413 violations
3. Still flag methods callable from AJAX/REST/forms
