## Vấn đề

Trên WordPress Multisite, `switch_theme()` và `update_option('template', ...)` có thể **âm thầm fail**
— trả về success nhưng theme KHÔNG thực sự chuyển. `wp theme activate` qua WP-CLI cũng fail với
lỗi generic "Could not switch to theme".

Root cause: WordPress Multisite có thêm layers validation (network-enabled check, blog-specific
options table, object cache) khiến các API chuẩn không hoạt động trong một số trường hợp.

## Triệu chứng

```bash
# WP-CLI fail
wp theme activate wezone-marketplace --url=https://wp.wezone.vn/ --allow-root
# Error: Could not switch to 'WeZone Marketplace' theme.

# PHP API fail im lặng
switch_theme('wezone-marketplace');
// Trả về null (success) nhưng theme VẪN là theme cũ

# update_option fail
update_option('template', 'wezone-marketplace');
// Trả về false
```

## Fix

```bash
# Direct SQL update — luôn hoạt động
wp db query "UPDATE wp_options SET option_value='wezone-marketplace' WHERE option_name='template'" --allow-root
wp db query "UPDATE wp_options SET option_value='wezone-marketplace' WHERE option_name='stylesheet'" --allow-root

# Sau đó flush cache
wp rewrite flush --allow-root
wp cache flush --allow-root
```

## Checklist trước khi switch theme trên Multisite

1. Theme đã được **network-enable**: `wp theme enable <slug> --network --allow-root`
2. Theme đã có trong **allowedthemes** site option: kiểm tra `get_site_option('allowedthemes')`
3. Theme files tồn tại và có đúng permissions (nginx:nginx, 755)
4. PHP syntax OK: `php -l functions.php`
5. Nếu tất cả OK mà vẫn fail → dùng direct SQL
