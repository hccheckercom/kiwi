### Pattern sai

Nginx config trên wezone.vn chỉ handle `/wordpress/admin` (network admin):

```nginx
location = /wordpress/admin {
    return 302 https://wp.wezone.vn/wp-admin;
}
```

Nhưng KHÔNG handle `/wordpress/{slug}/admin` (per-theme admin). Request rơi vào Next.js catch-all → 404.

### Phải làm

Thêm location block **trước** network admin block:

```nginx
# Per-theme admin — redirect to WordPress multisite admin
location ~ ^/wordpress/([^/]+)/admin/?$ {
    return 302 https://wp.wezone.vn/$1/admin;
}
```

### Lý do

Router.php (`wezone-core`) đã có rewrite rule `^admin/?$` → `wz_page=admin` và nginx trên wp.wezone.vn đã có rule `^/([^/]+)/admin/?$` redirect tới wp-admin. Nhưng proxy layer wezone.vn → wp.wezone.vn thiếu rule tương ứng cho pattern `/wordpress/{slug}/admin`.

**Quy tắc chung:** Khi thêm route mới vào Router.php, phải kiểm tra cả 3 lớp:
1. `Router.php` — rewrite rule + handler
2. `nginx-wp.wezone.vn.conf` — friendly URL redirect
3. `webstore-vn/nginx/conf.d/wezone.conf` — proxy redirect từ wezone.vn

### Grep check

```
# Kiểm tra wezone.conf có redirect cho route mới chưa
grep -c "wordpress.*admin" webstore-vn/nginx/conf.d/wezone.conf
```
