## Bad
```php
print(f"DEBUG: user={user_id}, data={data}")
```

## Good
```php
import logging
logger = logging.getLogger(__name__)
logger.debug("user=%s, data=%s", user_id, data)
```

## Why
print() debug statements are not filterable and often leak to production
