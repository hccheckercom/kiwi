## Pattern

Admin product image/CSV upload validates only file extension (`.jpg`, `.csv`). Attacker renames `shell.php` to `shell.php.jpg` or uses double extension. Without MIME type validation + content sniffing, malicious files get uploaded to `wp-content/uploads/`.

## Bug

```php
// SAI — chỉ check extension, dễ bypass
public function upload_image( WP_REST_Request $request ): WP_REST_Response {
    $file = $request->get_file_params()['image'];
    $ext  = pathinfo( $file['name'], PATHINFO_EXTENSION );

    if ( ! in_array( $ext, array( 'jpg', 'jpeg', 'png', 'webp' ), true ) ) {
        return new WP_REST_Response( array( 'error' => 'Invalid file type' ), 400 );
    }

    // shell.php.jpg passes! Or Content-Type header spoofed
    move_uploaded_file( $file['tmp_name'], $upload_dir . '/' . $file['name'] );
}
```

## Fix

```php
public function upload_image( WP_REST_Request $request ): WP_REST_Response {
    $file = $request->get_file_params()['image'] ?? null;
    if ( ! $file || UPLOAD_ERR_OK !== $file['error'] ) {
        return new WP_REST_Response( array( 'error' => 'Upload failed' ), 400 );
    }

    // 1. Check extension
    $allowed_types = array( 'jpg' => 'image/jpeg', 'jpeg' => 'image/jpeg', 'png' => 'image/png', 'webp' => 'image/webp' );
    $filetype = wp_check_filetype_and_ext( $file['tmp_name'], $file['name'], $allowed_types );

    if ( ! $filetype['ext'] || ! $filetype['type'] ) {
        return new WP_REST_Response( array( 'error' => 'File type not allowed' ), 400 );
    }

    // 2. Verify actual content (not just header)
    $image_info = @getimagesize( $file['tmp_name'] );
    if ( false === $image_info ) {
        return new WP_REST_Response( array( 'error' => 'File is not a valid image' ), 400 );
    }

    // 3. Use WordPress media handler (sanitizes filename, generates unique path)
    require_once ABSPATH . 'wp-admin/includes/media.php';
    require_once ABSPATH . 'wp-admin/includes/file.php';
    require_once ABSPATH . 'wp-admin/includes/image.php';

    $attachment_id = media_handle_upload( 'image', 0 );
    if ( is_wp_error( $attachment_id ) ) {
        return new WP_REST_Response( array( 'error' => $attachment_id->get_error_message() ), 500 );
    }

    return new WP_REST_Response( array(
        'id'  => $attachment_id,
        'url' => wp_get_attachment_url( $attachment_id ),
    ) );
}
```

## Áp dụng

Admin upload PHẢI: (1) `wp_check_filetype_and_ext()` validate extension + MIME, (2) `getimagesize()` cho images, (3) dùng `media_handle_upload()` thay `move_uploaded_file()`. KHÔNG bao giờ trust client-sent Content-Type header.
