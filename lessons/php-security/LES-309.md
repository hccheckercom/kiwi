## Pattern

Admin bulk actions (delete multiple orders, change status of 10 products, bulk assign category) execute N operations in a loop without wrapping in a transaction. If operation #5 fails, items 1-4 are already committed — partial state that's hard to debug or reverse.

## Bug

```php
// SAI — nếu item #5 fail, 1-4 đã committed
public function bulk_delete( WP_REST_Request $request ): WP_REST_Response {
    $ids = $request->get_param( 'ids' );
    foreach ( $ids as $id ) {
        $wpdb->delete( $table, array( 'id' => (int) $id ) );
    }
    return new WP_REST_Response( array( 'deleted' => count( $ids ) ) );
}
```

## Fix

```php
public function bulk_delete( WP_REST_Request $request ): WP_REST_Response {
    global $wpdb;
    $ids = array_map( 'absint', (array) $request->get_param( 'ids' ) );

    $wpdb->query( 'START TRANSACTION' );
    try {
        foreach ( $ids as $id ) {
            $result = $wpdb->delete( $wpdb->prefix . 'wz_orders', array( 'id' => $id ) );
            if ( false === $result ) {
                throw new \RuntimeException( "Failed to delete order #{$id}" );
            }
        }
        $wpdb->query( 'COMMIT' );
    } catch ( \Throwable $e ) {
        $wpdb->query( 'ROLLBACK' );
        return new WP_REST_Response( array( 'error' => $e->getMessage() ), 500 );
    }

    return new WP_REST_Response( array( 'deleted' => count( $ids ) ) );
}
```

## Áp dụng

Mọi admin endpoint xử lý batch/bulk (bulk delete, bulk status change, bulk assign) PHẢI wrap trong transaction. Dùng `wz_bulk_insert()` cho insert, manual transaction cho update/delete.
