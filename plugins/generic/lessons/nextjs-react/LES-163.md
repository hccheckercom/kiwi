## Bad
```ts
// External API down → fetch hangs forever → worker blocked → all requests queue
const res = await fetch('https://api.resend.com/emails', {
  method: 'POST',
  body: JSON.stringify(payload),
})
```

## Good (timeout pattern used across webstore-vn)
```ts
const res = await fetch('https://api.resend.com/emails', {
  method: 'POST',
  body: JSON.stringify(payload),
  signal: AbortSignal.timeout(8000), // 8s max
})
```

## Timeout guidelines for webstore-vn
| Service | Timeout | Why |
|---------|---------|-----|
| Resend (email) | 8000ms | Non-critical, fire-and-forget |
| Telegram | 5000ms | Notification only, never block |
| Claude/Gemini AI | 30000ms | Long-running but user is waiting |
| VNPay/MoMo | 10000ms | Payment — important but bounded |
| Supabase (via SDK) | Default (no manual timeout needed) | SDK handles internally |

## Why
Real fix: FIX M-19 in resend.order.ts. Node.js fetch() has no default timeout — if the remote server accepts the TCP connection but never responds, the request hangs indefinitely. In a webhook handler, this means the payment gateway times out and retries. AbortSignal.timeout() is the standard way to bound external calls.
