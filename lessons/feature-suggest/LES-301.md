## Bad
```php
// single.php — out-of-stock product
<?php if ( ! $product['in_stock'] ) : ?>
    <span class="text-red-700">Hết hàng</span>
<?php endif; ?>
```
Dead end. Visitor sees "Hết hàng", has no action to take, leaves forever. Zero chance of recovery when product restocks.

## Good
```php
// template-parts/product/out-of-stock.php
<?php if ( ! $product['in_stock'] ) : ?>
<div class="wz-waitlist" data-product-id="<?php echo esc_attr( $product['id'] ); ?>">
    <p class="text-label-caps font-bold text-error">Tạm hết hàng</p>
    <p class="text-body-sm text-on-surface-variant mt-xs">Nhận thông báo khi có hàng trở lại:</p>
    <form class="wz-waitlist__form flex gap-sm mt-sm" data-ajax="wz_waitlist_subscribe">
        <?php wp_nonce_field( 'wz_waitlist', '_nonce' ); ?>
        <input type="email" name="email" placeholder="Email của bạn" required
               class="flex-1 px-md py-sm border border-outline rounded-lg" />
        <button type="submit" class="wz-btn wz-btn--primary px-lg">Thông báo cho tôi</button>
    </form>
    <p class="text-body-xs text-on-surface-variant mt-xs">
        <?php echo esc_html( wz_waitlist_count( $product['id'] ) ); ?> người đang chờ
    </p>
</div>
<?php endif; ?>
```

## Why
Blueprint `conversion-recovery.md` Section 5 requires: (1) waitlist email form on out-of-stock products, (2) subscriber count as social proof, (3) back-in-stock email notification trigger, (4) similar products fallback grid. Without waitlist, out-of-stock pages are dead ends — visitor intent is captured nowhere, and restocking generates zero automatic re-engagement.
