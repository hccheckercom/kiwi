## Pattern

Admin export (orders CSV, products Excel) loads ALL records into memory before generating file. With 50k+ orders, PHP hits memory_limit (256MB default) → fatal error or timeout. No pagination, no streaming.

## Bug

```php
// SAI — 50k orders = 500MB+ RAM
public function export_orders( WP_REST_Request $request ): void {
    $orders = $wpdb->get_results( "SELECT * FROM {$table}" ); // ALL records!
    
    header( 'Content-Type: text/csv' );
    $fp = fopen( 'php://output', 'w' );
    foreach ( $orders as $order ) {
        fputcsv( $fp, (array) $order );
    }
    fclose( $fp );
    exit;
}
```

## Fix

```php
public function export_orders( WP_REST_Request $request ): void {
    header( 'Content-Type: text/csv; charset=utf-8' );
    header( 'Content-Disposition: attachment; filename="orders-' . gmdate( 'Y-m-d' ) . '.csv"' );

    $fp = fopen( 'php://output', 'w' );
    fputcsv( $fp, array( 'ID', 'Order Number', 'Customer', 'Total', 'Status', 'Date' ) );

    $batch_size = 500;
    $offset     = 0;

    do {
        $rows = $wpdb->get_results( $wpdb->prepare(
            "SELECT id, order_number, customer_name, total, status, created_at
             FROM {$wpdb->prefix}wz_orders
             ORDER BY id ASC
             LIMIT %d OFFSET %d",
            $batch_size,
            $offset
        ) );

        foreach ( $rows as $row ) {
            fputcsv( $fp, (array) $row );
        }

        $offset += $batch_size;
        // Flush output buffer mỗi batch
        if ( ob_get_level() ) {
            ob_flush();
        }
        flush();
    } while ( count( $rows ) === $batch_size );

    fclose( $fp );
    exit;
}
```

## Áp dụng

Admin export PHẢI dùng chunked streaming: SELECT LIMIT/OFFSET + fputcsv + flush mỗi batch. Không bao giờ SELECT * toàn bộ table vào memory. Áp dụng cho: orders, products, customers, analytics export.
