    Controller methods that accept per_page param MUST clamp with min(100, max(1, ...)).
    Missing min() allows attacker to request all records in one call → memory exhaustion.

## Bad
```php
public function get_reviews( WP_REST_Request $request ): WP_REST_Response {
    $per_page = max( 1, (int) ( $request->get_param( 'per_page' ) ?? 20 ) );
    // No upper bound! per_page=99999 loads all reviews
```

## Good
```php
public function get_reviews( WP_REST_Request $request ): WP_REST_Response {
    $per_page = min( 100, max( 1, (int) ( $request->get_param( 'per_page' ) ?? 20 ) ) );
```

## Why
Without `min(100, ...)`, an attacker can pass `per_page=99999` to load thousands of records in a single request, causing memory exhaustion and potential DoS. Every paginated controller MUST cap per_page.
