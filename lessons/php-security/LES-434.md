## Pattern

Using `locate_template()` or `load_template()` with values derived from database or user input without sanitization allows path traversal attacks. An attacker who can write to the database (e.g., via SQL injection or compromised admin) could load arbitrary PHP files.

## Bug

```php
// SAI — $renderer comes from DB without sanitization
$config   = SectionRegistry::get( $type );
$renderer = $config ? $config['renderer'] : $type; // fallback to raw DB value
$template = locate_template( 'template-parts/sections/' . $renderer . '.php' );
if ( $template ) {
    load_template( $template, false );
}

// If $type = '../../wp-config' → loads wp-config.php
// If $type = '../../wp-includes/version' → leaks WP version info
```

## Fix

```php
// ĐÚNG — validate against registry, sanitize fallback
$config = SectionRegistry::get( $type );
if ( ! $config ) {
    return; // unknown type — skip entirely
}
$renderer = sanitize_file_name( $config['renderer'] );
$template = locate_template( 'template-parts/sections/' . $renderer . '.php' );
if ( $template ) {
    load_template( $template, false );
}
```

## Áp dụng

1. **Never use raw DB values** in `locate_template()` or `get_template_part()`
2. **Validate against allowlist** (e.g., SectionRegistry) — reject unknown types
3. **Use `sanitize_file_name()`** to strip directory traversal characters
4. **Never fallback** to raw user/DB input when registry lookup fails
