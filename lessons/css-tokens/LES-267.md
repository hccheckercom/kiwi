## Bad
```html
<style>
.wz-toast__title { color: #1a1a1a; }
.wz-toast__time { color: #888; }
</style>
```
Hardcoded dark text on assumed-white background — unreadable in dark mode.

## Good
```html
<style>
.wz-toast__title { color: var(--color-text, #1a1a1a); }
.wz-toast__time { color: var(--color-text-muted, #888); }
</style>
```

## Why
Dark mode constitution rule #1: "NEVER hardcode colors — always use `var(--color-*)` tokens". Text colors must use semantic tokens so they auto-adapt when `[data-theme="dark"]` is active. Keep the hex as fallback value inside `var()`.
