## Pattern

An API route is registered with `'permission' => 'public'` but the controller callback
immediately checks `is_user_logged_in()`. The route-level permission should enforce
authentication so unauthenticated requests are rejected before reaching the controller.

## Bad

```php
// Registration
array(
    'route'      => '/balance',
    'method'     => 'GET',
    'callback'   => array( $controller, 'get_balance' ),
    'permission' => 'public',  // ← any visitor can hit this
),

// Controller
public function get_balance( $request ) {
    if ( ! is_user_logged_in() ) {
        return new WP_REST_Response( array( 'error' => 'Chưa đăng nhập' ), 401 );
    }
    // ...
}
```

## Good

```php
// Registration — enforce auth at route level
array(
    'route'      => '/balance',
    'method'     => 'GET',
    'callback'   => array( $controller, 'get_balance' ),
    'permission' => 'logged_in',  // ← only authenticated users
),

// Controller — no need for redundant auth check
public function get_balance( $request ) {
    $user_id = get_current_user_id();
    // ...
}
```

## Why

Defense-in-depth: if someone forgets the `is_user_logged_in()` check in the controller,
the route-level permission still blocks unauthenticated access. Also reduces unnecessary
processing — the framework rejects unauthorized requests early.
