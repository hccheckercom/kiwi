## Bad
```tsx
<button className="p-2 rounded" onClick={onClose}>
  <svg viewBox="0 0 24 24"><path d="M6 18L18 6M6 6l12 12"/></svg>
</button>
```
Icon-only button with no accessible name — screen readers announce "button" with no context.

## Good
```tsx
<button className="p-2 rounded" onClick={onClose} aria-label="Close dialog">
  <svg aria-hidden="true" viewBox="0 0 24 24"><path d="M6 18L18 6M6 6l12 12"/></svg>
</button>
```
`aria-label` provides accessible name; `aria-hidden` on decorative SVG.

## Why
WCAG 4.1.2 — interactive elements must have an accessible name. Icon-only buttons are the #1 accessibility violation in SPAs. Screen reader users hear "button" with no indication of what it does.
