**Triệu chứng:**
- Click link "Đổi mật khẩu" trong sidebar → URL đúng `/my-account/change-password/` nhưng hiển thị trang Tổng quan (dashboard)
- Template `wezone-templates/account/change-password.php` tồn tại nhưng không bao giờ được load
**Root cause:**
- Router.php có rewrite rule catch-all `^my-account/([a-z-]+)/?$` → `wz_section=$matches[1]` — URL parse đúng
- Nhưng `$allowed_sections` array KHÔNG chứa `change-password` → fallback `$section = 'dashboard'`
- Template file tồn tại nhưng Router whitelist chưa cập nhật khi thêm page mới
**Pattern sai:**
```php
// Router.php
$allowed_sections = array( 'dashboard', 'orders', 'profile', 'addresses', ... );
// Thiếu 'change-password' → mọi request tới /my-account/change-password/ hiển thị dashboard
```
**Phải làm:**
```php
$allowed_sections = array( 'dashboard', 'orders', 'order-detail', 'profile', 'addresses', 'wishlist', 'notifications', 'vouchers', 'password', 'change-password', 'address', 'returns' );
```
**Lý do:** Khi thêm account sub-page mới (template + sidebar link), PHẢI đồng thời thêm slug vào Router `$allowed_sections`. Nếu không, rewrite rule match nhưng Router reject slug → silent fallback dashboard, không có error log.
**Grep check:** `allowed_sections` trong Router.php — so sánh với `wezone-templates/account/*.php` file list → mọi file phải có slug tương ứng trong array → CRIT
**Ảnh hưởng:** Mọi theme dùng wezone-core Router — bất kỳ account sub-page nào thiếu trong whitelist đều bị ẩn
