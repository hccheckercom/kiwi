## Bad
```php
$pending_orders = (int) $wpdb->get_var(
    "SELECT COUNT(*) FROM {$orders_table}
     WHERE status IN ('pending', 'processing', 'on-hold')"
);
// No prepare() — inconsistent with other queries in same file
```

## Good
```php
$pending_orders = (int) $wpdb->get_var( $wpdb->prepare(
    "SELECT COUNT(*) FROM {$orders_table}
     WHERE status IN (%s, %s, %s)",
    'pending', 'processing', 'on-hold'
) );
```

## Why
While the current hardcoded strings are safe, `$wpdb->prepare()` should be used consistently for all queries as defense-in-depth. If someone later makes the status list dynamic without adding prepare(), it becomes an injection vector.
