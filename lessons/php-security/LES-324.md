## Pattern

Admin can send push/email/SMS/Zalo notifications to customers from dashboard. No rate limit per recipient → admin accidentally (or maliciously) sends 50 notifications in 1 minute to same customer. No cooldown between bulk sends → 10,000 emails in one click without confirmation.

## Bug

```php
// SAI — không giới hạn, admin click 10 lần = customer nhận 10 email
public function send_notification( WP_REST_Request $request ): WP_REST_Response {
    $customer_ids = (array) $request->get_param( 'customer_ids' );
    $message      = sanitize_text_field( $request->get_param( 'message' ) );

    foreach ( $customer_ids as $id ) {
        $email = get_user_meta( (int) $id, 'billing_email', true );
        wp_mail( $email, 'Thông báo từ shop', $message );
    }

    return new WP_REST_Response( array( 'sent' => count( $customer_ids ) ) );
}
```

## Fix

```php
public function send_notification( WP_REST_Request $request ): WP_REST_Response {
    global $wpdb;
    $customer_ids = array_map( 'absint', (array) $request->get_param( 'customer_ids' ) );
    $message      = sanitize_text_field( $request->get_param( 'message' ) );
    $channel      = sanitize_text_field( $request->get_param( 'channel' ) ?? 'email' );

    // Rate limit: max 500 recipients per bulk send
    if ( count( $customer_ids ) > 500 ) {
        return new WP_REST_Response(
            array( 'error' => 'Tối đa 500 người nhận mỗi lần gửi. Vui lòng chia nhỏ.' ),
            422
        );
    }

    // Cooldown: same admin cannot bulk send within 5 minutes
    $last_bulk = get_transient( 'wz_bulk_notify_' . get_current_user_id() );
    if ( $last_bulk && ( time() - (int) $last_bulk ) < 300 ) {
        $wait = 300 - ( time() - (int) $last_bulk );
        return new WP_REST_Response(
            array( 'error' => "Vui lòng đợi {$wait}s trước khi gửi tiếp." ),
            429
        );
    }

    // Per-recipient rate limit: max 3 notifications per customer per day
    $sent   = 0;
    $skipped = array();
    $today  = gmdate( 'Y-m-d' );

    foreach ( $customer_ids as $id ) {
        $count = (int) $wpdb->get_var( $wpdb->prepare(
            "SELECT COUNT(*) FROM {$wpdb->prefix}wz_notification_log
             WHERE customer_id = %d AND channel = %s AND DATE(sent_at) = %s",
            $id, $channel, $today
        ) );

        if ( $count >= 3 ) {
            $skipped[] = $id;
            continue;
        }

        wz_queue_notification( $id, $channel, $message );
        $sent++;
    }

    set_transient( 'wz_bulk_notify_' . get_current_user_id(), time(), 300 );

    return new WP_REST_Response( array(
        'sent'    => $sent,
        'skipped' => count( $skipped ),
        'message' => $skipped ? sprintf( '%d khách đã đạt giới hạn 3 tin/ngày', count( $skipped ) ) : '',
    ) );
}
```

## Áp dụng

Mọi admin notification endpoint PHẢI có: (1) max recipients per request, (2) cooldown giữa các lần bulk send, (3) per-recipient daily limit. Log mọi notification gửi đi để audit + rate limit check.
