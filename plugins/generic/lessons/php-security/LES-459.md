# Bug: Raw $wpdb Query Without prepare() — SQL Injection

## Problem

Using `$wpdb->query()`, `$wpdb->get_results()`, `$wpdb->get_row()`, or `$wpdb->get_var()` with interpolated variables without `$wpdb->prepare()` allows SQL injection.

## AST Detection

Tree-sitter finds `member_call_expression` nodes where object contains `wpdb` and method is `query`/`get_results`/`get_row`/`get_var`, then checks if `prepare` is used in the arguments.

## Bad

```php
$wpdb->query("DELETE FROM {$wpdb->prefix}orders WHERE id = $order_id");
$wpdb->get_results("SELECT * FROM {$table} WHERE user_id = {$user_id}");
```

## Good

```php
$wpdb->query($wpdb->prepare("DELETE FROM {$wpdb->prefix}orders WHERE id = %d", $order_id));
$wpdb->get_results($wpdb->prepare("SELECT * FROM %i WHERE user_id = %d", $table, $user_id));
```
