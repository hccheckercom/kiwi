**NOTE:** Pattern disabled (type: manual) due to high false positive rate. Negative lookahead cannot detect wp_next_scheduled() check in separate method. Requires manual code review.

## Bad
```php
public function init(): void {
    add_action( 'wezone/order/completed', [ $this, 'earn_points_from_order' ] );
    wp_schedule_event( time(), 'daily', 'wezone_loyalty_expire_check' );
    // Every request creates 1 new cron job
}
```

## Good
```php
public function init(): void {
    add_action( 'wezone/order/completed', [ $this, 'earn_points_from_order' ] );
    add_action( 'init', [ $this, 'maybe_schedule_expire' ] );
}

public function maybe_schedule_expire(): void {
    if ( ! wp_next_scheduled( 'wezone_loyalty_expire_check' ) ) {
        wp_schedule_event( time() + HOUR_IN_SECONDS, 'daily', 'wezone_loyalty_expire_check' );
    }
}
```

## Why
wp_schedule_event() called every request without checking wp_next_scheduled() creates duplicate cron jobs. 1000 requests = 1000 cron entries, cron table bloat, expire_old_points() runs 1000 times/day.

## Grep
`wp_schedule_event\s*\([^)]+\)(?!.*if\s*\(\s*!\s*wp_next_scheduled)` — in `**/*.php`
