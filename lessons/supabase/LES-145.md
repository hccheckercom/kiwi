## Bad
```ts
window.addEventListener('message', (event) => {
  const { colors } = event.data
  // No validation — attacker iframe can inject arbitrary CSS
  document.documentElement.style.setProperty('--primary', colors.primary)
  // colors.primary = "red; } * { display:none } .x {"  ← CSS injection
})
```

## Good (security layer from lib/color/security.ts)
```ts
import { isValidColorValue, isValidColorScheme, getSecurityConfig } from '@/lib/color/security'

window.addEventListener('message', (event) => {
  const { trustedOrigins } = getSecurityConfig()
  if (!trustedOrigins.has(event.origin)) return // Origin check

  if (!isValidColorScheme(event.data)) return // Schema validation

  // Each color value validated against strict regex
  // HEX: /^#[0-9a-fA-F]{6}$/
  // HSL: /^hsl\(\s*\d{1,3}\s*,\s*\d{1,3}%\s*,\s*\d{1,3}%\s*\)$/
  // OKLCH: /^oklch\(\s*[\d.]+\s+[\d.]+\s+[\d.]+\s*\)$/
  // Max length: 100 chars (prevent DoS)
  // No extra keys allowed (prevent injection via unknown fields)

  applyColors(event.data.colors)
})
```

## Why
webstore-vn uses postMessage to apply color themes from parent frame to demo iframes. Without validation, a malicious parent can inject arbitrary CSS via color values. The security layer (lib/color/security.ts) validates: origin, schema shape, color format regex, max length, and allowed keys.
