# LES-126: Plugin with REST API inside if(!$is_rest) block

## Problem

Plugin có đăng ký REST API routes (qua `Api::register()`) nhưng `->boot()` bị đặt trong block `if ( ! $is_rest )`. Kết quả: routes không bao giờ register khi WordPress xử lý REST request → admin UI gọi API trả về 404.

## Root Cause

`wezone-core.php` có block `if ( ! $is_rest )` để skip các plugin chỉ cần cho frontend. Developer đặt nhầm Banner\Plugin (có REST API) vào block này.

## Detection

```php
// SAI — Plugin có REST API nhưng nằm trong !$is_rest
if ( ! $is_rest ) {
    ( new \WeZone\Core\Banner\Plugin() )->boot(); // REST routes never register!
}
```

## Fix

```php
// ĐÚNG — Plugin có REST API phải boot NGOÀI $is_rest check
( new \WeZone\Core\Banner\Plugin() )->boot();

if ( ! $is_rest ) {
    // Chỉ đặt plugins KHÔNG có REST API ở đây
    ( new \WeZone\Core\Account\Plugin() )->boot();
}
```

## Key Points

- Bất kỳ plugin nào gọi `Api::register()` trong `boot()` → PHẢI nằm ngoài `if (!$is_rest)`
- Triệu chứng: Admin UI báo "Không tải được..." nhưng PHP CLI test OK (vì CLI không set `$is_rest`)
- Debug: kiểm tra `$wp_rest_server->get_routes()` trong HTTP context vs CLI context

## Affected

- wezone-core/Banner\Plugin (fixed 2026-05-19)
