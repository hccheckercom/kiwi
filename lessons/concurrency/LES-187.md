## Bad
```ts
const now = new Date()
if (now < flashSale.startTime || now > flashSale.endTime) {
  throw new Error('Flash sale not active')
}
```
Application server clock may differ from database clock. User can also manipulate client-side time checks.

## Good
```ts
// Let the database enforce time boundaries
const { data, error } = await supabase.rpc('purchase_flash_sale', {
  p_product_id: productId,
  p_quantity: quantity
})
// RPC: WHERE flash_start <= now() AND flash_end >= now() AND stock > 0
```

## Why
Flash sale timing must be enforced at the database level using `now()` — a single source of truth. Application servers may have clock skew, and client-side JavaScript is trivially manipulated. Database `now()` is atomic with the update query.
