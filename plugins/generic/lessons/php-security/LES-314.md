## Pattern

Admin search endpoints pass user input to SQL LIKE clause. Characters `%` and `_` are SQL wildcards — attacker searches `%` to dump all records, or `___` to match 3-char entries. `$wpdb->prepare()` escapes quotes but NOT LIKE wildcards.

## Scan

```yaml
pattern: 'LIKE\s*[''"]%.*\$|LIKE.*\$wpdb->prepare.*%s'
scope: "**/AdminUI/**/*.php|**/src/**/*Controller*.php"
exclude_line: "wpdb->esc_like|esc_like"
message: "LIKE query without esc_like() — wildcard injection in admin search"
```

## Bug

```php
// SAI — user search "%" trả về toàn bộ DB
$search = sanitize_text_field( $request->get_param( 'search' ) );
$results = $wpdb->get_results( $wpdb->prepare(
    "SELECT * FROM {$table} WHERE name LIKE %s",
    '%' . $search . '%'  // $search = "%" → LIKE "%%%" = match ALL
) );
```

## Fix

```php
$search = sanitize_text_field( $request->get_param( 'search' ) );
$results = $wpdb->get_results( $wpdb->prepare(
    "SELECT * FROM {$table} WHERE name LIKE %s",
    '%' . $wpdb->esc_like( $search ) . '%'  // Escapes % and _
) );
```

## Áp dụng

Mọi LIKE query trong admin search PHẢI dùng `$wpdb->esc_like()` trước khi wrap với `%`. Áp dụng cho: product search, order search, customer search, coupon search.
