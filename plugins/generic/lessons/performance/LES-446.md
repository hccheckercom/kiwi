# Bug: get_page_by_path() Multiple Calls Without Cache

## Problem

`get_page_by_path()` executes a `SELECT` query each time. When checking multiple slugs in a loop, this creates unnecessary DB load.

**Example (BAD):**
```php
$pages = [
    'shipping' => ['giao-hang', 'van-chuyen', 'shipping', 'chinh-sach-giao-hang'],
    'return'   => ['doi-tra', 'hoan-tra', 'return', 'chinh-sach-doi-tra'],
    'privacy'  => ['bao-mat', 'privacy', 'chinh-sach-bao-mat', 'privacy-policy'],
];

foreach ($pages as $type => $slugs) {
    foreach ($slugs as $slug) {
        $page = get_page_by_path($slug); // DB query each time
    }
}
// Worst case: 3 × 4 = 12 queries
```

## Fix

Batch-load all published pages once, check slugs against the in-memory set:

```php
$all_pages = get_posts([
    'post_type'   => 'page',
    'post_status' => 'publish',
    'numberposts' => -1,
    'fields'      => 'ids',
]);

// Build slug lookup set
$slug_set = [];
foreach ($all_pages as $page_id) {
    $slug_set[get_post_field('post_name', $page_id)] = true;
}

$found = 0;
foreach ($pages as $type => $slugs) {
    foreach ($slugs as $slug) {
        if (isset($slug_set[$slug])) {
            ++$found;
            break;
        }
    }
}
// 1 query instead of 12
```

## Impact

- 12 queries → 1 query for policy page checks
- Applies to any code checking page existence by slug
