## Vấn đề

Khi restore snapshot từ history (stored trong `wp_options`), code lấy `$history[$index]['tokens']` với null coalescing `?? array()` nhưng **không validate** rằng `tokens` thực sự là array.

Nếu data bị corrupt hoặc tampered:
- `tokens` có thể là string, int, hoặc nested object
- Downstream code gọi `foreach` trên non-array → fatal error
- `write_store_config_grouped()` nhận non-array → data corruption

## Sai

```php
$tokens = $history[ $index ]['tokens'] ?? array();
// $tokens có thể là string/int nếu data corrupt
$this->write_store_config_grouped( $tokens );
```

## Đúng

```php
$tokens = $history[ $index ]['tokens'] ?? array();
if ( ! is_array( $tokens ) ) {
    wp_send_json_error( array( 'message' => 'Invalid snapshot data' ) );
}
$this->write_store_config_grouped( $tokens );
```

## Tại sao quan trọng

- `get_option()` → `json_decode()` chain có thể return bất kỳ type nào
- Data trong `wp_options` có thể bị corrupt bởi migration, plugin conflict, hoặc manual DB edit
- Defense in depth: validate type TRƯỚC khi dùng
