# Bad Code

```python
@app.get("/users/{user_id}")
async def get_user(user_id: int, db: Session = Depends(get_db)):
    # BAD: Returns raw ORM model with password hash
    user = db.query(User).filter(User.id == user_id).first()
    return user  # Exposes password_hash, internal fields!

@app.get("/orders")
async def list_orders(db: Session = Depends(get_db)):
    # BAD: No response schema, returns everything
    orders = db.query(Order).all()
    return orders  # Exposes internal IDs, timestamps, etc.
```

# Good Code

```python
from pydantic import BaseModel

class UserResponse(BaseModel):
    id: int
    name: str
    email: str
    # password_hash NOT included
    
    class Config:
        from_attributes = True  # Pydantic v2 (orm_mode in v1)

class OrderResponse(BaseModel):
    id: int
    user_id: int
    total: float
    status: str
    created_at: datetime
    
    class Config:
        from_attributes = True

@app.get("/users/{user_id}", response_model=UserResponse)
async def get_user(user_id: int, db: Session = Depends(get_db)):
    user = db.query(User).filter(User.id == user_id).first()
    if not user:
        raise HTTPException(404, "User not found")
    return user  # Pydantic filters fields automatically

@app.get("/orders", response_model=list[OrderResponse])
async def list_orders(db: Session = Depends(get_db)):
    orders = db.query(Order).all()
    return orders  # Only returns fields in OrderResponse
```

# Why

**Without response models:**
- **Security:** Exposes sensitive data (passwords, tokens, internal IDs)
- **Breaking changes:** Adding DB fields breaks API contract
- **No documentation:** OpenAPI schema incomplete
- **Type safety:** No validation on response data

**Real-world security issues:**
```python
# BAD: Exposes password hash
@app.get("/users/{user_id}")
async def get_user(user_id: int):
    return db.query(User).first()
# Response: {"id": 1, "email": "user@example.com", "password_hash": "$2b$12$..."}

# GOOD: Filters sensitive fields
@app.get("/users/{user_id}", response_model=UserResponse)
async def get_user(user_id: int):
    return db.query(User).first()
# Response: {"id": 1, "email": "user@example.com"}
```

**Response model features:**
```python
# Exclude fields dynamically
@app.get("/users/me", response_model=UserResponse, response_model_exclude={"email"})
async def get_current_user():
    ...

# Include only specific fields
@app.get("/users", response_model=list[UserResponse], response_model_include={"id", "name"})
async def list_users():
    ...

# Different models for different status codes
@app.post("/users", response_model=UserResponse, status_code=201)
async def create_user(user: UserCreate):
    ...
```

**OpenAPI documentation:**
- Response models generate accurate OpenAPI schemas
- Clients can auto-generate types
- API documentation shows exact response structure
