**Mô tả:**
- Sau khi sửa code React trong `packages/wezone-core/src/AdminUI/` và rebuild, browser vẫn load file JS cũ từ cache
- WordPress dùng `WEZONE_ADMIN_UI_VERSION` làm cache-busting key cho `wp_enqueue_script()`
- Nếu version không thay đổi, browser không biết có file mới → admin vẫn thấy UI cũ

**Root cause:**
- Vite dùng content-hash cho filename (`main-OfX6kSkL.js`) — hash chỉ thay đổi khi content thay đổi đủ để tạo hash khác
- WordPress enqueue dùng version string làm query param: `main-OfX6kSkL.js?ver=1.0.0`
- Nếu version không đổi, browser cache hit → file cũ được serve

**Detect pattern:**
- `WEZONE_ADMIN_UI_VERSION` không thay đổi sau khi rebuild admin-ui

**Fix:**
- Bump `WEZONE_ADMIN_UI_VERSION` trong `packages/wezone-core/wezone-core.php` sau mỗi lần rebuild
- Hoặc dùng build timestamp: `define('WEZONE_ADMIN_UI_VERSION', '1.0.' . filemtime(__DIR__ . '/dist/.vite/manifest.json'));`

**Files liên quan:**
- `packages/wezone-core/wezone-core.php` — định nghĩa `WEZONE_ADMIN_UI_VERSION`
- `packages/wezone-core/src/AdminUI/AdminPage.php` — dùng version để enqueue script/style

**Lợi ích:** Admin luôn thấy UI mới nhất sau khi rebuild, không cần hard refresh
