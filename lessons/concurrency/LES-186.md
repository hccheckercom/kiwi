## Bad
```ts
const { data: coupon } = await supabase.from('coupons').select('*').eq('code', code).single()
if (coupon.usage_count >= coupon.usage_limit) throw new Error('Coupon expired')
await supabase.from('coupons').update({ usage_count: coupon.usage_count + 1 }).eq('id', coupon.id)
```
Two users submit same coupon simultaneously → both pass the check → coupon used usage_limit + 1 times.

## Good
```ts
const { data, error } = await supabase.rpc('redeem_coupon', {
  p_code: code,
  p_order_id: orderId
})
// RPC: UPDATE coupons SET usage_count = usage_count + 1
//       WHERE code = p_code AND usage_count < usage_limit
//       RETURNING *
if (error || !data) throw new Error('Coupon invalid or expired')
```

## Why
Coupon abuse is a revenue leak. Atomic check-and-increment in a single SQL statement eliminates the race window. Never read a counter, check it in application code, then write it back.
