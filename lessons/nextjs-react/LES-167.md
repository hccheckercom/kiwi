## Bad
```yaml
# deploy script — lint blocking deploy
steps:
  - run: npm run lint    # 1363 type-safety warnings → blocks deploy
  - run: npm run build   # Never reached
  - run: ./deploy.sh     # Never reached
```

## Good (pattern from webstore-vn CI)
```yaml
steps:
  - run: npm run lint || true    # Non-blocking — log warnings but don't fail
  - run: npm run build           # Build is the real gate
  - run: ./deploy.sh
```

## Or: separate lint from deploy
```yaml
jobs:
  lint:
    runs-on: ubuntu-latest
    continue-on-error: true  # Advisory only
    steps:
      - run: npm run lint

  deploy:
    runs-on: ubuntu-latest
    needs: []  # Does NOT depend on lint
    steps:
      - run: npm run build
      - run: ./deploy.sh
```

## Why
Real fix: commit 546599a. webstore-vn has 1363 type-safety issues (mostly `as any` in legacy code). Making lint block deploy means no deploys until all 1363 are fixed — unrealistic. Pattern: build errors block deploy (they break the app), lint warnings are advisory (they don't break the app). Fix lint incrementally, don't let it block shipping.
