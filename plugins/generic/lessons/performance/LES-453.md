# Bug: N+1 Custom Table Queries In Foreach Loop

## Problem

Calling `$wpdb->get_var()` inside a foreach loop to get per-item aggregated data (revenue, download count, etc.) causes N queries. Should aggregate in one query.

**Example (BAD):**
```php
foreach ( $posts as $post ) {
    $revenue = (float) $wpdb->get_var( $wpdb->prepare(
        "SELECT COALESCE(SUM(amount), 0) FROM {$orders_table}
         WHERE theme_product_id = %d AND status IN ('completed')",
        $post['ID']
    ) );
    $downloads = (int) $wpdb->get_var( $wpdb->prepare(
        "SELECT COUNT(*) FROM {$downloads_table} WHERE theme_product_id = %d",
        $post['ID']
    ) );
}
// 10 themes × 2 queries = 20 queries
```

## Fix

Use a single aggregated query with GROUP BY:

```php
$ids = wp_list_pluck( $posts, 'ID' );
$placeholders = implode( ',', array_fill( 0, count( $ids ), '%d' ) );

$revenue_map = array();
$rows = $wpdb->get_results( $wpdb->prepare(
    "SELECT theme_product_id, COALESCE(SUM(amount), 0) as revenue
     FROM {$orders_table}
     WHERE theme_product_id IN ($placeholders) AND status IN ('completed', 'processing')
     GROUP BY theme_product_id",
    ...$ids
), ARRAY_A );
foreach ( $rows as $row ) {
    $revenue_map[ (int) $row['theme_product_id'] ] = (float) $row['revenue'];
}

foreach ( $posts as $post ) {
    $revenue = $revenue_map[ $post['ID'] ] ?? 0;
}
// 2 queries total regardless of count
```

## Impact

- 10 themes × 2 queries = **20 queries → 2 queries**
