## Pattern

Admin UI allows changing order status via dropdown without validating allowed transitions. OrderStateMachine exists but admin endpoint bypasses it → can set "completed" from "cancelled", or "pending" from "refunded" — breaking business logic and triggering wrong hooks.

## Bug

```php
// SAI — admin endpoint set status trực tiếp không qua state machine
public function update_order_status( WP_REST_Request $request ): WP_REST_Response {
    global $wpdb;
    $order_id = (int) $request->get_param( 'id' );
    $status   = sanitize_text_field( $request->get_param( 'status' ) );

    // Không validate transition! cancelled → completed = invalid
    $wpdb->update(
        $wpdb->prefix . 'wz_orders',
        array( 'status' => $status ),
        array( 'id' => $order_id )
    );
    return new WP_REST_Response( array( 'success' => true ) );
}
```

## Fix

```php
public function update_order_status( WP_REST_Request $request ): WP_REST_Response {
    $order_id   = (int) $request->get_param( 'id' );
    $new_status = sanitize_text_field( $request->get_param( 'status' ) );

    // Dùng OrderStateMachine — validate transition + fire hooks
    $state_machine = new OrderStateMachine( new OrderHistory() );
    $result = $state_machine->transition( $order_id, $new_status );

    if ( is_wp_error( $result ) ) {
        return new WP_REST_Response(
            array( 'error' => $result->get_error_message() ),
            422
        );
    }

    return new WP_REST_Response( array(
        'success' => true,
        'status'  => $new_status,
    ) );
}
```

## Áp dụng

Mọi order status change (admin, webhook, cron) PHẢI đi qua OrderStateMachine. Không bao giờ UPDATE status trực tiếp trong DB. State machine đảm bảo: (1) transition hợp lệ, (2) hooks fire đúng, (3) history ghi log.
