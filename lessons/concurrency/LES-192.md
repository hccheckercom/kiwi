## Bad
```ts
// Zustand store
updateQuantity: (itemId, newQty) => {
  const cart = get().cart
  const item = cart.find(i => i.id === itemId)
  if (item) {
    set({ cart: cart.map(i => i.id === itemId ? { ...i, quantity: newQty } : i) })
    // Sync to server
    fetch('/api/cart', { method: 'PUT', body: JSON.stringify({ itemId, quantity: newQty }) })
  }
}
```
Rapid clicks: user sets qty=2 then qty=3, server receives both in parallel → last-write-wins, may save qty=2.

## Good
```ts
updateQuantity: async (itemId, newQty) => {
  // Optimistic update
  set({ cart: get().cart.map(i => i.id === itemId ? { ...i, quantity: newQty } : i) })

  // Server sync with expected version
  const { data, error } = await fetch('/api/cart', {
    method: 'PUT',
    body: JSON.stringify({ itemId, quantity: newQty, version: get().cartVersion })
  }).then(r => r.json())

  if (error) {
    // Rollback optimistic update, refetch
    const fresh = await fetch('/api/cart').then(r => r.json())
    set({ cart: fresh.items, cartVersion: fresh.version })
  } else {
    set({ cartVersion: data.version })
  }
}
```

## Why
Cart is a high-contention resource — users rapidly change quantities, add/remove items. Without version tracking, concurrent updates cause the wrong quantity to persist. Optimistic locking with rollback ensures consistency.
