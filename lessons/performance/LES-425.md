## Problem

`get_theme_tokens()` builds CSS string on every login page request. The method loops through tokens, calls `get_token()` multiple times, and concatenates strings — all repeated work that produces the same output unless config changes.

## Bad

```php
public function enqueue_styles(): void {
    $tokens = $this->get_theme_tokens();  // Rebuilt every request
    $css    = ':root {' . $tokens . '}';
    wp_add_inline_style( 'wezone-login', $css );
}

private function get_theme_tokens(): string {
    $tokens = array(
        '--wz-primary'   => $this->get_primary_color(),
        '--wz-secondary' => $this->get_token( 'theme.secondary', '#7b5800' ),
        // ... 12 more tokens
    );
    $css = '';
    foreach ( $tokens as $prop => $value ) {
        $css .= $prop . ':' . $value . ';';
    }
    return $css;
}
```

## Good

```php
public function enqueue_styles(): void {
    $tokens = $this->get_cached_theme_tokens();
    $css    = ':root {' . $tokens . '}';
    wp_add_inline_style( 'wezone-login', $css );
}

private function get_cached_theme_tokens(): string {
    $cache_key = 'wezone_login_tokens_' . md5( serialize( wz_config( 'theme', array() ) ) );
    $cached    = wp_cache_get( $cache_key, 'wezone-login' );
    if ( false !== $cached ) {
        return $cached;
    }

    $tokens = $this->get_theme_tokens();
    wp_cache_set( $cache_key, $tokens, 'wezone-login', HOUR_IN_SECONDS );
    return $tokens;
}
```

## Why

- **Performance**: Eliminates repeated string building (saves 1-5ms per request)
- **Scalability**: Reduces CPU load on high-traffic sites
- **Cache invalidation**: Only rebuild when theme config changes

## How to fix

1. Wrap `get_theme_tokens()` in cache layer
2. Use `wp_cache_get/set` for object cache (Redis/Memcached)
3. Or use `get_transient/set_transient` for DB-backed cache
4. Cache key should include config hash to auto-invalidate on change
