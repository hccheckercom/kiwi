## Bad
```php
// ThemeProductController.php:322, 324, 327
$raw_addons = get_post_meta( $id, '_wz_addon_services', true );
$item['addon_services'] = $raw_addons ? json_decode( $raw_addons, true ) : array();

$raw_benefits = get_post_meta( $id, '_wz_purchase_benefits', true );
$item['purchase_benefits'] = $raw_benefits ? json_decode( $raw_benefits, true ) : array();
```

## Good
```php
$raw_addons = get_post_meta( $id, '_wz_addon_services', true );
if ( $raw_addons ) {
    $decoded = json_decode( $raw_addons, true );
    if ( JSON_ERROR_NONE === json_last_error() ) {
        $item['addon_services'] = $decoded;
    } else {
        $item['addon_services'] = array();
        error_log( "JSON decode failed for addon_services (post $id): " . json_last_error_msg() );
    }
} else {
    $item['addon_services'] = array();
}
```

## Why
`json_decode()` returns `null` on both empty input AND malformed JSON. Without `json_last_error()` check:
- Corrupted JSON â†’ silent `null` â†’ empty array fallback
- No error logging â†’ data loss goes unnoticed
- Debugging becomes impossible

Always check `json_last_error()` after decode and log failures with context (entity ID, field name).

## Grep
`json_decode\s*\([^)]+\)\s*(?!.*json_last_error)` — in `**/*.php`
