## Bad
```php
async def websocket_endpoint(websocket: WebSocket):
    await manager.connect(websocket)
    
    while True:
        data = await websocket.receive_text()
        # Process message
        
# Missing: WebSocketDisconnect exception handling
```

## Good
```php
from fastapi import WebSocket, WebSocketDisconnect

@app.websocket("/ws")
async def websocket_endpoint(websocket: WebSocket):
    try:
        await manager.connect(websocket)
        
        while True:
            data = await websocket.receive_text()
            # Process message
            
    except WebSocketDisconnect:
        manager.disconnect(websocket)
    except Exception as e:
        print(f"WebSocket error: {e}", file=sys.stderr)
        try:
            await websocket.close(code=1011, reason=str(e))
        except:
            pass
        manager.disconnect(websocket)
```

## Why
Without WebSocketDisconnect exception handling, when client disconnects, the endpoint crashes and connection is not cleaned up from manager. This causes memory leak as disconnected connections remain in active_connections list. Always catch WebSocketDisconnect and call manager.disconnect().
