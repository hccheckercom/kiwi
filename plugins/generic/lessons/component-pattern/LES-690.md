## Bad
```tsx
// app/admin/products/page.tsx
<button className="bg-blue-600 text-white px-4 py-2 rounded">Add Product</button>

// app/admin/products/[id]/page.tsx
<button className="bg-indigo-500 text-white px-3 py-1.5 rounded-md">Save</button>
```
Same section, different button styles — inconsistent visual language.

## Good
```tsx
// Both use shared Button component
<Button variant="primary">Add Product</Button>
<Button variant="primary">Save</Button>
```
Or consistent utility classes:
```tsx
<button className="btn-primary">Add Product</button>
<button className="btn-primary">Save</button>
```

## Why
Inconsistent button styles within the same section signal a lack of design system adoption. Users lose trust when UI elements look different for no reason. Primary actions should always look the same.
