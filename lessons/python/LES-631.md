## Bad
```python
line = args.get("line", 0)
min_occ = args.get("min_occurrences", 2)
```

## Good
```python
line = int(args.get("line", 0))
min_occ = int(args.get("min_occurrences", 2))
```

## Why

MCP/JSON-RPC arguments arrive as JSON values. Clients may send `"42"` (string) instead of `42` (int). Without explicit `int()` casting, downstream code like `violation.line - 1` will throw `TypeError: unsupported operand type(s) for -: 'str' and 'int'`. Always cast at the API boundary.
