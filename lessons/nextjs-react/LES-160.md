## Bad
```tsx
// Backend API returns: { ends_at, items: [...] }
// Frontend theme expects: { end_time, products: [...] }
const flashSale = await fetchFlashSale()
return <CountdownTimer endTime={flashSale.end_time} /> // undefined!
return <ProductGrid products={flashSale.products} />   // undefined!
```

## Good (transform layer pattern from apps/demo/app/[slug]/page.tsx)
```tsx
const raw = await fetchFlashSale()

// Transform API shape → theme expected shape
const flashSale = raw ? {
  ...raw,
  end_time: raw.ends_at,           // API: ends_at → Theme: end_time
  products: (raw.items ?? []).map(item => ({
    ...item,
    name: item.product_name,       // API: product_name → Theme: name
    image: item.product_image,     // API: product_image → Theme: image
  })),
} : null

return <CountdownTimer endTime={flashSale?.end_time} />
return <ProductGrid products={flashSale?.products ?? []} />
```

## Defensive guard in component
```tsx
// Always guard against null/undefined — API may return null when no active sale
{flashSale && flashSale.products?.length > 0 && (
  <FlashSaleSection sale={flashSale} />
)}
```

## Why
Real bug: commit 03f42ac. Platform API returns `{ends_at, items[]}` but theme components expect `{end_time, products[]}`. This caused TypeError on agriculture theme when flash sale existed. Fix: add explicit transform layer between API response and component props. Always add null guards for optional data.
