# Bug: Cart merge() Không Check Stock — Qty Vượt Tồn Kho

## Vấn đề

Khi user login, `merge()` cộng qty từ guest cart vào user cart mà không kiểm tra stock. Nếu guest cart có qty=5 và user cart có qty=5 nhưng stock chỉ còn 8 → merged qty=10 > stock=8.

**Ví dụ lỗi:**
```php
// ❌ SAI — cộng qty không check stock
foreach ( $guest_data['items'] as $guest_item ) {
    foreach ( $cart['items'] as &$item ) {
        if ( $item['key'] === $key ) {
            $item['qty'] += $qty;  // ← không check stock!
            break;
        }
    }
}
```

## Fix

```php
// ✅ ĐÚNG — check stock trước khi merge
foreach ( $guest_data['items'] as $guest_item ) {
    $product_id = (int) ( $guest_item['product_id'] ?? 0 );
    $qty        = (int) ( $guest_item['qty'] ?? 1 );

    $product = wz_get_product( $product_id );
    if ( ! $product || empty( $product['in_stock'] ) ) {
        continue;
    }

    $stock_qty = (int) ( $product['stock_quantity'] ?? PHP_INT_MAX );

    foreach ( $cart['items'] as &$item ) {
        if ( $item['key'] === $key ) {
            $item['qty'] = min( $item['qty'] + $qty, $stock_qty );
            $found = true;
            break;
        }
    }
    unset( $item );

    if ( ! $found && $qty <= $stock_qty ) {
        // add new item
    }
}
```
