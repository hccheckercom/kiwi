# LES-292: Missing --text-5xl token — incomplete type scale

## Symptom
- Theme's `_tokens.css` defines tokens from `--text-xs` to `--text-4xl` but missing `--text-5xl`
- Display/hero text that needs 40px→64px fluid scaling has no token to reference
- Blueprint requires 9 size tokens: xs, sm, base, lg, xl, 2xl, 3xl, 4xl, 5xl

## Root Cause
When scaffolding the token system, the largest display size (`--text-5xl`) is often omitted because it's rarely used. However, the blueprint mandates it for:
- Hero display text on landing pages
- Large promotional headings
- Error page (404/500) display numbers

## Wrong Pattern
```css
/* _tokens.css — SAI: scale stops at 4xl */
--text-4xl: clamp(2rem, 1.5rem + 2vw, 3rem);
/* no --text-5xl defined */
```

## Correct Pattern
```css
/* _tokens.css — ĐÚNG: complete scale through 5xl */
--text-4xl: clamp(2rem, 1.5rem + 2vw, 3rem);
--text-5xl: clamp(2.5rem, 1.7rem + 3vw, 4rem);
```

## Fix Checklist
1. Open `assets/css/partials/_tokens.css`
2. Add `--text-5xl: clamp(2.5rem, 1.7rem + 3vw, 4rem);` after `--text-4xl`
3. Verify formula matches blueprint: 40px (mobile 375px) → 64px (desktop 1440px)
4. Update any hardcoded large sizes (64px, 48px in error pages) to use `var(--text-5xl)`
