## Pattern

Developer check WooCommerce active trước khi chạy code. Check này luôn trả false trên Wezone.

## Bad

```php
// Plugin compatibility check
if ( ! class_exists( 'WooCommerce' ) ) {
    return; // Plugin silently deactivates — BUG
}

// Function guard
if ( function_exists( 'WC' ) ) {
    $cart = WC()->cart;
}

if ( function_exists( 'wc_get_product' ) ) {
    $product = wc_get_product( $id );
}

// Constant check
if ( defined( 'WC_VERSION' ) ) {
    // WooCommerce-specific code
}

if ( defined( 'WOOCOMMERCE_VERSION' ) ) {
    // Legacy check
}

// Class check for features
if ( class_exists( 'WC_Payment_Gateway' ) ) {
    class My_Gateway extends WC_Payment_Gateway { }
}
```

## Good

```php
// Plugin compatibility check — dùng Wezone guard
if ( ! function_exists( 'wezone_is_active' ) ) {
    return;
}

// Function guard — Wezone functions luôn available khi plugin active
if ( function_exists( 'wz_get_product' ) ) {
    $product = wz_get_product( $id );
}

// Version check
if ( defined( 'WEZONE_VERSION' ) ) {
    // Wezone-specific code
}

// Feature check — dùng Wezone interfaces
if ( interface_exists( 'Wz_Payment_Gateway_Interface' ) ) {
    class My_Gateway implements Wz_Payment_Gateway_Interface { }
}

// Hoặc đơn giản — trong Wezone ecosystem, guard bằng wezone_is_active()
if ( wezone_is_active() ) {
    // Safe to use wz_* functions
}
```

## Why

Patterns `class_exists('WooCommerce')` và `function_exists('wc_*')` là anti-patterns phổ biến khi migrate:
1. **Guard luôn false** → code bên trong không bao giờ chạy → feature bị disable silently
2. **Early return** → plugin/theme tự deactivate mà không báo lỗi
3. **Conditional class definition** → class không được define → fatal error ở nơi khác

Đây là bug khó phát hiện vì:
- Không có error log
- Feature đơn giản là "không xuất hiện"
- Developer nghĩ code đúng vì không có lỗi
