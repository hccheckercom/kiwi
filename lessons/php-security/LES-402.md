# Bug: wz_cart_add() Chấp Nhận Qty ≤ 0 — Price Manipulation

## Vấn đề

`wz_cart_add()` có default `$qty = 1` nhưng không validate khi caller truyền qty ≤ 0 hoặc qty âm. Attacker POST `qty=-5` → cart total âm → checkout refund thay vì charge.

**Impact:** Price manipulation, mất tiền.

## Bad

```php
// ❌ SAI — không check qty
function wz_cart_add( int $product_id, int $qty = 1, int $variation_id = 0, array $meta = array() ): string {
    return wz_cart()->add( $product_id, $qty, $variation_id, $meta );
}
```

## Good

```php
// ✅ ĐÚNG — validate ở gateway level
function wz_cart_add( int $product_id, int $qty = 1, int $variation_id = 0, array $meta = array() ): string {
    if ( $qty < 1 ) {
        return '';
    }
    if ( $product_id < 1 ) {
        return '';
    }
    return wz_cart()->add( $product_id, $qty, $variation_id, $meta );
}
```
