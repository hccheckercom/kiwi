## Bad
```css
/* main.css */
@import 'partials/_reset.css';
@import 'partials/_tokens.css';
@import 'partials/_typography.css';
@import 'partials/_layout.css';
```
Each @import creates a sequential network request — browser must download file A before discovering file B.

## Good
Use a build tool (PostCSS, Tailwind CLI) to bundle into `main.bundle.css`:
```json
// postcss.config.js
module.exports = { plugins: [require('postcss-import'), require('tailwindcss')] }
```
Then in functions.php, prefer the bundle:
```php
$bundle = THEME_DIR . '/assets/css/main.bundle.css';
$main   = file_exists( $bundle ) ? $bundle : THEME_DIR . '/assets/css/main.css';
```

## Why
Performance constitution explicitly bans `@import` in CSS — it blocks rendering chain. If `main.bundle.css` exists the theme uses it, but if missing (dev without build step), the fallback `main.css` with 40+ @imports will severely degrade FCP/LCP.

## Mitigation
If theme already has `main.bundle.css` AND `functions.php` prefers it, this is acceptable for development workflow. The scan flags it as a reminder to always build before deploy.
