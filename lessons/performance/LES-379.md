# Bug: resolveDiscount() Double Query Dù Đã Fetch coupon_row

## Vấn đề

`resolveDiscount()` gọi `getByCode($code)` rồi `validate($code, $subtotal)` — nhưng `validate()` nhận `$code` (string) nên phải query DB lại để lấy coupon row. Comment "FIX BUG-106" nói đã fix nhưng thực tế chỉ fix cho `calculateDiscount()`.

**Ví dụ lỗi:**
```php
// ❌ SAI — validate() vẫn query DB lại
$coupon_row = $this->couponEngine->getByCode( $code );  // query 1
$error = $this->couponEngine->validate( $code, $subtotal );  // query 2 (internally)
$discount = $this->couponEngine->calculateDiscount( $code, $subtotal, $coupon_row );  // no query (fixed)
```

## Fix

```php
// ✅ ĐÚNG — pass coupon_row vào validate
$coupon_row = $this->couponEngine->getByCode( $code );
if ( ! $coupon_row ) {
    return 0.0;
}
$error = $this->couponEngine->validateRow( $coupon_row, $subtotal );
if ( $error !== '' ) {
    return 0.0;
}
return $this->couponEngine->calculateDiscount( $code, $subtotal, $coupon_row );
```
