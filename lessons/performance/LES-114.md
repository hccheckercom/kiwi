# LES-114: Client-Side PHP Search Filtering

## Problem

Order search endpoint:
1. Fetch ALL orders matching status filter (có thể hàng nghìn records)
2. Load toàn bộ vào PHP memory
3. `array_filter()` với `str_contains()` để tìm matching records
4. Trả về filtered results

Vấn đề:
- Memory: load 1000+ orders × full JSON (billing, shipping, items) vào RAM
- CPU: iterate toàn bộ array trong PHP
- DB: query không có LIMIT khi search (hoặc LIMIT quá lớn)
- Bandwidth: transfer toàn bộ data từ DB → PHP chỉ để discard 95%

## Fix

Chuyển search logic sang SQL WHERE clause:

```php
// ĐÚNG — SQL search
if ( ! empty( $args['search'] ) ) {
    $like     = '%' . $wpdb->esc_like( $args['search'] ) . '%';
    $where[]  = '(order_number LIKE %s OR billing LIKE %s OR guest_email LIKE %s)';
    $params[] = $like;
    $params[] = $like;
    $params[] = $like;
}

// SAI — PHP filter
$rows = $this->engine->list( $args ); // fetch ALL
$rows = array_filter( $rows, function($row) use ($search) {
    return str_contains($row['order_number'], $search)
        || str_contains($row['billing']['email'], $search);
});
```

Lưu ý: `billing` column là JSON → `LIKE %search%` trên JSON string vẫn match được name/email/phone.

## Impact

- Memory: O(n) → O(page_size)
- Queries: full table scan → indexed LIKE with LIMIT
- Time: 500ms+ (1000 orders) → <50ms

## Detection

Tìm pattern: `array_filter` + `str_contains` hoặc `strpos` trong REST controller sau khi fetch data.
