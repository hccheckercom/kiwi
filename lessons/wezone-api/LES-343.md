## Bad
```php
// functions.php
require __DIR__ . '/functions/product.php';
require __DIR__ . '/functions/order.php';
require __DIR__ . '/functions/cart.php';
```

## Good
```php
// functions.php
require_once __DIR__ . '/functions/product.php';
require_once __DIR__ . '/functions/order.php';
require_once __DIR__ . '/functions/cart.php';
```

## Why
`require` will re-parse the file every time functions.php is loaded. While individual functions use `function_exists()` guards, `require_once` is safer and avoids wasted parsing. AdapterManager uses `require_once` for functions.php itself but the sub-includes use `require`.
