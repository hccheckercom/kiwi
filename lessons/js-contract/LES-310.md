## Pattern

Admin SPA fetches data once on mount. If admin opens 2 tabs, edits in tab A, switches to tab B — tab B shows stale data. Also after mutation (create/update/delete), list view not refetched → user sees old data until manual F5.

## Bug

```tsx
// SAI — fetch 1 lần, không bao giờ refresh
useEffect(() => {
  wzApi.get('products', { per_page: '20' }).then(setProducts);
}, []);

// Mutation xong nhưng không refetch list
const handleDelete = async (id: number) => {
  await wzApi.delete(`products/${id}`);
  toast('success', 'Đã xóa');
  // List vẫn hiển thị product đã xóa!
};
```

## Fix

```tsx
// Pattern 1: Manual refetch after mutation
const [version, setVersion] = useState(0);

useEffect(() => {
  wzApi.get('products', { per_page: '20', page: String(page) }).then(setProducts);
}, [page, version]);

const handleDelete = async (id: number) => {
  await wzApi.delete(`products/${id}`);
  setVersion(v => v + 1); // trigger refetch
  toast('success', 'Đã xóa');
};

// Pattern 2: Refetch on window focus
useEffect(() => {
  const onFocus = () => setVersion(v => v + 1);
  window.addEventListener('focus', onFocus);
  return () => window.removeEventListener('focus', onFocus);
}, []);
```

## Áp dụng

Mọi admin list view PHẢI refetch sau mutation (create/update/delete). Khuyến khích thêm refetch-on-focus cho data thay đổi thường xuyên (orders, stock).
