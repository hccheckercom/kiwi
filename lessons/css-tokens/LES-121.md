## Problem

Browser default styles apply `text-decoration: underline` to all `<a>` tags. Without a global reset in the theme's base CSS, every link (logo, nav, buttons, hotline) displays an ugly underline.

## Detection

In `src/main.css` (or equivalent base CSS file), check for:

```css
a {
  text-decoration: none;
  color: inherit;
}
```

If missing → this bug is present.

## Fix

Add to `src/main.css` after the `body` reset:

```css
a {
  text-decoration: none;
  color: inherit;
}
```

Then rebuild: `npm run build:css`

## Why `color: inherit`

Without it, links default to browser blue (`#0000EE`), which breaks the design system token colors applied via Tailwind classes.
