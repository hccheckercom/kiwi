## Bad
```php
<img src="<?php echo esc_url( $product['image'] ); ?>"
  style="width:100%;aspect-ratio:1/1;object-fit:contain;" loading="lazy" />
```
No `width`/`height` attributes — browser can't reserve space before CSS loads → CLS.

## Good
```php
<img src="<?php echo esc_url( $product['image'] ); ?>"
  width="400" height="400"
  style="width:100%;aspect-ratio:1/1;object-fit:contain;"
  loading="lazy" decoding="async" />
```

## Why
Even with CSS `aspect-ratio`, explicit `width`/`height` attributes are needed as a fallback for browsers that haven't parsed CSS yet. Without them, the performance.md lazy-load filter also skips these images (it only adds lazy/decoding to imgs with both attributes).
