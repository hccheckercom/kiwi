# Bug: Banner update không validate image_id/reference_id tồn tại

## Mô tả
`BannerRepository::create()` và `update()` nhận `image_id`, `reference_id` từ client nhưng không check xem attachment/product/category có tồn tại không → orphan references.

## Tại sao nguy hiểm
- **Data integrity**: Banner reference image đã bị xóa
- **Broken UI**: Frontend render banner với image_id không tồn tại → broken image
- **Cascade delete**: Khi xóa product, banner vẫn reference → stale data

## Code sai (BannerRepository.php:90-119)
```php
public static function create( array $data ): int {
    global $wpdb;

    $insert = array(
        'title'      => sanitize_text_field( $data['title'] ?? '' ),
        'position'   => sanitize_key( $data['position'] ?? 'home_hero' ),
        'type'       => sanitize_key( $data['type'] ?? 'image' ),
        'image_id'   => ! empty( $data['image_id'] ) ? absint( $data['image_id'] ) : null,
        // ... rest
        'reference_id' => ! empty( $data['reference_id'] ) ? absint( $data['reference_id'] ) : null,
    );

    $result = $wpdb->insert( self::table(), $insert );
    // ...
}
```

**Vấn đề**: 
- `absint($data['image_id'])` không check attachment tồn tại
- `absint($data['reference_id'])` không check product/category tồn tại

## Code đúng
```php
public static function create( array $data ): int {
    global $wpdb;

    // Validate image_id
    $image_id = null;
    if ( ! empty( $data['image_id'] ) ) {
        $id = absint( $data['image_id'] );
        if ( 'attachment' === get_post_type( $id ) ) {
            $image_id = $id;
        }
    }

    // Validate reference_id based on type
    $reference_id = null;
    if ( ! empty( $data['reference_id'] ) && ! empty( $data['type'] ) ) {
        $id = absint( $data['reference_id'] );
        $type = sanitize_key( $data['type'] );

        if ( 'product' === $type ) {
            $exists = $wpdb->get_var( $wpdb->prepare(
                "SELECT COUNT(*) FROM {$wpdb->prefix}wz_products WHERE id = %d",
                $id
            ) );
            if ( $exists ) {
                $reference_id = $id;
            }
        } elseif ( 'category' === $type ) {
            $exists = $wpdb->get_var( $wpdb->prepare(
                "SELECT COUNT(*) FROM {$wpdb->prefix}wz_categories WHERE id = %d",
                $id
            ) );
            if ( $exists ) {
                $reference_id = $id;
            }
        }
    }

    $insert = array(
        'title'      => sanitize_text_field( $data['title'] ?? '' ),
        'position'   => sanitize_key( $data['position'] ?? 'home_hero' ),
        'type'       => sanitize_key( $data['type'] ?? 'image' ),
        'image_id'   => $image_id,
        'reference_id' => $reference_id,
        // ... rest
    );

    $result = $wpdb->insert( self::table(), $insert );
    // ...
}
```

## Cách fix
1. Validate `image_id` với `get_post_type($id) === 'attachment'`
2. Validate `reference_id` với query tương ứng (products/categories)
3. Set `null` nếu không tồn tại thay vì insert invalid ID
4. Áp dụng cho cả `create()` và `update()`

## Áp dụng cho
- Mọi foreign key reference: product_id, category_id, user_id, attachment_id
- Đặc biệt quan trọng khi không có DB foreign key constraint
- Admin UI nên validate trước khi submit (UX tốt hơn)
