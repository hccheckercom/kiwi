# Bug: CartEngine N+1 Reads — Mỗi Getter Gọi get() Riêng

## Vấn đề

`getItems()`, `getTotal()`, `getSubtotal()`, `getShipping()`, `getCount()`, `isEmpty()` mỗi method đều gọi `get()` → `loadRow()` riêng. Khi template gọi 3-4 getters trong cùng request → 3-4 lần cache lookup (hoặc DB hit nếu cache miss).

**Ví dụ lỗi:**
```php
// ❌ SAI — mỗi method gọi get() riêng
public function getItems(): array {
    return $this->get()['items'] ?? array();  // get() → loadRow()
}
public function getTotal(): float {
    return (float) ( $this->get()['total'] ?? 0 );  // get() → loadRow() AGAIN
}
public function getCount(): int {
    foreach ( $this->getItems() as $item ) {  // getItems() → get() → loadRow() AGAIN
        $count += (int) ( $item['qty'] ?? 1 );
    }
}
```

## Fix

```php
// ✅ ĐÚNG — cache cart data per-request
private ?array $cached_cart = null;

public function get(): array {
    if ( $this->cached_cart !== null ) {
        return $this->cached_cart;
    }
    $row = $this->loadRow();
    if ( ! $row ) {
        return $this->emptyCart();
    }
    $data = json_decode( $row['cart_data'], true );
    $this->cached_cart = is_array( $data ) ? $data : $this->emptyCart();
    return $this->cached_cart;
}

// Invalidate cache khi write
private function save( array $cart ): bool {
    $this->cached_cart = null;  // ← invalidate
    // ... existing save logic
}
```
