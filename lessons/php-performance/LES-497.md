## Bad
```php
public function apply_tier_discount( float $price, int $product_id ): float {
    $discount = (int) get_option( 'wezone-loyalty_discount_silver', 3 );
    // Filter chay 50 lan/page -> 50 get_option() calls
}
```

## Good
```php
private static $tier_discounts = null;

public function apply_tier_discount( float $price, int $product_id ): float {
    if ( self::$tier_discounts === null ) {
        self::$tier_discounts = [
            'silver' => (int) get_option( 'wezone-loyalty_discount_silver', 3 ),
            'gold'   => (int) get_option( 'wezone-loyalty_discount_gold', 5 ),
        ];
    }
    $discount = self::$tier_discounts['silver'];
}
```

## Why
get_option() in filter hook runs every request (e.g. wezone/product/price filter runs for each product). No cache means N queries per page load. WP autoload caches options but still has overhead.

## Grep
`get_option\s*\([^)]+\).*?(?:foreach|while|for\s*\(|add_filter)` — in `**/*.php`
