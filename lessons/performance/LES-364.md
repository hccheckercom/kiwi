## Problem

Export endpoint loads entire result set into memory via `$wpdb->get_results()`. With 365 days of data and thousands of products, the response can be tens of MB, exhausting PHP memory and causing 500 errors.

## Bad

```php
$data = $wpdb->get_results(
    $wpdb->prepare( "SELECT * FROM {$table} WHERE date >= %s ORDER BY date DESC", $from ),
    ARRAY_A
);
return new \WP_REST_Response( array( 'rows' => $data ), 200 );
```

## Good

```php
$limit = min( 10000, max( 1, (int) ( $request->get_param( 'limit' ) ?? 5000 ) ) );
$data = $wpdb->get_results(
    $wpdb->prepare(
        "SELECT * FROM {$table} WHERE date >= %s ORDER BY date DESC LIMIT %d",
        $from,
        $limit
    ),
    ARRAY_A
);
return new \WP_REST_Response( array( 'rows' => $data, 'truncated' => count( $data ) >= $limit ), 200 );
```

## Why

Unbounded queries are a DoS vector (even for admin endpoints) and can crash the PHP process. Always cap with LIMIT and indicate if results are truncated.
