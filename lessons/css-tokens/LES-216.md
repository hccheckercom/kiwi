# LES-216: Mobile zoom not disabled — UX feels like browser, not native app

## Symptom
- User can pinch-to-zoom the page on mobile
- Tapping search/input fields causes iOS Safari to auto-zoom in (font-size < 16px)
- Page stays zoomed after closing input — user must double-tap or pinch to reset
- Overall feel is "website in browser" instead of "native app"

## Root Cause
Three layers must ALL be present to fully prevent zoom:

1. **Viewport meta** missing `maximum-scale=1, user-scalable=no` — allows pinch zoom
2. **Input font-size < 16px** on mobile — iOS auto-zooms to make text readable
3. **Missing `touch-action: manipulation`** — allows double-tap zoom

## Detection

### Next.js (App Router)
```typescript
// app/layout.tsx — check viewport export
export const viewport: Viewport = {
  width: 'device-width',
  initialScale: 1,
  maximumScale: 1,        // ← MUST have
  userScalable: false,    // ← MUST have
  viewportFit: 'cover',
}
```

### WordPress / Static HTML
```html
<!-- WRONG -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

<!-- CORRECT -->
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
```

### CSS (both platforms)
```css
/* Global — prevents double-tap zoom */
html {
  touch-action: manipulation;
}

/* Mobile — prevents iOS auto-zoom on input focus */
@media screen and (max-width: 767px) {
  input, textarea, select {
    font-size: 16px !important;
  }
}
```

## Fix Checklist
1. Add `maximum-scale=1, user-scalable=no` to viewport meta
2. Add `touch-action: manipulation` on `html` element (global, not just mobile)
3. Force `font-size: 16px !important` on all input/textarea/select at mobile breakpoint
4. Verify: `-webkit-text-size-adjust: 100%` on body (prevents iOS text inflation)
