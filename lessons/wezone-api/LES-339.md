## Bad
```php
// AdapterManager only has: product(), order(), cart(), customer(), coupon()
// Missing: email(), productRelations(), productWriter()
// Callers must instantiate directly:
$email = new WZ_Email(); // no lazy singleton
```

## Good
```php
class AdapterManager {
    private ?WZ_Email $email = null;

    public function email(): WZ_Email {
        if ( $this->email === null ) {
            $this->email = new WZ_Email();
        }
        return $this->email;
    }
    // same for productRelations(), productWriter()
}
```

## Why
AdapterManager provides lazy singleton pattern for 5 adapters but 3 are left out. This means Email/ProductRelations/ProductWriter get instantiated multiple times or require callers to manage their own singletons.
