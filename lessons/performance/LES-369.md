# Bug: Race Condition Trong Deploy Detection

## Vấn đề

Check-then-update pattern không atomic → concurrent requests trigger multiple purge cùng lúc.

**Ví dụ lỗi:**
```php
// ❌ SAI — race condition
$current_hash = md5_file( $hash_file );
$stored_hash  = get_option( self::DEPLOY_HASH_OPTION, '' );

if ( $current_hash && $current_hash !== $stored_hash ) {
    update_option( self::DEPLOY_HASH_OPTION, $current_hash, true );
    $this->purge_all(); // ← 100 requests cùng chạy đoạn này!
}
```

**Timeline:**
```
Request A: get_option() → "old_hash"
Request B: get_option() → "old_hash" (A chưa update_option)
Request C: get_option() → "old_hash"
Request A: update_option() + purge_all()
Request B: update_option() + purge_all() ← duplicate!
Request C: update_option() + purge_all() ← duplicate!
```

**Hậu quả:**
- 100 concurrent requests → 100 `opcache_reset()` calls → CPU spike
- 100 `DELETE FROM options WHERE LIKE` → DB lock contention
- Purge không idempotent → có thể gây inconsistency

## Root Cause

1. **Check-then-act không atomic** → window giữa `get_option()` và `update_option()`
2. **Không có lock/semaphore** → không prevent concurrent execution
3. **`update_option()` không return "was updated"** → không biết mình là request đầu tiên

## Fix

### 1. Dùng transient lock (recommended)

```php
// ✅ ĐÚNG — atomic lock với transient
private function auto_purge_on_deploy(): void {
    $hash_file = $this->get_deploy_hash_file();
    if ( ! $hash_file ) {
        return;
    }

    $current_hash = md5_file( $hash_file );
    $stored_hash  = get_option( self::DEPLOY_HASH_OPTION, '' );

    if ( $current_hash && $current_hash !== $stored_hash ) {
        // Try to acquire lock (atomic via set_transient with expiration)
        $lock_key = 'wz_deploy_purge_lock';
        if ( ! get_transient( $lock_key ) ) {
            // Set lock for 60s (longer than purge duration)
            set_transient( $lock_key, time(), 60 );
            
            // Double-check hash after acquiring lock
            $stored_hash = get_option( self::DEPLOY_HASH_OPTION, '' );
            if ( $current_hash !== $stored_hash ) {
                update_option( self::DEPLOY_HASH_OPTION, $current_hash, true );
                $this->purge_all();
            }
            
            delete_transient( $lock_key );
        }
        // Else: another request is purging, skip
    }
}
```

### 2. Dùng wp_cache_add() (nếu có persistent object cache)

```php
// ✅ ĐÚNG — wp_cache_add() is atomic (returns false if key exists)
private function auto_purge_on_deploy(): void {
    $hash_file = $this->get_deploy_hash_file();
    if ( ! $hash_file ) {
        return;
    }

    $current_hash = md5_file( $hash_file );
    $stored_hash  = get_option( self::DEPLOY_HASH_OPTION, '' );

    if ( $current_hash && $current_hash !== $stored_hash ) {
        $lock_key = 'wz_deploy_purge_lock';
        
        // wp_cache_add() returns false if key already exists (atomic)
        if ( wp_cache_add( $lock_key, time(), 'wezone', 60 ) ) {
            // We got the lock
            $stored_hash = get_option( self::DEPLOY_HASH_OPTION, '' );
            if ( $current_hash !== $stored_hash ) {
                update_option( self::DEPLOY_HASH_OPTION, $current_hash, true );
                $this->purge_all();
            }
            
            wp_cache_delete( $lock_key, 'wezone' );
        }
    }
}
```

### 3. Dùng DB row lock (heavy, chỉ khi cần strong guarantee)

```php
// ✅ ĐÚNG — DB row lock (MySQL GET_LOCK)
private function auto_purge_on_deploy(): void {
    global $wpdb;
    
    $hash_file = $this->get_deploy_hash_file();
    if ( ! $hash_file ) {
        return;
    }

    $current_hash = md5_file( $hash_file );
    $stored_hash  = get_option( self::DEPLOY_HASH_OPTION, '' );

    if ( $current_hash && $current_hash !== $stored_hash ) {
        // Try to acquire named lock (timeout 0 = non-blocking)
        $lock_name = 'wz_deploy_purge';
        $acquired  = (int) $wpdb->get_var( 
            $wpdb->prepare( 'SELECT GET_LOCK(%s, 0)', $lock_name ) 
        );
        
        if ( $acquired === 1 ) {
            $stored_hash = get_option( self::DEPLOY_HASH_OPTION, '' );
            if ( $current_hash !== $stored_hash ) {
                update_option( self::DEPLOY_HASH_OPTION, $current_hash, true );
                $this->purge_all();
            }
            
            $wpdb->query( $wpdb->prepare( 'SELECT RELEASE_LOCK(%s)', $lock_name ) );
        }
    }
}
```
