## Pattern

Admin panel sets custom cookies (remember-me, session token, CSRF token) without `Secure`, `HttpOnly`, or `SameSite` flags. On HTTP (dev) or mixed-content pages, cookies are sent in cleartext → session hijacking via network sniffing. Missing HttpOnly → XSS can steal admin session.

## Bug

```php
// SAI — cookie accessible via JS, sent over HTTP, no SameSite
setcookie( 'wz_admin_token', $token, time() + 86400, '/' );

// SAI — session cookie scope too broad (all subdomains)
setcookie( 'wz_session', $session_id, 0, '/', '.wezone.vn' );
// Attacker on any subdomain (blog.wezone.vn) can read admin cookie
```

## Fix

```php
// ĐÚNG — all security flags set
setcookie( 'wz_admin_token', $token, array(
    'expires'  => time() + 86400,
    'path'     => '/wp-admin/',  // Chỉ gửi cho admin pages
    'domain'   => '',            // Current domain only, không share subdomains
    'secure'   => is_ssl(),      // Chỉ gửi qua HTTPS
    'httponly'  => true,         // JS không đọc được
    'samesite' => 'Strict',     // Không gửi cross-site
) );

// WordPress auth cookie — đảm bảo constants đúng trong wp-config.php
// define( 'COOKIE_DOMAIN', 'wp.wezone.vn' );  // Không dùng .wezone.vn
// define( 'ADMIN_COOKIE_PATH', '/wp-admin' );
// define( 'FORCE_SSL_ADMIN', true );
```

```php
// Regenerate session ID sau login để chống session fixation
public function handle_admin_login(): void {
    // ... validate credentials ...

    // Regenerate session — old session ID invalid
    if ( session_status() === PHP_SESSION_ACTIVE ) {
        session_regenerate_id( true );
    }

    // Set new auth cookie with fresh token
    wp_set_auth_cookie( $user_id, false, true ); // $secure = true
}
```

## Áp dụng

- Mọi custom cookie trong admin PHẢI có: `secure=true` (production), `httponly=true`, `samesite=Strict`
- Cookie path giới hạn `/wp-admin/` — không cần gửi cho frontend
- KHÔNG dùng wildcard domain (`.wezone.vn`) cho admin cookies
- Regenerate session ID sau mỗi lần login/privilege escalation
