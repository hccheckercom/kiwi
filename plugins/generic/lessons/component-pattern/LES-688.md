## Bad
```tsx
export default function OrdersPage() {
  const { data } = useQuery(['orders'], fetchOrders)
  return <OrderTable orders={data ?? []} />
}
```
If fetch fails, user sees empty table with no explanation. No retry option.

## Good
```tsx
export default function OrdersPage() {
  const { data, error, refetch } = useQuery(['orders'], fetchOrders)
  if (error) return <ErrorState message="Không thể tải đơn hàng" onRetry={refetch} />
  return <OrderTable orders={data ?? []} />
}
```

## Why
Network failures, server errors, and timeouts are inevitable. Without error UI, users see blank pages and assume the app is broken. Error states with retry buttons let users recover without refreshing.
