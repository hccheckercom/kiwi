## Bad
```tsx
useEffect(() => {
  const channel = supabase.channel('orders')
    .on('postgres_changes', { event: 'INSERT', schema: 'public', table: 'orders' }, handleNewOrder)
    .subscribe()
}, [])
```
No cleanup — channel stays open after component unmounts. Each re-mount creates a new subscription.

## Good
```tsx
useEffect(() => {
  const channel = supabase.channel('orders')
    .on('postgres_changes', { event: 'INSERT', schema: 'public', table: 'orders' }, handleNewOrder)
    .subscribe()

  return () => { supabase.removeChannel(channel) }
}, [])
```

## Why
Each Supabase channel is a WebSocket connection. Without cleanup, navigating between pages accumulates connections — eventually hitting connection limits, causing memory leaks, and receiving duplicate event callbacks.
