## Bad
```ts
const { data: product } = await supabase.from('products').select('stock').eq('id', productId).single()
if (product.stock < quantity) throw new Error('Out of stock')
await supabase.from('products').update({ stock: product.stock - quantity }).eq('id', productId)
```
Read-then-write race condition: two concurrent orders both see stock=5, both succeed → stock goes negative.

## Good
```ts
// Atomic decrement with guard in single query
const { data, error } = await supabase.rpc('decrement_stock', {
  p_product_id: productId,
  p_quantity: quantity
})
// RPC function: UPDATE products SET stock = stock - p_quantity WHERE id = p_product_id AND stock >= p_quantity RETURNING *
if (error || !data) throw new Error('Out of stock')
```

## Why
Non-atomic inventory operations are the #1 cause of overselling in e-commerce. Flash sales amplify this — hundreds of concurrent checkouts hitting the same product. The fix is a single UPDATE with a WHERE guard, not a SELECT + UPDATE sequence.
