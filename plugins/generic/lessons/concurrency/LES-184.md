## Bad
```ts
export async function POST(req: Request) {
  const { orderId, amount } = await req.json()
  await supabase.from('orders').update({ status: 'paid', amount }).eq('id', orderId)
  await sendConfirmationEmail(orderId)
}
```
If VNPay/MoMo retries the webhook (network timeout), order processes twice — duplicate emails, double inventory deduction.

## Good
```ts
export async function POST(req: Request) {
  const { orderId, transactionId, amount } = await req.json()

  // Idempotency check — skip if already processed
  const { data: existing } = await supabase
    .from('processed_webhooks')
    .select('id')
    .eq('transaction_id', transactionId)
    .single()

  if (existing) return Response.json({ ok: true })

  // Process + record atomically
  const { error } = await supabase.rpc('process_payment', {
    p_order_id: orderId,
    p_transaction_id: transactionId,
    p_amount: amount
  })

  if (error) return Response.json({ error: error.message }, { status: 500 })
  return Response.json({ ok: true })
}
```

## Why
Payment gateways retry webhooks on 5xx or timeout. Without idempotency keys, each retry re-processes the payment. This causes double charges, duplicate inventory deductions, and multiple confirmation emails.
