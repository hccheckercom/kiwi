## Pattern

Admin saves settings (store name, announcement, custom CSS/JS) via REST API. Values stored in `wp_options`. When rendered on frontend or admin, output not escaped → stored XSS. Even though only admin can save, a CSRF or compromised admin session can inject persistent XSS affecting all visitors.

## Scan

```yaml
pattern: 'get_option\s*\(\s*[''"]wezone_admin_[^)]+\)\s*(?!.*esc_)'
scope: "**/*.php"
exclude_line: "esc_html|esc_attr|wp_kses|sanitize_"
message: "Admin setting output without escaping — stored XSS risk"
```

## Bug

```php
// SAI — admin có thể save <script>alert(1)</script> vào store_name
// Frontend render:
echo '<h1>' . get_option( 'wezone_admin_store_name' ) . '</h1>';

// Hoặc trong REST response trả về cho admin SPA:
$settings['custom_css'] = get_option( 'wezone_admin_custom_css' );
// SPA render: dangerouslySetInnerHTML={{ __html: settings.custom_css }}
```

## Fix

```php
// Output: luôn escape theo context
echo '<h1>' . esc_html( get_option( 'wezone_admin_store_name', '' ) ) . '</h1>';

// Input: sanitize khi save
$store_name = sanitize_text_field( $request->get_param( 'store_name' ) );
update_option( 'wezone_admin_store_name', $store_name, false );

// Custom CSS: dùng wp_kses với allowed properties
$custom_css = wp_strip_all_tags( $request->get_param( 'custom_css' ) );
update_option( 'wezone_admin_custom_css', $custom_css, false );
```

## Áp dụng

- Input: sanitize_text_field() cho text, wp_strip_all_tags() cho CSS, wp_kses_post() cho rich HTML
- Output: esc_html() trong HTML, esc_attr() trong attributes, esc_url() cho URLs
- KHÔNG bao giờ echo raw get_option() — dù chỉ admin mới save được
