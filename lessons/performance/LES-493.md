## Bad
```php
public function export_config( \WP_REST_Request $request ): \WP_REST_Response {
    $source = new AdminOverrideSource();
    $overrides = [];
    foreach ( $source->get_all() as $slot => $icon ) {
        $overrides[ $slot ] = [
            'type' => $icon->type,
            'name' => $icon->name,
        ];
    }
    return new \WP_REST_Response( [
        'overrides' => $overrides,
    ] );
}
```

## Good
```php
public function export_config( \WP_REST_Request $request ): \WP_REST_Response {
    $limit = (int) ( $request->get_param( 'limit' ) ?: 1000 );
    $source = new AdminOverrideSource();
    $overrides = [];
    $count = 0;
    foreach ( $source->get_all() as $slot => $icon ) {
        if ( ++$count > $limit ) {
            break;
        }
        $overrides[ $slot ] = [
            'type' => $icon->type,
            'name' => $icon->name,
        ];
    }
    return new \WP_REST_Response( [
        'overrides' => $overrides,
        'truncated' => $count > $limit,
    ] );
}
```

## Why
Export endpoints that fetch all records without LIMIT can exhaust memory if dataset grows large (e.g., 10k+ icon overrides). Attacker can trigger OOM by creating many records then calling export. Always add configurable limit with default cap (e.g., 1000 rows).
