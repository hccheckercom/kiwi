**Pattern sai:**
```php
// Login Plugin hardcode font, không inject CSS variables từ theme active
wp_enqueue_style('wezone-login-font', '...Be+Vietnam+Pro...');
// CSS file login.css không tồn tại → 404
```
**Phải làm:**
```php
// Đọc font từ wz_config('theme.font') của theme active
$font = wz_config('theme.font', 'Lexend');
wp_enqueue_style('wezone-login-font', '...family=' . rawurlencode($font) . '...');

// Inject toàn bộ --wz-* tokens vào :root trên login page
$tokens = '--wz-primary:' . wz_config('theme.primary', '#6366f1') . ';...';
wp_add_inline_style('wezone-login', ':root {' . $tokens . '}');

// CSS file dùng var(--wz-*) thay vì hardcode
background: var(--wz-primary);
font-family: var(--wz-font);
border-radius: var(--wz-radius);
```
**Lý do:** Login page nằm ngoài theme template hierarchy (wp-login.php là core WP), nên theme CSS không load. Plugin Login phải chủ động inject design tokens từ `store-config.php` của theme active để login page tuân thủ DNA. Nếu không, mỗi theme mới đều phải fix lại login styling.
**Grep check:** `login_enqueue_scripts` + thiếu `wz_config('theme.` trong cùng file — nếu Login Plugin không đọc theme tokens → HIGH
