# LES-290: Arbitrary Tailwind text-[Npx] bypasses fluid typography scale

## Symptom
- Templates use `text-[10px]`, `text-[14px]` etc. instead of semantic token classes
- Typography doesn't scale fluidly between mobile and desktop
- Inconsistent font sizes that don't follow the design system

## Root Cause
Tailwind's arbitrary value syntax (`text-[Npx]`) is convenient but creates fixed-size typography that:
1. Doesn't use `clamp()` fluid scaling
2. Bypasses the `--text-*` token system
3. Creates maintenance burden — values scattered across templates

## Wrong Pattern
```php
<!-- SAI — arbitrary px values -->
<span class="text-[10px] uppercase">Label</span>
<p class="text-[13px] text-gray-600">Description</p>
<h3 class="text-[15px] font-bold">Card Title</h3>
```

## Correct Pattern
```php
<!-- ĐÚNG — semantic token classes -->
<span class="text-xs uppercase">Label</span>
<p class="text-sm text-on-surface-variant">Description</p>
<h3 class="text-base font-bold">Card Title</h3>
```

## Fix Checklist
1. Grep for `text-\[\d+px\]` in all PHP and CSS files
2. Map each arbitrary value to nearest token: 10-12px→text-xs, 13-14px→text-sm, 15-16px→text-base
3. Replace with standard Tailwind size class that maps to CSS variable
4. Remove generated arbitrary classes from `src/main.css` after all references are updated
