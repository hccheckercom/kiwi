# Bad Code

```python
def process_data(data):
    try:
        result = transform(data)
        save_to_db(result)
        return result
    except:
        return None

def fetch_user(user_id):
    try:
        return api.get_user(user_id)
    except:
        logger.error("Failed to fetch user")
        return None
```

# Good Code

```python
def process_data(data):
    try:
        result = transform(data)
        save_to_db(result)
        return result
    except (ValueError, DatabaseError) as e:
        logger.error(f"Failed to process data: {e}")
        return None

def fetch_user(user_id):
    try:
        return api.get_user(user_id)
    except (APIError, TimeoutError) as e:
        logger.error(f"Failed to fetch user {user_id}: {e}")
        return None
    except Exception as e:
        logger.exception(f"Unexpected error fetching user {user_id}")
        raise
```

# Why

**Bare except catches EVERYTHING:**
- `KeyboardInterrupt` (Ctrl+C) - prevents graceful shutdown
- `SystemExit` - prevents program exit
- `MemoryError` - masks resource exhaustion
- `SyntaxError` - hides code bugs
- All other exceptions - makes debugging impossible

**Real-world impact:**
- **Production incidents:** Can't kill hung processes (catches KeyboardInterrupt)
- **Silent failures:** Errors disappear without logs
- **Debugging nightmare:** No stack traces, no error context
- **Resource leaks:** Catches cleanup exceptions

**Correct patterns:**
```python
# Specific exceptions only
except (ValueError, KeyError):
    ...

# Catch all but re-raise unexpected
except Exception as e:
    logger.exception("Unexpected error")
    raise

# Cleanup that must run
finally:
    cleanup()
```

**Never use bare except unless:**
- Top-level exception handler (main loop)
- Must log and continue (with explicit re-raise)
- Temporary debugging (remove before commit)
