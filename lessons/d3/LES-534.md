## Bad
```php
const node = svg.append('g')
  .selectAll('circle')
  .data(nodes) // Original nodes array without x/y properties
  .join('circle')
  .attr('r', 20)
  .attr('fill', (d) => getColor(d.severity))

simulation.on('tick', () => {
  node
    .attr('cx', (d) => d.x) // d.x is undefined
    .attr('cy', (d) => d.y) // d.y is undefined
})
```

## Good
```php
// Add initial positions to nodes
const nodesWithPositions = nodes.map((d, i) => ({
  ...d,
  x: width / 2 + (Math.random() - 0.5) * 100,
  y: height / 2 + (Math.random() - 0.5) * 100
}))

const simulation = d3.forceSimulation(nodesWithPositions)
  .force('link', d3.forceLink(links).id((d) => d.id))
  .force('charge', d3.forceManyBody().strength(-300))
  .force('center', d3.forceCenter(width / 2, height / 2))

const node = svg.append('g')
  .selectAll('circle')
  .data(nodesWithPositions) // Use nodes with positions
  .join('circle')
  .attr('r', 20)
  .attr('fill', (d) => getColor(d.severity))

simulation.on('tick', () => {
  node
    .attr('cx', (d) => d.x) // d.x exists
    .attr('cy', (d) => d.y) // d.y exists
})
```

## Why
D3 forceSimulation requires nodes to have x and y properties. If you bind original data without positions, simulation updates d.x/d.y but they start as undefined, causing NaN in attr() calls and invisible nodes. Solution: map nodes to add initial x/y before passing to simulation and .data() binding. This ensures nodes have valid coordinates from first tick.
