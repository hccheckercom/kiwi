## Bad
```tsx
export default function ProductsPage() {
  const { data } = useQuery(['products'], fetchProducts)
  return (
    <div className="grid grid-cols-3 gap-4">
      {data?.map(p => <ProductCard key={p.id} {...p} />)}
    </div>
  )
}
```
No loading state — page shows empty grid then jumps when data arrives (CLS).

## Good
```tsx
export default function ProductsPage() {
  const { data, isLoading } = useQuery(['products'], fetchProducts)
  if (isLoading) return <ProductGridSkeleton />
  return (
    <div className="grid grid-cols-3 gap-4">
      {data?.map(p => <ProductCard key={p.id} {...p} />)}
    </div>
  )
}
```
Skeleton preserves layout dimensions — no content shift.

## Why
Pages that fetch data without loading states cause Cumulative Layout Shift (CLS). Users see empty space then content "pops in" — feels broken. Skeleton screens maintain spatial stability and signal that content is coming.
