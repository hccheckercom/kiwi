## Pattern

`completePayment()` updates `payment_txn_id` but discards the return value. If the DB update fails (connection issue, table locked), the transaction ID is silently lost — making payment reconciliation impossible.

## Wrong

```php
public function completePayment(int $id, string $txn_id = ''): bool {
    if ($txn_id !== '') {
        $wpdb->update(
            $wpdb->prefix . 'wz_orders',
            ['payment_txn_id' => $txn_id],
            ['id' => $id]
        );
        // ← return value ignored, no error handling
    }
    return $this->updateStatus($id, 'processing', 'Payment complete');
}
```

## Correct

```php
public function completePayment(int $id, string $txn_id = ''): bool {
    global $wpdb;

    if ($txn_id !== '') {
        $updated = $wpdb->update(
            $wpdb->prefix . 'wz_orders',
            ['payment_txn_id' => $txn_id],
            ['id' => $id]
        );
        if ($updated === false) {
            wz_log("completePayment: failed to save txn_id={$txn_id} for order #{$id}", 'error');
            return false;
        }
    }

    $result = $this->updateStatus($id, 'processing', "Payment complete (txn: {$txn_id})");

    if ($result) {
        do_action(Hooks::PAYMENT_COMPLETE, $id);
        do_action(Hooks::CHECKOUT_PAYMENT_COMPLETED, $id);
    }

    return $result;
}
```

## Why

Lost `payment_txn_id` means you cannot reconcile payments with the gateway. This leads to manual investigation during chargebacks or refund disputes.
