## Pattern

Config arrays với default values chứa hardcoded hex colors thay vì design tokens.

## Bug Example

```php
// ❌ BAD — hardcoded colors trong config
return array(
    'popup_bg_color' => array(
        'type'    => 'color',
        'label'   => 'Màu nền',
        'default' => '#ffffff',  // hardcoded
    ),
    'popup_accent_color' => array(
        'type'    => 'color',
        'label'   => 'Màu nhấn',
        'default' => '#2271b1',  // hardcoded
    ),
);
```

## Why It's Wrong

1. **Không consistent với design system** — colors không sync với CSS tokens
2. **Khó maintain** — đổi brand colors phải sửa nhiều nơi
3. **Không support dark mode** — hardcoded colors không adapt
4. **Không accessible** — không guarantee contrast ratios

## Correct Approach

```php
// ✅ GOOD — dùng design token references
return array(
    'popup_bg_color' => array(
        'type'    => 'color',
        'label'   => 'Màu nền',
        'default' => 'var(--wz-surface-primary)',  // token reference
    ),
    'popup_accent_color' => array(
        'type'    => 'color',
        'label'   => 'Màu nhấn',
        'default' => 'var(--wz-accent-primary)',   // token reference
    ),
);

// Hoặc nếu cần hex — reference từ centralized config
return array(
    'popup_bg_color' => array(
        'type'    => 'color',
        'label'   => 'Màu nền',
        'default' => wz_get_design_token('surface.primary'),  // centralized
    ),
);
```

## Detection

```regex
'default'\s*=>\s*'#[0-9a-fA-F]{3,6}'
```

Trong context: config array với `'type' => 'color'`

## Impact

- **Maintainability**: Mỗi lần đổi brand colors phải tìm và sửa tất cả hardcoded values
- **Consistency**: Colors trong config có thể khác với CSS tokens
- **Dark mode**: Không thể auto-adapt theo theme
