## Pattern

Admin creates coupon with code "SALE50". Another admin (or same admin in different tab) creates another coupon with same code. No uniqueness check → duplicate codes in DB. When customer applies "SALE50", system picks first match (unpredictable) or applies wrong discount.

## Scan


## Bug

```php
// SAI — không check trùng code
public function create_coupon( WP_REST_Request $request ): WP_REST_Response {
    global $wpdb;
    $code = strtoupper( sanitize_text_field( $request->get_param( 'code' ) ) );

    $wpdb->insert( $wpdb->prefix . 'wz_coupons', array(
        'code'       => $code,
        'discount'   => (float) $request->get_param( 'discount' ),
        'type'       => sanitize_text_field( $request->get_param( 'type' ) ),
        'expires_at' => sanitize_text_field( $request->get_param( 'expires_at' ) ),
    ) );

    return new WP_REST_Response( array( 'id' => $wpdb->insert_id ) );
}
```

## Fix

```php
public function create_coupon( WP_REST_Request $request ): WP_REST_Response {
    global $wpdb;
    $code = strtoupper( trim( sanitize_text_field( $request->get_param( 'code' ) ) ) );

    if ( empty( $code ) || strlen( $code ) < 3 ) {
        return new WP_REST_Response( array( 'error' => 'Mã giảm giá phải có ít nhất 3 ký tự' ), 422 );
    }

    // Check uniqueness
    $exists = $wpdb->get_var( $wpdb->prepare(
        "SELECT COUNT(*) FROM {$wpdb->prefix}wz_coupons WHERE code = %s",
        $code
    ) );

    if ( $exists ) {
        return new WP_REST_Response(
            array( 'error' => "Mã \"{$code}\" đã tồn tại. Vui lòng chọn mã khác." ),
            409
        );
    }

    $wpdb->insert( $wpdb->prefix . 'wz_coupons', array(
        'code'       => $code,
        'discount'   => abs( (float) $request->get_param( 'discount' ) ),
        'type'       => sanitize_text_field( $request->get_param( 'type' ) ),
        'expires_at' => sanitize_text_field( $request->get_param( 'expires_at' ) ),
    ) );

    return new WP_REST_Response( array( 'id' => $wpdb->insert_id ), 201 );
}
```

## Áp dụng

Mọi entity có unique identifier (coupon code, SKU, slug) PHẢI check uniqueness trước INSERT. Thêm UNIQUE constraint ở DB level làm safety net, nhưng vẫn cần application-level check để trả error message thân thiện.
