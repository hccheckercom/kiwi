## Bad
```python
def _execute(command: str) -> Dict:
    result = subprocess.run(
        command,
        shell=True,
        capture_output=True,
        text=True,
    )
```

## Good
```python
import shlex

def _execute(command: str) -> Dict:
    result = subprocess.run(
        shlex.split(command),
        capture_output=True,
        text=True,
    )
```

## Why

`shell=True` passes the command string to the system shell for interpretation. If any part of the command contains user-controlled input (e.g., MCP tool arguments, form data, filenames), an attacker can inject shell metacharacters like `; rm -rf /` or `$(malicious_command)` to execute arbitrary code. Use `shlex.split()` with `shell=False` (default) to pass arguments as a safe list.
