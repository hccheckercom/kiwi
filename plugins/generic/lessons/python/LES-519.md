# Bad Code

```python
def process_items(items):
    if len(items) == 0:
        return "No items"
    return f"Processing {len(items)} items"

def validate_input(data):
    if len(data) > 0:
        return True
    return False

def check_empty(collection):
    if len(collection) != 0:
        print("Has items")
```

# Good Code

```python
def process_items(items):
    if not items:
        return "No items"
    return f"Processing {len(items)} items"

def validate_input(data):
    return bool(data)  # Or just: if data

def check_empty(collection):
    if collection:
        print("Has items")
```

# Why

**Pythonic way: Use truthiness**
- Empty sequences are falsy: `[]`, `{}`, `""`, `()`
- Non-empty sequences are truthy
- More readable and idiomatic

**Performance:**
- `if not items:` is O(1) - checks internal flag
- `if len(items) == 0:` is O(1) but slower - calls `__len__`

**Readability:**
```python
# Clear intent
if items:
    process(items)

# Verbose
if len(items) > 0:
    process(items)
```

**When to use `len()`:**
- Checking specific length: `if len(items) == 5:`
- Comparing lengths: `if len(a) > len(b):`
- Need actual count: `print(f"{len(items)} items")`

**Edge cases:**
```python
# Works for all sequences
if items:  # list, tuple, set, dict, str

# Custom objects need __bool__ or __len__
class MyCollection:
    def __bool__(self):
        return self.has_items()
```
