**Pattern sai:**
```php
// change_password() — không check password reuse
$current_password = $request->get_param('current_password');
$new_password = $request->get_param('new_password');
if ( strlen($new_password) < 6 ) {
    return new WP_REST_Response( array( 'error' => 'Too short' ), 400 );
}
wp_update_user( array( 'ID' => $user->ID, 'user_pass' => $new_password ) );
```
**Phải làm:**
```php
$current_password = trim( (string) $request->get_param('current_password') );
$new_password = trim( (string) $request->get_param('new_password') );
if ( $new_password === $current_password ) {
    return new WP_REST_Response( array( 'error' => 'New password must differ' ), 400 );
}
```
**Lý do:** User có thể "đổi" password thành chính password hiện tại — vô nghĩa và có thể bypass audit trail
**Grep check:** `$new_password === $current_password` — thiếu trong REST API password endpoint → HIGH
