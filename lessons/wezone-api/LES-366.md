# Bug: Banner reorder dùng N queries thay vì batch update

## Mô tả
`BannerRepository::reorder()` dùng `foreach` + `$wpdb->update()` → N queries. Với 50 banners = 50 UPDATE queries. Cần dùng batch update hoặc `CASE WHEN`.

## Tại sao nguy hiểm
- **Performance**: 50 banners = 50 round-trips DB
- **Lock contention**: Mỗi UPDATE lock row riêng lẻ
- **Race condition**: Giữa các UPDATE có thể có concurrent request khác

## Code sai (BannerRepository.php:191-200)
```php
public static function reorder( array $ids ): bool {
    global $wpdb;
    $table = self::table();

    foreach ( $ids as $order => $id ) {
        $wpdb->update( $table, array( 'sort_order' => (int) $order ), array( 'id' => absint( $id ) ) );
    }

    return true;
}
```

**Vấn đề**: 
- N queries thay vì 1 query
- Không check update success
- Không có transaction

## Code đúng (Option 1: CASE WHEN)
```php
public static function reorder( array $ids ): bool {
    global $wpdb;
    $table = self::table();

    if ( empty( $ids ) ) {
        return false;
    }

    // Build CASE WHEN statement
    $cases = array();
    $id_list = array();
    foreach ( $ids as $order => $id ) {
        $id = absint( $id );
        $cases[] = $wpdb->prepare( 'WHEN %d THEN %d', $id, (int) $order );
        $id_list[] = $id;
    }

    $case_sql = implode( ' ', $cases );
    $ids_sql = implode( ',', $id_list );

    $query = "UPDATE {$table} SET sort_order = CASE id {$case_sql} END WHERE id IN ({$ids_sql})";
    
    return false !== $wpdb->query( $query );
}
```

## Code đúng (Option 2: Transaction + batch)
```php
public static function reorder( array $ids ): bool {
    global $wpdb;
    $table = self::table();

    if ( empty( $ids ) ) {
        return false;
    }

    $wpdb->query( 'START TRANSACTION' );

    try {
        foreach ( $ids as $order => $id ) {
            $result = $wpdb->update( 
                $table, 
                array( 'sort_order' => (int) $order ), 
                array( 'id' => absint( $id ) ) 
            );
            
            if ( false === $result ) {
                $wpdb->query( 'ROLLBACK' );
                return false;
            }
        }
        
        $wpdb->query( 'COMMIT' );
        return true;
    } catch ( Exception $e ) {
        $wpdb->query( 'ROLLBACK' );
        return false;
    }
}
```

## Cách fix
1. **Preferred**: Dùng `CASE WHEN` → 1 query duy nhất
2. **Alternative**: Wrap trong transaction nếu cần rollback
3. Check update success và return false nếu fail

## Áp dụng cho
- Mọi reorder/sort operation
- Bulk update operations
- Mass status change
