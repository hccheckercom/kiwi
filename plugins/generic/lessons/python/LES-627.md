## Bad

```python
def get_stats():
    conn = get_connection()
    rows = conn.execute("SELECT * FROM scans").fetchall()
    conn.close()  # Never reached if execute() throws
    return rows
```

## Good

```python
def get_stats():
    conn = get_connection()
    try:
        return conn.execute("SELECT * FROM scans").fetchall()
    finally:
        conn.close()

# Or better — use context manager
def get_stats():
    with get_connection() as conn:
        return conn.execute("SELECT * FROM scans").fetchall()
```

## Why

If an exception occurs between `get_connection()` and `conn.close()`, the connection leaks. SQLite has limited concurrent connections, so leaked connections can cause `database is locked` errors. Always use `try/finally` or a context manager.
