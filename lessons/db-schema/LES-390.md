## Bad
```php
// wz_customers
'user_id' => 'bigint(20) unsigned DEFAULT NULL',
// indexes:
'idx_user' => 'user_id',  // allows duplicates
```

## Good
```php
'user_id' => 'bigint(20) unsigned DEFAULT NULL',
// indexes:
'UNIQUE idx_user' => 'user_id',
```

## Why
Without UNIQUE on user_id, one WordPress user can have multiple customer records. This causes: wrong total_spent aggregation, duplicate entries in customer list, and inconsistent order_count. NULL values are excluded from UNIQUE in MySQL, so guest customers (user_id=NULL) are not affected.
