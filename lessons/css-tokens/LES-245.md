# LES-245: Duplicate CSS Selector Override — Banner Biến Mất

## Pattern sai

```css
/* Slider layout section — đúng */
.haven-hero__slide {
    grid-area: 1 / 1;
    visibility: hidden;
}

/* Banner Effects section — SAI: redefine cùng selector */
.haven-hero__slide {
    display: none;          /* override grid-area, làm banner biến mất */
    position: absolute;
    inset: 0;
}
```

## Pattern đúng

```css
/* Chỉ định nghĩa base rule MỘT LẦN trong slider layout section */
.haven-hero__slide {
    grid-area: 1 / 1;       /* CSS grid stack — tất cả slides cùng cell */
    visibility: hidden;     /* ẩn nhưng vẫn chiếm không gian (giữ height) */
    pointer-events: none;
    transition-property: opacity, transform, filter, clip-path;
    transition-duration: 600ms;
    transition-timing-function: ease-in-out;
}

.haven-hero__slide.is-active {
    visibility: visible;
    pointer-events: auto;
}

.haven-hero__slide.leaving {
    visibility: visible;    /* visible để animate ra */
    pointer-events: none;
}

/* Effects section chỉ thêm transition states, KHÔNG redefine base */
.haven-hero--fade-in .haven-hero__slide           { opacity: 0; }
.haven-hero--fade-in .haven-hero__slide.is-active { opacity: 1; }
.haven-hero--fade-in .haven-hero__slide.leaving   { opacity: 0; }
```

## Lý do

Khi thêm CSS section mới (effects, animations), dev dễ copy-paste base selector từ nơi khác và vô tình redefine nó. CSS cascade áp dụng rule cuối cùng — `display: none` ở block sau override `grid-area: 1/1` ở block trước, làm tất cả slides ẩn hoàn toàn kể cả slide `is-active`.

**Kiến trúc đúng cho slider:**
- Dùng CSS grid stack (`grid-area: 1/1`) thay vì `display: none` / `position: absolute`
- `visibility: hidden` (ẩn nhưng giữ không gian) thay vì `display: none` (xóa khỏi layout)
- Effects section chỉ thêm per-transition states, không redefine base selector

## Checklist

- [ ] Mỗi selector chỉ có 1 base definition trong toàn bộ CSS file
- [ ] Slider dùng CSS grid stack (`grid-area: 1/1`) không dùng `display: none`
- [ ] Effects section chỉ thêm modifier classes, không override base
- [ ] Sau khi thêm CSS section mới → grep để kiểm tra duplicate selectors
