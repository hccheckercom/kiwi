## Bad
```php
public function get_pending(): array {
    global $wpdb;
    return $wpdb->get_results( $wpdb->prepare( "SELECT ..." ), ARRAY_A );
    // get_results() returns null on DB error → type error (null vs array)
}
```

## Good
```php
public function get_pending(): array {
    global $wpdb;
    $rows = $wpdb->get_results( $wpdb->prepare( "SELECT ..." ), ARRAY_A );
    return is_array( $rows ) ? $rows : array();
}
```

## Why
`$wpdb->get_results()` returns `null` when the query fails (DB connection lost, table missing, syntax error). Returning `null` from a method declared `: array` causes a TypeError. All other methods in the codebase already guard this (BUG-006, BUG-007, BUG-009) — this is a missed spot. Always assign to a variable and guard before returning.
