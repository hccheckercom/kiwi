## Bad

```python
logger.debug(f"Processing {len(items)} items from {source}")
logger.info("User {} logged in from {}".format(user_id, ip))
```

## Good

```python
logger.debug("Processing %d items from %s", len(items), source)
logger.info("User %s logged in from %s", user_id, ip)
```

## Why

f-strings and `.format()` evaluate immediately, even if the log level is disabled. Using `%s`/`%d` style with positional args lets the logging module skip string interpolation entirely when the level is filtered out. This matters in hot paths with DEBUG/TRACE logging.

## Grep

```bash
rg 'log(?:ger)?\.\w+\s*\(\s*f["\x27]' --type py
```
