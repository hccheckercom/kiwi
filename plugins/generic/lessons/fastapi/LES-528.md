# Bad Code

```python
import requests
import time

@app.get("/external-data")
async def get_external_data():
    # BAD: Blocks event loop for 2 seconds
    response = requests.get("https://api.example.com/data")
    return response.json()

@app.post("/process")
async def process_data(data: dict):
    # BAD: Blocks event loop
    time.sleep(5)  # Simulate processing
    result = heavy_computation(data)
    return result

@app.get("/file")
async def read_file():
    # BAD: Blocking file I/O
    with open("data.txt") as f:
        content = f.read()
    return {"content": content}
```

# Good Code

```python
import httpx
import asyncio
from aiofiles import open as aio_open

@app.get("/external-data")
async def get_external_data():
    # GOOD: Non-blocking HTTP client
    async with httpx.AsyncClient() as client:
        response = await client.get("https://api.example.com/data")
        return response.json()

@app.post("/process")
async def process_data(data: dict):
    # GOOD: Non-blocking sleep
    await asyncio.sleep(5)
    
    # For CPU-bound work, use thread pool
    loop = asyncio.get_event_loop()
    result = await loop.run_in_executor(None, heavy_computation, data)
    return result

@app.get("/file")
async def read_file():
    # GOOD: Async file I/O
    async with aio_open("data.txt") as f:
        content = await f.read()
    return {"content": content}
```

# Why

**Blocking I/O in async functions:**
- Blocks entire event loop
- Other requests wait
- Server becomes unresponsive
- Defeats purpose of async

**Performance impact:**
```python
# BAD: 10 concurrent requests take 20 seconds
@app.get("/data")
async def get_data():
    response = requests.get(url)  # 2s each, sequential
    return response.json()

# GOOD: 10 concurrent requests take 2 seconds
@app.get("/data")
async def get_data():
    async with httpx.AsyncClient() as client:
        response = await client.get(url)  # 2s total, parallel
        return response.json()
```

**When to use sync vs async:**
```python
# Use async for I/O-bound operations
- HTTP requests (httpx, aiohttp)
- Database queries (asyncpg, motor)
- File I/O (aiofiles)
- Network operations

# Use thread pool for CPU-bound operations
import asyncio
from concurrent.futures import ThreadPoolExecutor

executor = ThreadPoolExecutor(max_workers=4)

@app.post("/compute")
async def compute(data: dict):
    loop = asyncio.get_event_loop()
    result = await loop.run_in_executor(executor, cpu_intensive_task, data)
    return result
```

**Common blocking operations:**
- `requests.get()` → use `httpx.AsyncClient()`
- `time.sleep()` → use `asyncio.sleep()`
- `open()` → use `aiofiles.open()`
- `psycopg2` → use `asyncpg`
- `pymongo` → use `motor`
