## Bad
```ts
const useAIStore = create<AIStore>((set) => ({
  responses: [],
  isLoading: false,
  addResponse: (r) => set(s => ({ responses: [...s.responses, r] })),
  resetSession: () => set({ isLoading: false }),
  // BUG: responses not cleared — old AI results show in new session
}))
```

## Good
```ts
const useAIStore = create<AIStore>((set) => ({
  responses: [],
  isLoading: false,
  addResponse: (r) => set(s => ({ responses: [...s.responses, r] })),
  resetSession: () => set({ responses: [], isLoading: false }),
  // All cached state cleared on reset
}))
```

## Why
Real bug: commit c235d72. AI response cache in Zustand store persisted across session resets. User clicks "New Scan" but sees old results mixed with new ones. Any reset/logout action must clear ALL derived state, not just flags.
