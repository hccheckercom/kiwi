## Bad
```html
<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no">
```
```css
body { overflow-x: hidden; } /* Hides horizontal scroll but masks layout bugs */
```
Blocking zoom violates WCAG 2.1 and signals poor mobile UX to Google. `overflow-x: hidden` on body masks broken layouts instead of fixing them.

## Good
```html
<meta name="viewport" content="width=device-width, initial-scale=1">
```
```css
/* Fix the actual overflow source instead of hiding it */
.container { max-width: 100%; }
img { max-width: 100%; height: auto; }
```

## Why
Google Ads Quality Score heavily weights mobile experience. Landing pages that block zoom, have horizontal scroll, or use tiny tap targets get penalized with higher CPC and lower ad rank. Google's Mobile-Friendly Test is used during ad review. Key requirements: (1) no zoom blocking, (2) no horizontal scroll, (3) tap targets ≥48px, (4) text readable without zooming, (5) content fits viewport width.
