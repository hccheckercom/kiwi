## Bad
```php
$counter = (int) get_option( 'wezone_sku_counter', 1000 );
++$counter;
update_option( 'wezone_sku_counter', $counter );
// Two concurrent requests read same value → duplicate SKU
```

## Good
```php
global $wpdb;
$wpdb->query( "UPDATE {$wpdb->options} SET option_value = option_value + 1 WHERE option_name = 'wezone_sku_counter'" );
$counter = (int) get_option( 'wezone_sku_counter' );
// Atomic increment — no race condition
```

## Why
get_option/update_option is not atomic. Two concurrent API calls can read the same counter value and generate identical SKUs. Use atomic SQL UPDATE to increment, then read the result.
