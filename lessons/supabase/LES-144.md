## Bad
```ts
if (body.transferAmount !== order.amount) {
  return Response.json({ error: 'Amount mismatch' }, { status: 422 })
}
// Bank transfers may have ±10,000 VND fee difference
// 500000 sent, 499000 received → legitimate payment rejected
```

## Good (tolerance pattern from webhook.handler.ts)
```ts
const BANK_FEE_TOLERANCE = 10_000 // VND — covers bank transfer fees
const receivedAmount = Math.round(body.transferAmount)
const expectedAmount = Math.round(order.amount)
const amountDiff = Math.abs(receivedAmount - expectedAmount)

if (amountDiff > BANK_FEE_TOLERANCE) {
  console.error(`[Webhook] Amount mismatch: expected ${expectedAmount}, got ${receivedAmount}`)
  return Response.json({ error: 'Amount mismatch' }, { status: 422 })
}

if (amountDiff > 0) {
  console.warn(`[Webhook] Minor diff: ${amountDiff} (within tolerance)`)
}
```

## Why
Real pattern from webstore-vn payment webhook. Vietnamese bank transfers deduct fees (3,000-10,000 VND). Strict equality rejects legitimate payments. Use Math.round() + tolerance threshold. Log warnings for non-zero diffs within tolerance for audit trail.
