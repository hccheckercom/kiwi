# LES-289: Hardcoded font-size in inline styles bypasses typography tokens

## Symptom
- PHP templates use `style="font-size: 17px"` or similar inline declarations
- Typography cannot be updated globally by changing CSS variables
- Different font sizes scattered across template files without consistency

## Root Cause
Quick prototyping leads to inline font-size declarations instead of using the token system (`var(--text-*)` or Tailwind classes). Unlike LES-241 which catches combined font-family+font-size+color, this catches any standalone font-size in inline styles.

## Wrong Pattern
```php
<!-- SAI — hardcoded font-size inline -->
<span style="font-size: 13px; font-weight: 600;">Badge</span>
<h2 style="font-size: 28px;">Title</h2>
<input style="font-size: 17px; padding: 8px;">
```

## Correct Pattern
```php
<!-- ĐÚNG — dùng Tailwind classes hoặc CSS token classes -->
<span class="text-xs font-semibold">Badge</span>
<h2 class="wz-h2">Title</h2>
<input class="text-base p-2">
```

## Fix Checklist
1. Identify all inline `font-size` declarations in PHP templates
2. Map each px value to the nearest token: 11-12px→text-xs, 13-14px→text-sm, 15-16px→text-base, 17-18px→text-lg, 20-22px→text-xl, 24-28px→text-2xl
3. Replace inline style with appropriate Tailwind class or `.wz-*` class
4. Exception: email templates and invoice templates may keep inline styles (email clients need them)

## Scanner Check
```
style="[^"]*font-size:\s*\d+px
```
