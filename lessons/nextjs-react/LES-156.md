## Bad
```ts
export async function sendEmail(to: string, subject: string, html: string) {
  const res = await fetch('https://api.resend.com/emails', {
    method: 'POST',
    body: JSON.stringify({ from: 'no-reply@wezone.vn', to: [to], subject, html }),
  })
  // Resend returns 422 for invalid email — but we don't check
  // Wastes API call + quota for obviously invalid addresses
}
```

## Good (pattern from resend.order.ts + resend.ts)
```ts
export async function sendOrderConfirmEmail(params: SendOrderConfirmParams): Promise<boolean> {
  const apiKey = process.env.RESEND_API_KEY
  if (!apiKey || apiKey === 're_...') return false // No key = skip (dev/CI)

  const EMAIL_RE = /^[^\s@]+@[^\s@]+\.[^\s@]+$/
  if (!EMAIL_RE.test(params.to)) {
    console.error('[Resend] Invalid email address:', params.to)
    return false // Don't waste API call
  }

  try {
    const res = await fetch('https://api.resend.com/emails', {
      method: 'POST',
      headers: { 'Authorization': `Bearer ${apiKey}`, 'Content-Type': 'application/json' },
      body: JSON.stringify({ ... }),
      signal: AbortSignal.timeout(8000),
    })
    if (!res.ok) { console.error('[Resend] Send failed:', await res.text()); return false }
    return true
  } catch (err) {
    console.error('[Resend] Network error:', err)
    return false
  }
}
```

## Checklist for email functions
```
✓ Guard: if (!apiKey || apiKey === 're_...') return false
✓ Validate: EMAIL_RE.test(to) before API call
✓ Timeout: AbortSignal.timeout(8000)
✓ Return boolean (not throw) — caller decides how to handle
✓ Log errors with context: [Resend] + what failed
```

## Why
Real pattern from all email functions in webstore-vn. Resend charges per API call. Invalid emails (null, empty, malformed from form bugs) waste quota and return 422. The guard pattern (key check + regex + timeout + boolean return) is consistent across all 4 email functions in the codebase.
