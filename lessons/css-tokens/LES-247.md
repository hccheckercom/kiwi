## Pattern sai

```css
.hero__visual {
    position: relative;
    height: 520px;
    overflow: hidden; /* clips img AND absolute children like products */
}

.hero__products {
    position: absolute;
    bottom: 16px;
    left: 12px;
    right: 12px; /* bị clip bởi overflow:hidden của cha */
}
```

## Phải làm

```css
/* Container ngoài clip horizontal scroll */
.haven-hero {
    overflow: hidden;
}

/* Visual tăng height để chứa products strip */
.haven-hero__visual {
    position: relative;
    height: calc(var(--hp-height-mobile, 520px) + 160px); /* mobile: banner + products */
    overflow: hidden;
}

/* Products absolute bottom:0 — nằm gọn trong visual đã mở rộng */
.haven-hero__products {
    position: absolute;
    bottom: 0;
    left: 0;
    right: 0;
}

/* Desktop: reset về height bình thường */
@media (min-width: 768px) {
    .haven-hero__visual {
        height: var(--hp-height-desktop, 560px);
    }
}
```

## Lý do

`overflow: hidden` trên container cha clip **tất cả** children kể cả `position: absolute`. Trên mobile, products bị ẩn hoàn toàn. Đồng thời nếu không có `overflow: hidden` ở wrapper ngoài, flex children tràn ra ngoài viewport gây horizontal scroll làm banner bị co lại.

Giải pháp: tăng height của `__visual` để bao gồm cả products strip, đặt products `bottom: 0` thay vì `bottom: 16px` để nằm gọn trong bounds.

## Grep check

Tìm pattern `overflow: hidden` trong hero/banner sections kết hợp với `position: absolute` trên children → kiểm tra xem children có bị clip không.
