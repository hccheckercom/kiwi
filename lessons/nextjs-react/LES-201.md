## Bad
```tsx
const Component = () => {
  const store = useCartStore() // Subscribes to ALL state changes
  return <span>{store.itemCount}</span>
}
```
Component re-renders on every store change (cart total, items array, etc.) even though it only uses `itemCount`.

## Good
```tsx
const Component = () => {
  const itemCount = useCartStore(state => state.itemCount)
  return <span>{itemCount}</span>
}
```
Only re-renders when `itemCount` actually changes.

## Why
Zustand without selectors subscribes to the entire store. In a cart with frequent updates (quantity changes, price recalculations), this causes cascading re-renders across all consuming components. Always use selectors to pick only what you need.
