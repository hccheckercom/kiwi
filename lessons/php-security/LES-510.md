## Bad
```php
public function add( string $old_url, string $new_url ): bool {
    global $wpdb;
    $table = $wpdb->prefix . 'wezone_migrator_redirects';
    
    return (bool) $wpdb->insert(
        $table,
        array(
            'old_url' => $old_url,
            'new_url' => $new_url,
        )
    );
}
```

## Good
```php
public function add( string $old_url, string $new_url ): bool {
    global $wpdb;
    $table = $wpdb->prefix . 'wezone_migrator_redirects';
    
    // Validate URLs
    $old_url = esc_url_raw( $old_url );
    $new_url = esc_url_raw( $new_url );
    
    if ( ! $old_url || ! $new_url ) {
        return false;
    }
    
    // Prevent open redirect â€” new_url must be internal or whitelisted
    if ( ! wp_validate_redirect( $new_url, home_url() ) ) {
        return false;
    }
    
    return (bool) $wpdb->insert(
        $table,
        array(
            'old_url' => $old_url,
            'new_url' => $new_url,
        )
    );
}
```

## Why
Attacker injects external URL â†’ open redirect â†’ phishing. Must validate new_url is internal or whitelisted before storing.
