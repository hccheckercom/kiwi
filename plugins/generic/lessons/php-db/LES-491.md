## Bad
```php
public function delete( string $name ): bool {
    $path = $this->get_upload_dir() . '/' . $name . '.svg';
    wp_delete_file( $path );
    return ! file_exists( $path );
}
```

## Good
```php
public function delete( string $name ): bool {
    // 1. Delete DB references first
    $registry = IconRegistry::instance();
    $registry->get_svg_registry()->remove( $name );
    
    // 2. Then delete file
    $path = $this->get_upload_dir() . '/' . $name . '.svg';
    wp_delete_file( $path );
    return ! file_exists( $path );
}
```

## Why
If file is deleted first but DB cleanup fails, DB references orphaned file â†’ 404 errors. Correct order: (1) delete DB records, (2) delete file. If DB delete fails, file remains and can be retried. If file delete fails after DB cleanup, file is orphaned but no broken references.
