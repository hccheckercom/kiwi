# LES-241: Inline styles in template-parts bypass design token system

## Symptom
- Section headings, eyebrow labels, and layout containers in PHP template-parts use `style="..."` attributes with hardcoded values
- Changing brand colors or typography requires searching through all `.php` files instead of updating CSS variables
- Multiple template-parts repeat the same `font-family`, `color`, `font-size`, `letter-spacing` values inline

## Root Cause
When scaffolding template-parts quickly, inline styles are written directly for speed. This creates:
1. **Token bypass** — hardcoded `#8B7355`, `font-size:28px`, `Cormorant Garamond` are copied everywhere instead of using `var(--wz-primary)`, `var(--wz-font-heading)`
2. **No reusability** — same heading pattern duplicated across flash-sale, categories, featured-products
3. **Unmaintainable** — brand refresh requires grep+replace across 10+ PHP files

## Wrong Pattern
```php
<!-- Section heading inline — SAI -->
<h2 style="font-family:var(--wz-font-heading,'Cormorant Garamond',serif);font-size:28px;font-weight:600;color:var(--wz-text,#3D2B1F);margin:0;">
    <?php esc_html_e( 'Flash Sale', 'wezone-haven' ); ?>
</h2>

<!-- Eyebrow label inline — SAI -->
<p style="font-size:11px;font-weight:600;letter-spacing:0.12em;text-transform:uppercase;color:var(--wz-text-muted,#8B7355);margin-bottom:4px;">
    <?php esc_html_e( 'Giới hạn thời gian', 'wezone-haven' ); ?>
</p>
```

## Correct Pattern
```css
/* _components.css — define once */
.haven-eyebrow {
    display: inline-flex;
    align-items: center;
    gap: 8px;
    font-size: 10px;
    font-weight: 700;
    letter-spacing: 0.18em;
    text-transform: uppercase;
    color: var(--wz-primary, #8B7355);
    margin-bottom: 8px;
}
.haven-eyebrow::before,
.haven-eyebrow::after {
    content: '';
    display: block;
    width: 20px;
    height: 1px;
    background: var(--wz-primary, #8B7355);
    opacity: 0.6;
}
.haven-heading {
    font-family: var(--wz-font-heading, 'Cormorant Garamond', serif);
    font-size: clamp(22px, 4vw, 30px);
    font-weight: 600;
    color: var(--wz-text, #3D2B1F);
    margin: 0;
}
```

```php
<!-- template-part — ĐÚNG -->
<div class="haven-section-header">
    <p class="haven-eyebrow"><?php esc_html_e( 'Giới hạn thời gian', 'wezone-haven' ); ?></p>
    <h2 class="haven-heading"><?php esc_html_e( 'Flash Sale', 'wezone-haven' ); ?></h2>
</div>
```

## Fix Checklist
1. Create `.haven-eyebrow`, `.haven-heading`, `.haven-section-header` in `_components.css`
2. Replace all `<h2 style="font-family:...">` in template-parts with class-based markup
3. Replace all `<p style="font-size:11px;...text-transform:uppercase">` eyebrows with `.haven-eyebrow`
4. Ensure CSS file uses only `var(--wz-*)` tokens — no hardcoded hex in class definitions

## Scanner Check
```
style="font-family:.*font-size:
style=".*letter-spacing:.*text-transform:
style=".*color:.*font-size:.*font-weight:
```

## Feature Suggestion
Each theme should scaffold a `_components.css` heading system before writing any template-parts:
- `.{theme}-eyebrow` — small-caps label with decorative lines
- `.{theme}-heading` — serif h2 with accent underline
- `.{theme}-section-header` — wrapper with margin
- `.{theme}-view-all` — "Tất cả" uppercase link
