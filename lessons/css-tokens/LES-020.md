**Triệu chứng:** Layout vỡ trên tablet/desktop — responsive classes (`md:`, `lg:`, v.v.) có trong PHP template nhưng KHÔNG có trong compiled `tailwind.css`. Tailwind purge đã xóa class vì file chưa được build lại sau khi thêm template mới.

**Nguyên nhân gốc:** Deploy script không build Tailwind CSS cho themes. File `tailwind.css` là bản cũ từ lần build trước — class mới thêm vào PHP templates không được generate.

**3 tình huống gây lỗi:**
1. Thêm file PHP mới có responsive class → quên `npm run build:css`
2. Thêm class mới vào file đã có → quên rebuild
3. Deploy lên VPS mà không có bước build (đã fix trong `scripts/deploy.ps1` Step 2b)

**Fix:**
```powershell
# Local dev — watch mode
cd themes\wezone-funilux  # hoặc wezone-trunganh-v2
npm run watch:css

# Trước khi commit — production build
npm run build:css

# Deploy tự động build (scripts/deploy.ps1 Step 2b đã có)
.\scripts\deploy.ps1
```

**Verify:**
```powershell
# Check class cụ thể có trong compiled CSS không
Select-String -Path "assets/css/tailwind.css" -Pattern "md\\:grid-cols-4"
# 0 matches → CSS stale → rebuild
```
