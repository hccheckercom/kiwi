# LES-293: --text-base below 15px minimum — Vietnamese readability violation

## Symptom
- Body text appears too small on mobile devices (14px instead of 15px minimum)
- Vietnamese diacritics (ă, ơ, ư, ệ, ỹ) are hard to read at small sizes
- Users complain about readability on 375px screens

## Root Cause
The `--text-base` token uses a `clamp()` formula where the minimum value is below 0.9375rem (15px). Vietnamese characters are more complex than Latin — they have stacked diacritics that require larger minimum sizes for legibility.

Blueprint mandates: `--text-base: clamp(0.9375rem, 0.89rem + 0.2vw, 1rem)` = 15px → 16px

## Wrong Pattern
```css
/* SAI — min 14px, quá nhỏ cho Vietnamese */
--text-base: clamp(0.875rem, 0.83rem + 0.18vw, 0.9375rem);
/* 0.875rem = 14px mobile → 0.9375rem = 15px desktop */
```

## Correct Pattern
```css
/* ĐÚNG — min 15px theo blueprint */
--text-base: clamp(0.9375rem, 0.89rem + 0.2vw, 1rem);
/* 0.9375rem = 15px mobile → 1rem = 16px desktop */
```

## Fix Checklist
1. Open `assets/css/partials/_tokens.css`
2. Find `--text-base` declaration
3. Replace with blueprint formula: `clamp(0.9375rem, 0.89rem + 0.2vw, 1rem)`
4. Verify body text renders at minimum 15px on 375px viewport
5. Check that `--text-md` (if exists) is adjusted accordingly or removed if redundant
