## Bad
```php
public function delete_backup( \WP_REST_Request $request ): \WP_REST_Response {
    $id = (int) $request->get_param( 'id' );
    // No current_user_can() check â€” relies on registration permission only
    $manager = new BackupManager();
    $deleted = $manager->delete_backup( $id );
    return new \WP_REST_Response( array( 'deleted' => $deleted ), $deleted ? 200 : 404 );
}
```

## Good
```php
public function delete_backup( \WP_REST_Request $request ): \WP_REST_Response {
    if ( ! current_user_can( 'manage_options' ) ) {
        return new \WP_REST_Response( array( 'error' => 'Unauthorized' ), 403 );
    }
    $id = (int) $request->get_param( 'id' );
    $manager = new BackupManager();
    $deleted = $manager->delete_backup( $id );
    return new \WP_REST_Response( array( 'deleted' => $deleted ), $deleted ? 200 : 404 );
}
```

## Why
If \WeZone\Core\Api::register() has a bug or is bypassed, attackers can call controller methods directly. Defense in depth requires capability checks inside methods, not just at registration.
