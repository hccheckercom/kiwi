# Bug: Unsanitized Superglobal Access

## Problem

Accessing `$_GET`, `$_POST`, `$_REQUEST`, or `$_SERVER` values without sanitization allows XSS, SQL injection, or path traversal.

## AST Detection

Tree-sitter finds `subscript_expression` nodes accessing superglobal arrays where the containing line lacks sanitize/escape functions.

## Bad

```php
$name = $_POST['name'];
$page = $_GET['page'];
echo $_REQUEST['search'];
```

## Good

```php
$name = sanitize_text_field( wp_unslash( $_POST['name'] ) );
$page = absint( $_GET['page'] );
echo esc_html( sanitize_text_field( wp_unslash( $_REQUEST['search'] ) ) );
```
