## Bad
```sql
CREATE TABLE customer_notes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  customer_id UUID REFERENCES customers(id),
  note TEXT,
  created_by UUID REFERENCES auth.users(id)
);
-- Forgot RLS → any authenticated user can read all customer notes
```

## Good
```sql
CREATE TABLE customer_notes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  customer_id UUID REFERENCES customers(id),
  note TEXT,
  created_by UUID REFERENCES auth.users(id)
);

ALTER TABLE customer_notes ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can only see their own notes"
  ON customer_notes FOR SELECT
  USING (created_by = auth.uid());
```

## Why
Supabase tables without RLS enabled are accessible to all authenticated users through the client SDK. This is the #1 security mistake in Supabase projects. Every new table MUST have RLS enabled and at least one policy, even if it's a deny-all policy for the admin-only table.
