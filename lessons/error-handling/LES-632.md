## Bad
```python
def get_data():
    try:
        conn = get_connection()
        return conn.execute("SELECT ...").fetchall()
    except Exception:
        return []
```

## Good
```python
import sys

def get_data():
    try:
        conn = get_connection()
        return conn.execute("SELECT ...").fetchall()
    except Exception as e:
        print(f"[module] get_data error: {e}", file=sys.stderr)
        return []
```

## Why

Catching `Exception` and returning an empty result hides database errors, schema mismatches, and connection failures. The caller sees empty data and assumes "no results" when the real cause is a crash. At minimum, log the exception to stderr so developers can diagnose issues.
