# Bug: get_post_meta() N+1 — AST Verified

## Difference from LES-445

LES-445 uses regex matching — catches `get_post_meta()` on any line, leading to false positives on single-post templates and helper functions.

LES-457 uses **tree-sitter AST parsing** to:
1. Parse PHP into syntax tree
2. Find all `get_post_meta()` call nodes
3. Find all loop nodes (foreach, for, while)
4. Only flag calls that are **inside a loop** AND the file has **no `update_meta_cache()` call**

## When to use which

- **LES-445**: fast regex scan, good for broad coverage with context_guard
- **LES-457**: precise AST scan, zero false positives for loop detection

## Example (flagged by LES-457)

```php
// AST detects: get_post_meta inside foreach, no update_meta_cache in file
foreach ($products as $product) {
    $price = get_post_meta($product->ID, '_price', true); // N+1!
}
```

## Example (NOT flagged by LES-457)

```php
// AST sees: get_post_meta NOT inside any loop — standalone, safe
$title = get_post_meta($post_id, '_title', true);
```
