## Problem

Login plugin enqueues Google Fonts on every page load (line 43-48). Each login request makes an external HTTP request to `fonts.googleapis.com`, adding latency and creating a SPOF (single point of failure).

## Bad

```php
public function enqueue_styles(): void {
    $font = $this->get_font();
    wp_enqueue_style(
        'wezone-login-font',
        'https://fonts.googleapis.com/css2?family=' . rawurlencode( $font ) . ':wght@400;500;600;700&display=swap',
        array(),
        null
    );
}
```

**Issues:**
- External request on every login page load
- Blocks rendering if Google Fonts is slow/down
- Privacy concern: Google tracks login page visits

## Good

```php
public function enqueue_styles(): void {
    // Use local font file bundled with plugin
    $font = $this->get_font();
    wp_enqueue_style(
        'wezone-login-font',
        WEZONE_LOGIN_URL . 'assets/fonts/' . sanitize_file_name( $font ) . '.css',
        array(),
        WEZONE_LOGIN_VERSION
    );
}
```

**Or use system fonts:**

```css
/* In login.css */
body {
    font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
}
```

## Why

- **Performance**: Eliminates external HTTP request (saves 100-300ms)
- **Reliability**: No dependency on Google Fonts uptime
- **Privacy**: No third-party tracking on login page
- **Offline**: Works without internet connection

## How to fix

1. Download font files from Google Fonts
2. Place in `assets/fonts/` folder
3. Generate local `@font-face` CSS
4. Enqueue local CSS file instead of external URL
5. Or use system font stack for zero-latency rendering
