## Bad
```php
useEffect(() => {
  loadGraph()  // Calls async API
}, [])

// When WebSocket reconnects â†’ component re-renders â†’ useEffect runs again â†’ infinite loading
```

## Good
```php
const hasLoadedRef = useRef(false)

useEffect(() => {
  if (!hasLoadedRef.current) {
    hasLoadedRef.current = true
    loadGraph()
  }
}, [])

// Or: track loading state in deps
useEffect(() => {
  if (!loading && !graphData) {
    loadGraph()
  }
}, [loading, graphData])
```

## Why
When WebSocket reconnects after tab visibility change, it triggers component re-render. If useEffect has empty deps [] and calls async data loading without tracking whether data is already loading/loaded, the component enters infinite loading state because the effect runs on every render caused by WebSocket state changes.
