# Bad Code

```python
import requests

def fetch_data(url):
    # BAD: No timeout, hangs forever if server doesn't respond
    response = requests.get(url)
    return response.json()

async def fetch_async(url):
    async with httpx.AsyncClient() as client:
        # BAD: No timeout
        response = await client.get(url)
        return response.json()
```

# Good Code

```python
import requests
import httpx

def fetch_data(url):
    # GOOD: 10 second timeout
    response = requests.get(url, timeout=10)
    return response.json()

async def fetch_async(url):
    async with httpx.AsyncClient(timeout=10.0) as client:
        response = await client.get(url)
        return response.json()

# Or separate connect/read timeouts
response = requests.get(url, timeout=(3.0, 10.0))  # 3s connect, 10s read
```

# Why

**Without timeout:**
- Requests hang forever
- Thread/worker pool exhaustion
- Cascading failures
- Poor user experience

**Real-world impact:**
- One slow API blocks all workers
- Memory leaks from hung connections
- Service appears frozen
