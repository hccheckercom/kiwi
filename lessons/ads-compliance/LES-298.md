## Bad
```php
// thankyou.php — fires purchase event on every page load
<script>
dataLayer.push({
    event: 'purchase',
    ecommerce: { transaction_id: '<?php echo $order_id; ?>', value: <?php echo $total; ?> }
});
</script>
```
User refreshes thank-you page → duplicate purchase event → inflated revenue in GA4 + double-counted conversions in Google Ads.

## Good
```php
// thankyou.php — dedup via sessionStorage
<script>
(function() {
    var orderId = '<?php echo esc_js( $order['order_number'] ); ?>';
    var key = 'wz_order_tracked_' + orderId;
    if (sessionStorage.getItem(key)) return;
    sessionStorage.setItem(key, '1');
    dataLayer.push({ ecommerce: null });
    dataLayer.push({
        event: 'purchase',
        ecommerce: {
            transaction_id: orderId,
            value: <?php echo (int) $order['total']; ?>,
            currency: 'VND',
            items: <?php echo wp_json_encode( wz_tracking_order_items( $order ) ); ?>
        }
    });
})();
</script>
```

## Why
Thank-you page reload is the #1 cause of duplicate purchase events. GA4 does NOT auto-dedup by transaction_id on the client side. Result: (1) revenue inflated 10-30%, (2) Google Ads over-reports conversions → overbids, (3) ROAS calculations are wrong. Blueprint explicitly requires sessionStorage dedup flag per transaction_id.
