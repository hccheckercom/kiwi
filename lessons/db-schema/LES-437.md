## Pattern

Migration or seeder functions that create multiple database records without wrapping them in a transaction leave the system in a partial state if any individual insert fails. The function may report partial success, making debugging difficult.

## Bug

```php
// SAI — partial failure leaves inconsistent state
public static function import_home_page(): array {
    $imported = array();
    $order    = 0;

    // If create #3 fails, we have sections 1-2 but not 3-8
    // No way to retry without duplicating 1-2
    if ( ! in_array( 'top_bar', $existing_types, true ) ) {
        $id = PageSectionRepository::create( array( ... ) );
        if ( $id ) $imported[] = 'top_bar';
    }

    if ( ! in_array( 'hero_banners', $existing_types, true ) ) {
        $id = PageSectionRepository::create( array( ... ) );
        if ( $id ) $imported[] = 'hero_banners';
    }

    // ... 6 more sections, any can fail silently
    return array( 'imported' => $imported );
}
```

## Fix

```php
// ĐÚNG — all-or-nothing with transaction
public static function import_home_page(): array {
    global $wpdb;
    $wpdb->query( 'START TRANSACTION' );

    $imported = array();
    $failed   = false;

    $sections = array(
        array( 'type' => 'top_bar', 'title' => 'Top Bar', 'data' => array( ... ) ),
        array( 'type' => 'hero_banners', 'title' => 'Hero Banners', 'data' => array( ... ) ),
    );

    $existing       = PageSectionRepository::get_page_sections( 'home' );
    $existing_types = array_column( $existing, 'section_type' );
    $order          = count( $existing );

    foreach ( $sections as $section ) {
        if ( in_array( $section['type'], $existing_types, true ) ) {
            continue;
        }

        $id = PageSectionRepository::create( array(
            'page_id'      => 'home',
            'section_type' => $section['type'],
            'title'        => $section['title'],
            'data'         => $section['data'],
            'sort_order'   => $order++,
        ) );

        if ( 0 === $id ) {
            $failed = true;
            break;
        }

        $imported[] = $section['type'];
    }

    if ( $failed ) {
        $wpdb->query( 'ROLLBACK' );
        return array( 'success' => false, 'error' => 'Insert failed', 'imported' => array() );
    }

    $wpdb->query( 'COMMIT' );
    return array( 'success' => true, 'imported' => $imported );
}
```

## Áp dụng

1. **Wrap batch inserts in transaction** — all succeed or all roll back
2. **Check each insert result** — stop on first failure
3. **Return clear success/failure** — don't report partial success as success
4. **Idempotency** — check existing records before insert to allow retry
