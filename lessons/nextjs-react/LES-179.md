## Old pattern (React 18 — still works but verbose)
```tsx
'use client'
import { useState, useEffect } from 'react'

function ProductList() {
  const [products, setProducts] = useState<Product[]>([])
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    fetch('/api/products').then(r => r.json()).then(data => {
      setProducts(data)
      setLoading(false)
    })
  }, [])

  if (loading) return <Skeleton />
  return <Grid items={products} />
}
```

## New pattern (React 19 — use() with Suspense)
```tsx
'use client'
import { use, Suspense } from 'react'

const productsPromise = fetch('/api/products').then(r => r.json())

function ProductList() {
  const products = use(productsPromise) // Suspends until resolved
  return <Grid items={products} />
}

// Parent wraps with Suspense
export default function Page() {
  return (
    <Suspense fallback={<Skeleton />}>
      <ProductList />
    </Suspense>
  )
}
```

## When to use which in webstore-vn
| Scenario | Pattern |
|----------|---------|
| Server Component with DB query | async function + loading.tsx |
| Client Component, data at mount | use() + Suspense |
| Client Component, data on interaction | useState + fetch in handler |
| Client Component, complex state | Zustand store + useEffect |

## Why
React 19 (used in webstore-vn) introduces use() hook that integrates with Suspense. It eliminates the useState+useEffect+loading boilerplate for data fetching. The promise must be created outside the component (or memoized) to avoid re-creating on every render. For webstore-vn, prefer Server Components for initial data and use() only when client-side fetching is required.
