## Bad
```php
// Check if user explicitly opted in via query param (for testing)
if ( isset( $_GET['ads_consent'] ) && $_GET['ads_consent'] === '1' ) {
    return true; // GDPR violation â€” no explicit user action
}
// â†’ Any user can bypass: example.com?ads_consent=1
```

## Good
```php
// Remove bypass entirely OR restrict to admin
if ( current_user_can( 'manage_options' ) && isset( $_GET['ads_consent'] ) ) {
    $consent = sanitize_text_field( wp_unslash( $_GET['ads_consent'] ) );
    if ( $consent === '1' ) {
        return true; // Only for admin testing
    }
}
```

## Why
Allowing consent bypass via GET param violates GDPR â€” user must explicitly opt-in via UI. Also unsanitized $_GET can cause XSS if value is echoed. Remove bypass or restrict to admin/testing environments only.
