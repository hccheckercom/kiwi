## Bad
```tsx
<div className="cursor-pointer p-4" onClick={handleSelect}>
  {item.name}
</div>
```
Clickable div — not focusable, no keyboard activation, no role announced.

## Good
```tsx
<button className="p-4 text-left w-full" onClick={handleSelect}>
  {item.name}
</button>
```
Or if div is required:
```tsx
<div role="button" tabIndex={0} onClick={handleSelect} onKeyDown={(e) => e.key === 'Enter' && handleSelect()}>
  {item.name}
</div>
```

## Why
WCAG 2.1.1 — all functionality must be keyboard accessible. Divs with onClick are invisible to keyboard users and screen readers. Use semantic `<button>` or add role + tabIndex + keyDown handler.
