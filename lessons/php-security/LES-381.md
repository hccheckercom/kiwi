# Bug: GET /cart Public Endpoint Không Rate Limit

## Vấn đề

`GET /wezone/v1/cart` có `permission_callback => '__return_true'` — bất kỳ ai cũng gọi được, không cần nonce. Attacker có thể:

1. **Session enumeration:** brute-force session keys qua cookie
2. **DoS:** spam GET requests → cache thrashing → DB pressure
3. **Cart scraping:** đọc cart content qua session key leak

## Fix

```php
// ✅ ĐÚNG — thêm rate limiting
register_rest_route( $ns, '/cart', array(
    'methods'             => \WP_REST_Server::READABLE,
    'callback'            => array( $this, 'get_cart' ),
    'permission_callback' => function() {
        return wz_rate_limit( 'cart_read', 30, MINUTE_IN_SECONDS );
    },
));
```

Hoặc minimal approach — chỉ cho phép khi có valid session cookie:
```php
'permission_callback' => function( WP_REST_Request $request ) {
    $cookie = sanitize_text_field( wp_unslash( $_COOKIE['wz_session'] ?? '' ) );
    return $cookie !== '' && preg_match( '/^[a-f0-9]{32}$/', $cookie );
},
```
