## Bad
```tsx
<div className="md:grid-cols-3 md:gap-6">
```
Has `md:grid-cols-3` but no base `grid-cols-1` — on mobile, grid has no column definition.

## Good
```tsx
<div className="grid-cols-1 gap-4 md:grid-cols-3 md:gap-6">
```
Base mobile style defined first, responsive overrides layered on top.

## Why
Mobile-first means base styles = mobile. Responsive prefixes only ADD behavior at larger breakpoints. Without a base class, mobile renders with browser defaults — often broken layout.
