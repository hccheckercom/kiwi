# Bug: N+1 Insert/Update Queries — Phải Dùng Bulk Insert Hoặc Temp Table Swap

## Vấn đề

Khi insert/update nhiều rows trong loop, mỗi iteration gọi `$wpdb->insert()` hoặc `$wpdb->update()` → **N queries** thay vì 1 bulk query → chậm, tốn connection pool, dễ timeout khi N lớn.

**Ví dụ lỗi:**
```php
// ❌ SAI — 1000 contacts = 1000 queries
foreach ( $contacts as $contact ) {
    $wpdb->insert(
        $wpdb->prefix . 'wezone_crm_contacts',
        array(
            'email'      => $contact['email'],
            'full_name'  => $contact['name'],
            'created_at' => current_time( 'mysql' ),
        )
    );
}

// ❌ SAI — 5 segments = 5 UPDATE queries
foreach ( $built_in as $slug => $name ) {
    $wpdb->query(
        $wpdb->prepare(
            "INSERT INTO {$segments_table} (name, slug, count) VALUES (%s, %s, %d)
             ON DUPLICATE KEY UPDATE count = VALUES(count)",
            $name, $slug, $counts[ $slug ] ?? 0
        )
    );
}
```

## Fix

### Option 1: Dùng `wz_bulk_insert()` (Shared Helper)

```php
// ✅ ĐÚNG — 1 query cho 1000 contacts
$rows = array();
foreach ( $contacts as $contact ) {
    $rows[] = array(
        'email'      => $contact['email'],
        'full_name'  => $contact['name'],
        'created_at' => current_time( 'mysql' ),
    );
}

wz_bulk_insert(
    $wpdb->prefix . 'wezone_crm_contacts',
    $rows,
    array( '%s', '%s', '%s' ) // format cho 3 columns
);
```

**`wz_bulk_insert()` signature** (từ `shared/src/Helpers.php`):
```php
function wz_bulk_insert( string $table, array $rows, array $format ): bool;
```

### Option 2: Temp Table Swap (Cho Rebuild Index)

Khi rebuild toàn bộ table (ví dụ: segment sync, analytics aggregation), dùng temp table để tránh mất data nếu insert fail giữa chừng:

```php
// ✅ ĐÚNG — atomic swap, không mất data nếu fail
global $wpdb;
$real_table = $wpdb->prefix . 'wezone_crm_segments';
$tmp_table  = $wpdb->prefix . 'wezone_crm_segments_tmp';

// 1. Tạo temp table giống structure
$wpdb->query( "CREATE TABLE {$tmp_table} LIKE {$real_table}" );

// 2. Bulk insert vào temp
$rows = array();
foreach ( $built_in as $slug => $name ) {
    $rows[] = array( 'name' => $name, 'slug' => $slug, 'count' => $counts[ $slug ] ?? 0 );
}
wz_bulk_insert( $tmp_table, $rows, array( '%s', '%s', '%d' ) );

// 3. Atomic swap
$wpdb->query( "RENAME TABLE {$real_table} TO {$real_table}_old, {$tmp_table} TO {$real_table}" );
$wpdb->query( "DROP TABLE {$real_table}_old" );
```

**Khi nào dùng temp table swap:**
- Rebuild toàn bộ table (xóa cũ + insert mới)
- Cần atomic operation (không có trạng thái giữa chừng)
- Data quan trọng (không được mất nếu fail)

**Khi nào dùng `wz_bulk_insert()` thôi:**
- Append rows mới (không xóa cũ)
- Upsert (INSERT ... ON DUPLICATE KEY UPDATE)
- Không cần atomic guarantee

## Impact

- **Performance:** 1000 contacts từ 5-10s → <500ms
- **Connection pool:** Giảm từ N connections → 1 connection
- **Timeout risk:** Tránh timeout khi N > 1000
